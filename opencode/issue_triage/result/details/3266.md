## Root Cause

RFC/tracking issue to migrate ~33 math functions in torch-xpu-ops/src/ATen/native/xpu/sycl/ from
std::/::/bare libm calls (e.g., std::exp, ::expf, sqrtf) to sycl:: / sycl::native:: namespace
equivalents for correctness, portability and Intel-GPU performance. Examples:
ForeachUnaryKernels.cpp:181 (STD_FUNCTOR macro), Philox4x32.h (bare expf/logf/sqrtf),
XPUMathCompat.h. Complex types and host code must keep std::. Also requests adding XPU fast-math
branches to upstream aten/src/ATen/NumericUtils.h.

## Fix Approach

- Execute the task list in the RFC one-PR-per-function: start with infra (XPUMathCompat.h additions and STD_FUNCTOR macro refactor to dispatch sycl:: for real / std:: for complex), then migrate kernels alphabetically with accuracy + perf tests per PR.
- File the NumericUtils.h change as a separate pytorch/pytorch PR.
- Carefully verify nearbyintf->rint semantics before swap.
