## Root Cause

RFC tracking issue to migrate ~33 math functions from std::/::/bare libm calls to
sycl::/sycl::native:: namespace across all SYCL kernels in
third_party/torch-xpu-ops/src/ATen/native/xpu/sycl/ (e.g., ForeachUnaryKernels.cpp:181 STD_FUNCTOR
macro, XPUMathCompat.h, NumericUtils.h). Goal is correctness/portability/perf alignment with
CUDAMathCompat.h conventions; complex types must keep std:: since sycl:: math does not accept
c10::complex.

- Per task list in RFC: land one PR per math function with accuracy + perf test.
