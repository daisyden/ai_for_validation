## Root Cause

test_dlpack.py::test_dlpack_exchange_api fails at test_dlpack.py:790-794 because
api->current_work_stream returns stream_out == nullptr for an XPU tensor. The implementation in
torch/csrc/Module.cpp:781-799 (TorchDLPackExchangeAPI::CurrentWorkStream) only handles
kDLCUDA/kDLROCM under #ifdef USE_CUDA and falls through to *out_stream = nullptr for all other
device types, including kDLOneAPI (XPU). No XPU branch is registered, so the stream exchange test
correctly flags a missing implementation.

## Fix Approach

- Add a USE_XPU branch in CurrentWorkStream (torch/csrc/Module.cpp:786) that returns at::xpu::getCurrentXPUStream(device_id).queue() (the sycl::queue pointer) for device_type == kDLOneAPI, matching the CUDA path.
- This is the upstream fix already proposed in pytorch/pytorch#173760.
- After landing, remove the skip from torch-xpu-ops.
- No SYCL kernel work is needed.
