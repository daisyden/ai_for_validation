## Root Cause

torch.ops.aten._flash_attention_forward has no XPU kernel registered. torch-xpu-ops only implements
the internal flash_attention_forward/flash_attention_backward wrappers
(src/ATen/native/transformers/xpu/flash_attn/flash_api.cpp:33, sycltla/mha_fwd.cpp:373) and
registers the SDPA dispatch path (Attention.cpp:210), but the public aten op
_flash_attention_forward (used by schema_check and meta tests) falls back to CPU, which has no
implementation. Duplicates #2285.

## Fix Approach

- Add an XPU registration for aten::_flash_attention_forward in yaml/native/native_functions.yaml (XPU dispatch) and implement the wrapper in src/ATen/native/transformers/xpu/flash_attn/flash_api.cpp that translates the schema arguments (cum_seq_q/k, max_q/k, is_causal, return_debug_mask, scale) to the existing flash_attention_forward_sycltla call.
- Also register _flash_attention_backward similarly, then remove the skip entry.
