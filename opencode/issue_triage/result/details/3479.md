test_profiler_xpu.py:2116 asserts traceEventProfiler['name'] == 'PyTorch Profiler (0)' but the
chrome trace produced on XPU contains a different profiler-section name (e.g. parametrized by device
or thread id). Multiple TestProfiler/TestExperimentalUtils cases (test_basic_chrome_trace,
test_dynamic_toggle, test_disable_external_correlation, test_utils_get_optimizable_events,
test_profiler_pattern_matcher_json_report) fail because Kineto/profiler emits different metadata on
XPU vs CPU/CUDA. Profiler logic is upstream in torch/csrc/profiler.
