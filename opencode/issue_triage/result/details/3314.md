## Root Cause

test/test_xpu.py::TestMemPool::test_mempool_id (test/test_xpu.py:3051) crashes on Windows during
MemPool destruction: stack shows c10::xpu::MemPool::~MemPool -> XPUCachingAllocator::releasePool ->
c10::xpu::syncStreamsOnDevice throwing through sycl8.dll, terminating in ucrtbase. Aborted at
sycl::queue boundary on Windows indicates a SYCL/Level-Zero driver-layer exception during stream
sync at MemPool teardown.

## Fix Approach

- Wrap syncStreamsOnDevice / releasePool in MemPool destructor with try/catch logging (destructors must not throw across SYCL DLL boundary on Windows) in c10/xpu/XPUCachingAllocator.cpp.
- in parallel, file a Level-Zero/SYCL Windows driver bug with the call stack and skip test_mempool_id on Windows via run_test_win_with_skip_mtl.py until the runtime fix lands.
