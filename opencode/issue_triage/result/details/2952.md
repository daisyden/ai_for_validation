## Root Cause

convnextv2_nano training bf16 fails accuracy on BMG (PVC passes). Bisect identified upstream commit
bc4d0bf39cc56f4ff3517d503dd80e225143580b which changes the Triton reduction heuristic for
triton_red_fused_convolution_native_layer_norm_permute_* from ReductionHint.DEFAULT to INNER. The
numeric difference (variance 1.9934 → 1.9933 in one column) raises gradient RMSE from 0.00135 to
0.00228, breaching 0.001 tolerance. It is bf16-only and BMG-only, suggesting a Triton XPU
INNER-reduction codegen accuracy issue on BMG.

## Fix Approach

- File a Triton-XPU bug for INNER reduction-hint codegen accuracy on BMG with bf16 reductions.
- Workarounds: (a) loosen tolerance for this model on BMG, (b) revert the heuristic for XPU until Triton fix lands, or (c) force higher-precision accumulator in the affected reduction.
- Coordinate with intel-xpu-backend-for-triton.
