## Root Cause

test_state_dict_stager.py::test_tensor_pinned_and_shared fails on XPU because
torch/distributed/checkpoint/_state_dict_stager.py:36 forces self.pin_memory=False whenever
`torch.cuda.is_available()` is False, and the subsequent pinning call at line 170 uses
`torch.cuda._pin_memory_utils.pin_memory` which is CUDA-only. On an XPU-only host pin_memory is
silently dropped, so cpu_tensor.is_pinned() returns False although the test expects True.

## Fix Approach

- Generalize the stager to be device-agnostic: replace the torch.cuda.is_available() guard in _state_dict_stager.py:36 with `torch.accelerator.is_available()` (or check cuda||xpu), and route pinning through a device-aware helper that calls torch.xpu host_alloc/pin APIs (pin_memory_utils has an XPU variant or use Storage.pin_memory(device='xpu')).
- Update _pin_memory_utils import accordingly and add an XPU is_pinned verification path.
