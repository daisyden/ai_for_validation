## Root Cause

test_int64_upsample3d_xpu_bfloat16 hits TORCH_CHECK 'output.numel() <= int32_t::max()' from UpSample
3d kernels because XPU implementation uses int32 indexing while the test exercises >2^31 element
output to verify int64 indexing fallback. CUDA skips this case due to insufficient memory; XPU runs
and trips the int32 guard. The check originates in upstream UpSample.h size guard reused by XPU sycl
upsample3d kernels.

## Fix Approach

- Either add int64-indexing path in XPU upsample3d nearest/linear sycl kernels, or follow CUDA and skip the test on XPU via skipIfInsufficient memory/large-tensor decorator in the upstream test_torch.py allowlist.
