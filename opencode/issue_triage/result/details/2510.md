## Root Cause

The slow test test_int64_upsample3d_xpu_bfloat16 exercises an upsample_nearest3d output whose numel
exceeds INT32_MAX. The XPU kernel enforces a hard 32-bit indexing limit via TORCH_CHECK at
third_party/torch-xpu-ops/src/ATen/native/xpu/sycl/UpSampleNearest3dKernels.cpp:174 (and 387-388 for
backward), so the check fires before the kernel can run. CUDA avoids the failure by skipping the
test due to insufficient memory, so the XPU path has no matching memory-based skip.

## Fix Approach

- Either (a) mirror CUDA by registering a largeTensorTest / memory-guard skip for this slow case in the XPU skip list, or (b) add a 64-bit-indexing fallback path in UpSampleNearest3dKernels.cpp (dispatch between int32/int64 index types based on output.numel()) analogous to the CUDA implementation, removing the unconditional TORCH_CHECK.
