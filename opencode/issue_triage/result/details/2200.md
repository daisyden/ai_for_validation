## Root Cause

On XPU, scaled_dot_product_attention dispatches via the oneDNN graph SDP primitive
(sdp_primitive_v1_kernel_t) — see torch-xpu-ops src/ATen/native/transformers/Attention.cpp /
sdp_utils. Profiler shows aten::_scaled_dot_product_fused_attention_overrideable consuming ~4ms on
B60 vs ~1ms flash_fwd_kernel on 4090D for the same shape. SDPBackend.FLASH_ATTENTION /
EFFICIENT_ATTENTION fall back to math because no native XPU flash/efficient kernel exists; the only
fast path is the oneDNN graph kernel which is slower than CUDA's flash-attn for these shapes. oneDNN
v3.10-rc shows no improvement (4.10ms). Request for aten::_flash_attention_forward was split out to
#2344.

## Fix Approach

- Performance optimization belongs to oneDNN team (sdp_primitive_kernel_t for B60/BMG).
- Push oneDNN to improve fp16 SDP performance on Xe2/B60.
- in parallel, register a real XPU flash_attention backend so SDPBackend.FLASH_ATTENTION dispatches to oneDNN/SYCL path instead of math (tracked under #2344).
- No torch-xpu-ops kernel change needed for this issue itself.
