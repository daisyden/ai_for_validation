## Root Cause

XPU lacks a native flash-attention backend. SDPA on XPU dispatches through
_scaled_dot_product_fused_attention_overrideable (the generic fused path) rather than a
flash-attention kernel, so SDPBackend.FLASH_ATTENTION silently falls back to the math/overrideable
path. No flash-attention kernel exists under
third_party/torch-xpu-ops/src/ATen/native/transformers/sycl/ (only AttentionKernels.cpp for
transform_bias_rescale_qkv), and native_functions.yaml registers only
_scaled_dot_product_fused_attention_overrideable for XPU. Result: B60 is ~4-5x slower than 4090D on
the same SDPA call.

## Fix Approach

- Implement a flash-attention forward/backward via oneDNN SDPA primitive (micro-kernel SDPA available in oneDNN 3.5+) and register it as _flash_attention_forward/_flash_attention_backward for XPU (or route SDPBackend.FLASH_ATTENTION via sdp::sdp_utils to a new XPU flash path).
- Add corresponding entries in third_party/torch-xpu-ops/yaml/native/native_functions.yaml and new kernels under src/ATen/native/transformers/sycl/FlashAttentionKernels.cpp.
- Gate support via can_use_flash_attention heuristic similar to the CUDA side.
