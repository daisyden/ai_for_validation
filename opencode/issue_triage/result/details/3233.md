## Root Cause

split_group at torch/distributed/distributed_c10d.py:5478-5483 raises 'No backend ... does not
support splitting' because the parent ProcessGroupXCCL backend reports
parent_backend.supports_splitting == False. ProcessGroupXCCL
(third_party/torch-xpu-ops/src/xccl/ProcessGroupXCCL.cpp) has not implemented the
supportsSplitting() override / comm_split_count() that NCCL provides, so DeviceMesh 2D init and
backend-override-by-index init both fail when constructing the sub-group via
ncclCommSplit-equivalent path.

## Fix Approach

- In third_party/torch-xpu-ops/src/xccl/ProcessGroupXCCL.{hpp,cpp}, implement supportsSplitting() to return true and add a split() / comm_split_count() implementation backed by ccl::create_communicator with the color/key arguments (oneCCL split-comm API).
- Also expose 'comm_split_count' in the Backend pybind so Python distributed_c10d.py can read it.
- Add UT coverage for DeviceMesh 2D eager init.
