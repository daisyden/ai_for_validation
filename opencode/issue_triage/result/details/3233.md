## Root Cause

DeviceMesh init_process_groups path calls torch.distributed.distributed_c10d.split_group
(distributed_c10d.py:5485), which raises 'No backend for the parent process group or its backend
does not support splitting' because ProcessGroupXCCL does not implement the splitGroup hook required
by eager-init DeviceMesh. Reproduced on tests test_2d_mesh_eager_init_subgroup and
test_backend_override_argument_dict_with_idx_and_backend_eager.

## Fix Approach

- Implement splitGroup support in ProcessGroupXCCL (third_party/torch-xpu-ops/src/xccl/ProcessGroupXCCL.cpp/.hpp) mirroring ProcessGroupNCCL::splitGroup, including oneCCL comm split via ccl::create_communicator with subgroup ranks.
- Register the split_group dispatch entry in Backend so split_group() recognizes the XCCL backend.
- Re-enable the two device_mesh UTs once landed.
