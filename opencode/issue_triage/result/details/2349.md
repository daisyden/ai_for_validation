## Root Cause

PyTorch XPU wheel ships pinned oneAPI runtime libs (libsycl.so.8, libur_loader, etc.) under
torch/lib with RPATH, but when a user sources /opt/intel/oneapi/2025.2/oneapi-vars.sh,
LD_LIBRARY_PATH overrides RPATH and an older system libur_loader.so.0 lacking symbol version
`LIBUR_LOADER_0.11' is loaded, causing ImportError on `from torch._C import *`
(torch/__init__.py:416). This is a backward-compat / library resolution problem between the bundled
oneAPI 2025.x runtime and the user-sourced environment.

## Fix Approach

- Either (a) make the bundled torch/lib path take precedence by using DT_RUNPATH + linking oneAPI deps with -Wl,-rpath at wheel build time so they aren't overridden by LD_LIBRARY_PATH.
- (b) use dlopen of bundled libsycl.so.8 with RTLD_DEEPBIND in torch/_C init.
- or (c) document version pinning.
- Coordinate with oneAPI release engineering for libur_loader symbol versioning compatibility.
