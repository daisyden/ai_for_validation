## Root Cause

symm_mem::fused_matmul_reduce_scatter (and friends) is registered only for CUDA in
torch/csrc/distributed/c10d/SymmetricMemory; XPU lacks a SymmetricMemory backend and the fused TP
collectives used by test/distributed/tensor/parallel/test_micro_pipeline_tp.py (MicroPipelineTP
fused matmul+reduce_scatter/all_gather+matmul) raise NotImplementedError. Per maintainer comment
this is the same blocker as #1551 — symmetric memory support is missing on XPU (requires an
XPU/XCCL-backed implementation plus kernel registrations).

## Fix Approach

- Implement an XPU SymmetricMemory backend (either ProcessGroupXCCL-backed or IPC/L0-based) and register the _async/fused symm_mem ops (fused_matmul_reduce_scatter, fused_all_gather_matmul, fused_scaled_matmul_reduce_scatter).
- Track together with #1551.
- until then skip the MicroPipelineTPTest cases in XPU CI via skip lists.
