## Root Cause

test_module_share_memory_xpu calls Module.share_memory() which invokes Storage._share_fd_() on each
parameter. _share_fd_ is guarded at torch/csrc/StorageSharing.cpp:208 with
`TORCH_CHECK(storage.device_type() == at::kCPU, "_share_fd_: only available on CPU")`, so any XPU
tensor hits this error. There is no XPU implementation of fd-based shared memory (only CPU has
THPStorage_shareFd registered at StorageSharing.cpp:656).

## Fix Approach

- Either (a) teach share_memory_ to move storage to CPU transparently for non-CPU devices (matches CUDA behaviour, which also errors), or more realistically (b) add XPU to the skip list for test_module_share_memory (aligns with CUDA which skips) - the upstream test should gate on `device == 'cpu'`.
- If true shared-memory IPC support for XPU is desired, add XPU branch in torch/csrc/StorageSharing.cpp using level-zero IPC handles.
- this is out of scope for a UT-only fix.
- Recommend short-term skip via test/xpu/skip_list_common.py.
- Related to #1678.
