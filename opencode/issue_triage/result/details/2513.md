## Root Cause

test_module_share_memory_xpu calls Tensor.share_memory_() on an XPU tensor, which routes to
StorageImpl::_share_fd_; this is implemented only for CPU storage (see torch/csrc/StorageSharing.cpp
/ aten/src/ATen/MapAllocator). XPU storage has no shared-memory IPC backend, so the runtime
correctly raises 'RuntimeError: _share_fd_: only available on CPU'. The test was ported from CUDA
without realizing CUDA uses cudaIpc, not _share_fd_; XPU has no equivalent IPC path yet.

## Fix Approach

- Either (a) skip test_module_share_memory for XPU in third_party/torch-xpu-ops/test/xpu/skip_list_common.py with reason 'XPU IPC/share_memory not implemented', or (b) implement an XPU IPC path (level-zero ipc handle) in c10/xpu and torch/csrc/xpu to support share_memory_().
- Short-term: skiplist entry.
- long-term feature gap tracked separately.
