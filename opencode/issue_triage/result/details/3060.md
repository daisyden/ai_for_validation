## Root Cause

torch._scaled_grouped_mm (aten::_scaled_grouped_mm) is not registered for the XPU dispatch key, so
MoE flows fail with NotImplementedError and cannot even fall back without
PYTORCH_ENABLE_XPU_FALLBACK=1. There is no XPU kernel implementation under
third_party/torch-xpu-ops/src/ATen/native/xpu/ for this op, and no entry in the XPU yaml
registration (src/ATen/native/xpu/XPUFallback.template / native_functions yaml). Maintainer Stonepia
scheduled this for ~PyTorch 2.15.

## Fix Approach

- Implement an FP8 scaled grouped matmul XPU kernel — either (a) add an oneDNN-backed path mirroring the existing _scaled_mm/_grouped_mm implementations in src/ATen/native/xpu/Blas.cpp, or (b) add a SYCL kernel under src/ATen/native/transformers/sycl/.
- Register the op via TORCH_LIBRARY_IMPL(aten, XPU, ...).
- Until then, add aten::_scaled_grouped_mm to the explicit CPU fallback list in src/ATen/native/xpu/XPUFallback.template so MoE workloads run (slowly) without setting the env var.
