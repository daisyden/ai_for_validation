## Root Cause

aten::_scaled_grouped_mm (declared in aten/src/ATen/native/native_functions.yaml:7402) only has a
CUDA dispatch (_scaled_grouped_mm_cuda at line 7405) and no XPU registration, so calling it on XPU
raises NotImplementedError. torch-xpu-ops has no corresponding kernel under
third_party/torch-xpu-ops/src/ATen/native/xpu/ (no ScaledGroupedMM*.cpp or SYCL kernel) and no yaml
entry in xpu_functions.yaml, which is required for the Llama/MoE FP8 flow (fp8 rowwise +
offsets-based grouped gemm).

## Fix Approach

- Implement _scaled_grouped_mm for XPU.
- Add src/ATen/native/xpu/ScaledGroupedMM.cpp with a registration for the op, and a SYCL kernel src/ATen/native/xpu/sycl/ScaledGroupedMMKernels.cpp/.h that handles fp8 (e4m3/e5m2) inputs with per-tensor/per-row scale tensors and an offs tensor to stride through groups.
- First milestone can dispatch to oneDNN matmul in a per-group loop (iterate over offs, call oneDNN int8/fp8 matmul with scale post-ops).
- a second milestone adds a fused SYCL kernel for better perf.
- Register under XPU in native_functions.yaml (or via the torch-xpu-ops out-of-tree yaml) and add UT under test/xpu mirroring CUDA's test_matmul_cuda::test_scaled_grouped_mm.
- Scheduled for PT2.15 per assignees Stonepia/liangan1.
