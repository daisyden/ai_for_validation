## Root Cause

Tracking/feature request (not a bug) to mirror CUDA's TorchAO benchmark enabling on XPU, following
pytorch/ao#3576. There is no failing test or root cause; the issue exists because TorchAO's
benchmark harness under ao/benchmarks currently hard-codes `cuda` device selection and lacks an XPU
branch, so `torchao==0.16.0+xpu` cannot be exercised by the CI performance suite on Intel GPUs.

## Fix Approach

- Port the TorchAO benchmark driver to accept `xpu` as a device (parameterize device string, replace torch.cuda.* with torch.accelerator or torch.xpu equivalents for sync/event/memory), wire it into torch-xpu-ops CI under .ci/ similar to the existing dynamo benchmarks, and open the corresponding PR against pytorch/ao per issue 3576.
- No kernel-level changes needed in torch-xpu-ops.
