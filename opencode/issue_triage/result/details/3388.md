## Root Cause

Dynamo lowers torch.xpu.Stream.record_event/wait_stream to the primitive ops
torch.ops.streams.record_event/wait_stream which require an int stream_index, but the variable
tracker passes None (see torch/_dynamo/variables/streams.py and the StreamVariable lowering).
Mirrors pytorch#180179 'torch.cuda.Stream traced incorrectly by torch.compile'; the bug is in
upstream Dynamo's stream tracing, not in torch-xpu-ops.
