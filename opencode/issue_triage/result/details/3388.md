## Root Cause

In test/xpu/dynamo/test_ctx_manager_xpu.py (`test_cuda_event_method` ~L554-591,
`test_cuda_stream_context_manager1`, `test_cuda_stream_method`), Dynamo traces
`Stream.record_event()` / `wait_event()` and lowers to the primitive
`torch.ops.streams.record_event(stream_id, stream_index)` / `streams.wait_stream(...)` ops. The XPU
`Stream` variable in pytorch/torch/_dynamo/variables/streams.py constructs the call with
`stream_index=None`, but the schema declares `int`, so dispatch fails with `RuntimeError:
streams::record_event() Expected a value of type int for argument stream_index but instead found
type NoneType`. Same root cause as upstream pytorch#180179 — Dynamo's stream-tracking does not yet
populate `stream_index` for non-CUDA streams.

## Fix Approach

- Fix is in pytorch (not torch-xpu-ops).
- Patch `torch/_dynamo/variables/streams.py` (StreamVariable / EventVariable) to populate `stream_index` from `Stream.stream_id`/`device_index` for XPU as it does for CUDA, and/or change the schema of `streams::record_event` and `streams::wait_stream` to accept `Optional[int]` and have the runtime resolve to current stream when None.
- Add an XPU-specific test in `test/dynamo/test_streams.py` mirroring `test_cuda_event_method`.
- Until upstream lands, keep these three cases dynamic-skipped via this issue and cross-link pytorch#180179.
