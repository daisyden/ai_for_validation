## Root Cause

TestCompositeComplianceXPU.test_cow_input_* failures across many linalg/matmul ops (addmm, addmv,
baddbmm, bmm, cholesky*, einsum, cdist, jiterator, etc.). The COW (copy-on-write) compliance harness
in torch/testing/_internal/composite_compliance.py instruments inputs and expects ops to materialize
before in-place mutation; XPU registrations (oneDNN matmul, oneMKL cholesky, etc.) bypass the
composite path and mutate storage that COW expected to be untouched, so the harness flags missed
materialization.

## Fix Approach

- Either extend XPU op implementations to call lazy_clone_storage()/materialize on aliased inputs the same way CUDA does (matches CUDA dispatch behavior), or update the test allowlist (test/test_ops.py / common_methods_invocations.py) to skip COW for ops that legitimately don't support it on XPU.
- Ports the upstream CUDA pattern to XPU.
