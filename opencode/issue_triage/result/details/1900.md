## Root Cause

torch.linalg.qr has no XPU kernel and falls back to CPU (RegisterXPU_0.cpp:54525); on PEFT LoRA EVA
the op consumes ~37% of weight-init time. Implementation PR intel/torch-xpu-ops#2399 exists but is
blocked on oneMKL performance optimizations for geqrf/orgqr GPU paths, so the milestone was moved
from PT2.11 -> PT2.13 by pbielak.

## Fix Approach

- Unblock PR #2399 by: (1) tracking the oneMKL geqrf/orgqr perf request to completion, (2) landing the kernel even if perf is suboptimal with a CPU-fallback heuristic for problem sizes where MKL GPU path regresses, (3) adding dtype/batching coverage tests in test/xpu/test_linalg_xpu.py, (4) verifying end-to-end LoRA EVA speedup before closing.
