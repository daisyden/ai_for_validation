## Root Cause

test/test_spectral_ops.py:337 hard-codes the expected error message as 'cuFFT only supports
dimensions whose sizes are powers of two'. The XPU FFT path routes through oneMKL-DFT and raises a
different RuntimeError (MKL/DFT specific wording) when fp16/chalf inputs have non-power-of-two
sizes, so assertRaisesRegex fails on XPU. Test is @onlyNativeDeviceTypes so it runs on XPU too.

## Fix Approach

- Make the error regex device-aware in test_spectral_ops.py test_fft_half_and_chalf_not_power_of_two_error (e.g. 'cuFFT|MKL|powers? of two' pattern) or add the test to xpu skiplist (test/xpu/skip_list_common.py) until the MKL FFT error message is normalized.
- Upstream change preferred.
- requires cherry-pick into torch-xpu-ops test overrides.
