## Root Cause

test_fft_half_and_chalf_not_power_of_two_error in test/test_spectral_ops.py expects the
fp16/complex32 'half-precision FFT requires power-of-two signal length' RuntimeError, which is
raised by CUDA's cuFFT path; XPU's oneMKL DFT-backed FFT does support non-power-of-two half/chalf
lengths, so no error is raised and assertRaises fails. Pure CUDA-specific test that was never gated
by @onlyCUDA. Maintainer-confirmed (CuiYifeng) to be skipped via pytorch PR #2996.

## Fix Approach

- Land upstream PR #2996 to add @onlyCUDA / dtypeIfCUDA gating on test_fft_half_and_chalf_not_power_of_two_error (or skip on XPU).
- No XPU kernel change required.
- remove the skip entry from torch-xpu-ops once upstream lands.
