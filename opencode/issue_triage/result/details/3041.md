## Root Cause

test_fake_crossref_backward_amp on torch.normal with scalar mean/std (e.g. SampleInput(1.6, 0.3,
[2,3])) has no inputs with requires_grad=True, so composite_compliance.compute_expected_grads raises
'Expected len(flat_diff_results) > 0' at torch/testing/_internal/composite_compliance.py:443. CUDA
already skips this via DecorateInfo at torch/testing/_internal/common_methods_invocations.py for op
'normal'; XPU was not added to the skip. Triggered by PyTorch PR #176690 generalizing TestFakeTensor
across accelerators.

## Fix Approach

- Add device_type='xpu' (or replace with allowed_dtypes/skip across non-CPU) to the existing DecorateInfo skip for TestFakeTensor on the 'normal' OpInfo in torch/testing/_internal/common_methods_invocations.py.
- submit upstream PyTorch PR.
