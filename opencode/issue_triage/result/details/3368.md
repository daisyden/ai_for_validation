## Root Cause

DeepSeek-R1-Distill-Llama-8B RTN INT4 first-token latency regresses ~7% on BMG (B580) after
upgrading transformers 4.55.4 → 5.4.0; v5.4.0 also requires --disable-skip-guard-eval since
skip-guard-eval is no longer supported, which forces full Dynamo guard evaluation per call and
increases inductor recompile/guard overhead on the woq-int4 path. The XPU torchao RTN INT4 quant
kernels themselves are unchanged; the slowdown stems from transformers' modeling/cache-init changes
interacting with the torch.compile guard system rather than a kernel regression. The report also
notes possible accuracy drift in v5.4.0 output text, suggesting upstream model-code behavior change.

## Fix Approach

- Profile both runs (already linked) to confirm whether the 7% gap lives in (a) Dynamo guard eval overhead from --disable-skip-guard-eval, (b) extra graph breaks/recompiles from transformers v5.4 model code, or (c) attention/cache changes.
- if (a), file an upstream pytorch issue to restore a guard-eval skip mechanism or reduce per-step guard cost.
- if (b), pin the offending change in transformers and either patch the modeling file or update the run script.
- Separately validate accuracy by diffing generated tokens against 4.55.4 baseline.
- if numerics differ, isolate the transformers commit responsible.
- No torch-xpu-ops kernel change expected.
