## Root Cause

DeepSeek-R1-Distill-Llama-8B RTN INT4 first-token latency regressed ~7% on BMG after upgrading
transformers 4.55.4 -> 5.4.0; user had to add --disable-skip-guard-eval because v5.4.0 dropped
skip_guard_eval support, which forces full Dynamo guard re-evaluation each step and likely retraces
the int4 woq path. Output content also indicates accuracy drift, suggesting Dynamo/Inductor cache
invalidation interacts with torchao int4 weight-only quant lowering registered in
pytorch/torch/_inductor/kernel and torchao XPU backend.

## Fix Approach

- Bisect first-token cost between the two transformers versions with TORCH_LOGS=guards,recompiles to identify which guard set forces recompile.
- restore an equivalent of skip_guard_eval (or keep static-cache + cudagraph-style replay) for the int4 path.
- Investigate accuracy by comparing dequant output of torchao RTN int4 kernel against transformers 4.55.4 reference.
- file companion issue against transformers if the regression is from generation_utils, otherwise patch torchao XPU dequant path.
