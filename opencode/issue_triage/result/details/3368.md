## Root Cause

After upgrading HuggingFace transformers from 4.55.4 to 5.4.0 the DeepSeek-R1-Distill-Llama-8B RTN
INT4 first-token latency on BMG regressed ~7%. transformers 5.4.0 dropped support for the
skip_guard_eval fast-path (the script must pass --disable-skip-guard-eval), forcing Dynamo to
re-evaluate guards on every step; the reporter also notes output accuracy degradation in the 5.4.0
log, indicating a graph-break/recompilation or dtype-handling change in the new transformers
modeling code that the XPU INT4 WOQ path is sensitive to.

## Fix Approach

- Profile the two runs (links 99 vs 100) to pinpoint whether the regression comes from extra guard evaluation, additional recompiles/graph breaks, or changed attention/cache code in transformers 5.4.0, then either restore the skip-guard-eval equivalent for XPU Inductor or patch the WOQ INT4 pattern/fusion so the new transformers modeling (e.g., static cache / attention impl) still hits the fused INT4 path.
- also investigate the reported accuracy delta separately.
