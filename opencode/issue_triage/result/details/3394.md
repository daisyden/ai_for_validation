## Root Cause

GPU segfault (drm_neo.cpp abort, NotPresent PDE write) in XPU SDPA path when seq_len exceeds ~16K
(e.g., 16413, 16611) with fp32 Q/K/V; the oneDNN/SYCL SDPA kernel used via
aten::scaled_dot_product_attention (see torch-xpu-ops
src/ATen/native/transformers/sycl/AttentionKernels.cpp and sdp_utils selection) allocates/addresses
workspace beyond valid GPU virtual memory for large sequences, causing out-of-bounds writes.

## Fix Approach

- Add a sequence-length/workspace-size guard in the XPU SDPA dispatch (sdp_utils / can_use_* checks) to fall back to the math backend when seq_len*head_dim exceeds a safe threshold, and track the underlying fix in oneDNN (JIRA MFDNN-14479) to correctly size and bounds-check memory for very long sequences.
