## Root Cause

test_foreach_max_empty_tensor_error expects _foreach_max on an empty tensor to raise the standard
'max(): Expected reduction dim...' error, but in
torch-xpu-ops/src/ATen/native/xpu/ForeachReduceOp.cpp the empty-tensor TORCH_CHECK is gated only on
the fast route (line 104-116): when can_use_fast_route returns false for e.g. bool/complex/integer
dtypes the code falls through to foreach_tensor_max_slow without performing the empty-numel check,
so the expected error is not raised for those dtypes.

## Fix Approach

- Hoist the numel>0 TORCH_CHECK in foreach_tensor_max_xpu above the can_use_fast_route branch so it runs for both fast and slow paths (matching CUDA/CPU behavior), so that the test passes for all dtypes (bfloat16/bool/complex/float/int/uint8).
