## action_reason

- XPU-native test in torch-xpu-ops/distributed (test_allgather_uneven)
- XCCLTraceTest.test_allgather_uneven exists at test_c10d_xccl.py:1248 but is parametrized with @parametrize('timing_enabled', [False]) at line 1247 -- only the _timing_enabled_False variant is generated; _timing_enabled_True variant does not exist.

## Root Cause

XCCLTraceTest::test_allgather_uneven expects the flight-recorder trace t['entries'] to contain
world_size+1 = 3 entries but only 2 are recorded. The XCCL flight recorder path in
third_party/torch-xpu-ops/src/xccl/ProcessGroupXCCL.cpp (enqueueWorkObject / record path used by
allgather) is missing an entry that NCCL records for the uneven allgather case, so the trace
diverges from the common test expectation.

## Fix Approach

- Apply the fixes already in flight: torch-xpu-ops PR #3332 (align XCCL flight-recorder entry emission with NCCL for uneven allgather) plus daisyden/pytorch#34 on the distributed_2.11 branch (which also disables/adjusts companion XCCL tests).
- Re-run test_allgather_uneven_timing_enabled_{True,False} to confirm 3 entries are emitted.
