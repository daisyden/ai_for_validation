## Root Cause

XCCLTraceTest::test_allgather_uneven_timing_enabled_{False,True} fails at
test/distributed/test_c10d_xccl.py:6003 with assertEqual(len(t['entries']), self.world_size + 1) —
the FlightRecorder for XCCL does not record the expected number of entries for an uneven all_gather
collective. Implementation in third_party/torch-xpu-ops/src/xccl/FlightRecorderXCCL.cpp and
ProcessGroupXCCL.cpp does not emit a trace entry per per-rank send/recv when
allgather_into_tensor_coalesced/uneven decomposes into multiple primitives, so the entry count is
short by world_size compared to NCCL.

## Fix Approach

- Audit ProcessGroupXCCL::allgather (uneven path, src/xccl/ProcessGroupXCCL.cpp) and ensure each underlying p2p/coalesced op increments FlightRecorderXCCL via record() once per logical entry — mirror NCCL's FlightRecorder.cpp behavior.
- Add unit-level trace recording inside coalesced begin/end and verify test_allgather_uneven asserts match world_size+1 entries.
