## Root Cause

Tracking issue enumerating SYCL/oneAPI build warnings across torch-xpu-ops sources (e.g.,
`src/ATen/native/xpu/sycl/MultiTensorApply.h:170` deprecated `at::xpu::HostAlloc`,
`Dequant_int4.cpp:33` unused var, `DilatedMaxPool3d.cpp:259` uninitialized `out_cl_h_stride`,
`BatchNormKernels.cpp:5420` deprecated `c10::value_or_else`, `mkl/SpectralOps.cpp:84` deprecated
oneMKL `set_value`, plus several `Wunused-variable`/`Wdeprecated-declarations` items). Most boxes
have been ticked via PR #1595 and #1828; remaining items are oneMKL deprecation and one IGC
generic-address-space verbose warning. Not a functional bug.

## Fix Approach

- Continue migrating off deprecated APIs: replace `at::xpu::HostAlloc`/`CachingHostAllocator_recordEvent` with `at::getHostAllocator(at::kXPU)->...`, swap `c10::value_or_else` for `std::optional::value_or`, update `oneapi::mkl::dft::descriptor::set_value` to the new overload, and clean unused vars.
- File a follow-up for the remaining IGC generic-address-space warning if it indicates unintended SLM/generic casts.
