## Root Cause

test_qrelu/test_qgelu fail with NotImplementedError: aten::dequantize.self for QuantizedXPU backend.
PyTorch has no QuantizedXPU dispatch implementation — quantize_per_tensor produces a QuantizedXPU
tensor but consumers (dequantize, quantized relu/gelu) only have QuantizedCPU registrations. Per
maintainer comment from @Silv3S, the QuantizedXPU dispatch key will not be added because the legacy
quantized-tensor API is deprecated upstream (pytorch/pytorch#173923).

## Fix Approach

- Do not add QuantizedXPU kernels.
- Follow the approach in torch-xpu-ops PR #3424: either (a) skip test_qrelu and test_qgelu permanently in the XPU port with a comment pointing to the upstream deprecation, or (b) rewrite the test bodies in test/xpu/quantization/core/test_quantized_op_xpu.py to use the new torchao/PT2E quantization flow on plain XPU float tensors instead of legacy QuantizedXPU tensors.
- Update parent issue #2531 to drop these from the cuda→xpu string-fix scope.
