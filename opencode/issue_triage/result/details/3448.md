## Root Cause

torch.compile(backend='eager', fullgraph=True) on a custom autograd.Function whose forward stashes
an aliased intermediate via a Python list side-effect produces all-zero gradient on XPU; eager gives
all-ones. Issue explicitly maps this to upstream Dynamo/AOTAutograd alias-filtering bug fixed by
pytorch/pytorch#180964 (issue 180642). The path is device-agnostic Dynamo handling of aliased side
effects in custom autograd.Functions; XPU only inherits the bug.

## Fix Approach

- Pull in pytorch/pytorch#180964 in the next PyTorch pin update.
- no torch-xpu-ops change required.
- Add an XPU autograd.Function compile test covering aliased intermediate captured via Python side-effect to guard against regression.
