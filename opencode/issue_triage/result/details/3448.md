## Root Cause

torch.compile(backend='eager', fullgraph=True) on XPU returns all-zero gradients for an
autograd.Function that stores an aliased intermediate via Python-list side effect, while eager
returns the correct ones-gradient. This is a Dynamo/AOTAutograd alias-filtering bug (custom
autograd.Function input/output aliasing through side effects) tracked upstream as
pytorch/pytorch#180642 with fix in PR #180964. Since backend='eager' is used, no Triton/SYCL kernel
is involved — the defect is purely in upstream Dynamo's variable tracker for autograd.Function side
effects.
