## Root Cause

Failure originates in transformers/integrations/flash_paged.py:51 calling cache.update() where cache
is None, because the paged-attention path expects a PagedCache object that XPU/torchao integration
does not construct; the immediate Python error is 'NoneType has no attribute update' (the title's
'assert vr is not None' refers to a downstream variable). Root cause is the paged_attention backend
wiring in PyTorch + transformers integration not initializing the kv cache when --attn-type
paged_attention is used on XPU with torchao woq. No XPU SYCL kernel is involved at the failure
point; the issue has been deferred to 2.13 because paged attention is not a 2.12 target.

## Fix Approach

- Implement/enable PagedAttention kv-cache plumbing on XPU: ensure transformers paged_attention_forward receives a non-None PagedCache by hooking up XPU-specific cache class in pytorch's attention backend registry (torch/nn/attention) or guarding the path to fall back to flash/SDPA when cache is None.
- Track jointly with paged-attention enablement effort planned for 2.13.
