## Root Cause

Running Phi-3.5/LLaMA with --attn-type paged_attention on BMG fails inside
transformers/integrations/flash_paged.py:51 at `cache.update(k, v, module.layer_idx, ...)` with
AttributeError: 'NoneType' object has no attribute 'update' (reported as 'assert vr is not None').
The paged_attention_forward path expects a PagedCache instance to be injected via
kwargs['past_key_values'], but the XPU generation setup (run_generation.py with --use-static-cache
and WOQ) does not construct a PagedCache, so cache is None. There is no XPU backend for HF's paged
attention yet, and no torch-xpu-ops kernel registered for flash_attn_varlen_func that
paged_attention_forward delegates to.

## Fix Approach

- Two-part fix: (1) In the client run_generation.py (gpu-models) ensure a transformers PagedCache/DynamicCache is created and passed when --attn-type paged_attention is selected instead of relying on static cache + WOQ path. (2) In torch-xpu-ops, implement an XPU paged-attention kernel (src/ATen/native/xpu/transformers/PagedAttention.cpp) or register a CPU-fallback ATTENTION_BACKEND for XPU so HF's paged_attention_forward can dispatch.
- Stonepia has already deferred to PT 2.13, aligning with ongoing FlashAttention XPU enablement.
