## Root Cause

aten::hspmm / hspmm.out has no SparseXPU dispatch.
aten/src/ATen/native/native_functions.yaml:7776-7784 only registers SparseCPU (hspmm_sparse_cpu) and
SparseCUDA (hspmm_sparse_cuda / hspmm_out_sparse_cuda in
aten/src/ATen/native/sparse/cuda/SparseCUDATensorMath.cu:182). The torch-xpu-ops yaml
(third_party/torch-xpu-ops/yaml/native/native_functions.yaml) has no entry for hspmm; although
SparseTensorMathKernels.cpp:41 includes hspmm_native.h, no XPU kernel/function is implemented, so
calls on SparseXPU raise NotImplementedError from the dispatcher.

## Fix Approach

- Port hspmm_sparse_cuda / hspmm_out_sparse_cuda from SparseCUDATensorMath.cu into torch-xpu-ops/src/ATen/native/sparse/xpu/ using SYCL kernels for the coalesced COO -> hybrid sparse conversion and subsequent matmul (can reuse sparse_mm path already present for XPU).
- Add SparseXPU dispatch entries to the XPU yaml under hspmm and hspmm.out.
- As a short-term unblock, add a CPU fallback registration so the op is at least runnable.
