## Root Cause

aten::hspmm has no SparseXPU registration. Only hspmm_native.h is included in
third_party/torch-xpu-ops/src/ATen/native/sparse/xpu/sycl/SparseTensorMathKernels.cpp (line 41) but
no kernel is registered for the SparseXPU dispatch key, so test_hsmm_xpu_float64 raises
NotImplementedError.

## Fix Approach

- Either implement hspmm for SparseXPU (port the CUDA hspmm kernel, register via TORCH_LIBRARY_IMPL(aten, SparseXPU, ...)) in SparseTensorMathKernels.cpp, or add a CPU fallback registration for aten::hspmm under SparseXPU using the standard fallback wrapper.
- Then unskip test_hsmm_xpu_float64.
