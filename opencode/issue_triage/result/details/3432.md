## action_TBD

- RETRIAGE_PRS | Address comment AR from LuFinch: tune/disable bmm+softmax->micro_sdpa fusion on XPU for tnt_s_patch16_224 (MFDNN-14987) | Address comment AR from jianyizh: gate max_pool2d_with_indices_backward atomic-add lowering off for XPU (regression from pytorch/pytorch#167318) for vgg16 | No action — investigate further

## action_reason

- All 6 PR-discovery vectors (V0 GraphQL, VA timeline, VB body refs, VC title-keyword, VD file-path, VE fix_approach) returned no VERIFIED fix PR. The only PR reference (pytorch/pytorch#167318) is the upstream cause of the vgg16 regression, not a fix. MFDNN-14987 is an internal Intel tracker, unverifiable on GitHub. | MEMBER-assigned owner @LuFinch is tracking the tnt_s_patch16_224 micro_sdpa regression internally; no public fix PR yet. | MEMBER-assigned owner @jianyizh has identified the vgg16 cause and proposed gating the atomic-add lowering for XPU; no fix PR filed yet. | Inception/dla102/phlippe_densenet/hf_Roberta_base/gmlp/xcit/ese_vovnet regressions have no root-cause comment yet and need kernel-level profiling and Inductor commit bisection between 2.11 and 2.12.

## Root Cause

Multiple torchbench/timm models regress >5% on PVC between 2.11 and 2.12 wheels. Maintainer comments
pin the dominant regressions to (a) Inductor's new bmm+softmax→micro_sdpa SDPA fusion path on
tnt_s_patch16_224 (tracked as MFDNN-14987) and (b) pytorch/pytorch#167318 changing
max_pool2d_with_indices_backward from CPU/oneDNN fallback to a Triton atomic-add op which is slow on
XPU (vgg16). Other inception/densenet regressions point to inductor codegen path changes.

## Fix Approach

- Two parallel actions: (1) tune/disable the bmm+softmax→micro_sdpa pattern on XPU (or fix the micro_sdpa Triton kernel perf) for tnt_s_patch16_224 — owner @LuFinch.
- (2) gate the upstream max_pool2d_with_indices_backward atomic-add lowering off for XPU and keep the ATen path.
- For inception_v3/dla102/phlippe_densenet collect kernel-level profiles and bisect upstream Inductor commits between 2.11 and 2.12.
