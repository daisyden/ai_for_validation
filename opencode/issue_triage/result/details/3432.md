* chuanqi129 (MEMBER) requested an acceptance methodology on 2026-04-29 before closing; assignees must respond.

## Root Cause

On PVC with the 2.12 RC wheel, multiple torchbench/timm models regress >5% vs 2.11. Triage in
comments points to several distinct causes: (1) tnt_s_patch16_224 regressed because the inductor
SDPA pattern (bmm+softmax→micro_sdpa) path changed (tracked as MFDNN-14987); (2) vgg16 regressed due
to upstream pytorch/pytorch#167318 changing max_pool2d_with_indices_backward from CPU/oneDNN
fallback to a Triton atomic-add kernel; (3) hf_Roberta_base hits oneDNN gemm regression MFDNN-14993
plus a broadcasted attention-mask stride-1 requirement that disables onednn micro-sdpa (MFDNN-14997
/ HUGGINGFAD-941).

## Fix Approach

- Split into per-model sub-tasks tracked by the assigned owners: pursue oneDNN fixes MFDNN-14987/14993/14997.
- for vgg16, evaluate whether to revert/disable the atomic-add lowering for max_pool2d_with_indices_backward on XPU or add an Inductor lowering choice.
- For hf_Roberta_base, coordinate with HuggingFace transformers to keep the attention mask contiguous on the last dim so the micro-sdpa fast path is selected.
- Establish an acceptance methodology (per chuanqi129's comment) before closing.
