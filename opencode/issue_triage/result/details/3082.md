## Root Cause

Large set of FSDP/HSDP/Replicate MultiThread tests (TestFullyShard*MultiThread,
TestFSDPMiscMultiThread, TestReplicate*) are blocked because
torch.testing._internal.common_distributed MultiThreadedTestCase / spawn_threads_and_init_comms
support for the 'xpu'/xccl backend is missing. ProcessGroupXCCL has no thread-local PG variant
equivalent to ProcessGroupGloo/NCCL needed for in-process multi-rank threaded tests in
torch/distributed/_composable and FSDP suites.

## Fix Approach

- Extend ProcessGroupXCCL (third_party/torch-xpu-ops/src/xccl/ProcessGroupXCCL.cpp) and torch.testing._internal.common_distributed to support the threaded-PG path used by MultiThreadedTestCase (register an XPU backend with init_process_group via 'threaded' fake-PG, or implement a per-thread XCCL group).
- Then enable the listed cases in third_party/torch-xpu-ops/test/xpu/skip_list_dist*.py.
