## Root Cause

Multiple Timm/Torchbench models (convnext_base, jx_nest_base, swin_base_patch4_window7_224,
twins_pcpvt_base, coat_lite_mini, tnt_s_patch16_224, hf_Reformer, timm_regnet) report
'eager_two_runs_differ' on ARC Windows 11 with public driver 32.0.101.8331 — two consecutive eager
runs of the same model produce different outputs, indicating non-determinism (likely RNG state,
atomicAdd ordering, or driver-side reordering on ARC). This is detected by benchmarks/dynamo
eager-vs-eager check before any compile path runs.

## Fix Approach

- Reproduce per-model on ARC Win11.
- bisect to identify the non-deterministic op (suspect dropout/RNG, atomic-based reductions, or scatter_add) and check if seeding/torch.use_deterministic_algorithms stabilizes.
- Compare against PVC Linux to isolate ARC-specific driver/IGC behavior.
- engage driver team if an op is deterministic on PVC but not ARC.
