## Root Cause

pyhpc_turbulent_kinetic_energy fp32 inductor inference fails accuracy (RMSE res 12.15 vs ref 0.61)
on release/2.11 and nightly 2.12.dev, while release/2.10 passes. Reduction-heavy fluid-dynamics
kernel; large RMSE blow-up in inductor-only path indicates a triton-xpu codegen or inductor lowering
bug (likely reduction/atomic/masking) introduced between 2.10 and 2.11. Issue is labelled
'dependency component: Triton' by the reporter.

## Fix Approach

- Dump inductor output_code.py and run kernels in isolation.
- compare fp64 reference.
- Bisect intel-xpu-backend-for-triton commits between 2.10 and 2.11 pins, then inductor lowerings (associative_scan / split_reduction).
- File minimal triton repro to intel-xpu-backend-for-triton.
- as workaround set TORCHINDUCTOR_DETERMINISTIC=1 or disable the offending lowering.
- Accuracy regression on production benchmark → P0.
