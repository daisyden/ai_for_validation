## Root Cause

test_abort_in_destroy_pg expects DistBackendError to be raised on aborted collective, but
ProcessGroupXCCL never propagates an abort/error state because abort() is unimplemented
(ProcessGroupXCCL.hpp:107). Similarly test_individual_send_recv expects FlightRecorder entry state
transition to 'completed' but XCCL trace stops at 'started' since work-completion update path is
missing in ProcessGroupXCCL.cpp.

## Fix Approach

- Wire abort() to set work exception_ptr to DistBackendError and rethrow on wait()/synchronize().
- update FlightRecorderXCCL to mark entries 'completed' from the watchdog upon SYCL event completion.
- Mirror ProcessGroupNCCL::abortCommsFromMap and WorkNCCL::handleException.
