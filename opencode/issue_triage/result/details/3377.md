## Root Cause

Same root cause as #3378: ProcessGroupXCCL::abort is not implemented (ProcessGroupXCCL.hpp:107
throws 'not implemented'), so after the test aborts the PG, subsequent collectives do not observe an
aborted comm and thus DistBackendError is never raised. The related FlightRecorder state never
transitions from 'started' to 'completed' because WorkXCCL completion/state-update (isCompleted /
event finish callback) is missing in the async path in ProcessGroupXCCL.cpp.

## Fix Approach

- Implement ProcessGroupXCCL::abort (see #3378 fix) so post-abort collectives raise DistBackendError, and add FlightRecorder state updates in WorkXCCL::finish/synchronize (transition entries to 'completed' once xcclEndEvent_ is observed ready), aligning with ProcessGroupNCCL's update_state_and_trace flow in FlightRecorderXCCL.cpp.
