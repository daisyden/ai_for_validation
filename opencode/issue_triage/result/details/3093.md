## Root Cause

NestedTensor SDPA path in torch/nested/_internal/sdpa.py:934 raises 'No viable backend for
scaled_dot_product_attention'. The XPU backend has no jagged/nested-tensor SDPA dispatch (only CPU
and CUDA flash/efficient/cudnn paths are wired). Additionally
test/xpu/functorch/test_vmap_xpu.py:109-110 appends CUDNN_ATTENTION to PLATFORM_SPECIFIC_SDPA via
PLATFORM_SUPPORTS_CUDNN_ATTENTION (test_vmap.py:101-102) which is never true for XPU, so
sdpa_kernel(CUDNN_ATTENTION) leaves no viable backend. Comment confirms blocked by issue #3140.

## Fix Approach

- Two-part fix: (1) upstream torch/nested/_internal/sdpa.py: extend backend selection to recognize XPU and route to math/efficient backend on XPU NJT.
- (2) torch-xpu-ops test_vmap_xpu.py: filter CUDNN_ATTENTION out of PLATFORM_SPECIFIC_SDPA when device==xpu, or skip the parametrize entries.
- Track behind #3140 nested-tensor SDPA enablement.
