## Root Cause

jagged/NestedTensor SDPA dispatches through torch/nested/_internal/sdpa.py::_select_sdp_backend
(line 293) which only probes can_use_flash_attention / can_use_efficient_attention; both are
CUDA-only, so for XPU tensors it falls through and raises 'No viable backend for
scaled_dot_product_attention' at sdpa.py:934. XPU has no jagged flash/efficient attention kernel
registered (torch-xpu-ops src/ATen/native/xpu/Attention.cpp only implements dense SDPA), blocking
test_dummy_mha_with_nt, test_sdpa_with_packed_in_proj_xpu_bfloat16 and vmap sdpa_backend3 cases.

## Fix Approach

- Add an XPU branch in torch/nested/_internal/sdpa.py::_select_sdp_backend that either (a) routes to a new jagged-NT SDPA overload implemented in torch-xpu-ops (extend src/ATen/native/xpu/Attention.cpp with a NestedTensor-aware kernel using the existing oneDNN/SDPA fused kernel plus offsets handling), or (b) unbinds the NT into padded dense, calls dense XPU SDPA, and re-wraps (math-fallback style).
- Register SDPBackend.MATH path for jagged XPU as a first, low-risk enablement.
