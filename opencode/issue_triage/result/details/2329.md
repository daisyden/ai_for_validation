## Root Cause

test_flop_counter_op_options0_xpu_float16 in test/inductor/test_inductor_scheduler.py needs
`torch._inductor.utils.get_device_tflops` and `get_dram_gbps`, which only have CUDA/CPU branches;
XPU is unsupported. Original SPIR-V `SPV_INTEL_subgroup_matrix_multiply_accumulate` error is fixed
in Triton 3.6.x, but the underlying feature gap (peak TFLOPS / DRAM bandwidth query for XPU) remains
and depends on intel-xpu-backend-for-triton#5792.
