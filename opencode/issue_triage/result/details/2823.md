## Root Cause

Llama-3.2-1B-Instruct dynamic-INT8 next-token throughput dropped 20% on BMG between nightly 0115 and
0122. Comments confirm root cause: pytorch/pytorch PR #166698 changed SDPA inputs to host scalars,
and oneDNN SDPA on BMG falls into the reference (non-ukernel) path for these shapes which now incurs
an extra H2D memcpy per call (MFDNN-14745). PVC hits the ukernel and is unaffected.

## Fix Approach

- Upstream fix in oneDNN to handle host-scalar inputs in the BMG SDPA reference path without H2D copies (track MFDNN-14745).
- Short term, in the XPU SDPA dispatch wrapper avoid wrapping scale/dropout as host scalars when oneDNN reference path is selected on BMG, or revert the host-scalar change for XPU until oneDNN is fixed.
