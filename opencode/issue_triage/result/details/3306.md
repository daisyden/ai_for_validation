## Root Cause

test/distributed/test_c10d_xccl.py XCCLTraceTest cases (test_fr_record_reset_*) call
torch._C._distributed_c10d._reset_fr_recording_xccl(), but only _reset_fr_recording_nccl is bound in
PyTorch (torch/csrc/distributed/c10d/init.cpp:4249); no XCCL counterpart is registered. The
FlightRecorder reset entry-point was never plumbed for ProcessGroupXCCL, so the Python attribute
lookup raises AttributeError inside the spawned worker (process exit code 10).

## Fix Approach

- Land xpu-ops PR #3332 plus daisyden/pytorch#34: add a _reset_fr_recording_xccl pybind in torch/csrc/distributed/c10d/init.cpp that calls FlightRecorderXCCL::get()->reset_all() (mirroring the NCCL binding), and update test_c10d_xccl.py to import it conditionally.
- Fix tracked by assignee frost-intel.
