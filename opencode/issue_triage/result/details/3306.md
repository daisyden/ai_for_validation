## Root Cause

FlightRecorder reset tests in test/distributed/test_c10d_xccl.py (e.g., line 6213) call
torch._C._distributed_c10d._reset_fr_recording_xccl(), but only the NCCL variant is bound in
torch/csrc/distributed/c10d/init.cpp:4249 ('_reset_fr_recording_nccl'). The XCCL counterpart is not
exposed in pybind, so the attribute lookup fails. The XCCL FlightRecorder reset API needs to be
added in ProcessGroupXCCL and registered in init.cpp.

## Fix Approach

- Implement reset_fr_recording_xccl() in ProcessGroupXCCL (mirroring NCCL FlightRecorder reset path) and add the pybind binding '_reset_fr_recording_xccl' alongside the NCCL one in torch/csrc/distributed/c10d/init.cpp around line 4249.
- Track via referenced PRs #3332 and daisyden/pytorch#34.
- if not landing for 2.11, disable affected XCCLTraceTest cases via skip decorator.
