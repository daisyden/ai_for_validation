## Root Cause

test/distributed/test_c10d_xccl.py calls torch._C._distributed_c10d._reset_fr_recording_xccl(), but
only `_reset_fr_recording_nccl` is bound in pytorch/torch/csrc/distributed/c10d/init.cpp:4249. The
XCCL flight-recorder reset Python binding was never registered, so all 8
fr_record_reset/multiple_resets tests error out at module attribute lookup. Fix is in flight via PR
#3332 plus an upstream PR.

## Fix Approach

- Add a `_reset_fr_recording_xccl` pybind in torch/csrc/distributed/c10d/init.cpp mirroring the NCCL binding and wire it to FlightRecorderXCCL::reset().
- Land via the upstream PyTorch PR (referenced in #3332 / daisyden/pytorch#34).
- no torch-xpu-ops kernel work needed.
