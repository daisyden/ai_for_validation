## Root Cause

ProcessGroupXCCL has no abort() override; only WorkXCCL::abort exists and unconditionally throws
TORCH_CHECK(false, 'ProcessGroupXCCL::WorkXCCL::abort not implemented') at
ProcessGroupXCCL.hpp:107-108. Additionally the wait loop at ProcessGroupXCCL.cpp:277 still has a
'todo: abort comm and exit' comment, so calling pg._abort() never tears down the XCCL communicators
and the test hangs until the 300s watchdog timeout.

## Fix Approach

- Implement ProcessGroupXCCL::abort() (and WorkXCCL::abort) that calls ccl::comm::abort / destroys the cached XCCL communicators, cancels pending work and sets their exception, mirroring ProcessGroupNCCL::abort.
- wire it through the Backend interface so dist.destroy_process_group triggers the abort path cleanly.
