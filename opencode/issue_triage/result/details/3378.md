## Root Cause

test_abort_pg hangs (300s timeout) because ProcessGroupXCCL has no working abort path.
third_party/torch-xpu-ops/src/xccl/ProcessGroupXCCL.hpp:107-108 implements WorkXCCL::abort() as
TORCH_CHECK(false, 'not implemented'), and ProcessGroupXCCL.cpp:277 has a 'todo: abort comm and
exit' placeholder. Without comm abort + watchdog teardown, destroy_process_group blocks waiting for
in-flight collectives.

## Fix Approach

- Implement WorkXCCL::abort() and ProcessGroupXCCL::abort()/shutdown() to call ccl::comm abort/destroy and signal the watchdog thread.
- ensure destroy_process_group joins watchdog quickly.
- Add a heartbeat monitor (ProcessGroupXCCLMonitor) tear-down so abort_pg returns within timeout.
