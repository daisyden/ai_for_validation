## Root Cause

test_abort_pg hangs (300s timeout) on XPU because ProcessGroupXCCL's WorkXCCL::abort() is not
implemented and explicitly throws (third_party/torch-xpu-ops/src/xccl/ProcessGroupXCCL.hpp:107-108:
TORCH_CHECK(false, "ProcessGroupXCCL::WorkXCCL::abort not implemented")). Without a working abort
path, the destroy_process_group / abort_process_group flow cannot terminate in-flight collectives,
so the test process never exits and the harness times out (common_distributed.py:1130
_join_processes). This is a missing-feature gap in XCCL relative to NCCL semantics.

## Fix Approach

- Implement WorkXCCL::abort() in third_party/torch-xpu-ops/src/xccl/ProcessGroupXCCL.cpp/hpp to flush/cancel the underlying SYCL queue and CCL communicator (similar to NCCL's ncclCommAbort path).
- ensure ProcessGroupXCCL::abort() / shutdown() drives the per-work abort and joins the watchdog/heartbeat thread without hanging.
- Add a fast-path that signals the work as completed-with-error so abort_pg can return promptly.
- After landing, rerun pytest -v test/distributed/test_c10d_xccl.py::ProcessGroupXCCLGroupTest::test_abort_pg.
