## Root Cause

hf_Reformer fp16 inference on BMG regressed from 505ms to 623ms, bisected to upstream PyTorch commit
e2917a38a (Inductor codegen change) which interacts poorly with the Intel GPU Triton backend / IGC
compiler. The single comment 'IGC-14276' confirms the regression has been traced to an Intel
Graphics Compiler bug filed against IGC, not to an ATen kernel. There is no specific SYCL-kernel
file:line in torch-xpu-ops; the slow path lives in the Triton-compiled softmax/attention kernels
generated by torch/_inductor.

## Fix Approach

- Track the IGC fix (IGC-14276) and bump the minimum IGC / compute-runtime requirement in torch-xpu-ops CI once resolved.
- In the meantime, add a Triton-heuristics workaround in torch/_inductor/runtime/triton_heuristics.py for the BMG device (e.g. force smaller XBLOCK / fewer warps for the affected persistent-reduction shape) and retest hf_Reformer bs=128 fp16.
- Close when OOB latency returns under the 505 ms baseline.
