## Root Cause

torch/amp/autocast_mode.py:248 hardcodes device_supported_dtypes = [torch.bfloat16, torch.float16]
and the branch at line 297 warns `XPU Autocast only supports dtypes of torch.bfloat16, torch.float16
currently` and disables autocast when fp32 is passed. CUDA has a special-case branch at lines
283-296 that accepts fp32 as a no-op, but XPU has no equivalent, so using
@autocast(device_type='xpu', dtype=torch.float32) in libraries like diffusers silently disables
autocast. This is a feature gap, not a kernel bug.

## Fix Approach

- Add an XPU branch parallel to the CUDA one in torch/amp/autocast_mode.py around lines 283-297 (or extend device_supported_dtypes for 'xpu' to include torch.float32) so that fp32 is accepted and treated as a no-op / upcast target, matching CUDA semantics.
- Verify that all XPU ATen ops in the autocast policy (third_party/torch-xpu-ops/src/ATen/native/xpu/*) and the `to` cast kernel accept fp32 destinations.
- Add a unit test in test/xpu/test_autocast_xpu.py covering dtype=torch.float32 that confirms no warning is raised and autocast is enabled.
