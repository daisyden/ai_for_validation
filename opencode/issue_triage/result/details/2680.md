## Root Cause

torch.amp.autocast_mode.autocast on device='xpu' with dtype=torch.float32 emits 'In XPU autocast,
but the target dtype is not supported' and disables autocast (torch/amp/autocast_mode.py:297-305).
The device_supported_dtypes list at line 248 only includes bfloat16/float16, while CUDA's autocast
accepts fp32 effectively as a no-op. Diffusers' transformer_kandinsky.py passes torch.float32 into
autocast and hits this gating, breaking XPU parity.

## Fix Approach

- In torch/amp/autocast_mode.py, allow torch.float32 through autocast as a no-op for XPU (matching CUDA behavior): either add float32 to device_supported_dtypes for XPU, or short-circuit before the gating check when fast_dtype==torch.float32 by simply not enabling cast.
- Fix is in upstream pytorch/pytorch.
