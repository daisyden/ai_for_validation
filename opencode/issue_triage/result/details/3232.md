## Root Cause

test_attention_shard_without_cp in test/distributed/tensor/test_attention.py:922 expects
assertNotEqual sharding behavior, but on XPU the SDPA efficient-attention path is unsupported and
silently falls back to the math backend ('SDPA Memory Efficient Attention backend is not supported
on XPU, falling back to math backend'), so outputs match and the AssertionError is not raised. The
capability check originates from torch/testing/_internal/common_cuda.py:98
(PLATFORM_SUPPORTS_MEM_EFF_ATTENTION).
