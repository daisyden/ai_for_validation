## Root Cause

test_attention_shard_without_cp (test/distributed/tensor/test_attention.py:922) expects that
sharding the attention on dim=2 is invalid and uses assertNotEqual (i.e. expects an assertion
failure). The test targets the efficient-attention backend. XPU does not implement SDPA
memory-efficient attention, so the SDPA dispatcher emits 'SDPA Memory Efficient Attention backend is
not supported on XPU, falling back to math backend'; the math backend accepts Shard(dim=2) and
produces equal output, so the expected failure does not occur. The support gate in
torch/testing/_internal/common_cuda.py:~98 is not restrictive enough for XPU.

## Fix Approach

- Skip the efficient-attention branch of test_attention_shard_without_cp on XPU (the path already agreed in the issue comments) by gating on PLATFORM_SUPPORTS_MEM_EFF_ATTENTION/xpu, or add a proper XPU check in common_cuda.py so the efficient backend is reported unsupported on XPU.
- Long-term: implement mem-efficient SDPA backend on XPU.
