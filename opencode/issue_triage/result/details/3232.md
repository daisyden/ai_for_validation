## Root Cause

test_attention_shard_without_cp (test/distributed/tensor/test_attention.py:893, assertNotEqual at
line 922) expects efficient-attention SDPA to produce a different result than the reference, but on
XPU it silently falls back to math backend ('SDPA Memory Efficient Attention backend is not
supported on XPU, falling back to math backend'), so results match the reference and assertNotEqual
fails. The capability gate in torch/testing/_internal/common_cuda.py:98
(PLATFORM_SUPPORTS_MEM_EFF_ATTENTION) does not exclude XPU, so the test path is taken even though
XPU has no mem-eff kernel.

## Fix Approach

- Two options: (1) preferred—update torch/testing/_internal/common_cuda.py to exclude XPU from PLATFORM_SUPPORTS_MEM_EFF_ATTENTION (or add an explicit @skipIfXPU on the DTensor attention shard tests in test/distributed/tensor/test_attention.py).
- (2) interim—skip these cases via torch-xpu-ops skip_list_dist.py as suggested in the comments.
- Long-term, implement an XPU efficient-attention kernel in torch-xpu-ops/src/ATen/native/transformers/sycl/ to remove the math fallback.
