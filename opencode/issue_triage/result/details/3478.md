## Root Cause

CppThreadTestXPU.test_with/without_enable_profiler_in_child_thread_xpu fail at assert_text
(test_cpp_thread_xpu.py:269) with 'baseline event count incorrect' — the chrome-trace event count
emitted from a C++ child thread under torch.profiler does not match the XPU expectation. This
indicates the libkineto XPU activity profiler does not record events from non-main C++ threads with
the same fidelity as CUDA/CPU (likely missing per-thread registration or the child-thread
profiler-enable propagation is incorrect on XPU). The test is the upstream
profiler/test_cpp_thread.py mirrored for XPU and has no XPU-specific overrides.
