## Root Cause

test_cpp_thread_xpu.py:269 asserts a baseline event count from the C++-thread profiler probe; on XPU
the event count differs ('baseline event count incorrect'). The cpp-thread profiler test expects a
fixed number of events recorded when profiler is enabled/disabled in child threads, but XPU's
profiler path records a different number of host/device events than CPU/CUDA. Logic lives in
upstream torch profiler (PyTorchThreadLocalState / kineto integration), not in torch-xpu-ops
kernels.

## Fix Approach

- Trace which extra/missing events appear on XPU with KINETO logging, then either fix the XPU profiler binding upstream so child-thread event counts match, or relax the test's expected count to be device-aware.
- Upstream PR against torch/csrc/autograd/profiler_kineto.cpp likely needed.
