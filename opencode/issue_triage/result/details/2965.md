## Root Cause

Umbrella of random UT failures for WW10 2026. (1) test_dispatch_meta_outplace_to_sparse_xpu_bfloat16
in test_meta_xpu.py is a recurring sparse-meta dispatch mismatch (history: #2188, #2618, #2675,
#2897, #2964). (2) test_jvpvjp_nn_functional_conv_transpose3d_xpu_float32 in
functorch/test_ops_xpu.py: conv_transpose3d backward through oneDNN is non-deterministic producing
cross-run numeric drift (BBBela confirmed on #2988 via deterministic-mode experiment). (3)
test_linalg_solve_triangular_large_xpu_complex128 in test_linalg_xpu.py: oneMKL triangular-solve
complex128 accuracy drifts above rtol on large matrices, passes/fails between runs. All three are
precision/non-determinism not correctness.

## Fix Approach

- Split remaining cases out of this umbrella (conv3d/max_pool already moved to #3103 and #2676).
- For conv_transpose3d jvpvjp: either raise tolerance in the OpInfo (tol overrides in third_party/torch-xpu-ops/test/xpu/xpu_test_utils.py) or enable deterministic oneDNN algo for convolution backward in aten/src/ATen/native/mkldnn/xpu/Conv.cpp.
- For linalg_solve_triangular_large complex128: raise rtol/atol for XPU in common_methods_invocations OpInfo override.
- For test_dispatch_meta_outplace_to_sparse_bfloat16: add XPU entry to the meta-dispatch skip list in test_meta_xpu.py (sparse bfloat16 kernels are not implemented on XPU, CPU fallback dispatch check is the correct behavior).
