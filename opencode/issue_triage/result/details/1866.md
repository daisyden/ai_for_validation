## Root Cause

vision_maskrcnn amp_bf16/fp16 inference reports fail_accuracy under inductor on BMG (Win/Linux). Per
@BartoszKokoszko investigation, the issue is also reproducible on CUDA: detection scores tie within
fp tolerance, so compile vs eager output ordering of (boxes/labels/scores/masks) differs, breaking
the elementwise comparison in benchmarks/dynamo/torchbench.py around line 304. Not an XPU kernel
correctness bug.

## Fix Approach

- Use IoU-based comparison for vision detection models in benchmarks/dynamo accuracy harness (draft already proposed by Bartosz) upstream in pytorch/pytorch.
- once landed, update XPU expected-accuracy list.
- No torch-xpu-ops kernel fix needed.
