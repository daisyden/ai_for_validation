## Root Cause

test_state_dict_with_cuda_params_Muon_xpu_float32 fails by ~2e-5 abs / ~1.9e-5 rel which marginally
exceeds the float32 tolerance (1e-5 / 1.3e-6). Muon optimizer (torch/optim/_muon.py) uses
Newton-Schulz orthogonalization composed of XPU foreach/matmul ops; tiny FMA/order-of-reduction
differences vs CUDA reference accumulate. This is an upstream test tolerance issue, not a functional
defect in any specific torch-xpu-ops SYCL kernel.

## Fix Approach

- Bump tolerance for Muon in test_optim (e.g., add XPU-specific atol/rtol overrides or relax the float32 tolerance for state_dict cross-device test) upstream.
- alternatively raise accumulator dtype to float in the Newton-Schulz inner matmuls.
- Re-enable test once tolerance lands.
