## Root Cause

torch.logcumsumexp on complex64 input returns NaN at the last index. In LogcumsumexpKernel.cpp the
dispatch at line 141 uses AT_DISPATCH_FLOATING_AND_COMPLEX_TYPES_AND2 with init = -infinity of
scalar_t (line 148); for complex types scalar_t is c10::complex and the -inf-imag combination
together with min/max selection in _logcumsumexp_minmax (lines 27-41, comparing only real parts)
causes NaN propagation when intermediate (min - max) hits inf-inf. CUDA uses opmath_t inside the
scan and a different init handling.

## Fix Approach

- In launch_logcumsumexp_kernel use opmath_t for the scan accumulator and set init to {-inf, 0} (matching CUDA's LogcumsumexpKernel.cu).
- Alternatively guard _log_add_exp_helper for complex when min and max share -inf real part (already partially handled at lines 106-112) but ensure imag is zeroed.
- Add the test_logcumsumexp_complex_xpu_complex64 case after fix.
