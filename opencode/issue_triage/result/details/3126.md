## Root Cause

Two NestedTensor-related gaps: (1) Four failure-mode tests (test_nested_fails_on_padding_head_dim,
test_flash_fail_fp32, test_fused_kernels_nested_broadcasting_requires_grad_failure,
test_flash_attention_fail_with_non_square_causal_attention) expect specific messages like 'For
NestedTensor inputs, Flash attention requires...' or 'Both fused kernels do not support training
with broadcasted NT inputs', but XPU emits the generic 'Flash attention kernel not used because:'
warning from aten/src/ATen/native/mkldnn/xpu/Attention.cpp:241 because the XPU fused-kernel selector
lacks the CUDA-specific input-validation messages. (2)
test_flash_attention_vs_math_ref_grads_nestedtensor fails with NotImplementedError:
aten::_scaled_dot_product_flash_attention has no 'NestedTensorXPU' dispatch key (only XPU dense is
registered), so nested-tensor SDPA is entirely unsupported on XPU.

## Fix Approach

- Two-part fix: (a) In Attention.cpp's can_use_flash_attention / can_use_overrideable_attention debug-print logic, add NestedTensor-specific constraint messages matching the CUDA text ('For NestedTensor inputs, Flash attention requires...', fp32 rejection, non-square causal, broadcasted NT training) so the four failure-mode tests match expected regex. (b) Register _scaled_dot_product_flash_attention (and backward) for the NestedTensorXPU dispatch key, either by routing to the dense XPU kernel via NestedTensor-aware wrapping or by adding a nested_tensor/xpu implementation.
- Short-term: skip test_flash_attention_vs_math_ref_grads_nestedtensor variants on XPU.
