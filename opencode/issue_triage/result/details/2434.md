## Root Cause

Of the originally listed cases only TestFakeQuantizeOpsXPU.test_fq_module_per_tensor_xpu remains
failing (pruning/pooling cases now pass). For very large inputs (~4.98e14) the XPU fake-quantize
per-tensor SYCL kernel rounds the quant step differently from the Python reference (kernel returns
[127., 0.] vs ref [126., -0.]) — same discrepancy reproduces on CPU, BMG, PVC. Root cause is
rounding-mode/precision in the fake_quantize_per_tensor SYCL kernel under
third_party/torch-xpu-ops/src/ATen/native/xpu/sycl/FakeQuantizeKernels.cpp vs the Python reference
using extreme magnitudes generated by hypothesis.
