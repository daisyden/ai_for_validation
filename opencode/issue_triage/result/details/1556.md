## Root Cause

test_tp_examples.py::test_transformer_req_grad_seq_parallel_float32_thaw_norm__output routes through
aten._scaled_dot_product_fused_attention_overrideable.default because XPU registers only the
overrideable SDPA backend for training (no efficient/flash kernel). DTensor's sharding propagation
table (torch/distributed/tensor/_ops/) has no strategy registered for this op, so DTensor raises
NotImplementedError when running TP. The UT setting requires_grad_(False) on attention Q/K/V weights
forces the fused-attention path. Fix requires both a real fused SDPA kernel (planned via oneDNN) and
a DTensor sharding rule for the overrideable op.

## Fix Approach

- 1) Land the oneDNN-based fused SDPA kernel for XPU so the generic overrideable op is backed by a real forward+backward implementation. 2) In torch/distributed/tensor/_ops register a sharding strategy for aten._scaled_dot_product_fused_attention_overrideable.default mirroring CUDA's _scaled_dot_product_efficient_attention rule (shard on batch/head dim, replicate mask). 3) Re-enable the skipped test in third_party/torch-xpu-ops/test/xpu/skip_list_distributed*.py.
- Reassign to the kernel team as requested by kpjeeja.
