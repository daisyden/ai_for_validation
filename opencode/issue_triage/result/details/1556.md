## Root Cause

DTensor sharding propagation has no rule registered for
aten._scaled_dot_product_fused_attention_overrideable.default. XPU's SDPA path (oneDNN-backed)
decomposes to this overrideable op, but torch/distributed/tensor/_ops/ contains no
register_op_strategy for it (only _scaled_dot_product_efficient_attention/_flash_attention have
strategies). On CUDA the model uses _flash_attention/_efficient_attention which have sharding
strategies registered, so the same TP test passes; XPU dispatches into
_scaled_dot_product_fused_attention_overrideable which is unhandled.

## Fix Approach

- Register a sharding strategy for aten._scaled_dot_product_fused_attention_overrideable(.default and _backward) in torch/distributed/tensor/_ops/_matrix_ops.py (or a new attention ops module) mirroring the strategy used for _scaled_dot_product_efficient_attention.
- Land the registration upstream in pytorch/pytorch since the operator is shared by CPU/XPU overrideable backends.
