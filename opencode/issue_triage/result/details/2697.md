## Root Cause

test_effn_attn_bias_padding_misaligned (test/inductor/test_cuda_repro.py:250-288) is a CUDA-specific
regression check: it forces SDPBackend.EFFICIENT_ATTENTION and asserts the generated wrapper
contains 'buf0 = ... , 0, ...' – the padded-bias expanded-stride marker produced by the CUDA-path
lowering of aten._scaled_dot_product_efficient_attention. On XPU, efficient_attention is not
registered, so SDPA is decomposed into bmm+softmax+bmm and the wrapper instead emits
empty_strided_xpu without the padded stride pattern, causing FileCheck to fail at line 282.

## Fix Approach

- Short term: skip this test for XPU (add @requires_cuda or xfail_if_xpu) since the codegen signature it checks is CUDA-EFFICIENT_ATTENTION-specific.
- Long term: add XPU support for _scaled_dot_product_efficient_attention (or have XPU's SDPA lowering reuse the same padded-bias expansion pass) so that the EFFICIENT_ATTENTION backend produces the expected wrapper code on XPU.
