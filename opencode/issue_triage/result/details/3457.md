## Root Cause

Two regression clusters on BMG B70 in 2.12: (1) 5 huggingface amp_fp16 inference models output bf16
instead of fp16 (autocast dtype handling regression on XPU inference path) - mengfei25 verified PR
pytorch/pytorch#180309 fixes all 5; same root cause as #3290. (2) 3 timm batchnorm-related training
RMSE failures (adv_inception_v3, fbnetv3_b x2) - fbnetv3_b is tracked in
intel-xpu-backend-for-triton#6489 (triton). The dtype-mismatch cluster lives in upstream pytorch
autocast/inductor.

## Fix Approach

- Cherry-pick pytorch/pytorch#180309 into 2.12 release branch to fix the 5 huggingface fp16->bf16 mismatches.
- For timm fbnetv3_b track triton issue #6489.
- For adv_inception_v3 batchnorm RMSE on amp_bf16 training, investigate batch_norm backward accumulator dtype on XPU.
