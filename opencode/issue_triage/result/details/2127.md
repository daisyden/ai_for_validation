## Root Cause

This is a meta enhancement filed after PR #2073 fixed an out-of-bounds bug in an error-handling code
path in third_party/torch-xpu-ops/src/ATen/native/xpu/mkl/BatchLinearAlgebra.cpp that was never
exercised by existing tests. Current CI tracks line coverage only via pytest plus skip_list gating,
which cannot surface dead error branches (catch blocks, MKL failure rethrows, TORCH_CHECK negative
paths). No tooling or policy currently flags such uncovered branches.

## Fix Approach

- Integrate branch/path coverage instrumentation into the xpu-ops CI: enable --coverage (gcov) on the C++ build and pytest-cov --branch on the Python side, publishing reports per nightly run.
- Add a targeted error-path audit pass over src/ATen/native/xpu/mkl and src/ATen/native/xpu (exceptions, TORCH_CHECK, fallback branches) and add negative-input unit tests.
- Consider a periodic 'run skipped UTs' job to surface regressions in dormant code.
