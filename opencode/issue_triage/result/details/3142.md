## Root Cause

FlashAttention SYCL-TLA forward kernel
(third_party/torch-xpu-ops/src/ATen/native/transformers/xpu/flash_attn/sycltla/mha_fwd.cpp) uses
sycl_ext_oneapi_work_group_scratch_memory, which the SYCL Graph extension explicitly forbids inside
graph nodes (synchronous 'invalid' exception per intel/llvm sycl_ext_oneapi_graph.asciidoc:2351).
Therefore every fused_kernel0 cudagraph SDPA test throws 'sycl_ext_oneapi_work_group_scratch_memory
feature is not yet available for use with the SYCL Graph extension'. Tracked compiler-side as
CMPLRLLVM-72057, planned for oneAPI 26.0.

## Fix Approach

- External oneAPI compiler dependency — wait for oneAPI 2026.0 / CMPLRLLVM-72057 to lift the work_group_scratch_memory + SYCL Graph restriction.
- Until then keep the tests skipped and, if needed, fall back to math backend when a SYCL Graph (cudagraph) capture context is detected in the FlashAttention dispatcher.
