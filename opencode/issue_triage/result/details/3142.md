## Root Cause

test_fused_attention_vs_math_ref_grads_cudagraph with fused_kernel0 (flash attention) builds a SYCL
Graph around the FlashAttention forward kernel. The sycltla FlashAttention kernels
(third_party/torch-xpu-ops/src/ATen/native/transformers/xpu/flash_attn/sycltla/mha_fwd.cpp) rely on
the sycl_ext_oneapi_work_group_scratch_memory extension, and the SYCL Graph spec explicitly states
'The new property defined by sycl_ext_oneapi_work_group_scratch_memory cannot be used in graph
nodes. A synchronous exception will be thrown with error code invalid if a user tries to add such a
kernel'. Therefore the whole test family throws at cudagraph capture time. Tracked in oneAPI as
CMPLRLLVM-72057.

## Fix Approach

- Pure oneAPI/driver-side fix: wait for oneAPI 2026.0 which lifts the graph restriction on work_group_scratch_memory (per LuFinch/daisyden).
- Meanwhile, keep these cudagraph SDPA cases skipped in test/xpu/skip_list_common.py with a reference to CMPLRLLVM-72057, and optionally teach the torch.cuda.graph replacement for XPU to fall back to the math SDPA backend when the compiled stream has an active SYCL-Graph capture so that user code does not crash.
