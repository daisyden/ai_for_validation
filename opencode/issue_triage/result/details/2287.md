## Root Cause

test_python_ref__refs_logspace_tensor_overload_xpu_* fails with 'no _refs support for
aten.copy.default'. The stack shows torch/_refs/__init__.py:5486 (logspace) calling torch.linspace
which inside _refs context (torch/_prims/context.py:155) hits torch/_refs/__init__.py:5409 that
invokes torch.ops.aten.copy.default -- a non-prim aten op not registered in _refs, so
TorchRefsMode(strict=True) rejects it. This is an upstream PyTorch decomposition bug, not
XPU-specific; CUDA is unaffected only because its OpInfo lists a smaller dtype set for
logspace_tensor_overload (bfloat16/float16/complex/... aren't tested). Assignee yucai-intel has
landed upstream fixes (pytorch/pytorch#169565 and #178734).

## Fix Approach

- Land upstream fix pytorch/pytorch PR #169565 (replace the bare aten.copy.default call in _refs.linspace around torch/_refs/__init__.py:5409 with a _refs-compatible implementation such as prims.copy_to or an elementwise assignment, matching CUDA behavior) and the follow-up #178734 to align logspace_tensor_overload OpInfo with CUDA.
- Then unskip the listed test_python_ref__refs_logspace_tensor_overload_xpu_* cases in third_party/torch-xpu-ops/test/xpu/skip_list_common.py.
