## Root Cause

test_python_ref__refs_logspace_tensor_overload fails with 'no _refs support for aten.copy.default'.
The error originates in upstream PyTorch _refs (torch/_refs/__init__.py:5409 calls
torch.ops.aten.copy.default inside linspace/logspace ref) but TorchRefsMode strict=True raises in
torch/_prims/context.py:157. This is a generic _refs/_prims gap unrelated to XPU; same failure would
occur on CUDA. Test pulled in via upstream PR pytorch/pytorch#161246.
