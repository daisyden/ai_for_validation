## Root Cause

On B70 (BMG) the XPU profiler trace shows an abnormally long copy kernel that overlaps other
activities; perfetto does not render it because it is considered invalid/overlapping. PVC does not
reproduce it and a similar BMG issue was fixed before. The assignee (aostrowski-hbn) confirmed via
internal PTI-384 that the wrong timestamp originates in the Level-Zero/GPU driver; PTI 0.16 already
propagates the raw data, so the defect is below torch-xpu-ops (no file:line in this repo). Surface
in torch: kineto/XPU profiler path under torch/csrc/profiler/ that consumes PTI events.

## Fix Approach

- Blocked on the Intel GPU driver fix tracked as PTI-384.
- Once the driver delivers corrected kernel timestamps, re-validate the trace with the updated BMG driver + PTI >= 0.16.
- Meanwhile add a workaround in the XPU profiler post-processing to clamp/skip copy events whose duration exceeds a sanity threshold and log a warning, and add a BMG profiler smoke test to CI to catch regressions.
