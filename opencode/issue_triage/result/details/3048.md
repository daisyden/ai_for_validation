## Root Cause

On B70 (BMG) the profiler trace shows an erroneous long-duration memcpy entry that overlaps other
ops; PVC is unaffected and a similar bug was previously fixed on BMG. PTI 0.16 is already in use, so
the issue is not in torch-xpu-ops profiler glue (third_party/torch-xpu-ops/src/ATen/xpu/Profiler.* /
pytorch torch/csrc/profiler/stubs/xpu.cpp) but in the underlying L0/PTI driver layer reporting
incorrect memcpy timestamps on B70. Owner aostrowski-hbn confirmed it is tracked internally
(PTI-384) and waiting on a driver team fix.

## Fix Approach

- No torch-xpu-ops code change required.
- Track internal PTI-384 / driver fix.
- once a fixed level-zero / PTI release is available, bump the minimum supported driver/PTI version in torch-xpu-ops CI manifest and re-validate the user's profiling reproducer on B70.
- Document the B70 profiler caveat in the README until then.
