## Root Cause

INT4 RTN flex-attention decode kernel regressed ~5% E2E on BMG/B580 between triton 3.6 -> 3.7.
Investigation in comments shows two root causes: (1) Inductor flex_attention triton template grid
ordering changed from {8;4;1} to {1;8;4} raising register pressure, (2) triton 3.7 register spilling
jumped from 3072 to 11392 bytes/thread (intel-xpu-backend-for-triton#6625) increasing per-call
latency from 15.8us to 24.2us. Only B580 reproduces because PVC/B60 have larger register file
budgets.

## Fix Approach

- Primary fix: track upstream triton issue intel/intel-xpu-backend-for-triton#6625 to reduce register spill in 3.7.
- Secondary: tune Inductor flex_attention triton template kernel-config heuristic in pytorch/torch/_inductor/kernel/flex_attention.py to keep {8;4;1}-style ordering on BMG (or add a BMG-specific autotune choice).
- No torch-xpu-ops kernel change needed.
