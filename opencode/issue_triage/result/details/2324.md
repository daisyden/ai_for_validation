## Root Cause

Feature request (PT2.13) for FP8 convolution support in TorchAO on XPU, tracking pytorch/ao PR
#3315. There is currently no FP8 conv kernel registered for XPU in torch-xpu-ops (no `_scaled_conv*`
symbol found under third_party/torch-xpu-ops/src) and oneDNN convolution primitives with FP8 scales
are not yet exposed through the ATen XPU bindings.

## Fix Approach

- Add an FP8 convolution op in torch-xpu-ops (e.g. `src/ATen/native/xpu/ScaledConv.cpp`) backed by oneDNN convolution with per-tensor/per-channel FP8 scales, register a `_scaled_conv2d` (or equivalent) XPU dispatch in native_functions.yaml, and enable the AO conv FP8 UTs from PR #3315 on XPU.
