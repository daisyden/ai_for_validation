## Root Cause

ProcessGroupXCCL Python binding does not expose _set_default_timeout, while ProcessGroupNCCL binds
it at torch/csrc/distributed/c10d/init.cpp:3302 and :3405.
test_dynamo_distributed.py::test_asymmetric_compilation calls backend._set_default_timeout(...)
which fails with AttributeError on XCCL. The XCCL Register.cpp / ProcessGroupXCCL.cpp in
third_party/torch-xpu-ops/src/xccl/ has not added the equivalent pybind method.

## Fix Approach

- Add _set_default_timeout method on ProcessGroupXCCL (mirror NCCL implementation in ProcessGroupNCCL.cpp) and expose it via py::class_ binding in third_party/torch-xpu-ops/src/xccl/Register.cpp similar to init.cpp:3302.
- then re-enable the two test_asymmetric_compilation cases.
