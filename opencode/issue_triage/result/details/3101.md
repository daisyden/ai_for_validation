## Root Cause

torch/csrc/distributed/c10d/init.cpp:3660 registers the ProcessGroupXCCL pybind class but does NOT
expose the '_set_default_timeout' method, unlike ProcessGroupGloo (same file line 3302) and
ProcessGroupNCCL. The underlying C++ method exists - ProcessGroupXCCL::setTimeout is defined at
third_party/torch-xpu-ops/src/xccl/ProcessGroupXCCL.hpp:197 - so tests calling
backend._set_default_timeout (test_dynamo_distributed.py::test_asymmetric_compilation line 1367) hit
AttributeError.

## Fix Approach

- Add a .def('_set_default_timeout', &::c10d::ProcessGroupXCCL::setTimeout, py::arg('timeout'), py::call_guard<py::gil_scoped_release>()) binding on the processGroupXCCL class in torch/csrc/distributed/c10d/init.cpp around line 3660 (mirror the Gloo/NCCL pattern at lines 3301-3305).
- Trivial one-line PyTorch patch.
- no torch-xpu-ops change required.
