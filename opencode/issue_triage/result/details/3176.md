## Root Cause

torch.sparse._triton_ops.check_device hardcodes the device type to 'cuda'
(torch/sparse/_triton_ops.py:32-36: `t.device.type == "cuda"`), so the Triton BSR SDPA kernel
rejects XPU tensors with 'all inputs are expected to be on the same GPU device'. The test
test_triton_scaled_dot_product_attention in test_sparse_csr_xpu.py:5013 passes valid xpu tensors
that still fail this check. Upstream PyTorch fix was landed (accept xpu as well), and the nightly
run on 2026-04-17 shows the cases now pass; the issue is already CLOSED.

## Fix Approach

- No code change needed — issue already resolved by the upstream change broadening check_device to accept xpu.
- Remove the skip entries from test/xpu/skip_list_common.py and close the tracking issue if not already done.
