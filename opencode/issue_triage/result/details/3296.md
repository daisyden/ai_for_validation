## Root Cause

test_comprehensive_stft_xpu_float16 fails with ~0.0002 abs / 0.0046 rel mismatch on 11/100 elements
(tolerance 1e-5 abs / 1e-3 rel). The XPU MKL FFT path in
third_party/torch-xpu-ops/src/ATen/native/xpu/mkl/SpectralOps.cpp:323-329 (promote_fft_input)
upcasts Half->Float for the MKL DFT, runs in fp32, then casts back to ComplexHalf at lines 381-382 /
461-462 / 550-551, so the result has fp16 round-trip error larger than CPU's tighter tolerances. The
opinfo tolerance for stft fp16 is too tight for the MKL/fp32-roundtrip pattern used on XPU.

## Fix Approach

- Either widen the stft fp16 tolerance for XPU in OpInfo (atol~5e-4, rtol~5e-3) similar to existing CUDA overrides, or skip stft fp16 in test/xpu/skip_list_common.
- alternative is to align numerics by routing stft fp16 through the same fp32 reference path on CPU baseline.
- Track in ut_upstream and submit OpInfo tolerance bump PR mirroring pytorch/pytorch#179550.
