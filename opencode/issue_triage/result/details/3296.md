## Root Cause

test_comprehensive_stft_xpu_float16 fails with 11/100 mismatched elements (max abs diff 2.44e-4)
against fp32 reference. stft on XPU dispatches to the oneMKL-backed _fft_r2c implementation in
third_party/torch-xpu-ops/src/ATen/native/xpu/mkl/SpectralOps.cpp; for fp16 inputs the kernel
performs internal accumulation/twiddle multiplies in a precision that differs slightly from CPU
reference, producing diffs above the default 1e-5/1e-3 tolerance used by the inductor opinfo
harness.

## Fix Approach

- Either (a) bump the per-op tolerance for stft fp16 in the XPU opinfo skip/xfail/tolerance list (e.g., test/inductor/test_torchinductor_opinfo.py inductor_expected_failures or torch-xpu-ops test/xpu skiplist) to ~5e-4 abs / 5e-3 rel, or (b) promote fp16 input to fp32 inside _fft_r2c_mkl in torch-xpu-ops/src/ATen/native/xpu/mkl/SpectralOps.cpp before the MKL DFT call and cast back, matching CPU/CUDA reference precision.
