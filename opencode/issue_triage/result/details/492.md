## Root Cause

timm_efficientdet in pytorch/benchmark hard-codes CUDA:
torchbenchmark/models/timm_efficientdet/__init__.py:55 raises NotImplementedError for non-CUDA
devices, and the upstream rwightman/efficientdet-pytorch loader also uses hard-coded `cuda` strings.
This is not a torch-xpu-ops code defect but a benchmark harness gap; the XPU runner can't even load
the model. PR https://github.com/pytorch/benchmark/pull/2374 has stalled in review.

## Fix Approach

- Drive pytorch/benchmark PR #2374 to landing (generalize device string, replace `.cuda()` with `.to(device)`) and upstream similar changes to rwightman/efficientdet-pytorch (effdet/data/loader.py).
- After merge, re-enable timm_efficientdet in the XPU torchbench accuracy suite and verify across amp_bf16/amp_fp16/fp16/fp32/bf16 training configs.
