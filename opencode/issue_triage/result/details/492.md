## Root Cause

torchbench timm_efficientdet raises NotImplementedError('The original model code forces the use of
CUDA.') from benchmark/torchbenchmark/models/timm_efficientdet/__init__.py:55, and the underlying
rwightman/efficientdet-pytorch (effdet/data/loader.py) hardcodes .cuda() calls in its DataLoader.
The model never reaches any XPU kernel; failure is purely in the upstream torchbench harness /
third-party effdet code. Tracking PR pytorch/benchmark#2374 was opened to generalize the device.

## Fix Approach

- Land pytorch/benchmark PR #2374 (and follow-up effdet patch) to replace hardcoded 'cuda' with the harness device argument in timm_efficientdet __init__ and effdet/data/loader.py.
- After merge, refresh the torchbench pin and re-enable timm_efficientdet in the XPU torchbench accuracy list.
- No torch-xpu-ops code change.
