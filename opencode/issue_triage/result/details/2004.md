## Root Cause

The 12 failing test_sharded_tensor.py cases rely on torch.distributed.rpc with the tensorpipe SHM
transport, which is not supported on XPU. The failure 'eof ...
tensorpipe/transport/shm/connection_impl.cc:259' occurs on remote_shard.to_here()
(test/distributed/_shard/sharded_tensor/test_sharded_tensor.py:999) because RPC has no XPU backend
registered. Per maintainer comments (kpjeeja 2026-01-05), ShardedTensor is deprecated and RPC for
XPU is not prioritized; these tests should simply be excluded from the XPU suite rather than fixed.

## Fix Approach

- Add these 12 sharded_tensor test cases to the XPU skip/exclude list (e.g., third_party/torch-xpu-ops skip_list_common.py or the distributed test skip config).
- Confirm deprecation status with the distributed team and document the skip reason referencing ShardedTensor deprecation.
- Re-enable only if/when RPC (tensorpipe) gains XPU support (tracked for PT2.10 per kpjeeja comment).
