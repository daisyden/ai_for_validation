## Root Cause

test_fake_autocast_linalg_pinv/pinverse_xpu_float32 fail because linalg.pinv/pinverse eventually
call at::matmul under autocast; the FakeTensor path doesn't apply the float16 cast that real tensors
get, yielding a dtype mismatch in prims.utils.compare_tensor_meta
(torch/_prims_common/__init__.py:175). CUDA already skips these via fake_autocast_device_skips
(upstream PR 114531). test_fake_crossref_backward_amp_nn_functional_bilinear_xpu_float32 reproduces
on CUDA too (alias-info mismatch for aten.view at torch/_subclasses/fake_utils.py:56) and is tracked
separately. Per pbielak (2026-03-24), skips land via upstream PRs 178277 and 175965 (torch-xpu-ops
issue 2250).

## Fix Approach

- Confirm PR pytorch/pytorch#178277 has merged and adds fake_autocast_device_skips["xpu"] = {"linalg.pinv", "pinverse"} in test/test_ops.py.
- confirm pytorch/pytorch#175965 handles the bilinear crossref case.
- After both land in the pinned PyTorch used by torch-xpu-ops CI, re-run the three tests and close the issue.
- No xpu kernel change required.
