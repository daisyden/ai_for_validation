## Root Cause

On BMG B60, hf_T5/hf_T5_base inductor inference latency regressed ~30-33% in PyTorch 2.12 vs 2.11
across both amp_bf16 and amp_fp16, while eager latency is unchanged (<0.2%), proving the regression
lives in the inductor-compiled path. The most plausible delta is the triton-xpu version bump 3.7.0
-> 3.7.1 between releases, which changes Triton codegen/autotuning for the T5
matmul+softmax+layernorm fused kernels on Battlemage. No torch-xpu-ops kernel changed in this perf
path; eager invariance rules out aten/oneDNN.

## Fix Approach

- 1) Bisect triton-xpu between 3.7.0 and 3.7.1 (intel-xpu-backend-for-triton) on hf_T5 amp_bf16 BMG to find the offending commit. 2) Compare TORCHINDUCTOR_TRACE/TORCH_LOGS=output_code generated kernels and autotune choices between 2.11 and 2.12 wheels to identify which kernel/config regressed. 3) File the bisected fix in intel-xpu-backend-for-triton.
- if codegen-side, add BMG-specific autotune configs to torch-xpu-ops inductor backend overrides as a stop-gap. 4) Add hf_T5 BMG perf threshold to weekly perf dashboard.
