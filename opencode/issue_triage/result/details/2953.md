## Root Cause

Running benchmarks/dynamo/huggingface.py with --bfloat16 for TrOCRForCausalLM/XGLMForCausalLM raises
'value cannot be converted to type float without overflow' from
transformers/modeling_attn_mask_utils.py:164 where torch.full((tgt_len,tgt_len),
torch.finfo(dtype).min) is called. The dtype passed in is bfloat16 but somewhere a float-typed
tensor is being filled with the bf16 finfo.min (~-3.39e38) which overflows float; the regression
appears between PyTorch 2.10 → 2.11 and reproduces on Linux WSL and native Windows, so it is a
benchmark/dynamo harness regression in upstream.

## Fix Approach

- Bisect upstream PyTorch between 2.10 and 2.11 to find the dtype-handling regression in benchmarks/dynamo/huggingface.py or torch/_dynamo handling of finfo(min).
- Likely fix is to ensure torch.full with bf16 finfo.min doesn't get coerced into float32 tensor in the model wrapper.
- Track upstream.
- no XPU-specific kernel involved.
