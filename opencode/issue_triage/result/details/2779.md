- **GitHub**: https://github.com/intel/torch-xpu-ops/issues/2779

- **action_Type**: TRACK_PR

## action_TBD

* Track PR pytorch/pytorch#173634 to merge
* Track PR pytorch/pytorch#178734 to merge

## action_reason

* Primary fix PR #173634 is OPEN and APPROVED but still not merged; merging it will close the issue.
* Secondary related PR #178734 (same logspace XFAIL family, also APPROVED) should also be tracked.

## Root Cause

logspace integer kernel at torch-xpu-ops/src/ATen/native/xpu/sycl/RangeFactoriesKernel.cpp:273-298
(AT_DISPATCH_INTEGRAL_TYPES path, "logspace_xpu") computes pow(base, start + i*step) in the integer
scalar type, leading to one-off rounding errors at certain steps versus the reference decomposition.
Failures hit int16/int32/int64 in test_decomp_xpu logspace cases.

## Fix Approach

- In RangeFactoriesKernel.cpp logspace integer branch, compute pow in double (or accumulate type) and round to integer at the end, matching the CUDA reference.
- add unit-test parity with test_decomp_xpu int variants.
