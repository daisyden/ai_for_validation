## Root Cause

test_float8_scale_fast_accum_xpu fails with ~25% element mismatch (greatest abs diff 1.5) in
scaled_mm fp8 fast-accumulate path. Assignee confirmed it is an oneDNN defect (MFDNN-14743); the
fast-accumulate reduction in the oneDNN XPU matmul primitive drops scaling in certain tile
configurations. Fix is in uxlfoundation/oneDNN PR #4923 (backport to 3.10.2). Failure originates in
oneDNN's ocl/sycl matmul kernel used by torch-xpu-ops src/ATen/native/xpu/Blas.cpp::scaled_mm
dispatch.

## Fix Approach

- Bump bundled oneDNN in cmake/External/oneDNN.cmake to v3.10.2 (hash f1d47193..) which contains PR #4923.
- Then re-enable the test in test/xpu/skip_list_common.py / test/xpu/skip_list_win.py and verify on BMG and PVC as the assignee already did locally.
