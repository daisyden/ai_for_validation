## Root Cause

test_decomp_xpu.HasDecompTest.test_has_decomposition (and test_aten_core_operators) compare runtime
decomposition list against a snapshot file (HasDecompTest.test_has_decomposition.expect) copied from
pytorch/pytorch into torch-xpu-ops/test/xpu/. Upstream changes (e.g. addition of
aten::_flash_attention_forward_no_dropout_inplace) flip the snapshot, breaking the local copy until
it is re-synced. Failure at test/xpu/test_decomp_xpu.py:1342 assertExpected mismatch.

## Fix Approach

- Either (a) stop vendoring the .expect file and load it at test time from the installed PyTorch (torch.testing._internal location), or (b) auto-sync the .expect file during build/CI from upstream pytorch.
- Short-term: keep skip with 'random' label as currently applied in torch-xpu-ops skip list.
