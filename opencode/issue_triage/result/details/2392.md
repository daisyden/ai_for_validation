## Root Cause

On Windows test runner with a 16.54 GiB Arc/iGPU (total_capacity=16.54GB, 8.27GB allowed),
test_set_per_process_memory_fraction at test/test_xpu.py:457 computes application_memory from device
total memory and tries to allocate ~8.11 GiB as int8. The allocation exceeds the fraction-limited
allocator budget so XPU OOM is raised. This is an environment/capacity mismatch for small-memory
Windows GPUs, not a kernel bug.

## Fix Approach

- Guard the test with a minimum-memory check (e.g., require device total_memory >= 24 GB) or scale application_memory down when the allowed budget is small.
- alternatively mark with @largeTensorTest / skip on Windows client GPUs.
- Update test/test_xpu.py:test_set_per_process_memory_fraction to compute allocation size from `allowed = total * fraction` rather than `total` and add a skipIf for low-memory devices.
