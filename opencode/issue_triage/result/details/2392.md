## Root Cause

test/test_xpu.py:592 test_set_per_process_memory_fraction allocates ~49% of total device memory then
sets fraction=0.5 and tries to allocate again. On a Windows iGPU/16.5 GiB device the prior test
residue plus the 8.11 GiB allocation exceeds the 8.27 GiB allowed budget, triggering OOM during
setup (not the intended OOM check). Test design assumes a clean memory state and a discrete-GPU
sized device.

## Fix Approach

- Either skip on Windows/iGPU (current `skipped_windows` label) or harden the test: call `gc.collect().
- torch.xpu.empty_cache()` more aggressively, lower the fraction (e.g., 0.25), and compute application_memory using `memory_reserved()+memory_allocated()` after cleanup.
- Update test in pytorch/pytorch test/test_xpu.py.
