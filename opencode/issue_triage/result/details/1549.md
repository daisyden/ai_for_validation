## Root Cause

test_micro_pipeline_tp asserts that the inductor micro-pipeline-TP pass rewrites all_gather +
scaled_mm into a fused_all_gather_scaled_matmul node in the post-grad FX graph. The fusion pass
(torch/_inductor/fx_passes/micro_pipeline_tp.py) only emits
torch.ops.symm_mem.fused_all_gather_scaled_matmul when the device has a registered XPU/CUDA impl and
supports symmetric memory; on XPU there is no XPU registration for fused_all_gather_scaled_matmul
(torch/distributed/_symmetric_memory/__init__.py:1097 is CUDA-only) so the pass leaves the graph
un-fused and the assertion fails.

## Fix Approach

- Same path as #1551: register an XPU implementation (or fallback) for symm_mem.fused_all_gather_scaled_matmul, and ensure the inductor micro-pipeline-TP pass enables fusion on XPU device type.
- Until full symm_mem support exists, register a Python fallback impl that decomposes to all_gather + scaled_mm so the inductor pattern matcher can emit the fused node, or skip these tests on XPU.
