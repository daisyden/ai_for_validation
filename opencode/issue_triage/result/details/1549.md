## Root Cause

test_fuse_all_gather_scaled_matmul_* asserts that the Inductor micro-pipeline-TP pass rewrites an
all_gather+scaled_mm pattern into symm_mem::fused_all_gather_scaled_matmul. The pattern-matcher in
torch/_inductor/fx_passes/micro_pipeline_tp.py only performs the fusion when a
SymmetricMemory-backed implementation is available; since XPU lacks SymmemMemory (same blocker as
#1551, needing Level Zero / upcoming SYCL symm-memory in oneAPI 2026.0), the fused op never appears
in the traced graph and the assertion fails.

## Fix Approach

- 1) Depends on #1551: enable XPU SymmetricMemory via SYCL 2026 APIs, then register fused_all_gather_scaled_matmul for XPU and ensure the Inductor async-TP pass is enabled for the xpu device key (check should_use_async_tp). 2) Meanwhile skip these cases in the torch-xpu-ops distributed skip list to keep CI green.
