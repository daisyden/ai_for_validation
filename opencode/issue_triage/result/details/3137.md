## Root Cause

test_transformerencoder_fastpath_*_use_autocast_True_* fails with 'expected scalar type Half but
found Float' inside torch._transformer_encoder_layer_fwd (test_transformers_xpu.py:556). Under
autocast, the XPU transformer-encoder fastpath in torch/nn/modules/transformer.py:917 dispatches to
a fused kernel whose dtype-promotion handling on XPU diverges from CUDA, yielding mixed Half/Float
inputs. Owner LuFinch confirmed the fastpath is being deprecated upstream and PR
pytorch/pytorch#179701 was closed without merging; cases were marked not_target and skipped via
torch-xpu-ops PR #3510 commit 8722c7c0 in test/xpu/skip_list_common.py.

## Fix Approach

- No code fix needed in torch-xpu-ops: keep the skip entries from PR #3510 in test/xpu/skip_list_common.py since the upstream nn.TransformerEncoder fastpath is deprecated.
- If reactivated later, fix dtype handling in torch/nn/modules/transformer.py and the _transformer_encoder_layer_fwd dispatch (revive pytorch/pytorch#179701) so XPU autocast promotes Q/K/V consistently before the fused call.
- Track via the upstream deprecation.
- no Sycl-kernel work required.
