## Root Cause

_TorchScriptTesting::queue_push is registered only for CPU and CUDA backends in
pytorch/test/cpp/jit/test_custom_class_registrations.cpp:747-758 (TORCH_LIBRARY_IMPL for CPU and
CUDA, no XPU registration). When test_torchbind_xpu.py runs the autocast-device variant on XPU, the
dispatcher cannot find an XPU implementation and raises NotImplementedError. This is a
test-harness/custom-class registration gap in upstream PyTorch test code, not a real operator
missing in torch-xpu-ops.

## Fix Approach

- Add a TORCH_LIBRARY_IMPL(_TorchScriptTesting, XPU, m) block in pytorch/test/cpp/jit/test_custom_class_registrations.cpp registering queue_push/queue_pop/queue_size for XPU (the kernels are device-agnostic Tensor moves, identical to the CPU/CUDA impls).
- Submit upstream PR to pytorch/pytorch.
