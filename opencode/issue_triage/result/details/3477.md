## Root Cause

The test test_compile_obj_torchbind_op_with_autocast_device in
third_party/torch-xpu-ops/test/xpu/export/test_torchbind_xpu.py:1582,1591 calls
torch.ops._TorchScriptTesting.queue_push, but this custom test op is registered only for CPU/CUDA in
upstream pytorch/test/cpp/jit/test_custom_class_registrations.cpp; no XPU TORCH_LIBRARY_IMPL exists,
so the dispatcher raises NotImplementedError on XPU. This is a test-fixture/registration gap in
upstream PyTorch test code, not a real operator missing from torch-xpu-ops. Pure host-side Tensor
moves — no XPU kernel work required.

## Fix Approach

- Add a TORCH_LIBRARY_IMPL(_TorchScriptTesting, XPU, m) block in pytorch/test/cpp/jit/test_custom_class_registrations.cpp registering queue_push/queue_pop/queue_size for XPU (reuse the CPU implementation, since the ops just move Tensor handles into a queue).
- Submit upstream PR to pytorch/pytorch.
- once merged, remove the skip from torch-xpu-ops.
