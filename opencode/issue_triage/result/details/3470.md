## Root Cause

The Linux UT summary job reads the live list of issues with the `skipped` label at summary time
rather than at build start. Because torch-xpu-ops CI build+UT runs span hours, an issue can be
closed mid-run when its fix lands on main; the build under test does not contain the fix, the test
still fails for the known reason, but the summary no longer recognises the failure as expected and
reports a false regression. The race is in the CI summary script (.github/ci_commit_pins /
.github/scripts that run `gh issue list --label skipped` at summary time) — see PR #3429 for the
proposed fix.

## Fix Approach

- Snapshot the skipped-issue allowlist at workflow start (e.g. an early job that runs `gh issue list --label skipped --state all` and uploads the result as a workflow artifact), and have the summary job consume that artifact instead of re-querying GitHub.
- Land #3429.
