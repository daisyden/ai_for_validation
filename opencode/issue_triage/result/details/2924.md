## Root Cause

xcit_large_24_p8_224 timm training amp_bf16 fails accuracy on patch_embed.proj.0.0.weight.grad (RMSE
res 0.00603 vs ref 0.00074) on release/2.11 and nightly 2.12.dev, while 2.10 passes. Failure is in
the conv backward grad of the first patch-embed conv when compiled by inductor + triton-xpu; conv
backward in bf16 is lowered to triton kernels whose accumulator/precision handling regressed between
releases. Reporter tagged 'dependency component: Triton'.

## Fix Approach

- Reproduce, dump TORCHINDUCTOR output_code for the conv-backward kernel of patch_embed.proj.0.0.
- compare to 2.10 generated kernel for accumulator dtype (fp32 acc on bf16 inputs) and tile layout.
- Bisect intel-xpu-backend-for-triton between the 2.10 and 2.11 pins.
- file minimized triton repro upstream.
- If accumulator-dtype regression, force fp32 accumulation via inductor config.
- P0 — accuracy failure on training benchmark.
