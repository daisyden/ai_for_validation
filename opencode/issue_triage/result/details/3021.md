## Root Cause

On B60 + XCCL, dist.all_to_all_single corrupts data for small/simple tensor cases (Test 1 in repro
fails: each rank gets wrong values), while
torch.distributed._functional_collectives.all_to_all_single succeeds and large-tensor dist path also
works — pointing to a size/contiguity-dependent bug in the XCCL alltoall implementation in
third_party/torch-xpu-ops/src/xccl/ProcessGroupXCCL.cpp (alltoall_base / alltoall) atop oneCCL
2021.16.1. Issue causes Ulysses attention to produce black images in sglang-diffusion. Comment
confirms it is fixed with DLE 2026.0 + level-zero 1.27.0 + intel-compute-runtime 26.05.37020.3.

## Fix Approach

- Verify fix on DLE 2026.0 stack using the provided test_alltoall_single.py and close once green.
- then add a regression UT that exercises small tensors and the 4D Ulysses pattern.
- If still seen, inspect alltoall_base in ProcessGroupXCCL.cpp for split-size / element-count handling on small messages and align with ft_c semantics.
