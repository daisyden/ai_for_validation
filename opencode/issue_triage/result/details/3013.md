## Root Cause

Same upstream UT (test_pdist_norm_large_xpu) raises 'RuntimeError: Kernel is incompatible with all
devices in devs' — a SYCL/Level-Zero JIT failure when launching the pdist SYCL kernel for the
'large' configuration. Source:
third_party/torch-xpu-ops/src/ATen/native/xpu/sycl/DistanceKernels.cpp. Typical causes are a
work-group size or local-memory request that exceeds device limits (Arc/PVC
max_work_group_size=1024, SLM 64KB), or a kernel template instantiation that uses an unsupported
feature for the target sub-device — the JIT then rejects the binary on every device in the context.

## Fix Approach

- Add bounds checks before kernel launch in DistanceKernels.cpp: clamp work-group size against syclMaxWorkGroupSize() / device max, and clamp local-memory usage against syclLocalMemSize().
- fall back to a smaller-tile variant or chunked outer loop for 'large' inputs.
- Reproduce with SYCL_UR_TRACE=1 to confirm the failing kernel and the requested vs available WG/SLM sizes, then add a guard so the kernel either resizes or routes to the chunked path instead of attempting an oversized launch.
