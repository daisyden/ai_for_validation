## Root Cause

Same test `test_pdist_norm_large_xpu` (torch-xpu-ops/test/xpu/test_torch_xpu.py:5186) fails on Intel
client cards (e.g. Arc) with 'Kernel is incompatible with all devices in devs'. PdistKernelFunctor
(DistanceKernels.cpp:737) hard-codes fp64 paths: `device_sqrt<double>` at line 746 and
`accscalar_t=acc_type_device<scalar_t,kXPU>` resolves to `double` for fp32 input (line 813), so the
produced SPIR-V image requires `cl_khr_fp64`. On devices without fp64 (Arc A-series, Battlemage
consumer SKUs) the Level-Zero driver cannot select any compatible device for the SPV module, raising
this error.

## Fix Approach

- Either (a) keep the index math in double but guard the kernel with `if (!syclDeviceHasFP64()) TORCH_CHECK(false, 'pdist needs fp64')` / provide a fp32 fallback using `device_sqrt<float>`, or (b) skip the test on non-fp64 XPU devices via `@skipIfXpu` or `@largeTensorTest('5GB', device='xpu')` plus a runtime fp64 skip decorator.
- Preferred: switch `device_sqrt<double>` to a templated variant chosen by `std::is_same_v<accscalar_t,double> && device_has_fp64`, falling back to float sqrt on non-fp64 parts.
