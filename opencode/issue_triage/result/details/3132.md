## Root Cause

Failure is `TORCH_CHECK(!print_debug, "No available kernel. Aborting execution.")` raised from
aten/src/ATen/native/transformers/sdp_utils_cpp.cpp:101 because XPU's select_sdp_backend
(third_party/torch-xpu-ops/src/ATen/native/transformers/SDPUtils.cpp) rejects every registered
backend for the given params. Sub-classes: (a) cuDNN cases (test_cudnn_attention_*) – XPU has no
cuDNN backend so the check is expected; (b) nested-tensor fused-kernel cases – same root cause as
#3133, nested SDPA not implemented; (c) test_singelton_head_dim_stride_ne_1_xpu – SDPUtils.cpp:77-78
passes `ignore_singleton_dim=false` to `sdp::check_last_dim_stride_equals_1_dense`, so a
query/key/value with head_dim==1 but stride!=1 is wrongly rejected (template definition at
aten/src/ATen/native/transformers/sdp_utils_cpp.h:491-504); (d)
test_fused_backwards_throws_determinism_warning_* – oneDNN and SYCLA SDPA backends do not honor the
deterministic-algorithms flag.

## Fix Approach

- For the singleton-stride case, land/forward-port pytorch/pytorch#179800 which flips `ignore_singleton_dim` to true in SDPUtils.cpp:77-78 (mirroring CUDA behavior).
- Skip all cuDNN-specific cases (no XPU cuDNN backend) and all nested-tensor fused-kernel cases (tracked under #3133).
- Skip the determinism-warning case until an SDPA backend with deterministic support is available.
- record all skips in test/xpu/skip_list_common.py with issue links.
