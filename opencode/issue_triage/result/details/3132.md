## Root Cause

Bulk SDPA UT failures with 'No available kernel. Aborting execution.': cuDNN-attention cases are
unsupported on XPU (cuDNN backend will not be ported); nested-tensor SDPA cases
(test_fused_kernels_nested_broadcasting*, test_fused_kernels_seq_len_1_inputs,
test_scaled_dot_product_attention_fused_kernels_packed_accuracy_type_nested) hit no-kernel because
oneDNN/sycltla SDPA paths in third_party/torch-xpu-ops/src/ATen/native/transformers/ do not support
nested tensors; test_fused_backwards_throws_determinism_warning fails because neither oneDNN nor
sycltla supports deterministic SDPA. test_singelton_head_dim_stride_ne_1 was already fixed by
pytorch/pytorch#179800.

## Fix Approach

- Per maintainer LuFinch: skip cuDNN cases permanently (CUDA-only backend).
- skip nested-tensor SDPA cases (low priority, will migrate to _flash_attention_forward varlen path).
- skip determinism cases (backend gap).
- remove skip for test_singelton_head_dim_stride_ne_1 after PR #179800 propagates.
- Long-term: implement varlen/nested + deterministic SDPA in oneDNN / sycltla.
