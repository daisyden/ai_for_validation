## Root Cause

test_invalid_input (test/test_sparse_csr.py:898) expects the generic message 'Expected all tensors
to be on the same device, but found at least two devices, xpu:0 and xpu:1'. The XPU sparse
compressed validation in aten/src/ATen/native/sparse/SparseCsrTensor.cpp
(_validate_sparse_compressed_tensor_args / at::sparse_csr::CheckSparseTensorInvariants) emits the
more specific 'device of crow_indices (=xpu:0) must match device of col_indices (=xpu:1)', so the
assertRaisesRegex fails. The CSR codepath validates component devices separately before PyTorch's
common device-check runs.

## Fix Approach

- Update the XPU/sparse test expectations in torch-xpu-ops skip/override list to match the actual emitted message (regex 'device of .* must match device of'), or upstream a fix so sparse_compressed_tensor validation uses the canonical 'Expected all tensors to be on the same device' message from c10 before emitting the component-level error.
- Preferred: adjust the test/error-regex in test/xpu reference to accept both phrasings.
