## Root Cause

Sparse CSR add and BSR-dense gemm Triton kernels fail on XPU because
torch/sparse/_triton_ops_meta.py:141 hard-codes torch.cuda.get_device_name() in get_meta(), so XPU
dispatches into CUDA path and raises 'Torch not compiled with CUDA enabled'. The
bsr_dense_addmm/mm/linear meta-tuned Triton kernels and sparse_add CSR have no XPU device-name
routing or pre-tuned configs.

## Fix Approach

- Upstream-pytorch fix in torch/sparse/_triton_ops_meta.py to dispatch on tensor.device.type (use torch.xpu.get_device_name() for XPU) and add an XPU device_name table.
- ensure bsr_dense_addmm Triton kernels run on intel-xpu-backend-for-triton.
- Add sparse_csr add XPU coverage or keep CPU fallback in third_party/torch-xpu-ops/yaml/native/native_functions.yaml.
- Then unskip in test/xpu/skip_list_common.py.
