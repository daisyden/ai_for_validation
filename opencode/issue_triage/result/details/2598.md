## Root Cause

On BMG, Qwen2.5-1.5B-Instruct first-token latency regresses >10% when max-new-tokens=1 vs 2 with WOQ
uint4/RTN/group=128 + inductor + static cache. Root cause is most likely a one-shot path in the int4
WOQ matmul (oneDNN matmul primitive cache cold-path / immediate-commandlist warm-up under
SYCL_PI_LEVEL_ZERO_USE_IMMEDIATE_COMMANDLISTS=2) that gets amortized away by the second token, so
measuring only the first iteration exposes the cold-cache cost.

## Fix Approach

- Profile with onednn_verbose and ze_tracer to confirm primitive-cache miss / kernel JIT on first call.
- ensure the int4 WOQ oneDNN matmul primitive is pre-created/warmed during model load (or extend torch.xpu primitive cache warm-up).
- Compare with num-warmup>=4 actually hitting the 1-token path.
