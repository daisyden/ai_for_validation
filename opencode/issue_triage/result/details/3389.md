## Root Cause

Dynamo's stream tracing lowers to streams::record_stream for tensors created on CPU (e.g. torch.ones
without device=xpu in test_stream_backward_simple), but aten::record_stream is only registered for
XPU/CUDA backends, so backward() raises NotImplementedError. Additionally, f-string format specs
with spaces (e.g. f'{ stream }') hit a Python format-spec validation error after torch.Stream
reconstruct. Evidence: torch/_dynamo/variables/streams.py:124/201 custom_ops expect device-side
streams; test/xpu/dynamo/test_streams_xpu.py:444-482 creates CPU tensors under torch.Stream context.

## Fix Approach

- Update test_streams_xpu tests to explicitly create inputs on the XPU device (device='xpu', requires_grad=True) so record_stream routes to the XPU backend, and fix the f-string reproducer to strip/trim the format specifier before evaluation.
- alternatively mark these tests as xfail pending upstream PyTorch fixes (pytorch#178155/#134746/#138557).
