## Root Cause

Tests in third_party/torch-xpu-ops/test/xpu/dynamo/test_streams_xpu.py (e.g.
`test_stream_backward_simple` at ~line 444-482, `test_record_stream_problem_basic`/`_interleaved`)
fail because Dynamo's stream/record_stream graph lowering paths in pytorch/torch/_dynamo (stream
variables and the `record_stream` proxy) produce CPU-backend tensors that then dispatch
`aten::record_stream`, which is only registered for XPU/CUDA — yielding `NotImplementedError: Could
not run aten::record_stream with arguments from the CPU backend`. Two of the four cases additionally
fail on hard-coded `cuda` device strings in `assertExpectedInline` golden output (fixed by PR #3383
commit 88801652), while the remaining two (`test_stream_backward_simple/_sync`) belong to the same
stream-id text-mismatch family as #3521. The `Cases:` filter format in the issue body is also
outdated (single-comma + `_xpu` suffix) and no longer matches `failures_op_ut.log`.

## Fix Approach

- 1) Re-emit the `Cases:` block in the new dotted-class / no-`_xpu`-suffix format documented in the latest comment so dynamic-skip filtering matches `failures_op_ut.log`. 2) Land PR #3383 commit 88801652 to remove the hard-coded `cuda`/`cuda:0` strings and parametrize on `GPU_TYPE`, then drop `test_record_stream_problem_basic`/`_interleaved` from this issue once rerun 25042442065 confirms green. 3) For `test_stream_backward_simple`/`_sync`, file/track an upstream pytorch fix in `torch/_dynamo/variables/streams.py` so that backward-graph tensors carrying `record_stream` are kept on XPU (or the op is short-circuited when the tensor is on CPU).
