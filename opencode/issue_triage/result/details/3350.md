## Root Cause

On Windows Lunar Lake (Arc 140V, driver 32.0.101.8508, oneAPI 2025.3, intel-pti 0.16.0),
torch.profiler creates the `__xpu_profiler__` track but emits no device kernels/memcopies in the
Chrome trace, while `unitrace` on the same env captures all events correctly. The XPU profiler
backend in PyTorch (torch/csrc/profiler/stubs/xpu.cpp + Kineto's XPUPTI plugin via intel-pti) fails
to subscribe to/flush Level-Zero device activity records on Windows; the host track is also nearly
empty, indicating the PTI view-kind subscription or trace finalization on Windows is broken.

## Fix Approach

- Reproduce on Linux to confirm Windows-specific.
- verify intel-pti 0.16.0 + driver 32.0.101.8508 LZ tracing API compatibility and check whether ptiViewEnable(KERNEL/MEMORY_COPY) succeeds on Windows.
- File against intel-pti / Kineto XPUPTI integration and add a Windows-side flush/stop ordering fix in torch/csrc/profiler XPU backend.
- As workaround, document unitrace as the supported path on Windows until the PTI Windows backend is fixed.
