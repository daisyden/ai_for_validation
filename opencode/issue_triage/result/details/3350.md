## Root Cause

On Windows (Lunar Lake, driver 32.0.101.8508, torch 2.11.0+xpu, intel-pti 0.16.0) torch.profiler
creates the __xpu_profiler__ track but emits no kernels/memcpy/host events, while unitrace on the
same setup captures everything correctly — so the driver/PTI events are produced, but the Kineto XPU
profiler integration in PyTorch fails to ingest/flush them on Windows. Fix PR identified at
pytorch/kineto#1381.

## Fix Approach

- Land upstream Kineto PR pytorch/kineto#1381 and uprev the kineto submodule in pytorch.
- Validate with the user's profile_llama_woq_compile.py reproducer on Windows + Arc 140V.
- ensure both Host and Device tracks are populated and compare against unitrace output.
- Add a Windows XPU profiler smoke test to CI.
