## Root Cause

On Windows Lunar Lake (Arc 140V, driver 32.0.101.8508, intel-pti 0.16.0, torch 2.11.0+xpu) the
torch.profiler XPU backend successfully creates the `__xpu_profiler__` track but emits zero
device-side kernel/memcpy events, while a parallel unitrace capture on the same workload shows full
Host+Device activity. This points at the PTI-SDK (Intel Profiling Tools Interface) collector on
Windows failing to subscribe to L0 device events; torch-xpu-ops simply forwards what PTI delivers,
so the gap is in the underlying intel-pti / level-zero tracing path rather than in PyTorch profiler
bindings.

## Fix Approach

- Reproduce with PTI debug logging enabled, file a bug against intel/pti-gpu (or upgrade to a newer intel-pti release that supports Windows LNL) and validate.
- meanwhile add a documented workaround pointing Windows users to unitrace.
- No torch-xpu-ops code change is expected.
