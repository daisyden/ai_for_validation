## Root Cause

test_triton_bsr_softmax (test/test_sparse_csr.py:3546) JIT-compiles a Triton BSR softmax kernel and
fails on XPU with 'Error during Intel loadBinary: ZE_RESULT_ERROR_INVALID_KERNEL_NAME' from the
Level Zero loader. The kernel name resolution fails in the intel-xpu-backend-for-triton runtime when
looking up the compiled SPV/binary against the L0 module, indicating a Triton-XPU compile/lowering
or kernel-name mangling regression rather than a sparse-op logic bug. Split from #2209 and currently
labeled 'skipped'.

## Fix Approach

- File a Triton-XPU backend bug with the failing BSR softmax kernel and the ZE_RESULT_ERROR_INVALID_KERNEL_NAME trace.
- in torch-xpu-ops, keep the @skipIfXPU/skip list entry until Triton fix lands, then unskip and re-run test_triton_bsr_softmax_xpu_{bf16,f16,f32}.
