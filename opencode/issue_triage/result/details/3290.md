## Root Cause

Running benchmarks/dynamo/huggingface.py with --amp --amp-dtype bfloat16 on openai/whisper-tiny
fails with 'dtype mismatch torch.float16, torch.bfloat16' from torch/_dynamo/utils.py:3365 during
the inductor accuracy comparison. The bug is in upstream PyTorch's dynamo/inductor benchmark harness
handling of AMP dtype expectations for the whisper model; fix is tracked by pytorch/pytorch PR
#180309 (verified passing). It is not an XPU kernel issue but surfaces on BMG/PVC via the XPU
benchmark suite.
