## Root Cause

Meta/tracking issue (module: infra) listing corner cases not covered by the automated XPU release
validation: (a) verifying AOT-compiled binaries load on all target platforms without JIT-compiling
SYCL kernels, and (b) scanning torch-xpu-ops and Triton build logs for new warnings/errors. These
are process gaps in the release validation pipeline, not a code bug.

## Fix Approach

- Add the two checklist items as dedicated jobs in the XPU release validation workflow: (1) a no-JIT AOT smoke test that sets `SYCL_CACHE_PERSISTENT=0` and asserts no JIT spilling on each target arch.
- (2) a log-diff job that compares current build warnings against a baseline and fails on new entries.
- Track completion of each checkbox in the issue.
