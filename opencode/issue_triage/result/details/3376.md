## Root Cause

ProcessGroupXCCL is missing multiple Python-bound backend APIs that NCCL exposes (_is_initialized,
comm_split_count, _verify_work_timeout, _set_enable_nan_check, _set_default_timeout,
_wait_for_pending_works, split_group support) and torch.xpu is missing the xccl submodule plus
torch._C._distributed_c10d._dump_xccl_trace_json, so the newly enabled test_c10d_xccl.py tests all
raise AttributeError (see test file lines 598, 972, 997, 1068, 1087, 5390, 5497, 5675). These are
XCCL feature gaps relative to NCCL parity.

## Fix Approach

- In ProcessGroupXCCL (torch-xpu-ops distributed backend) add the missing backend methods (_is_initialized, comm_split_count, _verify_work_timeout, _set_enable_nan_check, _set_default_timeout, waitForPendingWorks, split_group support) with pybind bindings mirroring ProcessGroupNCCL, and expose torch.xpu.xccl plus _dump_xccl_trace_json in the XPU Python module.
- for unsupported features use skip decorators on the corresponding tests until parity lands.
