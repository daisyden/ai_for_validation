## Root Cause

Multiple feature gaps in ProcessGroupXCCL pybind surface: missing _is_initialized, comm_split_count,
_verify_work_timeout, _set_enable_nan_check, _set_default_timeout, _dump_xccl_trace_json,
torch.xpu.xccl module, _wait_for_pending_works, and split_group support. None of these are exposed
in third_party/torch-xpu-ops/src/xccl/ProcessGroupXCCL.{hpp,cpp} or registered via Python init in
pytorch/torch/csrc/distributed/c10d/init.cpp for XCCL, while NCCL provides them. Tests in
test/distributed/test_c10d_xccl.py were enabled in the daisyden/distributed_2.11 branch ahead of the
implementation.

## Fix Approach

- Implement the missing methods in ProcessGroupXCCL (eager init, comm split, NaN check, default timeout, work timeout verifier, dump_xccl_trace_json, _wait_for_pending_works) and expose via pybind in init.cpp.
- add torch.xpu.xccl Python module mirroring torch.cuda.nccl.
- Track as a feature-gap umbrella.
- until implemented, skip the affected tests on XPU upstream.
