## Root Cause

On release/2.11 BMG, sebotnet33ts_256 fp32 training fails accuracy (RMSE 0.0099 vs ref 0.0038 on
stem.conv2.conv.weight.grad). Per @jianyizh, the guilty upstream commit pytorch/pytorch@9ec2063
changed Inductor's silu lowering so the Triton kernel calls std::exp instead of sigmoid (native
exp), and torch-xpu-ops PR #2462 further altered std::exp lowering from a native instruction to the
standard one, costing precision on Intel Triton backend. Numeric drift is in Inductor codegen /
Triton backend, not the SYCL operator.

## Fix Approach

- Restore higher-precision exp lowering in the Inductor XPU backend (torch-xpu-ops PR #2462 follow-up) — either keep the native exp instruction for silu/sigmoid lowerings, or relax sebotnet accuracy tolerance only after upstream fixes the silu lowering.
- Coordinate with intel-xpu-backend-for-triton on std::exp accuracy and verify on 2.12.
