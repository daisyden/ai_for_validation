## Root Cause

The XPU topk kernel hard-rejects any selected dimension with more than INT_MAX elements at
third_party/torch-xpu-ops/src/ATen/native/xpu/sycl/TensorTopKKernel.cpp:64 ('The dimension being
select can not have more than INT_MAX elements.'). The kernel uses 32-bit indexing internally and
never falls back to a 64-bit indexing path the way CUDA's topk now can, so legitimate >2^31 element
inputs immediately error out instead of running. This mirrors the upstream CUDA fix tracked in
pytorch/pytorch#173622 / PR #176095 but XPU has no 64-bit equivalent.

## Fix Approach

- Port the CUDA 64-bit-indexing topk path (pytorch#176095) to TensorTopKKernel.cpp: detect when slice_size > INT_MAX and dispatch a templated kernel instantiated with int64_t index_t (and stride/offset types) instead of erroring.
- Add a corresponding test in test_sorting_xpu for the >INT_MAX shape.
