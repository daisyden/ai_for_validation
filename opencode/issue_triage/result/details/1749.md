## Root Cause

`can_use_overrideable_attention` in `aten/src/ATen/native/mkldnn/xpu/Attention.cpp:44-52` rejects
any SDPA call where q/k/v have `requires_grad=True` via `check_no_grad`, because the oneDNN XPU SDPA
primitive has no backward implementation. This causes transformers tests
`MusicgenTest::test_sdpa_can_dispatch_on_flash` and
`MusicgenMelodyTest::test_sdpa_can_dispatch_on_flash` (and any training SDPA path) to raise
`RuntimeError: No available kernel`. Per comments, oneDNN SDPA backward has been deferred from v3.11
to v3.12 and the feature is now retargeted to PyTorch 2.13 (tracked internally as MFDNN-14834).

## Fix Approach

- Blocked on oneDNN 3.12 SDPA training support.
- Once oneDNN is upgraded, remove/relax `check_no_grad` in `Attention.cpp:44` and register the XPU SDPA backward op (both overrideable and default paths), then re-enable the two transformers tests.
- Interim: keep CPU fallback guidance and ensure the warning message directs users to disable grad or use `torch.no_grad()` for inference.
