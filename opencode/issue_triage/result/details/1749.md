## Root Cause

scaled_dot_product_attention with requires_grad=True on XPU raises 'No available kernel'. The check
at pytorch/aten/src/ATen/native/mkldnn/xpu/Attention.cpp:42-49 (check_no_grad) rejects the
OVERRIDEABLE/oneDNN path when any input requires grad because oneDNN SDPA backward is not yet
supported; FLASH_ATTENTION is unsupported on XPU and there is no fallback. Originally targeted
oneDNN 3.9, deferred to 3.11, then 3.12, now retargeted to PyTorch 2.13 per MFDNN-14834.

## Fix Approach

- Wait for oneDNN SDPA backward (now scheduled in oneDNN 3.12+/MFDNN-14834) and integrate the SDPA training BKW path in aten/src/ATen/native/mkldnn/xpu/Attention.cpp.
- Until then, register a math-backend fallback for the requires_grad case so transformers UTs (test_sdpa_can_dispatch_on_flash) don't hard-error.
