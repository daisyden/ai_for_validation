## Root Cause

torch.xpu exposes get_device_name/get_device_properties but has no public API to distinguish
integrated GPUs (iGPU) from discrete GPUs (dGPU). Per guangyey's comment, the capability must come
from a SYCL runtime extension (sycl_ext_intel_device_info) exposing device type/architecture so
bindings in torch/xpu/__init__.py and c10/xpu/XPUDeviceProp can surface an is_integrated flag.

## Fix Approach

- Check if sycl::ext::intel::info::device::device_id / architecture is sufficient.
- extend c10/xpu/XPUDeviceProp.h to include an is_integrated or device_type field populated from SYCL device info, plumb through torch/csrc/xpu/Module.cpp and add torch.xpu.get_device_type()/get_device_capability helper.
- Track oneAPI SYCL extension readiness before exposing.
