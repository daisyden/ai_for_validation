## Root Cause

torch.nn.functional.scaled_dot_product_attention with requires_grad=True on XPU dispatches to the
MATH backend because no fused SDPA backward kernel is available, causing 4x memory bloat and OOM in
AutoRound training/quantization. Per maintainer comments, oneDNN SDPA backward has accuracy issues
blocking PT 2.12, and SYCLTLA SDPA training was enabled in PT 2.11 but does not yet support dropout,
attn_mask, or deterministic mode (each forces MATH fallback). Code paths: torch-xpu-ops
src/ATen/native/transformers/Attention.cpp and the SYCLTLA/oneDNN backend selectors.

## Fix Approach

- Track and complete oneDNN SDPA backward (MFDNN-* fix for accuracy) targeting PT 2.13.
- meanwhile extend SYCLTLA SDPA backward to support dropout, attn_mask, and deterministic mode so it isn't bypassed to MATH.
- Update the backend selector in torch-xpu-ops Attention dispatcher to prefer SYCLTLA when grad is required.
