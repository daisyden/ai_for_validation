## Root Cause

XPU registers _scaled_dot_product_efficient_attention forward (native_functions.yaml:7945-7947,
Attention.cpp:341) but has no corresponding efficient_attention_backward kernel. As a result, SDPA
training dispatches to the math backend on the backward pass, materializing the full [B,H,S,S]
attention matrix and causing OOM for 4K-seq Llama-shape workloads. Flash-attn backward exists in
flash_attn/sycltla/mha_bwd.cpp but is only invoked when the forward picked the flash path via
SDPUtils.cpp, which does not happen here because efficient attention is preferred.

## Fix Approach

- Add _scaled_dot_product_efficient_attention_backward XPU implementation (or unify with flash_attention_backward_sycltla) and register it in yaml/native/native_functions.yaml next to the forward op.
- Update SDPUtils.cpp priority logic so training workloads reliably pick a memory-efficient backend whose backward is available, avoiding the math-backend fallback.
