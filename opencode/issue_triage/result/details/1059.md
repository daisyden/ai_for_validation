## Root Cause

torch-xpu-ops currently uses a manual `max work-group size` query in
src/comm/DeviceProperties.h:14-28 rather than the recommended kernel-specific shortcut API from SYCL
runtime (tracked by intel/llvm#15650). The shortcut API exists now but a DPC++/IGC performance
regression (CMPLRLLVM-71252) blocks adoption. Latest status (2026-04-01): dependency fix is expected
in oneAPI DLE 26.0.

## Fix Approach

- Wait for oneAPI DLE 26.0.
- once available, replace the custom query in src/comm/DeviceProperties.h with `sycl::ext::oneapi::experimental::info::kernel::max_work_item_sizes` / kernel-bundle shortcut API, re-benchmark to confirm CMPLRLLVM-71252 is closed, then migrate all call sites (syclMaxWorkGroupSize etc.) and remove the fallback path.
