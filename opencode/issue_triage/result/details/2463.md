## Root Cause

On Windows CI, test_xpu_pluggable_allocator (and follow-on test_set_per_process_memory_fraction)
fails because torch.utils.cpp_extension.load_inline() raises 'OSError: SYCL runtime is not detected'
from _join_sycl_home in torch/utils/cpp_extension.py:185 (called via _prepare_ldflags at line 2454).
The sycl_home detection logic does not pick up the oneAPI install path on the Windows runner; even
when found, the secondary failure (Stonepia's log) is MSVC default /std:c++14 without -fsycl causing
static_assert in sycl/detail/defines_elementary.hpp. Both bugs live in upstream PyTorch
cpp_extension.py.

## Fix Approach

- Fix in pytorch/pytorch torch/utils/cpp_extension.py: make _join_sycl_home robustly locate Windows oneAPI (env vars CMPLR_ROOT/ONEAPI_ROOT, default Program Files path) and ensure the SYCL extension build path passes /std:c++17 (or -fsycl when using icx) on Windows.
- Until fixed, keep test_xpu_pluggable_allocator and test_set_per_process_memory_fraction in the Windows skip list.
