## Root Cause

307 cases in test/inductor/test_cudagraph_trees_expandable_segments.py (CudaGraphTreeTests) depend
on CUDAGraph tree capture/replay machinery implemented in torch/_inductor/cudagraph_trees.py and
torch/cuda/graphs.py. XPU lacks an equivalent torch.xpu.Graph / graph-tree plumbing (no SYCL
command-graph capture path exposed to Inductor), so these tests cannot be enabled even though they
are the XPU-ported (expandable_segments) variant.

## Fix Approach

- Implement XPUGraph tree support: (1) expose a torch.xpu.graph/XPUGraph capture API backed by SYCL command-graph (or Level Zero command list immediate submission), (2) generalize torch/_inductor/cudagraph_trees.py into a device-agnostic graph_trees module and register an XPU backend, (3) add expandable-segment allocator hooks in XPU caching allocator.
- Track as a multi-sprint feature.
- in the interim mark the 307 tests ut_skipped under torch-xpu-ops.
