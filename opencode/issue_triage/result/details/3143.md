## Root Cause

test_transformers_xpu.TestSDPACudaOnlyXPU.test_fused_attention_vs_math_ref_grads_cudagraph with
fused_kernel1 (mem-efficient attention) calls out.backward(), which dispatches to
aten::_scaled_dot_product_efficient_attention_backward. That operator had no XPU kernel registered
in torch-xpu-ops (only the forward _scaled_dot_product_efficient_attention was wired up), so the
autograd engine raised NotImplementedError. The backward SYCL-TLA implementation exists in
third_party/torch-xpu-ops/src/ATen/native/transformers/xpu/flash_attn/sycltla/mha_bwd.cpp but it was
not bound to the efficient-attention backward schema.

## Fix Approach

- Already addressed by intel/torch-xpu-ops PR #3317 which adds the XPU registration for aten::_scaled_dot_product_efficient_attention_backward by routing it to the existing mha_bwd sycltla kernel.
- Action: verify the PR (@daisyden is asked to re-run the listed cudagraph test cases), then close the issue and unskip the tests in test/xpu/skip_list_*.py.
