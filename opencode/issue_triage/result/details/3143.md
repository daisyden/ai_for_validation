## Root Cause

128 fused_kernel1 (efficient-attention) cudagraph variants of
test_fused_attention_vs_math_ref_grads_cudagraph fail with
'aten::_scaled_dot_product_efficient_attention_backward not implemented for XPU' because the
backward was never registered for the efficient-attention path used by SDPA fused_kernel1 in
third_party/torch-xpu-ops/src/ATen/native/transformers/sycl/AttentionKernels.cpp. PR #3317 (merge
42e79dfe) adds the registration but as of 2026-04-23 the pytorch/pytorch xpu.txt pin (5aa1a01) is 19
commits behind, so the nightly still reproduces.

## Fix Approach

- No further code change needed in torch-xpu-ops.
- bump third_party/xpu.txt in pytorch/pytorch to a commit >= 42e79dfe, then remove the @skipIfXpu(msg='torch-xpu-ops: #3140') guard at test/xpu/test_transformers_xpu.py:5743 and re-run the 128 fused_kernel1 variants to close the issue.
