## Root Cause

model.share_memory() calls _share_fd_ which at torch/storage.py:447-526 is gated to CPU; XPU falls
through because there is no XPU IPC implementation analogous to _share_cuda_ in
torch/csrc/StorageSharing.cpp:281 and no XPU equivalent of CudaIPCTypes/CUDACachingAllocator IPC
hooks. Per riverliuintel/MarekStrachacki, SYCL runtime IPC is deferred to oneAPI 2026.1 (targeting
PT2.14), so the underlying primitive is not yet available.

## Fix Approach

- Short-term: add 'xpu' to the non-POSIX-shared-memory device list in torch/storage.py:393 so share_memory_() no longer errors.
- raise a clear NotImplementedError on pickling XPU storage through reductions.py.
- Long-term: once SYCL IPC lands in oneAPI 2026.1, implement _share_xpu_ in torch/csrc/StorageSharing.cpp and an XPUIPCTypes mirroring CudaIPCTypes, and register an XPU reducer in torch/multiprocessing/reductions.py.
