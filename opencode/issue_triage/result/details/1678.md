## Root Cause

model.share_memory() ultimately calls UntypedStorage.share_memory_() in torch/storage.py:518 which
invokes _share_fd_ that is gated to CPU only; XPU path is not in the device-skip list
(torch/storage.py:393) and there is no XPU IPC implementation analogous to CUDA's _share_cuda_ in
torch/csrc/StorageSharing.cpp:281 plus CudaIPCTypes.cpp. Per maintainers, full IPC requires SYCL
runtime IPC support landing in oneAPI 2026.1 (target PT 2.14).

## Fix Approach

- Short-term: add 'xpu' to the non-POSIX-shm device list in torch/storage.py:393 to surface a clearer error.
- Long-term: implement XPU IPC mirroring CudaIPCTypes.cpp and add _share_xpu_/StorageSharing entry once SYCL IPC ships in oneAPI 2026.1.
- register XPU reduction in torch/multiprocessing/reductions.py.
