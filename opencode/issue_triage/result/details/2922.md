## Root Cause

Seven tests in test/inductor/test_compile_subprocess.py (GPUTests on xpu, e.g.
test_conv1d_depthwise_xpu, test_shape_prop_torch_ones_xpu, test_functionalize_rng_wrappers_xpu) fail
on Windows with 'AssertionError: pass_fds not supported on Windows'. This is a regression in the PT
2.11 Inductor subprocess test harness: it launches a child Python via subprocess.Popen with
pass_fds=..., which CPython explicitly forbids on Windows (subprocess.py Popen.__init__). CUDA runs
do not hit it because Windows CUDA CI does not exercise these GPUTests. The failing harness is the
new compile-subprocess test added upstream for the 2.11 cycle.

## Fix Approach

- Fix upstream in torch/_inductor/test_operators or test/inductor/test_compile_subprocess.py (and the helper in torch/_inductor/compile_worker/subproc_pool.py if that is the caller): branch on platform.system()=='Windows' to avoid passing pass_fds (use an alternative fd/handle-inheritance scheme or skip subprocess isolation on Windows).
- Short term in torch-xpu-ops, add a Windows skip decorator for these seven tests in the XPU test list until the upstream harness fix lands.
