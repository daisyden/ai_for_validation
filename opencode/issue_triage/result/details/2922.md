## Root Cause

Several inductor subprocess UTs in test/inductor/test_compile_subprocess.py (e.g.
test_conv1d_depthwise_xpu, test_weight_norm_conv2d_xpu) regressed in PyTorch 2.11 with
'AssertionError: pass_fds not supported on Windows'. The failure originates from upstream PyTorch's
subprocess-based compile harness (TestCase.run_in_subprocess / Popen using pass_fds) which is not
portable to Windows. Issue is purely Windows + upstream test-infra; no XPU kernel involvement.

## Fix Approach

- File upstream pytorch fix to guard pass_fds usage on Windows in the inductor subprocess test harness (or skip these tests on win32).
- Mirror the skip in torch-xpu-ops Windows skip list until upstream patch lands.
