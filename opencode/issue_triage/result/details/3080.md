## Root Cause

AOTCudagraphs / cudagraph dynamo backend tests (test_dynamo/test_cudagraphs.py,
test_inductor/test_cuda_repro.py, test_foreach big_num_tensors w/ use_cuda_graph=True) are blocked
because XPU has no equivalent of cudagraph capture (xpugraph). torch._inductor.cudagraph_trees and
torch.cuda.CUDAGraph are CUDA-only; torch.xpu has no Graph/stream-capture API in
pytorch/torch/xpu/__init__.py.

## Fix Approach

- Implement xpugraph (SYCL command-graph capture) as a torch.xpu.Graph runtime API in upstream pytorch (torch/csrc/xpu/ + torch/xpu/graphs.py) backed by SYCL ext_oneapi_command_graph, then plumb into _inductor/cudagraph_trees device-agnostic path.
- Tests will be enabled by replacing TestAotCudagraphs with a device-generic variant via instantiate_device_type_tests.
