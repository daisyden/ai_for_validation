## Root Cause

A batch of ~26 cudagraph tests (test/dynamo/test_cudagraphs.py,
test_cudagraphs_expandable_segments.py, test_backends.py::test_aot_cudagraphs_cuda,
test_structured_trace.py::test_cudagraphs, test_foreach.py foreach_max/foreach_norm with
use_cuda_graph=True, inductor/test_cuda_repro.py::test_cpu_index) cannot run on XPU because they are
hard-wired to CUDA graphs. While torch/xpu/graphs.py exposes a torch.xpu.XPUGraph class, the
Dynamo/Inductor cudagraph backend paths (torch/_inductor/cudagraph_trees.py,
torch/_dynamo/backends/cudagraphs.py) and the test harness (@onlyCUDA decorators / device='cuda'
hardcoding) have no XPU equivalent, so these tests cannot be re-instantiated for XPU.

## Fix Approach

- This is a tracker/feature-gap issue.
- Extend Inductor+Dynamo cudagraph trees to dispatch through a generic device-graph abstraction (CUDAGraph vs XPUGraph) analogous to how streams/events are abstracted via c10::Stream/Event.
- Then port tests under test/dynamo/test_cudagraphs*.py to device-generic form with instantiate_device_type_tests + dtypesIfXPU, or fork XPU variants.
- In the short term keep the skip list in torch-xpu-ops UT config and file granular sub-issues per test family (foreach + cudagraph, structured_trace, aot_cudagraphs, test_cpu_index).
