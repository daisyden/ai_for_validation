## Root Cause

test_micro_pipeline_tp.py::test_fuse_scaled_matmul_reduce_scatter_* exercise async-TP fusion which
lowers to symm_mem::fused_scaled_matmul_reduce_scatter, an op that requires SymmetricMemory
(peer-to-peer VA aliasing). XPU has no registered implementation because Intel SymmetricMemory needs
Level Zero IPC primitives that cannot be upstreamed (per Chao1Han 2025-12-24); the equivalent SYCL
symm-memory API is only expected in oneAPI 2026.0. Thus the aten dispatcher throws
NotImplementedError for the XPU key.

## Fix Approach

- 1) Wait for SYCL symmetric memory support in oneAPI 2026.0 and implement an XPU SymmetricMemoryAllocator + register fused_scaled_matmul_reduce_scatter through ProcessGroupXCCL, mirroring the CUDA/NVSHMEM path in torch/csrc/distributed/c10d/SymmetricMemory.cpp. 2) Until then, skip these async-TP tests in the XPU skip list and document the gap in torch-xpu-ops release notes.
