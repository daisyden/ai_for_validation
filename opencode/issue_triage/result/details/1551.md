## Root Cause

torch.ops.symm_mem.fused_scaled_matmul_reduce_scatter has only Meta and CUDA implementations
registered (torch/distributed/_symmetric_memory/__init__.py:464, 1339); there is no XPU
implementation nor any dispatch fallback to the meta/decomposed path on XPU. Symmetric memory based
fused TP collectives rely on CUDA-specific NVSHMEM/IPC primitives and are not yet available in
oneCCL/XPU symmetric memory, so calls from test_micro_pipeline_tp.py raise NotImplementedError on
XPU.

## Fix Approach

- Either (a) register an XPU implementation under torch.library.impl(lib, 'fused_scaled_matmul_reduce_scatter', 'XPU') that calls the Python fallback _fused_scaled_matmul_reduce_scatter_fallback (matmul + dist.reduce_scatter via XCCL), gated until a real XPU symm_mem fused kernel exists.
- or (b) skip these tests on XPU until torch-xpu-ops symmetric memory backend implements fused matmul+RS.
- Short-term fix is the fallback registration upstream.
