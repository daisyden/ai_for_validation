## Root Cause

The upstream PyTorch tests being pulled in (test/nn/test_pooling.py:317/342/2024/2031,
test/test_foreach.py:1369, test/test_nn.py:7467, test/test_transformers.py:1954) hard-code
device='cuda' or call torch.cuda APIs, so when executed by torch-xpu-ops' op_ut harness on an
XPU-only build they crash in torch/cuda/__init__.py:404 with 'Torch not compiled with CUDA enabled'.
The xpu test wrapper does not override the device string for these large-tensor/launch-config tests
and there is no skip entry in the XPU skip list. Issue is marked duplicate of #2444.

## Fix Approach

- Either (a) add these test IDs to test/xpu/skip_list_common.py with a clear TODO so CI stays green, or (b) preferred: send an upstream PR replacing hard-coded 'cuda' with instantiate_device_type_tests / self.device_type so the tests run on XPU.
- For test_pool3d_large_size_int64 and test_pooling_large, also check whether XPU has the 49GB device memory headroom they need before enabling.
- Close as duplicate of #2444 after the skip list lands.
