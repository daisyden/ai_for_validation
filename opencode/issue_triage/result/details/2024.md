## Root Cause

Several upstream tests reused via torch-xpu-ops (test/nn/test_pooling.py:317, :342, :2024-2031;
test/test_nn.py:7467; test/test_foreach.py:1369; test/test_transformers.py:1954) hardcode
device='cuda' or call torch.randint(..., device='cuda') instead of using the device-generic harness,
so when run under XPU they raise 'AssertionError: Torch not compiled with CUDA enabled'. Bug lives
in upstream PyTorch test sources, not in any XPU kernel.

## Fix Approach

- Submit upstream PRs replacing hardcoded device='cuda' with the parameterized device argument (or @onlyCUDA / dtypes()-aware @largeTensorTest) in test_pooling.py (test_adaptive_pooling_avg_nhwc_launch_config_*, test_pooling_large, test_pool3d_large_size_int64), test_nn.py (test_layer_norm_large_tensor), test_foreach.py, and test_transformers.py.
- until merged, keep these in the torch-xpu-ops skip list.
