## Root Cause

Cluster of upstream UTs reach CUDA-only runtime APIs that torch.xpu does not expose: (a)
DataParallel/scatter_gather path needs torch._C._scatter / torch._C._gather
(test_autograd_xpu.py::test_checkpointing_without_reentrant_dataparallel,
test_dataparallel_saved_tensors_hooks); (b) torch.xpu lacks typed Storage classes
FloatStorage/DoubleStorage/HalfStorage/BFloat16Storage/Byte/Char/Short/Int/Long/BoolStorage (see
torch/xpu/__init__.py, no legacy storage subclasses registered, while torch/cuda/__init__.py defines
them) reached via test_torch_xpu.py test_tensor_storage_type_xpu_* and test_storage_setitem_xpu_*;
(c) torch.xpu.tunable submodule does not exist (test_linalg_xpu.py::__tunableop_ctx at line 448
calls torch.xpu.tunable.enable); (d) torch.xpu.amp / GradScaler pickling missing
(test_pickle_gradscaler_xpu, test_grad_scaler_pass_itself_xpu). Maintainer (CuiYifeng) confirmed
_scatter/_gather will NOT be ported (DataParallel deprecated).

## Fix Approach

- Split into three subgroups: (1) DataParallel/_scatter/_gather cases - permanently skip via skip_list_common.py with WONT_FIX note per maintainer. (2) Storage type cases - either add legacy typed Storage subclasses to torch/xpu/__init__.py (mirroring torch/cuda/__init__.py FloatStorage etc., bound to untyped XPU storage) or skip these CUDA-style legacy-storage tests for XPU. (3) tunableop cases - blocked by issue 2066/814.
- skip until torch.xpu.tunable is implemented (needs oneDNN/XeTLA GEMM tuner bindings in torch/csrc/xpu plus torch/xpu/tunable.py). (4) amp GradScaler - ensure torch.xpu.amp.GradScaler is exposed and picklable in torch/xpu/amp/__init__.py.
- Update skiplist entries and reference issue 2508 (duplicate) for tracking.
