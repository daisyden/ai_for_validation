## action_TBD

* Track PR intel/torch-xpu-ops#3018 to merge
* Address CI failures on PR intel/torch-xpu-ops#3018

## action_reason

* PR #3018 (Support sampled_addmm in SparseCsrXPU) is OPEN and authored by assignee jenniew; merging will fix the missing-kernel root cause.
* linux-ut (op_ut) summary check is failing on PR #3018 and must be resolved before merge.

## Root Cause

aten::sparse_sampled_addmm has no SparseCsrXPU registration, so all test_sampled_addmm_* and
gradcheck_mm_SparseCSR_* tests raise NotImplementedError on the SparseCsrXPU backend. Additionally,
sample input creation via .to_sparse_csr() raises 'expected col_indices to be a contiguous tensor
per batch', indicating XPU CSR conversion path is also broken. No kernel exists under
third_party/torch-xpu-ops/src/ATen/native/sparse/.

## Fix Approach

- Either register a CPU fallback for aten::sparse_sampled_addmm on SparseCsrXPU (analogous to other sparse fallbacks in torch-xpu-ops/src/ATen/native/sparse/) and fix to_sparse_csr contiguity for XPU, or implement a native SYCL sampled_addmm CSR kernel.
- Skip UTs in skip_list_common.py until coverage lands.
