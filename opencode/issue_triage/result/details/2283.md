## action_reason

- Verified PR(s) open: intel/torch-xpu-ops#3018

## Root Cause

aten::sparse_sampled_addmm has no SparseCsrXPU implementation.
third_party/torch-xpu-ops/src/ATen/native/sparse/xpu/SparseCsrTensorMath.cpp and its kernel pair
SparseCsrTensorMathKernels.cpp do not register sparse_sampled_addmm (unlike CUDA's
SparseCsrTensorMath.cu, which implements it via cuSPARSE SDDMM). As a result,
test_sparse_csr_xpu.py::test_sampled_addmm_* and test_sparse_xpu.py::test_gradcheck_mm_SparseCSR_*
raise NotImplementedError, and test_ops dtype/out_requires_grad tests also fail (secondary error
'expected col_indices to be a contiguous tensor per batch' originates from to_sparse_csr in
sample_inputs_sparse_sampled_addmm at common_methods_invocations.py:1215, but the primary blocker is
the missing kernel). Likely duplicate of issue #2214.

## Fix Approach

- Implement sparse_sampled_addmm for SparseCsrXPU: add a sampled_addmm_out_sparse_csr_xpu kernel in src/ATen/native/sparse/xpu/sycl/SparseCsrTensorMathKernels.cpp backed by oneMKL sparse SDDMM (mkl::sparse::sddmm) or a hand-written SYCL kernel iterating the CSR sparsity pattern of the output tensor, register it in SparseCsrTensorMath.cpp via TORCH_LIBRARY_IMPL(aten, SparseCsrXPU, ...) for sparse_sampled_addmm and sparse_sampled_addmm.out, and remove the corresponding skips from test/xpu/skip_list_common.py once passing.
