## Root Cause

ProcessGroupXCCL does not advertise shrink support, so torch.distributed.shrink_group raises
'Process group backend for 0 does not support shrinking operations' at
torch/distributed/distributed_c10d.py:6208 in _validate_shrink_backend_requirements. The capability
is gated on backend bind() options that XCCL never sets because the underlying oneCCL runtime has no
abort/shrink primitive (confirmed by maintainer frost-intel in the issue thread).

## Fix Approach

- Won't-fix at backend level.
- permanently skip the ten test_shrink_group_* cases in the XCCL test list (daisyden/pytorch#34) and document the limitation in ProcessGroupXCCL README.
- Re-evaluate only if oneCCL adds a comm-shrink API.
