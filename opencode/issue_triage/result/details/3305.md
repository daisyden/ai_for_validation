## Root Cause

test_c10d_xccl.py shrink_group tests fail with `Process group backend for '0' does not support
shrinking operations` raised from
torch/distributed/distributed_c10d.py:_validate_shrink_backend_requirements. ProcessGroupXCCL does
not implement the shrink/abort fault-tolerance API because the underlying oneCCL runtime does not
support abort or shrink. Per maintainer (frost-intel), there is no plan to add this feature to XCCL.

## Fix Approach

- Won't-fix at the kernel level.
- Disable/skip the test_shrink_group_* cases in test/distributed/test_c10d_xccl.py (already proposed in daisyden/pytorch#34) and document the unsupported capability in ProcessGroupXCCL until oneCCL adds shrink primitives upstream.
