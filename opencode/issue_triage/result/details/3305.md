## action_reason

- Maintainer declared shrink unsupported on XCCL (oneCCL has no abort/shrink primitive); the test-disable PR daisyden/pytorch#34 merged 2026-04-28 removes the affected shrink_group_* tests, so verify the cases are no longer collected and close.

## Root Cause

ProcessGroupXCCLGroupTest.test_shrink_group_* invoke c10d.shrink_group(), which checks
backend_impl.supports_shrinking (torch/csrc/distributed/c10d/init.cpp:2820 and
torch/distributed/distributed_c10d.py:6217). XCCL backend does not set supports_shrinking=True
because oneCCL has no abort/shrink primitive, so _validate_shrink_backend_requirements raises
TypeError at distributed_c10d.py:6208. This is a feature gap, not a regression.

## Fix Approach

- Per maintainer (frost-intel): no plan to add shrink to XCCL since oneCCL lacks the underlying support.
- Disable the test_shrink_group_* cases for XCCL in test/distributed/test_c10d_xccl.py (already proposed in daisyden/pytorch#34).
- Revisit only if oneCCL adds fault-tolerant shrink primitives upstream.
