## Root Cause

test_fn_gradgrad for nn.functional.linear_cross_entropy float64 fails with 'Backward is not
reentrant' (gradcheck.py:813): repeated backward passes through the op produce non-deterministic
gradients with tolerance 0.0. The op is a newly added upstream fused linear+cross-entropy whose
backward (or its double-backward decomposition) introduces non-determinism, likely via atomic
accumulation in the matmul/scatter portion; the same nondeterminism would show on CUDA with the same
kernel choice, so the OpInfo should declare nondeterministic_alert/gradcheck_nondet_tol rather than
rely on tol=0.

## Fix Approach

- Upstream fix in the linear_cross_entropy OpInfo: set gradcheck_nondet_tol to a small positive value (matching other reduction-using ops) or mark the backward as nondeterministic.
- If determinism is achievable, replace atomic-add reductions in the backward with a deterministic reduction.
- Keep the case skipped in torch-xpu-ops until the upstream OpInfo is updated.
