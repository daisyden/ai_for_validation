## Root Cause

test_scaled_mm_vs_emulated reads torch.xpu.get_device_properties(0).major, but _XpuDeviceProperties
currently exposes no 'major'/'minor' arch-version attribute. Per maintainer comment, exposing GPU
arch major/minor requires DPC++ runtime support tracked in CMPLRLLVM-72166 and is targeted for
oneAPI 2026.1 / DPC++ 2026.2.

## Fix Approach

- Wait for oneAPI 2026.1 (CMPLRLLVM-72166) to expose arch major/minor via SYCL device info, then add 'major'/'minor' fields to _XpuDeviceProperties in torch/csrc/xpu/Module.cpp and the Python binding.
- Short-term: in torch-xpu-ops upstream test harness, gate test_scaled_mm_vs_emulated with hasattr(props,'major') or skipIfXpu until the property lands.
