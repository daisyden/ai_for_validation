## Root Cause

MultiheadAttention's nested-tensor fast-path gate _check_arg_device at
torch/nn/modules/activation.py:1056 only accepts cpu/cuda/privateuseone, so XPU tensors fail the
device whitelist and hit the assertion at activation.py:1449 (`assert not any_nested`). Issue author
confirmed root cause and plans an upstream PR to add 'xpu' to the allowed device list.

## Fix Approach

- Submit upstream PyTorch PR to extend torch/nn/modules/activation.py:_check_arg_device to include 'xpu' in the accepted device-type list (mirroring the cuda branch).
- After the upstream lands, remove the affected tests from torch-xpu-ops skip lists.
