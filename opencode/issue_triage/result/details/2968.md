## Root Cause

Multiple XcclErrorHandlingTest cases (test_xccl_blocking_wait_with_barrier,
test_xccl_errors_blocking, test_xccl_non_blocking_wait_with_barrier,
test_param_layout_mismatch_error) hit `[E ProcessGroupXCCL.cpp:231] Work ran time out after 3009
milliseconds`. The desync/abort path in third_party/torch-xpu-ops/src/xccl/ProcessGroupXCCL.cpp
around the watchdog/Work::wait timeout handling does not propagate the expected exception type
(DistBackendError/timeout) before the test's barrier, leaving ranks stuck so the test harness times
out instead of seeing the structured error.

## Fix Approach

- In ProcessGroupXCCL.cpp Work::wait/checkAndSetException and the watchdog (ProcessGroupXCCLMonitor.cpp), ensure that on timeout the work is marked completed-with-exception and abort() is called on the comm so subsequent barrier() unblocks.
- align error-handling semantics with ProcessGroupNCCL (blocking_wait, abort_on_timeout).
- Add explicit error type matching what test_xccl_errors_blocking expects.
