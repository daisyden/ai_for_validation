## Root Cause

The failing tests (test_xccl_blocking_wait_with_barrier, test_xccl_errors_blocking,
test_xccl_non_blocking_wait_with_barrier, test_param_layout_mismatch_error) exercise blocking-wait
error handling and desync detection by calling process_group.barrier().wait(timeout=...).
ProcessGroupXCCL does not implement the NCCL-equivalent blocking-wait / error-detection path (see
watchdog/abort handling in third_party/torch-xpu-ops/src/xccl/ProcessGroupXCCL.cpp, e.g. around line
231 where 'Work ran time out after' is logged), so the barrier legitimately hangs to its timeout
instead of surfacing the expected error, producing RuntimeError: Work ran time out.

## Fix Approach

- Short term: skip these tests on XCCL (daisyden/pytorch#34 already does this) since the underlying blocking-wait/error-handling feature is not in XCCL yet.
- Long term: implement the blocking-wait / errors-blocking semantics in ProcessGroupXCCL (watchdog thread raising async errors, abort + exception propagation on wait) mirroring ProcessGroupNCCL, then re-enable the tests.
