## Root Cause

test_sparse_xpu.py:1965-1979 unconditionally skips test_bmm_windows_error for XPU via
`@unittest.skipIf(TEST_XPU, 'bmm sparse-dense XPU is not yet supported')`, but bmm sparse-dense is
now implemented on XPU (SparseTensorMath/SparseBlas). The test is a Windows+CUDA<11.0 negative test
that should simply be `@onlyOn(['cuda'])` or remain skipped on XPU for a different reason — no XPU
functional bug.

## Fix Approach

- In third_party/torch-xpu-ops/test/xpu/test_sparse_xpu.py:1970 remove the `@unittest.skipIf(TEST_XPU, ...)` decorator (leaving the pre-existing Windows/CUDA gating intact so the test is a no-op off-Windows).
- Verify on Windows XPU runner per daisyden's comment before closing.
