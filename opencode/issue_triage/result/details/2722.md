## Root Cause

test_flip_xpu_float32 fails with NotImplementedError: 'aten::flip' not runnable for QuantizedXPU
backend. The QuantizedXPU dispatch key is not yet plumbed through torchgen/native_functions.yaml
(see aten/src/ATen/native/native_functions.yaml:6352 for flip's dispatch block). Upstream PR
pytorch/pytorch#173923 adds QuantizedXPU as a supported torchgen dispatch key; until then no XPU
kernel can be registered for the quantized variant of flip. Currently flip's XPU kernel lives in
third_party/torch-xpu-ops/src/ATen/native/xpu/TensorTransformations.cpp with QuantizedCPU fallback
but no QuantizedXPU.

## Fix Approach

- Blocked on pytorch/pytorch#173923 (enable QuantizedXPU in torchgen).
- Once merged: (1) add QuantizedXPU dispatch entry for flip in aten/src/ATen/native/native_functions.yaml (mapping to the existing CPU quantized implementation since flip is a memory layout op, or to the XPU kernel with a quantized-aware wrapper).
- (2) register the QuantizedXPU kernel in third_party/torch-xpu-ops/yaml/native/native_functions.yaml or the backend-specific registration file.
- (3) remove skip from test/xpu/skip_list for test_flip_xpu_float32.
- Minimal implementation can reuse the existing flip XPU kernel since flip only permutes indices and does not require dequantization.
