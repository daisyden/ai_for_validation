## Root Cause

test_ldexp_xpu fails with 'DispatchStub: missing kernel for xpu' at
aten/src/ATen/native/DispatchStub.cpp:275. torch.ldexp is implemented in
aten/src/ATen/native/BinaryOps.cpp via the structured binary stub mul_stub composed with ldexp_stub;
the XPU backend has no XPU dispatch registered for ldexp_stub (no entry in
third_party/torch-xpu-ops/src/ATen/native/xpu/BinaryOps.cpp / sycl). Pre-existing WAs in pytorch PR
#171238 confirm the missing XPU registration.

## Fix Approach

- Add an XPU kernel registration for ldexp_stub in torch-xpu-ops: implement at third_party/torch-xpu-ops/src/ATen/native/xpu/sycl/BinaryGeometricKernels.cpp (or a new BinaryLdexpKernel.cpp), register via REGISTER_XPU_DISPATCH(ldexp_stub, ...), and add wiring in src/ATen/native/xpu/BinaryOps.cpp.
- Then remove the WAs from upstream PR #171238 and re-enable test_ldexp_xpu.
