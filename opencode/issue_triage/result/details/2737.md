## Root Cause

torch/nn/parallel/comm.py:258 calls torch._C._gather(...) unconditionally. This Python binding is
registered only in the CUDA-only translation unit torch/csrc/cuda/python_comm.cpp:88
(initCommMethods registers _broadcast/_broadcast_coalesced/_scatter/_gather/_gather_out on the
top-level torch._C module). In XPU-only builds without CUDA, python_comm.cpp is not compiled, so
torch._C._gather does not exist, yielding AttributeError when dp.gather() is called on XPU tensors.
Same missing symbol will affect _scatter/_broadcast in DataParallel.

## Fix Approach

- Make the comm bindings device-agnostic: move _gather/_scatter/_broadcast(_coalesced)/_gather_out definitions from torch/csrc/cuda/python_comm.cpp into a device-neutral module (e.g. torch/csrc/Module.cpp or a new torch/csrc/python_comm.cpp) that is always compiled.
- The underlying comm.cpp uses generic at::Tensor and stream handling, so only the optional CUDAStream plumbing is CUDA-specific (guard with USE_CUDA and accept generic c10::Stream otherwise, or add XPUStream support).
- Alternative: add torch/csrc/xpu/python_comm.cpp mirroring the CUDA file and register on torch._C.
- Then DataParallel gather() works for XPU.
