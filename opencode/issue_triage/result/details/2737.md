## Root Cause

test_data_parallel.py::test_gather_gpu fails with AttributeError: module 'torch._C' has no attribute
'_gather'. torch/nn/parallel/comm.py:gather() (line 258) calls torch._C._gather(tensors, dim,
destination), which is a CUDA-only Python binding (registered via torch/csrc/cuda/comm.cpp /
python_nn_functions). On the daisyden distributed_2.10 branch built without CUDA, the symbol is not
exposed, and there is no XPU equivalent registered, so DataParallel.gather is unusable on XPU.

## Fix Approach

- Upstream torch/nn/parallel/comm.py needs an XPU-aware gather path: either expose torch._C._gather for XPU by registering an XPU implementation in torch/csrc/Module/comm bindings (mirroring CUDA), or add a Python fallback using torch.cat/torch.stack with target_device when CUDA is unavailable.
- Track via a pytorch/pytorch PR.
- torch-xpu-ops only needs to depend on the fix.
