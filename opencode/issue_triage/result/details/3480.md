## Root Cause

test_custom_ops_xpu.py:1470 calls assertRaisesRegex expecting message 'either None or a Tensor', but
autograd raises a different upstream message 'element 0 of tensors does not require grad and does
not have a grad_fn' first. The expected error string in upstream PyTorch's custom_op infrastructure
changed (or is never reached on XPU because the autograd guard fires earlier), so the regex no
longer matches. This is a test-expectation/upstream message-drift issue, not an XPU kernel bug.
