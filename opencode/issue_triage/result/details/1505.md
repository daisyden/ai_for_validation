## Root Cause

Training-accuracy failures on ARC-WSL (Windows) for 10+ timm/HF models under bf16/fp16 using
Inductor+Triton; failures are weight-gradient cosine-similarity drops (e.g., sebotnet33ts_256
sim=0.26, Albert sim=-0.13). Only reproduces on ARC-WSL Windows driver, not Linux, so root cause is
on the WSL GPU driver / IGC codegen path for low-precision training kernels; issue persists through
driver 32.0.101.8136 / WSL 24.52.32224 and pt2.9.

## Fix Approach

- Reduce repro to a minimal failing op per model (e.g., conv backward, embedding backward for Albert token_type_embeddings), compare WSL vs Linux vs eager, then file a driver/IGC JIRA with kernel dumps.
- Re-test on BMG WSL + pt2.10 as requested by chuanqi129.
- add these models to XPU-WSL skip list until driver ships the fix.
