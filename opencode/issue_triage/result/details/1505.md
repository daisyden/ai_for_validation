## Root Cause

10+ Timm/HF models (gluon_inception_v3, mixer_b16_224, mobilenetv2_100, sebotnet33ts_256,
AlbertForQuestionAnswering, AlbertForMaskedLM, etc.) fail accuracy training on ARC-WSL with cosine
similarity drop on weight gradients (e.g., score=0.26 for sebotnet33ts_256). Failures are bf16/fp16
training-only and isolated to WSL/Windows client; backward-pass numerics from the inductor-generated
triton-xpu kernels diverge from eager. Likely a Triton-XPU codegen accuracy bug for low-precision
conv/matmul backward on Xe-LPG (ARC iGPU on LNL/MTL via WSL).

## Fix Approach

- Bisect against pytorch-triton-xpu commits on ARC-WSL.
- compare generated Triton IR for failing reduction/conv-bwd against Linux.
- File issue against intel-xpu-backend-for-triton with minimal repro (single failing op gradient).
- As mitigation, add models to dynamo_test_failures and validate on BMG WSL with 2.10.
