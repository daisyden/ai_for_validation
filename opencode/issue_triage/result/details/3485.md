## Root Cause

TestCompositeComplianceXPU.test_cow_input_nn_functional_linear_cross_entropy_xpu_float32 fails at
test/test_ops.py:2013 (check_cow_input asserting 'Argument 0 during forward call unexpectedly
materializes') for the new upstream OpInfo nn.functional.linear_cross_entropy. The op has no
XPU-specific kernel under torch-xpu-ops/src/ATen/native/xpu/ (grep returned no hits) and no native
impl was found under aten/src/ATen/native; it is a recently-added OpInfo whose
composite/decomposition path on XPU triggers a hidden materialization of the input (e.g., a
contiguous() or copy along the lowering route) that the COW-input compliance harness flags. This is
a CompositeCompliance-style OpInfo bookkeeping issue, not an XPU-specific kernel bug — the same
materialization would occur on any backend that exercises this composite path.

- Until upstream lands the OpInfo fix, keep the existing skip in third_party/torch-xpu-ops/test/xpu/skip_list_common.py.
