## Root Cause

GPTQ BKC script LLM/inference/gptq.py:18 imports 'from torchao._models.llama.model import
Transformer, prepare_inputs_for_model', but pytorch/ao PR #3552 removed the torchao._models package
entirely, so ModuleNotFoundError is raised on BMG with torchao 0.16.0.dev20260108+xpu. TorchAO
maintainers indicated (pytorch/ao#4191) that GPTQ is not yet stable and recommended XPU not enable
it at this time, so this is a script/BKC bitrot rather than an XPU kernel bug.

## Fix Approach

- Short-term: disable the INT4 GPTQ workload in the XPU weekly CI (TorchAO_LLM_XPU_Weekly) until torchao upstream stabilizes GPTQ, per xiaowangintel's comment.
- Medium-term: migrate the script to the new torchao GPTQ quantize_ API (torchao.quantization.quantize_ / Int4WeightOnlyConfig with GPTQ) and drop the torchao._models.llama.model dependency by vendoring a minimal Transformer or using transformers LLaMA directly.
- Reopen once torchao provides a supported GPTQ entry point.
