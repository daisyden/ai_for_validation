## Root Cause

torchao removed the torchao._models.llama package in pytorch/ao#3552, so the Intel BKC script
gptq.py fails at `from torchao._models.llama.model import Transformer` with ModuleNotFoundError.
This is purely a downstream test-script breakage; per @xiaowangintel comment referencing
pytorch/ao#4191, GPTQ is unstable upstream and XPU enabling is deferred.

## Fix Approach

- Disable/skip the INT4 GPTQ XPU BKC test until torchao re-stabilizes GPTQ.
- update the gptq.py script to use the new torchao quantize_ API path or pin to an older torchao version.
- No torch-xpu-ops kernel work needed.
