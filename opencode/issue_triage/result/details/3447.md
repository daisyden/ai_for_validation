## Root Cause

torch.native_channel_shuffle on XPU diverges from CPU for invalid inputs: groups=0 causes SIGFPE
(integer divide-by-zero), and channels-not-divisible / rank<=2 silently succeed instead of raising.
The CPU TORCH_CHECKs live only in channel_shuffle_cpu / channel_shuffle
(aten/src/ATen/native/ChanelShuffle.cpp:23-32 and 50-59); the math/composite path
native_channel_shuffle has no input validation so XPU dispatch falls through unchecked, hitting `c %
groups` (SIGFPE) or producing wrong-shape output. Upstream pytorch/pytorch#181029 adds the missing
checks to native_channel_shuffle.

## Fix Approach

- Land upstream pytorch/pytorch#181029 (adds dim/groups/divisibility checks to native_channel_shuffle in aten/src/ATen/native/ChanelShuffle.cpp).
- No torch-xpu-ops kernel change required since there is no XPU-specific channel_shuffle kernel.
- the fix is upstream only.
