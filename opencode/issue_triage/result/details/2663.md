## Root Cause

In torch/testing/_internal/common_utils.py (and the semi-structured test harness in
test/test_sparse_semi_structured.py), the SEMI_STRUCTURED_SUPPORTED_BACKENDS dict is only populated
under `if torch.cuda.is_available()` for `cutlass` (SM8X) and `cusparselt` (SM8X/SM9X/HIP). On XPU,
neither CUTLASS nor cuSPARSELt exist, so the dict stays empty and every parameterized case in
test_sparse_semi_structured.py is skipped silently. This is a test-coverage gap, not a kernel bug:
torch-xpu-ops does not provide a 2:4 semi-structured sparse kernel backend.

## Fix Approach

- Short-term: explicitly skip or xfail the whole test file for XPU to make the gap visible.
- Long-term: either (a) register a CPU-fallback / oneMKL-based semi-structured backend under an `xpu` key in SEMI_STRUCTURED_SUPPORTED_BACKENDS, or (b) implement a SYCL CUTLASS-equivalent 2:4 sparse MM kernel in torch-xpu-ops (src/ATen/native/sparse/) and gate the dict on `torch.xpu.is_available()`.
- Until a backend exists, keep the issue in UT Backlog and document the non-support.
