## Root Cause

test/test_sparse_semi_structured.py guards SEMI_STRUCTURED_SUPPORTED_BACKENDS population behind
torch.cuda.is_available() and SM8X/SM9X capability checks, with backends keyed only on CUTLASS and
cuSPARSELt. On XPU the dict remains empty so all parametrized cases are skipped/no-op. XPU has no
equivalent semi-structured sparse backend (no CUTLASS, no cuSPARSELt), making this a feature-gap /
test-coverage gap rather than a kernel bug.

## Fix Approach

- Either (a) explicitly skipIf(not _has_semi_structured_backend) at the suite level so coverage reporting is honest, or (b) extend the pytorch/test/test_sparse_semi_structured.py harness to include an XPU branch that registers a fallback/skip path.
- Long term, plan an XPU semi-structured 2:4 sparse backend.
- until then mark as feature gap in upstream pytorch tests.
