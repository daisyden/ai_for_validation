## Root Cause

Build/UX issue: PyTorch XPU wheels bundle pinned oneAPI runtime libs (libpti, libur_loader,
libsycl). Activating a different oneAPI env (e.g., 2025.1) before 'import torch' causes cryptic
loader errors like 'LIBUR_LOADER_0.10 not found' from torch/__init__.py:418, and SYCL cpp_extension
builds (torch/utils/cpp_extension.py SyclExtension path) fail similarly. There is no
version-mismatch detection or actionable diagnostic.

## Fix Approach

- Add an oneAPI version probe in torch/utils/cpp_extension.py (SyclExtension/include_paths) and in torch/xpu/__init__.py that compares the build-time oneAPI version (recorded at wheel build) against runtime libsycl/libpti versions, emitting a clear actionable error listing conflicting packages and recommending the matching oneAPI release.
- Coordinate with oneAPI for backward-compat (PREQS-2946).
