## Root Cause

PT2E perf benchmarks for demucs and hf_DistilBert fail. demucs hits TORCH_CHECK in
aten/src/ATen/native/mkldnn/xpu/qconv.cpp:56-63 which rejects unary_post_op fusion (e.g. 'relu') for
1D/3D inputs ('quantized pointwise conv1d doesn't support unary_post_op fusion. Got
unary_post_op:relu'). hf_DistilBert fails at torchao/quantization/pt2e convert with mixed-device
assertion ({xpu, cpu}) in _assert_and_get_unique_device because the prepare/convert path only
allowed cpu/cuda.

## Fix Approach

- Extend XPU oneDNN qconv to support unary post-op fusion for 1D/3D (or have inductor lowering avoid emitting fused post-op for non-2D quant conv on XPU).
- coordinate with torchao to allow XPU device in convert_pt2e device-uniqueness check (upstream torchao/PyTorch fix).
