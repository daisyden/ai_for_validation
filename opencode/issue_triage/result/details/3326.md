## Root Cause

test_mem_eff_attention_large_seq_len_uniform_attention_xpu in
test/xpu/test_transformers_xpu.py::TestSDPAFailureModesXPU fails sporadically (sometimes passes
nightly, then re-reproduces); marked with labels 'skipped, random'. The test exercises the
memory-efficient attention path on very large sequence lengths, which stresses the SYCL/level-zero
runtime on the CI machines. The same symptom has recurred across issues #2010, #2110, #2024, and
#3259, with no single source-code fix; the failure pattern (intermittent, environment-dependent) and
the fact previous fixes only masked it strongly indicate a GPU driver / Level-Zero timeout or
kernel-eviction issue rather than a logic bug in the attention SYCL kernel under
third_party/torch-xpu-ops/src/ATen/native/transformers/.

## Fix Approach

- Keep the test in the skip list and tag for driver investigation: collect ZE_DEBUG and dmesg output from a failing CI run, capture the seq_len/dtype combination that fails, and file with the GPU compute-runtime / level-zero team referencing prior issues #2010/#2110/#2024/#3259.
- Try reducing the test's tensor size or splitting into smaller chunks as a workaround.
- alternatively gate the large-seq-len variant behind a CI env flag until the driver fix lands.
