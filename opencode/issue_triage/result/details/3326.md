## Root Cause

test_mem_eff_attention_large_seq_len_uniform_attention_xpu intermittently fails with two issues per
maintainer comment: (1) UserWarning that SDPA Memory Efficient Attention backend is not supported on
XPU and falls back to math (test expectation mismatch), and (2) sporadic 'level_zero backend failed
with error: 40 (UR_RESULT_ERROR_OUT_OF_RESOURCES)' from the L0 driver when running large-seq SDPA in
test_transformers_xpu.py::TestSDPAFailureModesXPU. Bug recurs across issues #2010/2110/2024/3259,
suggesting GPU-machine memory pressure / driver resource exhaustion.

## Fix Approach

- Land warning-message alignment PRs (pytorch/pytorch#180646, torch-xpu-ops#3405, pytorch/pytorch#180857) to fix issue (1).
- For (2), gate the test with a memory-availability check or reduce sequence length on shared CI hosts.
- report the OOR to the Level-Zero/compute-runtime driver team and consider adding torch.xpu.empty_cache() before the test plus serializing it on CI to avoid concurrent allocation pressure.
