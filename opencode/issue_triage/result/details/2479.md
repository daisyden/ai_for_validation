## Root Cause

torch.rand produces materially different bf16 outputs at the same index across BMG vs PVC for
identical seed (0.1885 vs 1.5e-05), indicating the XPU Philox/uniform RNG kernel does not produce
architecture-invariant streams. The uniform_kernel in
third_party/torch-xpu-ops/src/ATen/native/xpu/sycl/DistributionUniform.cpp dispatches work based on
subgroup/EU geometry differing between PVC and BMG, so the per-element Philox offset mapping varies.
CUDA contract requires same seed -> same values regardless of GPU.

## Fix Approach

- Make the Philox per-element counter assignment independent of work-group/subgroup size by using a deterministic global element index (linear thread_id over numel, fixed UNROLL) as the Philox subsequence.
- Mirror CUDA's distribution_nullary_kernel layout in DistributionTemplates.h and audit DistributionUniform.cpp / DistributionTemplates.h to remove device-geometry-dependent striding.
