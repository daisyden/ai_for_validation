## Root Cause

test_distributed_spawn.py tests (test_compute_bucket_assignment_by_size_sparse_error_with_logger,
test_monitored_barrier_allreduce_hang_wait_all_ranks) hang and produce 'Work ran time out after 0
milliseconds' / 'barrier timeout' on BACKEND=xccl. monitored_barrier is implemented in PyTorch's
ProcessGroup base; ProcessGroupXCCL (third_party/torch-xpu-ops/src/xccl/ProcessGroupXCCL.cpp) lacks
proper monitored_barrier support / timeout plumbing, so the helper Barrier.sync in
distributed_test.py:571 fires the 30s timeout. Per #2701 some cases fix with
TORCH_XCCL_BLOCKING_WAIT env.

## Fix Approach

- In ProcessGroupXCCL.cpp implement monitoredBarrier (mirroring ProcessGroupNCCL) or set the appropriate XCCL env defaults (TORCH_XCCL_BLOCKING_WAIT=1) in the test runner.
- ensure WorkXCCL records start time so timeout reporting is non-zero.
- Coordinate with #2701 fix.
