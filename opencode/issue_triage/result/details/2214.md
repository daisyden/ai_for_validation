## Root Cause

test_gradcheck_mm in pytorch test/test_sparse.py:5297 wraps the call with assertRaisesRegex
expecting an error string about 'sparse_addmm_sparse_backward: unsupported combination of layouts
...'. On XPU the path triggers a different upstream error from at::native::empty_sparse_compressed
('expected sparse compressed (non-block) tensor layout but got SparseBsr'), so the regex match
fails. The discrepancy is in the upstream sparse layout dispatch / error message, not in any XPU
SYCL kernel; XPU simply has no specialized sparse compressed addmm backward and falls through to the
generic error path that is raised earlier than CUDA's MKL-gated message.

## Fix Approach

- Update the assertRaisesRegex pattern in upstream test_sparse.py to also accept the 'empty_sparse_compressed expected sparse compressed (non-block) tensor layout' message for non-MKL backends (XPU), or skip the test on XPU until sparse compressed addmm backward is implemented.
- Likely duplicate of #2283.
- coordinate fix via PR to pytorch/pytorch test_sparse.py.
