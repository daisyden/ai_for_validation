## Root Cause

test/test_sparse.py::test_gradcheck_mm (line 5297) uses assertRaisesRegex with a regex hard-coded
for CPU/CUDA error text ('sparse_addmm_sparse_backward: unsupported combination of layouts ...
addmm: computation on (CPU|CUDA) is not implemented ...'). On XPU, the failure path is reached
earlier via empty_sparse_compressed which raises 'empty_sparse_compressed expected sparse compressed
(non-block) tensor layout but got SparseBsr', so the regex does not match and the test fails. Likely
duplicate of #2283.

## Fix Approach

- Extend the expected-error regex in test/test_sparse.py (around line 5297) to also accept the XPU-specific message (e.g., add 'empty_sparse_compressed expected sparse compressed.*' alternative), or add an XPU branch that calls the test with the appropriate regex.
- Alternatively, upstream an XPU device entry in the sparse_addmm_sparse_backward dispatch so the thrown error matches the CPU/CUDA wording.
- Remove the hard-coded skip in test/xpu/skip_list_common.py once fixed.
