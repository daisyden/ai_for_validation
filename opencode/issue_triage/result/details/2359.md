## Root Cause

gradgradcheck for index_reduce (mean/prod) float64 on XPU fails with 'Backward is not reentrant'
(tolerance 0.0). The index_reduce double-backward computes a non-deterministic order of atomic
accumulations on XPU (kernel in
third_party/torch-xpu-ops/src/ATen/native/xpu/sycl/IndexingKernels.cpp / Indexing.cpp index_reduce
path), causing tiny nondeterministic differences. The fix is purely an OpInfo-level
gradcheck_nondet_tol bump as already proposed in pytorch/pytorch#178589.

## Fix Approach

- Land the OpInfo update in pytorch/pytorch#178589 setting gradcheck_nondet_tol for index_reduce mean/prod.
- then unskip the four cases in the torch-xpu-ops skip list.
- No kernel change needed.
