## Root Cause

torch.linalg.svd has no XPU backend; falls back to CPU at RegisterXPU_0.cpp:54525, blocking PEFT
LOFTQ/PISSA finetuning (~92% of total time). Benchmarks by @CuiYifeng show oneMKL XPU gesvd is
0.3-1.7x CPU (mostly slower) per oneMKL doc 'some computations may be performed on CPU'. The 'hang
forever' symptom was a NUMA binding artifact when CPU fallback ran multi-card.

## Fix Approach

- meanwhile keep CPU fallback with documented NUMA-bind guidance.
- If implementing, add linalg_svd via oneapi::mkl::lapack::gesvd in src/ATen/native/xpu/BatchLinearAlgebra.cpp and gate by shape/perf heuristic.
