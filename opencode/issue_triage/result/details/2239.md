## Root Cause

functorch test_ops_xpu vjp/grad/vmap variants on conv_transpose2d/3d and conv3d hit 'could not
create a primitive descriptor for the deconvolution forward propagation primitive' from the
oneDNN-backed XPU deconvolution path when given the SampleInput at index 2 (stride=(3,2),
padding=(1,2), output_padding=(2,3), groups=2, dilation=(4,4)). The XPU Deconvolution.cpp wrapper
passes parameter combinations that oneDNN's deconvolution primitive descriptor cannot create. Per
comments, upstream PR pytorch/pytorch#176420 fixes transpose3d and torch-xpu-ops PR #2995 fixes
conv3d.

## Fix Approach

- Land pytorch/pytorch#176420 and torch-xpu-ops#2995, which guard or reshape the parameters before calling the oneDNN deconvolution primitive descriptor (or skip incompatible OpInfo samples upstream).
- After landing, unskip the residual conv_transpose3d and conv3d functorch variants.
