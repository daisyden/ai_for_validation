## Root Cause

Upstream test expects nondeterministic alert string 'histc_xpu with floating point input' (mirroring
CUDA at aten/src/ATen/native/cuda/SummaryOps.cu:411 which emits '_histc_cuda with floating point
input'), but the XPU impl at third_party/torch-xpu-ops/src/ATen/native/xpu/SummaryOps.cpp:45
unconditionally calls `globalContext().alertNotDeterministic("_histc_xpu")` without the ' with
floating point input' suffix and without the float-dtype gating. Hence assertRaisesRegex fails to
match the message.

## Fix Approach

- Edit third_party/torch-xpu-ops/src/ATen/native/xpu/SummaryOps.cpp:_histc_xpu to call `globalContext().alertNotDeterministic("_histc_xpu with floating point input")` only when `self.is_floating_point()`, mirroring cuda/SummaryOps.cu:405-411.
- PR #3333 is already in flight implementing this.
- merge and close the issue.
