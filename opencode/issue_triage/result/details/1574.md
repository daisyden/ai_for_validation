## Root Cause

aten::_grouped_mm has no XPU registration; dispatcher raises NotImplementedError for XPU device.
Required for MoE models and downstream scaled_grouped_mm (FP8). Per LuFinch's comment the plan is to
implement via oneDNN v3.13 in PyTorch 2.15; FP8 scaled_grouped_mm targeted for 2.16 and depends on
this landing first.

## Fix Approach

- Add XPU kernel for _grouped_mm under src/ATen/native/xpu/ that dispatches to oneDNN matmul primitive with group descriptor (requires oneDNN >=3.13).
- Register in native_functions or yaml_native_functions.yaml, add op_ut coverage, then build on top for _scaled_grouped_mm (FP8).
- Track with milestone PT2.15.
