## Root Cause

`aten::_grouped_mm` (and `_scaled_grouped_mm`) has no XPU registration; calls fall through to a
NotImplementedError. No kernel exists under `third_party/torch-xpu-ops/src/ATen/native/xpu/` for
grouped_mm, while CUDA implements it via cuBLAS grouped GEMM and uses it in MoE / FP8 paths. Per
maintainers, plan is to back grouped_mm with oneDNN v3.13 in PT 2.15, with FP8 `_scaled_grouped_mm`
deferred to ~2.16.

## Fix Approach

- Add an XPU implementation of `aten::_grouped_mm` using oneDNN v3.13 grouped matmul primitives, register it via `XPU_OP_REGISTER`/native_functions.yaml override, then layer FP8 `_scaled_grouped_mm` once oneDNN exposes scaled grouped GEMM.
- Place the kernel under `src/ATen/native/xpu/GroupedMM.cpp` with oneDNN bridge in `src/ATen/native/xpu/mkldnn/`.
- Add op_ut coverage mirroring CUDA tests.
