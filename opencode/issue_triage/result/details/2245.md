## Root Cause

test_addmm_errors at test/test_sparse_csr.py:2143 expects a 'crow_indices[..., 0] == 0' RuntimeError
when addmm receives a malformed expanded tensor, but XPU dispatches to oneDNN matmul which raises a
different message ('could not create a primitive descriptor for the matmul primitive'). The XPU
backend lacks the upstream input-validation check that exists for CUDA/CPU before falling through to
the BLAS primitive. Fix PR https://github.com/pytorch/pytorch/pull/180985 adds the check on the
stock PyTorch side.

## Fix Approach

- Land upstream PyTorch PR #180985 to add the expanded-input shape validation in the addmm meta/dispatch path so XPU surfaces the same crow_indices error message before reaching oneDNN.
- then unskip the test in torch-xpu-ops.
