## Root Cause

Generic sparse_compressed_to_dense in aten/src/ATen/native/TensorConversions.cpp:699 only unsqueezes
crow/col indices when batch_ndim==0 (line 724). When the user constructs a sparse_csr_tensor with
size=(1,3,3) (batch_ndim=1) but 1-D crow_indices/col_indices/values (i.e., unbatched indices relying
on broadcasting), the function skips the unsqueeze branch, so at line 792 dense.index_add_(0,
offsets, values.flatten(0,1)) hits `values.flatten(0,1)` where values is 1-D of shape [5] and dense
has shape [3,3], producing the error 'source tensor shape must match self tensor shape, excluding
the specified dimension'. This is a generic aten bug (not XPU-specific) but surfaces on XPU because
CSR to_dense dispatches to this composite implementation.

## Fix Approach

- In sparse_compressed_to_dense (TensorConversions.cpp:699-803), detect when indices batch dims are missing relative to tensor batch dims and either (a) expand/unsqueeze+expand crow_indices/plain_indices/values to match self.size() batch dims before the existing flatten(0, batch_ndim-1) path, or (b) reject such inputs earlier with a clearer error and fix sparse_csr_tensor constructor to validate that indices carry batch dims.
- Mirror any handling already in CUDA/CPU convert_indices path.
- This is an upstream PyTorch fix.
- no XPU kernel changes required.
