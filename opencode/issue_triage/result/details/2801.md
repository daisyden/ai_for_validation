## Root Cause

Calling to_dense() on a batched sparse CSR tensor (size=(1,3,3)) on XPU fails with 'source tensor
shape must match self tensor shape... self.shape=[3,3] source.shape=[5]'. The XPU sparse-CSR path in
third_party/torch-xpu-ops/src/ATen/native/sparse/xpu/SparseCsrTensorMath.cpp (around L292
numBatchDimensions / getCompressedPlainIndices and the to_dense conversion in
SparseCsrTensorMathKernels.cpp) does not broadcast a single batch index set across the batch dim, so
values (length 5) are scattered into a single (3,3) self instead of into a (1,3,3) batched output.

## Fix Approach

- In the XPU sparse CSR to_dense kernel (SparseCsrTensorMathKernels.cpp), broadcast crow_indices/col_indices/values along the batch dimension before scatter (mirror the CUDA implementation in aten/src/ATen/native/sparse/cuda/SparseCsrTensorMath.cu).
- Alternatively, register a CPU fallback for to_dense on batched CSR until the kernel handles batch broadcasting.
- Add a regression test in test/xpu/test_sparse_csr_xpu.py.
