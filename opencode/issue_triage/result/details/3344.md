## Root Cause

test/test_linalg.py:1758-1762 expects exactly one Python warning ('An output with one or more
elements was resized') when calling torch.linalg.cond(a, p, out=out) with a wrong-shape out tensor;
on XPU two warnings are produced. torch.linalg.cond decomposes into multiple sub-ops (norm,
matrix_norm, linalg_inv/svd) and on XPU some of these intermediate ops fall back to CPU/oneMKL paths
and resize the out tensor an extra time, emitting a second resize warning. Labelled `ut_upstream`
and `random` by reporter; not an accuracy/numerics issue.

## Fix Approach

- Audit torch.linalg.cond's lowering on XPU (torch/_refs/linalg.py and aten/src/ATen/native/mkldnn/xpu/Blas.cpp / linalg fallbacks) to find the sub-op that issues the extra resize_output_check warning.
- either ensure the intermediate temporary uses an internally-allocated buffer (avoiding the public resize warning) or, if behavior is acceptable, mirror upstream and skip/expectedFailure the test in test/xpu/skip_list_common.py with a pointer to this issue.
