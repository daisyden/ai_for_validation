## Root Cause

DDP test test_ddp_apply_optim_in_backward[_grad_as_bucket_view_false] in
test/distributed/test_distributed_spawn.py reproducibly fails only with SGD optimizer (not Adam) on
XPU/XCCL backend: 'Tensor-likes are not close! Mismatched 16/3072, max abs diff 0.0070, Params not
equal at iteration 4'. Per maintainer triage, problem only appears when optim-in-backward is fused
with SGD; suggests bucket-reduction / gradient-accumulation ordering in ProcessGroupXCCL allreduce
path (third_party/torch-xpu-ops/src/xccl/ProcessGroupXCCL.cpp) interacts badly with the in-backward
optimizer hook so SGD applies updates against a not-yet-finalized gradient. Adam masks this because
of its momentum buffer averaging.

## Fix Approach

- Reproduce locally with the daisyden/distributed_2.10 branch wheels, compare allreduce ordering vs NCCL on the same DDP path, and check the optim_in_backward hook ordering in torch/nn/parallel/distributed.py against the XCCL Work completion semantics.
- Likely fix: ensure XCCL Work::wait()/finalize is called before the DDP comm hook fires the SGD step (sync point in ProcessGroupXCCL::allreduce).
