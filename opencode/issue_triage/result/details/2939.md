## Root Cause

gmlp_s16_224 inference AMP bf16/fp16 on PVC 1100 regressed ~15% between torch 2.10 and 2.11. Owner
jianyizh confirms the cause is an Inductor reduction-hint change: generated Triton reduction kernels
flipped from `ReductionHint.DEFAULT` to `ReductionHint.INNER`, which picks a smaller XBLOCK/RBLOCK
tiling that under-utilizes XPU SIMD lanes for gmlp's (128, inner-small) reductions. This is the same
class of regression as issue 2952 and is driven by upstream torch/_inductor/runtime heuristic
changes rather than a torch-xpu-ops kernel.

## Fix Approach

- In the Inductor XPU heuristics (torch/_inductor/runtime/triton_heuristics.py and hints.py), add an XPU-specific override that keeps DEFAULT hint (or picks a larger RBLOCK config) for reductions with small inner dim on PVC, or register extra autotune configs for ReductionHint.INNER on XPU so the tuner can reach the pre-regression shape.
- Validate with the reproducer command in the issue body against release/2.10 to confirm parity.
- File against upstream pytorch inductor with XPU perf evidence.
