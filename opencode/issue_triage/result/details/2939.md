## Root Cause

release/2.11 inductor inference regression on timm gmlp_s16_224 amp (~15% drop vs 2.10) reproduced
via benchmarks/dynamo/timm_models.py with --backend=inductor on PVC Max 1100. No SYCL/aten kernel
implicated; the 2.10→2.11 delta with identical model code points to triton-xpu 3.7.0 codegen /
inductor scheduler / pre-grad pass changes between releases. No torch-xpu-ops source line; failure
is in generated triton kernels invoked by inductor.

## Fix Approach

- Bisect torch-xpu-ops + intel-xpu-backend-for-triton commits between 2.10 and 2.11 release branches against benchmarks/dynamo/timm_models.py --only gmlp_s16_224 --inference --amp.
- capture TORCHINDUCTOR_TRACE / output_code.py for both versions and diff fused triton kernel shapes/num_warps/block sizes.
- File a triton-xpu issue with the offending kernel if heuristic regression confirmed.
- otherwise revert/tune the inductor lowering or max-autotune config that changed.
