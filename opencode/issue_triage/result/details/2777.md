Two intermittent/sporadic failures on PVC: (1)
test_meta_xpu.TestMetaXPU.test_dispatch_symbolic_meta_outplace_all_strides_to_sparse_xpu_float32
exercising the meta-tensor `to_sparse` stride dispatch path (test/test_meta.py) and (2)
test_sparse_xpu.TestSparseAnyXPU.test_reductions_backward_sum_SparseCOO_xpu_float64 which hits the
SparseCOO reduction backward path implemented in
third_party/torch-xpu-ops/src/ATen/native/sparse/xpu/SparseTensorMath.cpp. AKloniecki ran 100 reps
on PVC without reproducing, and on BMG the fp64 case fails simply for lack of fp64 support, so the
actual defect (if any) is a flakiness specific to some driver/toolchain combination and no
deterministic root cause in source has been identified.
