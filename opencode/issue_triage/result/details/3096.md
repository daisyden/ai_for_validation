## Root Cause

test/inductor/test_max_autotune.py:3980 is gated with @skipIfXpu(msg='XPU not support
VISIBLE_DEVICES') because Inductor's TuningProcessPool (torch/_inductor/autotune_process.py) only
masks CUDA_VISIBLE_DEVICES; on XPU the analogous env vars are ZE_AFFINITY_MASK /
ONEAPI_DEVICE_SELECTOR and the autotune sub-process spawn code does not translate device selection
for xpu. Additionally torch.xpu has no device-visibility wrapper equivalent to torch.cuda's
CUDA_VISIBLE_DEVICES handling.

## Fix Approach

- In torch/_inductor/autotune_process.py extend TuningProcessPool.initialize / _subproc_env to set ZE_AFFINITY_MASK (or ONEAPI_DEVICE_SELECTOR='level_zero:<idx>') for xpu devices when autotune_multi_device is enabled.
- factor device-visibility env selection behind a helper keyed on current device type.
- Update test_max_autotune.py::test_tuning_pool_multiple_devices to parametrize over device and use the corresponding env var, then remove the @skipIfXpu decorator.
- Verify torch.xpu.device_count() respects the env var (runtime support exists in Level-Zero driver).
