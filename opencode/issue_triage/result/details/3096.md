## Root Cause

test_max_autotune.py::TestTuningProcessPool::test_tuning_pool_multiple_devices spawns autotuning
subprocesses keyed by CUDA_VISIBLE_DEVICES. The Inductor TuningProcessPool in
torch/_inductor/autotune_process.py uses CUDA_VISIBLE_DEVICES exclusively to pin per-worker devices;
ZE_AFFINITY_MASK / ONEAPI_DEVICE_SELECTOR equivalents for XPU are not wired through, so multi-device
autotune cannot select an XPU per worker.

## Fix Approach

- Extend TuningProcessPool/select_device logic in torch/_inductor/autotune_process.py to set ZE_AFFINITY_MASK (or ONEAPI_DEVICE_SELECTOR) for XPU workers, and add an XPU branch in test_tuning_pool_multiple_devices.
- upstream the change to pytorch/pytorch.
