## Root Cause

vision::roi_align on XPU is implemented in
third_party/torch-xpu-ops/src/ATen/native/xpu/sycl/RoiAlignKernels.cpp (registered via
src/ATen/native/xpu/RoiAlign.cpp). Under autocast with x in fp32 and rois in bf16, 90.8% of elements
diverge from the reference but with small absolute deltas (max 1.3e-3 vs 1e-5 tolerance), indicating
the kernel performs bilinear-interpolation arithmetic in the bf16 rois dtype rather than promoting
to fp32 like the CUDA torchvision kernel. Only the rois_dtype1 (bf16) variants fail; rois_dtype0
passes.

## Fix Approach

- In RoiAlignKernels.cpp, ensure the kernel templates on x's dtype and casts rois coordinates to fp32 before computing bilinear-interpolation weights/positions.
- accumulate output in fp32 then cast back.
- Mirror the CUDA torchvision RoIAlign kernel template parameterization.
- Verify against test_torchvision_roi_ops.py::TestRoIAlign::test_autocast all parameterizations.
- Skip via Known_issue.log Cases: lines until kernel fix lands.
