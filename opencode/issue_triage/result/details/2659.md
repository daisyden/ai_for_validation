## Root Cause

`test_dist2.py::ProcessGroupNCCLTest::test_group_split` and `test_remote_group_merge` call
`group.split_group([0], timeout=...)`, which in torch/csrc/distributed/c10d/ProcessGroup.cpp:190
calls `parentBackend->getBackendOptions()`. The XCCL backend `ProcessGroupXCCL`
(third_party/torch-xpu-ops/src/xccl/ProcessGroupXCCL.hpp around line 179) defines its own `Options`
struct and a `getOptions()` accessor but never overrides the virtual `Backend::getBackendOptions()`
from torch/csrc/distributed/c10d/Backend.hpp:132, so the base implementation throws `RuntimeError:
Backend xccl does not implement getBackendOptions.` Additionally, `splitGroup()`/comm-split is not
yet implemented for XCCL (Chao1Han confirms WIP).

## Fix Approach

- In third_party/torch-xpu-ops/src/xccl/ProcessGroupXCCL.hpp, add an `override` right next to `getOptions()` mirroring ProcessGroupNCCL.hpp:797: `c10::intrusive_ptr<Backend::Options> getBackendOptions() override { return c10::static_intrusive_pointer_cast<Backend::Options>(options_).
- }`.
- Then implement `splitGroup` / `mergeRemoteGroup` in ProcessGroupXCCL.cpp using `ccl::create_communicator` with a sub-world split (oneCCL `comm_split`), matching NCCL's `ncclCommSplit` flow.
- Gate the new tests behind a capability flag until both pieces land.
