## Root Cause

On Windows + Arc B580 (BMG) with PyTorch 2.11.0+xpu, 5 e2e accuracy benchmarks (DistillGPT2,
AllenaiLongformerBase, openai/whisper-tiny in HF amp-bf16; pit_b_224 in timm amp-fp16; timm_regnet
in torchbench bf16 training) regress from pass on 2.10 to fail_accuracy on 2.11. The same failure
reproduces with old oneAPI, narrowing the regression to pytorch/torch-xpu-ops kernel changes between
2.10 and 2.11 rather than oneAPI/driver. The dominant ops in these models are conv/linear/SDPA which
on XPU go through oneDNN (`third_party/torch-xpu-ops/src/ATen/native/xpu/Conv.cpp`, `Linear.cpp`,
`Attention.cpp`).

## Fix Approach

- Bisect torch-xpu-ops commits between release/2.10 and release/2.11 against the 5 affected models to find the regressing PR.
- check oneDNN-backed conv/linear bf16 accumulator and SDPA bf16/fp16 paths.
- verify amp autocast lists are unchanged.
- inspect each failure log for the first divergent op and gate fix accordingly.
