## Root Cause

On BMG-Windows PT2.8, multiple UTs fail with accuracy mismatches. The complex128 prod reductions
(test_reductions_xpu.py) failed due to first-dim reduction producing incorrect 0+0j results in
third_party/torch-xpu-ops/src/ATen/native/xpu/sycl/ReduceSumProdKernels.cpp; these were fixed via PR
#2477. The remaining failure test_GroupNorm_general_xpu stems from accumulator float-rounding in
GroupNormKernels.cpp causing stacked rounding error vs CPU reference (per PR #3072).

## Fix Approach

- Land PR #3072 (GroupNorm accumulator/dtype fix) which addresses the remaining stacking float-rounding error in GroupNormKernels.cpp.
- Verify all listed prod tests pass post #2477, then close this issue.
