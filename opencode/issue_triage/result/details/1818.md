## Root Cause

Per kdrozd-dev comments, the prod complex128 failures (majority of the list) were float-rounding
issues in the XPU reduction path and are fixed by PRs #2477 / #3072 (pending merge). The remaining
offender, test_GroupNorm_general_xpu, fails due to accumulated float rounding error in the XPU
GroupNorm welford/row-reduction implementation under the specific shape exercised by this test — an
algorithmic numerics issue, not a correctness bug per se.

## Fix Approach

- Merge #3072 to resolve the complex128 prod cases.
- For GroupNorm, either (a) raise to fp64 accumulators in the welford two-pass reduction in GroupNormKernels.cpp like CUDA does, or (b) add a tolerance bump for this specific shape in the skip-list/xpu_test_base.
- re-run the failing UT on BMG Windows to confirm.
- Close the issue once both PR #3072 merges and the GroupNorm accumulator fix lands.
