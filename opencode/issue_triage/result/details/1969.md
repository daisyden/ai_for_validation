## Root Cause

Dynamo's VariableBuilder at torch/_dynamo/variables/builder.py:1043 calls store_user_object_weakref
on the torch.Event instance passed to stream.wait_event(e); torch/_dynamo/utils.py:4689 does
weakref.ref(obj), but torch.Event (a C-binding object) doesn't support weak references, raising
TypeError. This is an upstream PyTorch Dynamo limitation around the accelerator Event abstraction,
reproduced when the xpu test test_gpu_event_across_graph_break exercises the graph-break path
through torch.accelerator.current_stream().wait_event(). Per assignee guangyey (2026-04-02), the UT
is now skipped on main and torch.Event is correctly traced in current dynamo, so the symptom no
longer reproduces.

- any residual fix belongs in upstream PyTorch Dynamo (allow weakref on torch.Event or special-case it in store_user_object_weakref).
