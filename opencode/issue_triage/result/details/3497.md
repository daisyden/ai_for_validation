## Root Cause

Llama-3.2-1B-Instruct dynamic INT8 next-token latency is 8% slower on the 0421 nightly vs 0409
nightly on BMG (Arc B580). Maintainer @xiaowangintel confirmed in the issue comments that profiling
localizes the regression to the oneDNN INT8 gemm_kernel — the same dynamic-quant matmul path called
from torchao's int8_dynamic_activation_int8_weight lowering. No torch-xpu-ops SYCL kernel changed in
the window; the perf delta is entirely in the oneDNN library version pulled in between the two
nightlies.
