## Root Cause

torch.linalg.cholesky has no XPU backend implementation in torch-xpu-ops; calls fall back to CPU as
warned at PyTorch's RegisterXPU_0.cpp:54523. The op is not registered for XPU in
third_party/torch-xpu-ops/yaml/native/native_functions.yaml (no linalg_cholesky_ex entry), so the
dispatcher routes via CPU fallback. Used by llm-compressor GPTQ (gptq_quantize.py:160) and impacts
upstream UTs per maintainer comments.

## Fix Approach

- Implement linalg_cholesky_ex_out for XPU via oneMKL (potrf/potrs from oneapi::mkl::lapack).
- register in yaml/native/native_functions.yaml and add a wrapper in src/ATen/native/xpu/BatchLinearAlgebra.cpp following the existing oneMKL linalg pattern.
