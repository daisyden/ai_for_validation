## Root Cause

torch.linalg.cholesky is not registered for XPU in torch-xpu-ops (no Cholesky kernel under
third_party/torch-xpu-ops/src/ATen/native/xpu/, glob confirms no Cholesky* file). The dispatcher
therefore triggers the generic CPU fallback, producing the UserWarning from
build/xpu/ATen/RegisterXPU_0.cpp:54523 seen in llm-compressor GPTQ (gptq_quantize.py:160, H =
torch.linalg.cholesky(H)). This also blocks several upstream PyTorch linalg UTs on xpu (per daisyden
2025-11-19).

## Fix Approach

- Implement at::linalg_cholesky_ex (the structured op that cholesky is lowered to) for XPU by wrapping oneMKL LAPACK potrf (via sycl/onemkl::lapack::potrf or onemkl batched potrf for batched cases).
- Add the kernel under src/ATen/native/xpu/BatchLinearAlgebra.cpp (or new Cholesky.cpp), register the ex + info path, cover fp32/fp64/complex, and remove the CPU fallback entry from fallback_list.yaml / generated RegisterXPU.
- Owner mwiktor-intel is already assigned.
- validate with test_linalg.py cholesky suite.
