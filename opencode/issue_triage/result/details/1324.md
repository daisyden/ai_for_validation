## Root Cause

On Windows LNL, when XPU OOMs during allocation, the SYCL/Level-Zero driver corrupts existing tensor
contexts: subsequent access to previously-valid tensors returns UR_RESULT_ERROR_OUT_OF_RESOURCES
(40) and then a generic UR error. Per maintainer trace (Stonepia/guangyey), this is a
compute-runtime/UR backend bug - the OOM path inappropriately tears down or invalidates the context
queue. JIRA GSD-10738/GSD-11849/GSD-11670 track the driver fix; suspected related to driver
returning incorrect free memory.

## Fix Approach

- Fix in driver/compute-runtime (track GSD-11849, GSD-11670).
- PyTorch-side mitigation: in c10/xpu/XPUCachingAllocator.cpp, before calling sycl::malloc_device, query free memory and proactively raise OOM before driver corruption point.
- Depends on oneAPI 2026.1 + driver fix per chuanqi129.
