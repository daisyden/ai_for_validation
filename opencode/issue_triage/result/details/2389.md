## Root Cause

test_cond_errors_and_warnings_xpu_complex128 (test/test_linalg.py:1728) asserts exactly one warning
is raised when computing torch.linalg.cond on a singular complex128 matrix, but XPU emits two
warnings: both the expected ill-conditioned/singular warning from LAPACK path and an extra warning
(likely a duplicated emission from the oneMKL LAPACK wrapper getrf/gesvd path or from out= overload
warning filter reset). The initially reported 'Data corruption detected' error transitioned to an
assertEqual(len(w), 1) == 2 mismatch, indicating a duplicate runtime_warn emission in the XPU
linalg_cond implementation path.

## Fix Approach

- Audit torch-xpu-ops linalg kernels (src/ATen/native/xpu/LinearAlgebra*.cpp and the oneMKL wrappers in src/ATen/native/xpu/mkl/) for places where TORCH_WARN is emitted along both the factorization and the cond-number code paths.
- de-duplicate so only the singular-matrix warning fires once for complex128.
- Also inspect `linalg_cond` out-variant in aten/src/ATen/native/BatchLinearAlgebra.cpp for XPU-specific fallback that may double-warn.
- Verify with pytest -k test_cond_errors_and_warnings_xpu_complex128.
