## Root Cause

Four SDPA failure-mode tests assume CUDA-specific behavior.
test_fused_backwards_throws_determinism_warning_fused_kernel1_warn_only_True expects a UserWarning
about non-determinism, but XPU flash/mem-efficient backward path has no determinism support (would
hard-error instead). The other three (test_invalid_fused_inputs_dim_3_kernel2,
test_invalid_last_dim_stride_kernel2, test_mask_invalid_last_dim_stride_kernel1) exercise the cuDNN
attention backend (kernel2 / kernel1-cudnn) which XPU does not support; test_transformers_xpu.py
relies on PLATFORM_SUPPORTS_CUDNN_ATTENTION being True.

## Fix Approach

- Per maintainer guidance: set PLATFORM_SUPPORTS_CUDNN_ATTENTION=False for XPU in test_transformers_xpu.py so the three cuDNN-kernel tests are gated off, and add a skip for test_fused_backwards_throws_determinism_warning on XPU until oneDNN/SYCLTLA exposes a non-deterministic warning path.
- Add corresponding skip entries to xpu_test_utils.py / skiplist.
