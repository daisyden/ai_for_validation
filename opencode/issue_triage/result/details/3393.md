## Root Cause

In test/dynamo/test_activation_checkpointing.py::test_pattern_matcher (line 1883+), the test
hard-codes CUDA-only backend selection logic using torch.cuda.get_device_properties and checks
aten._scaled_dot_product_flash_attention, so on XPU the SDPA pattern matcher picks a different fused
op (or the debug inner-compile hook is never invoked), leaving aot_graphs empty and triggering
IndexError at line 1933.

## Fix Approach

- Generalize the test's backend-selection branch to handle XPU (use torch.xpu.get_device_properties and the XPU-appropriate sdpa op, or gate via SM80OrLater / requires_cuda), tracking the upstream fix in pytorch/pytorch#169241.
- meanwhile keep the test in the torch-xpu-ops skip list.
