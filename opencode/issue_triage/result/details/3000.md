## Root Cause

test_nested_tensor_multiprocessing_context_{forkserver,spawn}_xpu in
third_party/torch-xpu-ops/test/xpu/test_dataloader_xpu.py:223 fails because the DataLoader worker
pickling path calls storage._share_fd_cpu_() in torch/multiprocessing/reductions.py:615 to send
tensors across processes. XPU Storage does not implement _share_fd_ (it is CPU-only per
torch/storage.py:527), so shared-memory FD reduction raises 'RuntimeError: _share_fd_: only
available on CPU'. Existing skip in skip_list_common.py:97-100 only covers
test_torch_xpu.test_module_share_memory_xpu, not the two dataloader tests.

## Fix Approach

- Short-term: extend skip_list_common.py with 'test_dataloader_xpu.py': ('test_nested_tensor_multiprocessing_context_forkserver_xpu', 'test_nested_tensor_multiprocessing_context_spawn_xpu') to match the companion PR #2989.
- Long-term: implement XPU StorageImpl._share_fd_ via a SYCL shared-memory / ze_ipc_mem_handle path in torch-xpu-ops (mirror CUDA's MapAllocator flow) so tensors allocated on XPU can be serialized across worker processes, then remove the skip and register the function in torch/multiprocessing/reductions.py.
