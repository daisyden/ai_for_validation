## Root Cause

All 11 AOTAutogradCachePicklerTests in test/xpu/dynamo/test_aot_autograd_cache_xpu.py fail at
torch/_functorch/_aot_autograd/autograd_cache.py:749 with AttributeError: 'dict' object has no
attribute 'cudagraphs'. create_fx_config (autograd_cache.py:714) hardcodes cudagraphs =
BoxedBool(torch._inductor.config.triton.cudagraphs); CompilerConfigExtra exposes only a CUDA-named
'cudagraphs' field with no XPU equivalent ('xpugraphs') and no device-agnostic plumbing, so the XPU
port has no valid value to pass.

## Fix Approach

- Upstream PyTorch fix: either add an 'xpugraphs' field to CompilerConfigExtra and torch._inductor.config.triton, plumbed through create_fx_config, or generalise to a device-agnostic name (e.g. device_graphs) selected by GPU_TYPE.
- Until then on torch-xpu-ops side: pass compiler_config_extra=None (matches pytorch/test/dynamo/test_aot_autograd_cache.py:3211) or construct CompilerConfigExtra(cudagraphs=BoxedBool(False), forward_device=None).
- Skip the 11 cases for now.
