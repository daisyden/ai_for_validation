## Root Cause

eca_halonext26ts fp16 inductor training fails with 'L0 build module failed. Log: IGC: Internal
Compiler Error: Floating point exception' → ZE_RESULT_ERROR_MODULE_BUILD_FAILURE during Triton
kernel JIT. Crashes with libigc2 2.28.4-1228 / intel-opencl-icd 26.05.37020.3 (driver 26.05) on BMG;
last-known-good is libigc2 2.24.8 / 25.48.36300.8 (driver 25.48). The IGC compiler hits an FP
exception while lowering a Triton-generated SPIR-V kernel — pure compiler bug in newer IGC.

## Fix Approach

- File a driver-team bug against IGC 2.28.4 with the failing Triton-generated SPV/LLVM IR (extract via TRITON_CACHE_DIR) for eca_halonext26ts.
- Pin CI to libigc2 2.24.8 / driver 25.48 until IGC fix lands.
- no torch-xpu-ops code change.
- Track via dependency component: driver.
