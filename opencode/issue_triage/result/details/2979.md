## Root Cause

L0/IGC driver bug on BMG: when building a Triton-generated SPIR-V module for eca_halonext26ts, IGC
crashes with 'Internal Compiler Error: Floating point exception', causing
ZE_RESULT_ERROR_MODULE_BUILD_FAILURE at triton/backends/intel/driver.py:217 load_binary(). The crash
is environment-specific: it reproduces with intel-opencl-icd/intel-ocloc 26.05.37020.3 + libigc2
2.28.4 + libze1 1.27.0 on kernel 6.17, but not with the older 25.48.36300.8 + libigc2 2.24.8 stack.
No PyTorch/torch-xpu-ops code change is implicated.

## Fix Approach

- File a bug against the Intel compute-runtime/IGC driver team with the failing kernel's SPIR-V / AOT reproducer extracted from /tmp/torchinductor_root/ao/cao...py (already agreed in-thread by @chuanqi129).
- As a short-term workaround, pin the CI/runtime to the last-known-good driver stack (intel-opencl-icd 25.48.36300.8, libigc2 2.24.8, libze1 1.26.2).
- Once IGC ships a fix, upgrade and re-enable the test.
