## Root Cause

HF Deberta/DebertaV2 models crash under amp bf16/fp16 at
transformers/models/deberta/modeling_deberta.py:290 where `attention_scores.masked_fill(~mask,
torch.finfo(query_layer.dtype).min)` uses -inf / finfo.min from the original fp32 dtype and cannot
be represented in bf16, triggering 'value cannot be converted to type at::BFloat16 without overflow'
inside eager validate_model. This is an upstream HuggingFace transformers bug (discussed in
huggingface/transformers PR #35336), not a torch-xpu-ops kernel issue — it only surfaces first on
LNL Windows because that is where the benchmark runs bf16 eager.

## Fix Approach

- Wait for/land the transformers fix (PR #35336 or successor) that uses `torch.finfo(attention_scores.dtype).min` instead of query_layer.dtype.
- Pin a fixed transformers version in the LNL Windows E2E suite.
- no torch-xpu-ops code change required.
- Close once transformers release is adopted.
