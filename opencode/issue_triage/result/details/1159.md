## Root Cause

Eager run of HuggingFace DebertaForQuestionAnswering / DebertaV2ForMaskedLM fails in
transformers/models/deberta/modeling_deberta.py:290 at attention_scores.masked_fill(~attention_mask,
torch.finfo(query_layer.dtype).min) because under amp float16 the dtype-min sentinel is converted to
BFloat16 during a downstream cast and overflows. Owner Stonepia confirmed this is a
transformers-side bug being tracked by huggingface/transformers PR #35336, not an XPU kernel issue.
Reproduces on Windows LNL only because that branch enables amp float16 on Deberta.

## Fix Approach

- Wait for huggingface/transformers PR #35336 to land and bump pinned transformers version in the benchmarks/dynamo HuggingFace runner.
- no torch-xpu-ops change required.
- Once merged, re-run nightly LNL Windows accuracy job and close this issue.
