## Root Cause

timm xcit_large_24_p8_224 float16 training accuracy fails on PTL Windows under inductor with
similarity 0.974, key 'blocks.16.local_mp.conv2.weight.grad'. Same pattern as BMG Windows #2924. The
failure is in the conv backward gradient computation path under inductor on Windows — XCiT uses
depthwise/local_mp conv2 which on XPU lowers to oneDNN convolution_backward; fp16 accumulation
precision differs on Windows builds vs Linux, producing a gradient mismatch that Dynamo accuracy
comparator rejects.
