## Root Cause

The sycltla forward kernel dispatcher only instantiates specialized tile-shapes for headdim in
{64,96,128,192}; for any other value it falls into the else branch and aborts with
TORCH_CHECK(false, 'FlashAttentionForwardXPU only support headdim 64,96,128,192') at
third_party/torch-xpu-ops/src/ATen/native/transformers/xpu/flash_attn/sycltla/mha_fwd.cpp:348-349.
The Inductor SDPA tests (test_sdpfa_xpu, test_sdfpa_unbacked_strides_xpu,
test_fallback_kernel_with_symexpr_output_xpu) and the cudagraph fused_kernel0 transformer tests
exercise other head_dims (e.g. 32), so they get rejected before any kernel runs. Duplicate of #2698.

## Fix Approach

- Two-layer fix: (1) at the dispatcher in aten/native/transformers/xpu choose the math / mem-efficient SDPA backend when headdim is outside the supported set instead of forcing FlashAttention (the sdp::can_use_flash_attention heuristic needs an XPU-aware headdim check), as partially done in pytorch/pytorch#180646.
- (2) long-term, extend mha_fwd.cpp to add specialized TileShape instantiations for headdim 32 (and any other required dims) so FlashAttention is actually used instead of falling back.
