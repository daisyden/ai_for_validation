## Root Cause

FlashAttentionForwardXPU explicitly TORCH_CHECKs headdim in {64,96,128,192} at
third_party/torch-xpu-ops/src/ATen/native/transformers/xpu/flash_attn/sycltla/mha_fwd.cpp:349, but
the upstream cudagraph SDPA tests parametrize head_dim=32 which falls outside the supported set,
causing 'FlashAttentionForwardXPU only support headdim 64,96,128,192'. Marked as duplicate of #2698;
LuFinch reports the headdim selection has been fixed in pytorch/pytorch#180646 (dispatcher should
fall through to math/efficient backend for unsupported head_dim instead of erroring).

## Fix Approach

- Land/pick up pytorch/pytorch#180646 so the SDPA backend selector rejects flash for head_dim<64 before reaching mha_fwd.cpp:349 (fall through to efficient/math).
- Optionally widen the TORCH_CHECK message and gate the dispatcher in src/ATen/native/transformers/Attention.cpp.
- Note remaining cudagraph dependency on oneAPI 2026.0 (CMPLRLLVM-72057) for full enablement.
