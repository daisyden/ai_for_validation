## Root Cause

test_invalid_fused_inputs_invalid_dtype_kernel1 passes FP64 q/k/v and expects
scaled_dot_product_attention to raise RuntimeError, because CUDA FlashAttention/MemEfficient kernels
reject FP64. XPU's mem-efficient attention path in aten/src/ATen/native/mkldnn/xpu/Attention.cpp
falls back to the Math backend (see lines 222-228) which accepts FP64, so no error is raised and the
assertRaises fails. This is an expected behavioral divergence, not a bug.

## Fix Approach

- Skip the test on XPU (dtype-rejection expectation is CUDA-specific).
- Add test_invalid_fused_inputs_invalid_dtype_kernel1_xpu to the XPU skiplist in torch-xpu-ops test/xpu/skip_list_common.py with a comment explaining XPU mem-eff falls through to Math backend.
