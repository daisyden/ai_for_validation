## Root Cause

Tracking enhancement, not a bug. Several XPU kernels (e.g., LayerNorm, GroupNorm) under
third_party/torch-xpu-ops/src/ATen/native/xpu/ have drifted from the CUDA reference in
aten/src/ATen/native/cuda/ (e.g., layer_norm_kernel.cu, group_norm_kernel.cu). The XPU ports were
forked from older CUDA revisions and miss subsequent numerical/perf improvements, increasing
long-term maintenance cost and producing subtle numerical divergences with CUDA.

## Fix Approach

- Port-forward each listed kernel (LayerNorm first, then GroupNorm, etc.) by diffing the XPU .cpp against the current aten/src/ATen/native/cuda/*.cu equivalent and re-implementing parallelization primitives with SYCL/oneDNN counterparts while keeping the algorithmic structure identical.
- Validate each with op_ut tests and inductor accuracy runs.
- track as a series of PRs under this parent task.
