## Root Cause

Many ATen op helpers (shape/stride inference, reduction/broadcast utilities, etc.) are duplicated
locally in torch-xpu-ops/src/ATen/native/xpu instead of being shared with CPU/CUDA because the
upstream helpers are defined in .cpp files, not headers. This is a long-term refactor tracking item,
not a bug; noted by fengyuan14 as multi-release effort.

## Fix Approach

- Audit torch-xpu-ops/src/ATen/native/xpu for duplicated utilities (e.g., sum_to, permute_dims helpers, inferExpandGeometry variants), file upstream PyTorch PRs to promote them from .cpp to .h in aten/src/ATen/native, and then delete XPU copies and include the shared headers.
- Prioritize by copy-count.
