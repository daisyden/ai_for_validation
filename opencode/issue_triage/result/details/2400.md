## Root Cause

The `tf32_on_and_off` decorator is defined only in torch/testing/_internal/common_cuda.py:332 and
toggles `torch.backends.cuda.matmul.allow_tf32` / `torch.backends.cudnn.flags`. Upstream UTs (e.g.
test/test_nestedtensor.py:33) import and apply it unconditionally, so on XPU runs the decorator
either no-ops or leaves TF32 behavior inconsistent, and there is no corresponding XPU-side switch to
control matmul precision during tests.

## Fix Approach

- Add an XPU counterpart in torch/testing/_internal/common_cuda.py (or a new common_xpu.py) that, when device==xpu, toggles `torch.backends.mkldnn.matmul.fp32_precision` / the XPU tf32 equivalent (currently exposed via `torch.xpu` / oneDNN math-mode setting).
- Have `tf32_on_and_off` detect the active accelerator and switch the right backend, or introduce a `tf32_on_and_off_xpu` alias used by the XPU UT shards in torch-xpu-ops.
- Coordinate with CUDA team on a unified `@with_tf32` decorator.
