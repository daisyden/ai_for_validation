## Root Cause

Many upstream tests use `@tf32_on_and_off` from torch/testing/_internal/common_cuda.py:332 which is
CUDA-only. XPU has no equivalent decorator, so XPU enablement of tests like
test/test_nestedtensor.py:33 is blocked or runs without exercising the tf32 toggle.

## Fix Approach

- Add an XPU equivalent (e.g., `tf32_on_and_off` in torch/testing/_internal/common_xpu.py or generalize the existing one to dispatch by device type) that toggles `torch.backends.mkldnn.allow_tf32` / XPU-specific tf32 knobs.
- Submit upstream PR to pytorch/pytorch and update tests to use the device-generic decorator.
