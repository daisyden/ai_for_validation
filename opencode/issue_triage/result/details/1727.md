## Root Cause

FSDP test test_non_root_forward_backward calls torch.get_device_module(device_type)._sleep() at
test/distributed/_composable/fsdp/test_fully_shard_training.py:521, but torch.xpu module does not
expose a _sleep helper (CUDA registers it via THCPModule_initExtension in
torch/csrc/cuda/Module.cpp; XPU equivalent torch/csrc/xpu/Module.cpp lacks the binding). Per
maintainer guangyey, the underlying SYCL primitive needed for a busy-wait/sleep on device requires
oneAPI 2026 features.

## Fix Approach

- Add torch.xpu._sleep binding in torch/csrc/xpu/Module.cpp once oneAPI 2026 sycl extension is available.
- meanwhile gate or skip the FSDP test on XPU.
- Track via oneAPI dependency and follow up with upstream PyTorch PR mirroring CUDA's _sleep implementation.
