## Root Cause

Stale test list / test instantiation. test_conv2d_hipdnn_backend_selection_xpu and
test_conv2d_hipdnn_deterministic_error_xpu come from upstream test/nn/test_convolution.py and are
ROCm/hipDNN-specific backend tests that have no meaning on the XPU backend (there is no hipDNN on
Intel GPU); torch-xpu-ops' test wrapper in
third_party/torch-xpu-ops/test/xpu/nn/test_convolution_xpu.py instantiates them on XPU via
instantiate_device_type_tests. The third case
(test_mem_eff_attention_large_seq_len_uniform_attention_xpu) was confirmed in comments to be a
random crash and has been moved to issue #3326.

## Fix Approach

- Add the two hipdnn tests to skip_list_common.py under test_convolution_xpu (they are ROCm-only.
- @onlyROCM or equivalent guard is missing upstream), or push a decorator fix upstream so these backend-selection tests are gated by torch.version.hip.
- The SDPA random-crash case is tracked separately in #3326 and can be closed from this issue.
