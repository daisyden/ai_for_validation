## Root Cause

test_side_stream_backward_overlap (test/test_autograd.py:14275, check_ordering at line ~14301)
asserts that side-stream backward correctly overlaps with main-stream work. The XPU stream
synchronization/scheduling path lacks the deterministic ordering primitive (a torch.xpu._sleep /
sleep kernel equivalent to torch.cuda._sleep) that this test depends on, so check_ordering() returns
False. As noted by @pponikox, fix is blocked on adding the XPU sleep utility tracked in
torch-xpu-ops issue #1986.

## Fix Approach

- Implement torch.xpu._sleep (a busy-wait SYCL kernel that holds the stream for N cycles) as part of issue #1986, then audit autograd stream-overlap scheduling logic in PyTorch core (torch/csrc/autograd/engine.cpp around stream-readiness checks) to ensure XPU events/streams behave like CUDA.
- Re-enable the test once the sleep primitive lands.
- Until then, keep the test in skip_list.
