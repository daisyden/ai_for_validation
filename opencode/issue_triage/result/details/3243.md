## Root Cause

test_side_stream_backward_overlap in test/test_autograd.py (lines ~14280-14320) relies on
CUDA-specific _sleep_if_cuda to hold the side stream busy so that the ordering assertion
(side_backward_start < main_backward_start) becomes deterministic. On XPU there is no equivalent
device-side busy sleep, so the two backward streams run too fast for the timing-based assertThat
check and the ordering test returns False. This is a test-harness gap rather than an autograd bug.

## Fix Approach

- Add an XPU equivalent to _sleep_if_cuda (e.g. a torch.xpu._sleep helper that busy-loops on device) and call it in the side-stream branch, or skip/xfail the test on XPU until such a primitive exists.
- Currently the test is already in the skipped list (labels: module: ut, skipped).
