## Root Cause

test_ctx_manager_xpu.py::CtxManagerTests::test_cuda_device fails because Dynamo refuses to inline
torch.xpu.device.__init__: torch/xpu/__init__.py is on Dynamo's MOD_SKIPLIST, so when the test
invokes 'with torch.get_device_module(device_type).device(x.device.index - 1):' the polyfill
instantiate_user_defined_class_object hits 'Attempted to inline function marked as skipped'
(torch/_dynamo/polyfills/__init__.py:424). This is a Dynamo skip-list / polyfill gap in upstream
PyTorch, fixed by pytorch/pytorch PR #179905.

## Fix Approach

- Track upstream PR pytorch/pytorch#179905 (allow tracing torch.xpu.device or remove from MOD_SKIPLIST).
- Keep test skipped in torch-xpu-ops until merged, then unskip.
