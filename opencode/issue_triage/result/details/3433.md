## Root Cause

DCGAN inductor compilation fails at the Intel Triton backend's make_llir stage
(triton/backends/intel/compiler.py:401 `pm.run(mod, 'make_llir')` raising `PassManager::run failed`)
for the generated `triton_poi_fused_convolution_0` kernel. This is a 2.11→2.12 regression caused by
either Inductor codegen producing IR the Intel LLIR pass cannot handle or by the
intel-xpu-backend-for-triton version bundled in the 2.12 wheel.

## Fix Approach

- File against intel-xpu-backend-for-triton with the failing TTIR.
- have Triton dump the IR before make_llir and reduce to a minimal repro.
- Either fix the LLIR conversion pass or add a Triton workaround in Inductor's XPU codegen for fused pointwise convolution.
- revert to known-good Triton commit if a regression range can be bisected.
