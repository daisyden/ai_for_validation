## Root Cause

Inductor's pad_mm autoheuristic unconditionally calls torch.cuda.get_device_capability() at
torch/_inductor/autoheuristic/autoheuristic.py:88 inside AutoHeuristic.__init__, triggered from
torch/_inductor/fx_passes/pad_mm.py:766 run_autoheuristic -> :658 _should_pad -> :799 should_pad_mm.
On XPU-only build CUDA is unavailable so _lazy_init raises 'Torch not compiled with CUDA enabled',
breaking inductor compile for huggingface/timm/torchbench. Regression introduced between pytorch
commits a886dc9 (good) and ebd76107 (bad).

## Fix Approach

- Upstream PyTorch fix: in autoheuristic.py replace hardcoded torch.cuda.get_device_capability() with device-aware lookup using the input tensor's device (xpu/cuda/cpu), or guard with torch.cuda.is_available().
- Alternatively short-circuit run_autoheuristic in pad_mm when device is non-CUDA.
- File PR to pytorch/pytorch.
