## Root Cause

Windows op_ut/extended UT runs hang then crash with access violation (c0000005). Reproducer narrowed
to functorch test_vmapjvpvjp_max_pool2d_with_indices_backward_xpu_float32. Native stack shows heap
corruption inside SYCL kernel JIT pipeline:
sycl8::ext::oneapi::experimental::detail::build_from_source -> nested pipe_base::wait_non_blocking
(x4) -> ntdll!RtlFreeHeap, triggered when functorch dispatches into
max_pool2d_with_indices_backward_kernel from
torch-xpu-ops/src/ATen/native/xpu/sycl/DilatedMaxPool2d.cpp. Bug is in the SYCL/L0 Windows runtime,
not the kernel logic.

## Fix Approach

- File against SYCL/Level-Zero Windows runtime with the cdb stack and minimal repro.
- Workaround in torch-xpu-ops: add the failing functorch test (vmapjvpvjp + max_pool2d_with_indices_backward + float32 and related decomp tests) to the Windows skip list in test/xpu/skip_list_win* until the SYCL JIT heap-corruption fix lands.
