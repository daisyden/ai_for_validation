## Root Cause

test_fmod_remainder_by_zero_integral_xpu_int64 fails with 'Tensor-likes are not equal! Mismatched
44/100' because the integral fmod/remainder by-zero behavior on XPU does not match CPU. The XPU
integer fmod/remainder kernel at
torch-xpu-ops/src/ATen/native/xpu/sycl/BinaryRemainderKernel.cpp:72-83 dispatches
AT_DISPATCH_INTEGRAL_TYPES through FmodIntegralFunctor/RemainderIntegralFunctor which produces a
different sentinel value for divide-by-zero than the CPU reference (test expects
fn(x,zero)==4294967295). test_cudnn_weight_format scalar mismatch is a related smaller miscompare.
PyTorch has a workaround PR #171238 (per WA TODO comment) that needs to be removed once fixed.

## Fix Approach

- In FmodIntegralFunctor/RemainderIntegralFunctor inside BinaryRemainderKernel.cpp, special-case b==0 to return the same sentinel as CPU (max-of-type for unsigned-cast int64).
- Then remove the WA in pytorch/pytorch#171238 and unskip both cases.
