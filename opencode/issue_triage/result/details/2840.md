## Root Cause

test_depthwise_conv_64bit_indexing_xpu in test/nn/test_convolution.py fails with ~0.2% mismatched
elements and inf relative diff on tensors requiring 64-bit indexing (>2.15B elements). XPU depthwise
convolution dispatches via oneDNN (Convolution.cpp -> oneDNN brgconv/dw); the oneDNN path used for
XPU does not correctly handle 64-bit index/offset arithmetic for very large activations, producing
wrong outputs at large spatial offsets.

## Fix Approach

- Confirm via oneDNN whether dw conv primitive supports int64 offsets on XPU.
- if not, in torch-xpu-ops Convolution dispatch fall back to a SYCL kernel (or chunk along batch/channel) when total element count exceeds INT_MAX.
- File oneDNN issue with shape (B,1,H,W) repro and add explicit 64-bit-indexing guard in the XPU depthwise path.
