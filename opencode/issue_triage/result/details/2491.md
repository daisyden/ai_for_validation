## Root Cause

Grouped failures with 'AssertionError: False is not true' across several tests.
test_cpp_warnings_have_python_context_{cpu,xpu} (test/test_torch.py:826) fails because warnings
issued from XPU SYCL kernels don't carry a Python source context (TORCH_WARN emitted without the
warning handler that CUDA uses). test_pdist_norm_large_xpu (test_torch.py:4110) hits numerical /
tolerance issues in the XPU pdist kernel on large inputs.
test_storage_all_devices_non_blocking_True_xpu was a copy_ non-blocking bug already fixed by
pytorch/pytorch PR #176278.

## Fix Approach

- Split into sub-fixes: (1) land pytorch/pytorch#176278 and drop storage_all_devices skip.
- (2) extend the Python warning-context handler (torch/csrc/utils/cpp_stacktraces) so warnings raised from XPU kernels attach a Python frame, or skip test_cpp_warnings_have_python_context on XPU matching other non-CUDA backends.
- (3) investigate pdist XPU kernel (third_party/torch-xpu-ops/src/ATen/native/xpu/sycl/DistanceKernels.cpp) for accumulation precision on large norms, add a higher tolerance or use double accumulator for the norm reduction.
