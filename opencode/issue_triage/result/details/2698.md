## Root Cause

FlashAttentionForwardXPU only supports head_dim ∈ {64,96,128,192}; the explicit TORCH_CHECK at
third_party/torch-xpu-ops/src/ATen/native/transformers/xpu/flash_attn/sycltla/mha_fwd.cpp:349 throws
for any other head_dim. The inductor unbacked-symints / AOT fallback tests call
_scaled_dot_product_flash_attention with head_dim values outside this whitelist and have no fallback
path to math/efficient SDPA.

## Fix Approach

- In mha_fwd.cpp, instead of TORCH_CHECK(false, ...), return a status indicating unsupported head_dim and dispatch to math/efficient SDPA fallback at the SDPA selector layer.
- Alternatively register an XPU-side can_use_flash_attention check so inductor selects the fallback automatically for unsupported head dimensions.
