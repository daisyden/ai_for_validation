## Root Cause

The SYCL-TLA FlashAttention forward kernel only specializes head dimensions 64, 96, 128, 192
(third_party/torch-xpu-ops/src/ATen/native/transformers/xpu/flash_attn/sycltla/mha_fwd.cpp:293-349);
any other headdim trips the TORCH_CHECK at line 348. The inductor tests (test_sdpfa_xpu,
test_sdfpa_unbacked_strides_xpu, test_fallback_kernel_with_symexpr_output_xpu) construct SDPA inputs
whose head_dim is not in the supported set (or is unbacked/symbolic at trace time), so lowering
through aten._scaled_dot_product_flash_attention.default raises at runtime on XPU while the CUDA
backend silently falls back to math/efficient kernels.

## Fix Approach

- (1) In _scaled_dot_product_flash_attention_xpu dispatcher, check headdim eligibility before calling run_mha_fwd and fall back to math/efficient SDPA (or reject with can_use_flash_attention=false in the sdp_utils heuristics) when unsupported, rather than TORCH_CHECK-failing inside the kernel.
- (2) add missing headdim specializations (e.g., 32, 72, 256) in mha_fwd.cpp following the existing templated run_mha_fwd_specialized pattern.
- (3) skip the affected inductor tests on XPU until (1) lands.
