## Root Cause

test_monitored_barrier_allreduce_hang(_wait_all_ranks) raises 'barrier timeout' on XCCL backend.
ProcessGroupXCCL::barrier at third_party/torch-xpu-ops/src/xccl/ProcessGroupXCCL.cpp:1927 implements
only an allreduce-based barrier and does not support the monitored_barrier collective semantics
expected by the test (per-rank timeout reporting); the test relies on detecting hung ranks within
the user-supplied timeout, which XCCL does not honor for a basic allreduce barrier.

## Fix Approach

- Either add proper monitored_barrier support in ProcessGroupXCCL (mirroring NCCL's implementation with per-rank ack/timeout) or skip test_monitored_barrier_allreduce_hang* in test/distributed/test_distributed_spawn.py for the xccl backend.
- File request to oneCCL if backing API is missing.
