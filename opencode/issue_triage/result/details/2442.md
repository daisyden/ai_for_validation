## Root Cause

aten::_flash_attention_forward is registered in the CPU-fallback list at
third_party/torch-xpu-ops/src/ATen/native/xpu/XPUFallback.template:233, but the op has no CPU
implementation either, so test_meta/test_ops/test_decomp calls on XPU FlashAttention dispatch to CPU
and raise NotImplementedError: Could not run 'aten::_flash_attention_forward' with arguments from
the 'CPU' backend. Issue owner confirmed 'We will target this API', i.e. an XPU native kernel is
required.

## Fix Approach

- Implement a native XPU _flash_attention_forward SYCL kernel (e.g. under src/ATen/native/xpu/FlashAttention.cpp) and register it via native_functions.yaml, then remove _flash_attention_forward from XPUFallback.template.
- As an interim, change the fallback to explicitly error or use the SDPA math backend and update skip_list_common.py until the kernel lands.
