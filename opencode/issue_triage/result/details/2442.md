## Root Cause

test_meta/test_ops/test_decomp/test_out_warning over aten::_flash_attention_forward fail with 'Could
not run aten::_flash_attention_forward with arguments from the CPU backend'. The op has no XPU
registration and the meta/ref-comparison harness routes the reference call to CPU which also lacks
an implementation. _flash_attention_forward is a CUDA-only private op; XPU SDPA is dispatched
through scaled_dot_product_attention overrides, not through _flash_attention_forward
(third_party/torch-xpu-ops/src/ATen/native/transformers/Attention.cpp / sdp_utils.cpp).

## Fix Approach

- Add the four cases to the skip list (already labelled 'skipped').
- long-term, register an XPU dispatch for aten::_flash_attention_forward (or expectedFailure on these meta/decomp/out tests) so the upstream OpInfo harness no longer routes them through CPU.
- Per latest comment 'We will target this API' — schedule implementation in transformers SYCL kernels.
