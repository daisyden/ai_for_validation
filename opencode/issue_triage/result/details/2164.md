## Root Cause

third_party/torch-xpu-ops/test/xpu/test_torch_xpu.py:12484-12501 is a verbatim port of upstream
TestTorch.test_no_cuda_monkeypatch, which asserts that torch.cuda.Stream() raises 'Tried to
instantiate dummy base class Stream'. In an XPU-enabled build (cuda not built), torch.cuda.Stream()
instead raises 'torch.cuda.Stream requires CUDA support' because the XPU runtime registers real
accelerator stream/event shims that override the dummy base class. The test's purpose is strictly to
validate the CUDA dummy-class monkeypatch and is not meaningful on XPU, as confirmed by @daisyden on
2025-11-25.

## Fix Approach

- Remove or @unittest.skip the ported test_no_cuda_monkeypatch in third_party/torch-xpu-ops/test/xpu/test_torch_xpu.py (label it cuda-specific with an xfail/skip decorator referencing #2164).
- Optionally file a separate XPU-side test for the torch.xpu.Stream / torch.xpu.Event 'not built with xpu' paths, which is already prototyped as a commented-out test_no_xpu_monkeypatch at lines 12503-12509 and blocked by issue #302.
