# Issue #2285: Support efficient attention

- **GitHub**: https://github.com/intel/torch-xpu-ops/issues/2285
- **Category**: Flash Attention
- **Priority**: P1
- **Status**: open
- **Assignee**: chunhuanMeng
- **owner_transferred**: daisyden
- **Reporter**: daisyden
- **Labels**: skipped
- **Dependency**: SYCL kernel: AttentionEfficient.cpp
- **action_Type**: CHECK_CASES

## action_TBD

- check_case_avaliablity

## action_reason

- _efficient_attention_forward OpInfo (common_methods_invocations.py:17396) has dtypes=empty_types() with no dtypesIfXPU; only dtypesIfCUDA is set. No XPU-supported dtypes -> no XPU test variant generated.

## Root Cause

torch.nn.attention.SDPBackend.EFFICIENT_ATTENTION and torch.ops.aten._efficient_attention_forward
are not implemented for XPU. test_meta_xpu/test_ops_xpu tests for _efficient_attention_forward
(meta/fake/cow/out/variant_consistency) fail because there is no XPU kernel registration.
Additionally test_qtensor_*_cuda_xpu, test_ravel/test_flatten_xpu fail due to missing CUDA->XPU
test-name adaptation rather than op gaps. Duplicate umbrella with #2853.

## Fix Approach

- Implement _efficient_attention_forward/backward for XPU via a new SYCL kernel (e.g. third_party/torch-xpu-ops/src/ATen/native/xpu/sycl/AttentionEfficient.cpp) and register in native_functions.yaml/xpu_ops.yaml dispatch keys (XPU, Meta), reusing the existing Flash-Attention kernels under src/ATen/native/xpu/sycl/Attention*.cpp as a starting point.
- For the quantized/view test cases, port CUDA test variants to XPU naming (drop _cuda suffix or add XPU to DEVICES list in test_quantized_tensor_xpu.py) and enable test_ravel_xpu/test_flatten_xpu in test_view_ops_xpu.py.
- Track alongside duplicate issue #2853.
