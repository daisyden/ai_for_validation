## Root Cause

Inductor make_fallback() asserts that an op cannot be both a fallback and have a decomp.
aten.index_add.default hits this assertion at torch/_inductor/lowering.py:2444 (make_fallback)
because it is registered both via fallback path and listed under check_decomps (lowering.py:110
references aten::index_add). XLNetLMHeadModel bf16 training compile triggers the duplicate
registration. Issue is in upstream pytorch Inductor lowering registry, not torch-xpu-ops; XPU op is
registered (yaml/native/native_functions.yaml:825).

## Fix Approach

- File a fix in pytorch/pytorch _inductor/lowering.py to remove index_add from one of the two registries (either drop the fallback registration or pass override_decomp=True).
- Verify with the huggingface XLNetLMHeadModel benchmark reproducer.
