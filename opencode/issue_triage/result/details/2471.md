Other items in the same comment are 'In plan' or rename-to-XPU mappings, so this is a partial, not full, not-target decision; the issue should be labeled `not_target` but remain open for the in-plan items.

## Root Cause

test_cuda.py cannot be run end-to-end under XPU because torch.xpu is missing counterparts for
several CUDA runtime/memory APIs exercised by the test classes: TestCuda needs host_memory_stats,
cudart(), memory._set_allocator_settings, preferred_blas_library,
DoubleTensor/FloatStorage/IntStorage; TestCudaMallocAsync needs memory_snapshot; TestMemPool needs
MemPool; TestCUDAOptims needs CUDAGraph; TestGDS needs gds.gds_register_buffer; TestCudaAutocast
needs backends.cudnn.flags; TestCompileKernel/TestCudaDeviceParametrized need _compile_kernel;
TestFXMemoryProfiler needs memory._record_memory_history. These are all feature gaps in torch/xpu/
and torch/_C accelerator bindings rather than test bugs.

## Fix Approach

- Per @guangyey's triage in the issue, wire up the existing accelerator equivalents (torch._C._accelerator_setAllocatorSettings, torch.xpu.memory_snapshot, torch.xpu.MemPool, torch.xpu.XPUGraph, torch.xpu.memory._record_memory_history) and add the remaining targeted APIs (host_memory_stats, _compile_kernel) under torch/xpu/.
- For non-target APIs (cudart, DoubleTensor, gds, cudnn.flags) add explicit skip decorators in the XPU port of test_cuda.py keyed on torch.xpu.is_available().
- Track progress as sub-tasks for PT2.13 milestone.
