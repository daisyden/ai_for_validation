## Root Cause

test_cuda.py upstream tests cannot run on XPU due to multiple missing torch.xpu counterparts of CUDA
APIs: torch.xpu.host_memory_stats, torch.xpu.cudart, torch.backends.cuda.preferred_blas_library,
torch.xpu.MemPool/XPUGraph, torch.xpu.gds, torch.xpu._compile_kernel,
torch.xpu.memory._record_memory_history, torch.xpu.memory._set_allocator_settings. Maintainer
triaged each in two comments, marking some 'In plan' and others 'Not Target'. The gaps live in
pytorch/pytorch torch/xpu/* and torch/_C accelerator bindings.

## Fix Approach

- Feature-gap tracking issue.
- Land per-API parity PRs in pytorch/pytorch: wire up torch.xpu.memory_snapshot, torch.xpu.MemPool, torch.xpu.XPUGraph, torch.xpu.memory._record_memory_history, torch._C._accelerator_setAllocatorSettings.
- add host_memory_stats and _compile_kernel per the 'In plan' list.
- explicitly skip Not-Target tests (cudart, DoubleTensor, gds, cudnn.flags) in torch-xpu-ops skip list.
