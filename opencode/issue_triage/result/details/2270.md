## Root Cause

test_decomp_new.py::test_comprehensive_torch_ops_aten__flash_attention_forward_xpu_float16 fails
with NotImplementedError for aten::_flash_attention_forward on the CPU backend. The test uses
CrossRefFakeMode/__torch_dispatch__ (test/xpu/test_decomp_new.py:851, :1052) to re-dispatch the op
on meta/CPU tensors for cross-reference, but _flash_attention_forward has no CPU implementation
upstream. XPU itself registers _flash_attention_forward (RegisterXPU_0.cpp:54891), yet the decomp
harness forces a CPU rerun that cannot complete. Reported as duplicate of #2442; fix tracked in PR
#2341.

## Fix Approach

- Either (a) skip _flash_attention_forward in the decomp cross-ref OpInfo list for XPU (mirroring CUDA's skip in common_methods_invocations, adding the op to xfail_if_not_implemented / skipped_ops in test/xpu/test_decomp_new.py) so CrossRefFakeMode does not redispatch to CPU, or (b) land PR #2341 which adds the XPU flash-attention forward decomposition path so the cross-ref works without hitting CPU.
- Short term: extend the skip_list in test/xpu/ and align with issue #2442's resolution.
