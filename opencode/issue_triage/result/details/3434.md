## Root Cause

Speech_transformer with amp_bf16 hits a Tensor.item() graph break inside get_non_pad_mask
(speech_transformer/utils/utils.py:108) and then SIGSEGVs in native code while compiling/executing
the partial XPU graph. Maintainer bisected the regression to upstream pytorch/pytorch#177158
(Dynamo/Inductor change) and a fix is already proposed in pytorch/pytorch#181753; no Python
traceback is produced, indicating the segfault occurs in the C++ Dynamo/Inductor path, not in a
torch-xpu-ops kernel. Related to existing tracking issue intel/torch-xpu-ops#2057.

## Fix Approach

- Cherry-pick / land upstream PR pytorch/pytorch#181753 into release/2.12 to revert/repair the behavior introduced by #177158.
- Re-run the speech_transformer amp_bf16 inference reproducer (benchmarks/dynamo/torchbench.py --only speech_transformer --backend=inductor --amp --amp-dtype bfloat16) to confirm the segfault is gone, and add the model back to the 2.12 inductor passing list.
