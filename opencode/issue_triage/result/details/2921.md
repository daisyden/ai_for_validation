## Root Cause

test_sparse_csr_unary_inplace_abs_xpu_complex64 and test_sparse_csr_unary_out_abs_xpu_complex64 in
third_party/torch-xpu-ops/test/xpu/test_sparse_csr_xpu.py started failing after
pytorch/pytorch@c41320d0e936ce8e7826d96c2559918644c16bc4, which changed the semantics/expected
output of abs() for complex64 in the sparse CSR unary op machinery. Upstream already marks these as
XFAIL on CPU and CUDA; the XPU test list inherits the generic cases but was not updated with the
same XFAIL markers, so they appear as new failures only on XPU.

## Fix Approach

- Land pytorch/pytorch#177632 (already opened by AKloniecki) which adds the same XFAIL decorators for XPU that CPU/CUDA have.
- Alternatively, in third_party/torch-xpu-ops/test/xpu/skip_list_common.py (or the sparse CSR skip list) add entries for test_sparse_csr_unary_inplace_abs_xpu_complex64 and test_sparse_csr_unary_out_abs_xpu_complex64 until the PR merges.
- No kernel work required.
- purely a test-list alignment.
