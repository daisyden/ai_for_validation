## Root Cause

test_sparse_csr_unary_inplace_abs_xpu_complex64 and test_sparse_csr_unary_out_abs_xpu_complex64 in
third_party/torch-xpu-ops/test/xpu/test_sparse_csr_xpu.py started failing after upstream pytorch
commit c41320d0e936 changed abs() semantics for complex (returning real-typed output). The XPU
sparse CSR abs path inherits the unary kernel dispatch; the test expectations diverge from the new
dtype-promotion behavior.

## Fix Approach

- Update the XPU sparse CSR test reference / OpInfo expected output dtype to match the new abs(complex)->real semantics from upstream commit c41320d0e936, or temporarily skip the two complex64 cases in test_sparse_csr_xpu.py.
- Sync with upstream sparse abs handler if XPU implements its own.
