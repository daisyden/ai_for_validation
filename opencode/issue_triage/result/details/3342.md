## Root Cause

Running diffusers with torch.compile + AOT-inductor on XPU emits warnings 'aten.unsqueeze.default /
aten.expand.default / aten.split_with_sizes.default is missing a c-shim implementation, using proxy
executor as fallback'. The c-shim entries are autogenerated from torchgen/aoti/fallback_ops.py
(consumed by torchgen/gen_aoti_c_shim.py:9), and grep shows none of these three ops are listed
there, so neither torch/csrc/inductor/aoti_torch/generated/c_shim_xpu.h nor c_shim_cuda.h declares
them. The proxy executor then falls back through a CUDA-coded path which is invalid on XPU and
breaks the test.

## Fix Approach

- 1) Upstream PyTorch fix: add 'aten.unsqueeze.default', 'aten.expand.default', 'aten.split_with_sizes.default' (with proper schema versions) to inductor_fallback_ops in torchgen/aoti/fallback_ops.py and regenerate c_shim_xpu.h / c_shim_cuda.h via the AOTI shim codegen. 2) Verify the proxy-executor fallback is no longer triggered by re-running the diffusers test_compile_works_with_aot reproducer. 3) Independently fix the proxy executor so it does not blindly route to CUDA on XPU devices (use device-tagged dispatch) as a defense in depth. 4) Track via an upstream pytorch/pytorch PR.
- close torch-xpu-ops#3342 once it lands.
