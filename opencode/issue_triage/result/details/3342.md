## Root Cause

Triggered from torch/_inductor/ir.py:8729-8738: cpp_wrapper/AOTI codegen looks up the op in
torchgen/aoti/fallback_ops.py::inductor_fallback_ops and falls back to the proxy executor when
missing. aten.unsqueeze.default, aten.expand.default and aten.split_with_sizes.default are not
present in inductor_fallback_ops (file scanned, no match), so AOTI emits the proxy-executor path
which on XPU dispatches via the CUDA backend, breaking diffusers QwenImage compile. These are pure
view ops that should never need a fallback; inductor should lower them in-place rather than emitting
an extern call.

## Fix Approach

- Fix in upstream PyTorch inductor: ensure aten.unsqueeze/expand/split_with_sizes are lowered as views in torch/_inductor/lowering.py for the cpp_wrapper/AOTI path so no fallback is emitted (preferred).
- As a stop-gap, add these three ops to torchgen/aoti/fallback_ops.py::inductor_fallback_ops and re-run `python torchgen/gen.py --update-aoti-c-shim` to generate XPU c-shim entries (currently only aoti_torch_xpu__trilinear exists in torch/csrc/inductor/aoti_torch/generated/c_shim_xpu.h).
- Also investigate the proxy-executor device-routing bug that sends XPU tensors through the CUDA backend.
