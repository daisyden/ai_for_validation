## Root Cause

test_prims.test_philox_rand is decorated @onlyCUDA in pytorch/test/test_prims.py:281, but the XPU
port via XPUPatchForImport in third_party/torch-xpu-ops/test/xpu/test_prims_xpu.py neutralises
@onlyCUDA and re-dispatches the test on XPU. The test body unconditionally calls
CUDARngStateHelper.get_torch_state_as_tuple() (torch/_prims_common/__init__.py:2198), which raises
'CUDA not available' on XPU-only machines. There is no XPURngStateHelper exposed by
torch._prims_common, so the test cannot run faithfully on XPU.

## Fix Approach

- Either (a) preserve the @onlyCUDA effect through XPUPatchForImport so this test is skipped on XPU, or (b) add an XPURngStateHelper in torch/_prims_common/__init__.py mirroring CUDARngStateHelper (calling torch.xpu.get_rng_state / set_rng_state / philox seed-offset accessors) and port test_philox_rand to use it.
- Then drop the skip in test_prims_xpu.py.
