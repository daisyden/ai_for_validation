## Root Cause

test_prims.py:281-310 test_philox_rand is decorated @onlyCUDA upstream, but the XPU port re-routes
it to XPU (via XPUPatchForImport neutralizing onlyCUDA). The test body unconditionally calls
CUDARngStateHelper.get_torch_state_as_tuple() (torch/_prims_common/__init__.py:2198) which raises
RuntimeError('CUDA not available') on XPU-only machines. There is no XPURngStateHelper equivalent
exposed by _prims_common, so the test cannot run on XPU as-is.

## Fix Approach

- Two upstream options: (a) fix XPUPatchForImport in third_party/torch-xpu-ops to preserve @onlyCUDA so the test correctly skips on XPU.
- or (b) add an XPURngStateHelper in torch/_prims_common/__init__.py mirroring CUDARngStateHelper (using torch.xpu.get_rng_state/manual_seed and the XPU philox state), generalize the test to dispatch on device, and submit the PR to pytorch/pytorch.
- Option (a) is the minimal fix.
- option (b) gives real XPU coverage of the philox prim path.
