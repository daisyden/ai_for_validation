## Root Cause

Aggregated upstream UT failures where tests hard-code 'cuda' device or call torch.cuda.* APIs (e.g.,
test_sync_warning_xpu uses CudaSyncGuard at test_torch_xpu.py:2522/2567 → torch/cuda/__init__.py:417
raises 'Torch not compiled with CUDA enabled'). Other listed cases like
test_layer_norm_backwards_eps clone tensors to 'cuda', test_cufft_plan_cache is CUDA-specific, and
tunableop tests are not yet ported. These are test-portability/skiplist maintenance issues, not
kernel bugs.

## Fix Approach

- Port each test in third_party/torch-xpu-ops/test/xpu/ to substitute xpu for cuda (use TEST_XPU/torch.xpu APIs, replace CudaSyncGuard with an XPU equivalent or skip), or keep them in the XPU skiplist with proper reason.
- Submit fixes upstream where the original test in pytorch/test/* needs device-generic refactor (e.g., test_torch.py test_sync_warning, test_spectral_ops cufft_plan_cache).
