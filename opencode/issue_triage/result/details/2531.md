## Root Cause

Upstream tests hard-code torch.cuda.* calls and are run on an XPU-only build where CUDA lazy_init
asserts 'Torch not compiled with CUDA enabled' (torch/cuda/__init__.py:417). Examples:
test_torch_xpu.py:2522 uses CudaSyncGuard which calls torch.cuda.get_sync_debug_mode; test_nn_xpu.py
test_layer_norm_backwards_eps does x.to('cuda'); test_spectral_ops_xpu.py
test_cufft_plan_cache_xpu_float64 is CUDA-specific; test_linalg_xpu.py tunableop cases call
torch._C._cuda_tunableop_enable; test_nn_xpu.py::test_ctc_loss_cudnn_tensor_* uses cuDNN; quantized
test_qrelu_xpu / test_qgelu_xpu rely on CUDA-only quantized kernels.

## Fix Approach

- Triage per test: (a) genuinely CUDA-specific (cufft_plan_cache, ctc_loss_cudnn_*, numeric_check_leak_tunableop_rocm, gemm_bias_offline_tunableop, mm_submatrix_offline_tunableop) - skip in third_party/torch-xpu-ops/test/xpu/skip_list_common.py with 'CUDA/ROCm only'. (b) port-able (test_sync_warning, test_layer_norm_backwards_eps, test_sort_large_slice) - replace torch.cuda/CudaSyncGuard with an XPU-aware guard (add XpuSyncGuard in torch/testing/_internal/common_utils.py or parameterize CudaSyncGuard on device) and change x.to('cuda') to x.to('xpu') in the XPU copy of the test. (c) quantized qrelu/qgelu - add XPU fallback registration or skip until XPU quantized kernel lands.
- Items already passing in nightly (test_ctc_loss_cudnn_tensor_cpu_length) can be removed from skiplist.
