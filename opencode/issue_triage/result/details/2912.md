## Root Cause

Bulk UT regression: on PT2.11 Windows/BMG (Arc B580, driver 32.0.101.8331, oneAPI 2025.3.1) the
extended op UT suite has 220 Pass→Fail vs PT2.10 while simultaneously 1366 Fail→Pass (net
improvement). The changed_tests.log attached to the issue shows the 220 new failures span many op
kernels across test_ops_xpu / test_nn_xpu rather than a single root cause, and the same failures
appear on LNL (Arc 140V, driver 8132), pointing at a platform-wide regression most likely in the
PT2.11 oneAPI-2025.3/driver stack combined with the Windows build. Similar to #2744.

## Fix Approach

- This needs decomposition, not a single fix: (1) parse the attached changed_tests.log and cluster the 220 failures by op/error signature.
- (2) reproduce on Linux BMG with the same oneAPI 2025.3.1 to separate OS-specific vs driver-specific failures.
- (3) file per-cluster child issues and bisect PT2.10→PT2.11 (both torch-xpu-ops submodule and oneAPI 2025.3 driver upgrade are candidates).
- Likely triage: driver/oneAPI upgrade is the dominant cause as tests pass on PT2.10 with same HW.
- coordinate with driver team for 8331+ fixes and add release-blocker skip list entries in torch-xpu-ops/test/xpu/skip_list_win_bmg.py for any remaining failures before 2.11 GA.
