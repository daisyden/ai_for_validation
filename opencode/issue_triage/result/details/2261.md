## Root Cause

When a profiler test runs under a forked subprocess, PTI's sycl_collector.h:388 emits 'Another
subscriber already subscribed to Sycl runtime events ...'. The warning stems from PTI subscribing to
SYCL runtime events in both the parent and the forked child; it was fixed upstream in pti-gpu commit
a27ebeb (shipped in PTI 0.14/0.15, Deep Learning Essentials 2025.3.1). The PyTorch profiler unit
tests ported in pytorch/pytorch#165423 still carry workarounds tagged with this issue number.

## Fix Approach

- No code change needed in torch-xpu-ops.
- Require PTI >= 0.15 (Deep Learning Essentials 2025.3.1) in the CI image and merge pytorch/pytorch#174673, which removes the 'refer https://github.com/intel/torch-xpu-ops/issues/2261' WAs in test/profiler/test_profiler.py.
- Once merged, verify test_basic_chrome_trace and related fork-mode profiler cases no longer emit the PTI warning and close the issue.
