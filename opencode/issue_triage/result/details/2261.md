## Root Cause

When PyTorch profiler tests run in a forked subprocess, PTI's sycl_collector.h:388 emits 'Another
subscriber already subscribed to Sycl runtime events' warnings, causing tests added by
pytorch/pytorch#165423 to fail. Cause: SYCL runtime event subscription state is inherited across
fork, so PTI in the child sees an existing subscriber from the parent and refuses to subscribe.
CUDA's CUPTI does not have this fork-propagation behavior.

## Fix Approach

- Coordinate with oneAPI/PTI team to make sycl_collector resubscribe after fork (pthread_atfork handler) or to silently re-init in child.
- In the meantime keep the workaround in pytorch#165423.
- track removal once PTI fix is released.
- File bug against pti-gpu / unified-runtime.
