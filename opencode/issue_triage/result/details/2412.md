## Root Cause

Feature gap: NestedTensor binary ops support a CUDA-specific fast path for nested+dense in
aten/src/ATen/native/nested/NestedTensorBinaryOps.cpp:104, and similar CUDA-only branches exist in
NestedTensorBackward.cpp, NestedTensorMatmul.cpp, NestedTensorFactories.cpp. XPU is excluded, so
nested+dense add/mul (e.g. [B,*,D]+[B,1,D]) errors out.

## Fix Approach

- Add XPU support to the four nested-tensor files in pytorch/pytorch upstream by relaxing `is_cuda()` checks to also include `is_xpu()` and registering equivalent XPU kernels (likely re-using or porting the broadcast add/mul SYCL kernel in torch-xpu-ops).
- Coordinate with #2425.
