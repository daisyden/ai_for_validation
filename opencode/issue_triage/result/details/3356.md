## Root Cause

test_autocast_flash_attention_xpu at test/dynamo/test_activation_checkpointing.py:1834 fails the
eager-vs-compiled SDPA comparison with 94.3% mismatched elements (max abs diff ~1.1e-3 vs tol 1e-5).
The reference and result come from `scaled_dot_product_attention` under
autocast+activation_checkpointing; on XPU the SDPA forward/backward is dispatched to the
oneDNN-backed kernels (third_party/torch-xpu-ops/src/ATen/native/transformers/) where bf16/fp16
accumulation differs from the math-backend reference path used by ref. PyTorch PR #169241 (linked in
issue) addresses the tolerance/backend selection in the test for XPU.
