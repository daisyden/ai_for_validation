## Root Cause

Windows nightly 0209 introduced a large batch (~150) of new UT failures concentrated in: (1)
Conv1d/2d/3d/Transpose forward+backward at float64/complex128 (`test_convolution_xpu.py`,
`test_modules_xpu.py`, `test_ops_gradients_xpu.py`)—oneDNN XPU does not provide fp64/complex conv
kernels on Windows BMG; (2) functorch/vmap batchnorm and compile/hessian tests
(`test_eager_transforms_xpu.py`); (3) MaxUnpool/embedding norm/MultiheadAttention edge cases; (4)
complex32/complex128 numerics for unary ufuncs (atan/tanh/sigmoid/exp/cosh/sinh/tan). The common
thread is a Windows-only regression in driver 32.0.101.8331 + new oneDNN paths exposing
fp64/complex64 conv and complex transcendental accuracy gaps.

## Fix Approach

- Bulk-skip these cases in `third_party/torch-xpu-ops/test/xpu/skip_list_win.py` to unblock CI, then split the batch by root-cause cluster: (a) fp64/complex conv → oneDNN datatype gap, file feature request to oneDNN or implement SYCL native fp64 conv fallback.
- (b) complex unary numerics → revisit `UnaryOpsKernels.cpp` complex branches.
- (c) MaxUnpool/MHA edge errors → fix bounds checking in the corresponding SYCL kernels.
- (d) functorch batchnorm → check vmap rule registration for xpu batch_norm.
- Track each cluster as a sub-issue.
