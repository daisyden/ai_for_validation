## Root Cause

test_all_gather_base fails with 'input tensor must be the same size as output size times world size'
in reduce_scatter_tensor. Per maintainer comment, root cause is in
pytorch/torch/distributed/nn/functional.py _AllGatherBase.backward(): out_size[0] = out_size[0] //
dist.get_world_size(group=ctx.group) is executed twice, producing a wrongly-sized grad output. The
bug was introduced on the daisyden/pytorch distributed_2.10 branch via a bad merge from main (commit
5f979129d9...); main branch is correct.

## Fix Approach

- Re-merge main into the distributed_2.10 branch (daisyden/pytorch) so torch/distributed/nn/functional.py:_AllGatherBase.backward divides out_size[0] by world_size only once.
- Then re-run test_c10d_spawn_nccl.py::TestDistributedNNFunctionsNccl::test_all_gather_base.
- Pure upstream/branch fix.
- no torch-xpu-ops change required.
