## Root Cause

test/test_linalg.py::test_svd (around line 3131) defines drivers = {'cpu': (None,), 'cuda': (None,
'gesvd', 'gesvdj', 'gesvda')} and then does drivers[torch.device(device).type] at line 3143. When
instantiated for device_type='xpu' the dict lookup raises KeyError: 'xpu', which surfaces as 'Key
"xpu" is missing from dict "driver"' for test_svd_xpu_{float32,float64,complex64,complex128}. The
fix belongs upstream (pytorch/pytorch#172824 is already referenced).

## Fix Approach

- Upstream PR to test/test_linalg.py: add an 'xpu': (None,) entry to the drivers dict (and similarly in any other svd/lstsq helper that keys on device type).
- alternatively use drivers.get(type, (None,)).
- Until that lands, keep the four test_svd_xpu_* cases in third_party/torch-xpu-ops/test/xpu/skip_list_common.py.
- Track PR #172824 to close this out.
