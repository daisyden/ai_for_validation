## Root Cause

test_svd in test/test_linalg.py (around line 3123) uses an @skipCUDAIfNoMagmaAndNoLinalgsolver /
driver-dict decorator that looks up the test driver by device key; the dict only contains
'cuda'/'cpu' entries, so on XPU the lookup raises KeyError: 'xpu'. This is a test-harness gap in the
upstream test decorator, not a kernel bug.

## Fix Approach

- Upstream patch in pytorch/test/test_linalg.py: add an 'xpu' entry to the SVD driver dict (or extend the skip decorator to handle the xpu device), or in torch-xpu-ops add a test override/skip-list entry until upstream is fixed.
