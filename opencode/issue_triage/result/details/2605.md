## Root Cause

Feature gap: pytorch/pytorch#170341 added a CUDA-only Inductor freezing pattern that horizontally
fuses int4 QKV matmuls. XPU's int4 mm has a different op signature/layout (torchao
_weight_int4pack_mm_xpu via oneDNN), so the upstream pattern matcher does not fire on XPU. No
XPU-side equivalent exists in torch/_inductor/fx_passes/freezing_patterns.py.

## Fix Approach

- Add an XPU-specific freezing pattern in torch/_inductor/fx_passes (mirror cuda int4-mm horizontal-fusion pattern, but match the XPU int4 packed-mm op and its scale/zero-point layout).
- Add unit tests under test/inductor for the XPU pattern.
- Land in upstream pytorch.
