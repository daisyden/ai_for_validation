## Root Cause

torch.xpu.is_available() calls into the SYCL/Level-Zero driver to enumerate devices, which
initializes the GPU context before user code can set ZE_AFFINITY_MASK; this differs from CUDA where
PYTORCH_NVML_BASED_CUDA_CHECK=1 enables a fork-safe NVML-based path. XPU lacks an equivalent
fork-safe probe (xpu-smi-based) — see torch/xpu/__init__.py is_available/device_count and the
upstream effort in pytorch/pytorch#178496. Maintainer indicates the feature is gated on a concrete
use case.

## Fix Approach

- Implement a fork-safe device probe in torch.xpu via xpu-smi / sysfs (mirroring PYTORCH_NVML_BASED_CUDA_CHECK), reusing the device_count work in pytorch/pytorch#178496.
- in the meantime, document the workaround of setting ZE_AFFINITY_MASK before any torch.xpu API.
