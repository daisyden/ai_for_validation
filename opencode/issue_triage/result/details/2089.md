## Root Cause

torch.xpu.is_available() calls torch.xpu.device_count() which goes through c10::xpu::device_count()
in c10/xpu/XPUFunctions.cpp, enumerating devices via the Level-Zero/SYCL runtime; that runtime call
creates the per-process GPU context before ZE_AFFINITY_MASK is read, so later setting
ZE_AFFINITY_MASK has no effect (tensor still lands on device 0). CUDA avoids this with
PYTORCH_NVML_BASED_CUDA_CHECK which queries NVML without initializing the CUDA driver, but XPU has
no equivalent non-initializing probe because XPU-SMI / sysman-based enumeration is not yet wired in.

## Fix Approach

- Implement a fork-/init-safe device probe using level-zero sysman or XPU-SMI that reports device count without creating a SYCL context, mirroring the NVML path.
- Gate it behind an env var (e.g.
- PYTORCH_XPU_SMI_BASED_CHECK=1) and use it from torch.xpu.is_available / device_count.
- Track alongside pytorch/pytorch#178496 (fork-safe device_count).
- Until then, document the workaround of setting ZE_AFFINITY_MASK before importing torch / calling is_available.
