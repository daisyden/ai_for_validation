## Root Cause

torch.export with a data-dependent (unbacked) symbolic batch dim u0 fails on XPU with
`GuardOnDataDependentSymNode: Could not guard on data-dependent expression Eq(u0, 1)` at the SDPA
dispatch site; the XPU SDPA backend selection logic checks `q.size(0) == 1` (or similar) which
forces a guard on u0 instead of falling through symbolically as the CUDA path does. Issue reporter
LuFinch confirms it is fixed by pytorch/pytorch#181084, which is purely an upstream PyTorch
dispatch-logic change in the XPU SDPA selector. Eager works; only export tracing crashes.
