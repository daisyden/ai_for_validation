## Root Cause

torch.export with an unbacked symint (u0) batch dimension feeding scaled_dot_product_attention
raises GuardOnDataDependentSymNode 'Could not guard on data-dependent expression Eq(u0, 1)' on XPU
but not CUDA. The XPU SDPA dispatcher / meta path branches on an equality with the symbolic batch
dim during backend selection (e.g. checks Eq(batch, 1) to pick a fast path), while CUDA has
guard-free handling. Owner LuFinch confirmed it is fixed by pytorch/pytorch#181084 in upstream
PyTorch SDPA dispatch.
