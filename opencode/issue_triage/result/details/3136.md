## Root Cause

Three sub-cases fail with `AssertionError: False is not true`. (1) test_disable_fastpath_xpu: mocks
the MHA fastpath and asserts it is called; under xpu autocast the fastpath is bypassed (same code
path as #3137 in aten/src/ATen/native/transformers/transformer.cpp) so `fastpath_mock.called ==
True` fails — fixed by pytorch/pytorch#179701. (2,3)
test_fused_sdp_priority_order_use_compile_{False,True}_xpu: the priority-order test expects cuDNN
SDPA to be selectable; XPU has no cuDNN backend (sdp::SDPBackend::cudnn_attention is not registered
in third_party/torch-xpu-ops/src/ATen/native/transformers/SDPUtils.cpp), so the priority assertion
evaluates False.

## Fix Approach

- For test_disable_fastpath_xpu, land/forward-port pytorch/pytorch#179701 and re-enable.
- For the two fused_sdp_priority_order cases, add them to the XPU skip list in test/xpu/skip_list_common.py (cuDNN is not a supported XPU backend).
- optionally filter `SDPBackend.CUDNN_ATTENTION` out of the expected priority list in a future upstream XPU-aware test refactor.
