## Root Cause

test_disable_fastpath_xpu fails at test_transformers_xpu.py:1532/1508 because fastpath_mock.called
is False — the XPU MultiheadAttention fastpath path is not entered as the upstream test expects.
test_fused_sdp_priority_order_use_compile_{False,True}_xpu fails because it benchmarks cuDNN against
flash/mem-efficient SDPA backends, and XPU has no cuDNN backend. Maintainer confirmed nn fastpath is
deprecated (pytorch/pytorch#179701 abandoned) so all three cases were skipped via torch-xpu-ops PR
#3510 commit 475bdf37 in test/xpu/skip_list_common.py.

## Fix Approach

- Keep the skip entries added by PR #3510 in test/xpu/skip_list_common.py.
- For test_fused_sdp_priority_order_*, upstream the skip via @skipIfNoCuDNN-equivalent or device-availability guard in pytorch/test/test_transformers.py so XPU is not asked to compare against a cuDNN backend it does not implement. test_disable_fastpath should remain skipped on XPU until/unless nn fastpath is revived upstream.
