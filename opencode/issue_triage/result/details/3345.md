## Root Cause

Infrastructure/CI request — not a code bug. The torch-xpu-ops repo lacks a
.github/workflows/copilot-setup-steps.yml that exposes the pvc_rolling self-hosted runner with
oneAPI environment to GitHub Copilot's cloud agent, preventing Copilot from running XPU validation
steps.

## Fix Approach

- Add .github/workflows/copilot-setup-steps.yml as proposed in the issue with job name copilot-setup-steps, runs-on: pvc_rolling, oneAPI sourcing via setvars.sh, and uv/lintrunner/cmake setup.
- Disable the Copilot integrated firewall in repo settings (it's incompatible with self-hosted runners).
- No source-code changes needed.
