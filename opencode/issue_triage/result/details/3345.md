## Root Cause

Infrastructure/tooling request, not a code defect. The Copilot cloud agent has no access to an XPU
device, so it cannot run validation steps for fix-PR drafts. The repository lacks the
.github/workflows/copilot-setup-steps.yml workflow that GitHub Copilot Coding Agent uses to
provision a runtime environment (per GitHub docs on customize-the-agent-environment).

## Fix Approach

- Add .github/workflows/copilot-setup-steps.yml as drafted in the issue body, targeting the existing self-hosted runner label `pvc_rolling`, sourcing /opt/intel/oneapi/setvars.sh, installing uv + build deps + lintrunner, and verifying icpx/CMake.
- Disable the Copilot integrated firewall in repo Copilot settings as required for self-hosted runners.
- No source-code change.
