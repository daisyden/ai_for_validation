## Root Cause

test/inductor/test_torchinductor.py:9710 (test_randint) failed on XPU gpu_cpp_wrapper / dynamic
shapes paths with ~90% element mismatch, indicating the cpp_wrapper codegen for randint on XPU was
not producing the same RNG sequence as the eager/Python-wrapper path. The issue was tracked to the
inductor lowering/cpp_wrapper of aten.randint where the XPU Philox seed/offset plumbing did not
match the Python wrapper call, causing divergent outputs between cnts runs. A fix has been submitted
upstream in pytorch/pytorch#171773 and verified by the reporter.

## Fix Approach

- Land upstream PR pytorch/pytorch#171773 which aligns the XPU cpp_wrapper randint path with the eager path (seed/offset + argument marshalling).
- Once merged, remove the skip entry for test_randint_xpu_gpu_wrapper / test_randint_xpu_dynamic_shapes_gpu_wrapper in torch-xpu-ops skip lists and re-enable in CI.
