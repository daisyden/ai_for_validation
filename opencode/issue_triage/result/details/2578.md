## Root Cause

Int4WeightOnlyConfig (TinyGemm/TensorCoreTiledLayout) e2e SQNR is -2.9 (essentially garbage) on XPU,
indicating the int4 weight-only matmul path (_weight_int4pack_mm or its TensorCoreTiled-packed
equivalent) is producing wrong results. The XPU implementation lives in
third_party/torch-xpu-ops/src/ATen/native/xpu/sycl/LinearInt4.cpp; the tinygemm packing/unpacking
layout assumed by torchao does not match the XPU kernel.

## Fix Approach

- Audit LinearInt4.cpp packing convention (inner_k_tiles=8 TensorCoreTiledLayout) vs torchao's tinygemm pack.
- align scale/zero-point dequant order and group_size=128 handling.
- Add unit test that compares bf16 ref vs XPU int4 mm at op level.
