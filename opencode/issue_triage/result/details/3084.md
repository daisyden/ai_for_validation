## Root Cause

torch.library.register_autocast hard-codes the allowed device types to {'cpu','cuda'} at
torch/_library/custom_ops.py:875 and only dispatches to AutocastCPU/AutocastCUDA keys (lines
890-899). Passing device_type='xpu' raises ValueError('Unknown device type: xpu'), blocking the five
test_library_register_autocast* cases in test/test_custom_ops.py.

## Fix Approach

- Extend register_autocast to accept 'xpu' (and ideally any device type registered via torch._register_device_module): add self._autocast_xpu_dtype, update the allowed-list check at line 875, and register the kernel under DispatchKey.AutocastXPU.
- Also broaden the ExcludeDispatchKeyGuard keyset at line 890 to include AutocastXPU.
- The DispatchKey.AutocastXPU already exists in c10/core/DispatchKey.h, so this is purely a Python-side plumbing change.
