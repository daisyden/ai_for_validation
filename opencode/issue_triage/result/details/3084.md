## Root Cause

torch/library.py:910 hard-rejects any device_type other than 'cpu' or 'cuda' in register_autocast:
`if device_type not in ['cpu', 'cuda']: raise ValueError`. The five test_library_register_autocast*
tests in test/test_custom_ops.py either pass 'xpu' or rely on dispatch-key naming that only knows
AutocastCUDA/AutocastCPU, so XPU custom ops cannot register an autocast rule.

## Fix Approach

- Upstream PR to pytorch/pytorch: in torch/library.py register_autocast, accept all autocast-capable device types (xpu, hpu, mps, cpu, cuda) by validating against torch._C._autocast_supported_devices (or an allow-list).
- Map the device_type string to the corresponding AutocastXPU dispatch key in torch/_library/custom_ops.py register_autocast helper.
- Then re-enable the five tests in the XPU skip list.
