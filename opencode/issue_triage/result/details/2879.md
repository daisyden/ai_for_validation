## Root Cause

`test_share_memory_xpu` in `test/xpu/test_nestedtensor_xpu.py` calls `storage._share_fd_cpu_()`
which raises `RuntimeError: _share_fd_: only available on CPU` at `torch/storage.py:527`. The
`_share_fd_` IPC mechanism is a Linux/CPU-only file-descriptor-based shared-memory API; XPU storages
do not implement it (XPU IPC uses a different handle path). The upstream test (inherited from CUDA
NestedTensor suite) was written to skip on non-CPU storage devices but the test list still runs it
on XPU.

## Fix Approach

- Skip `test_share_memory_xpu` in `third_party/torch-xpu-ops/test/xpu/skip_list_common.py` (already noted at line 98) or add an `@onlyCPU`/`skipXPU` decorator upstream in `test/test_nestedtensor.py`.
- No XPU implementation needed—`_share_fd_` is a CPU-only concept.
