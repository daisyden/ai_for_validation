## Root Cause

Dynamo's autocast handling in torch/_dynamo/variables/ctx_manager.py does not propagate the autocast
dtype through torch.get_device_module(device_type).amp.autocast() when device_type='xpu', so the
exported tensor stays float32 instead of float64. The corresponding CUDA path is wired up via
AutocastModeVariable but the XPU module routing returns the wrong dtype, causing
assertEqual(exported.dtype, torch.float64) to fail.

## Fix Approach

- Extend Dynamo AutocastModeVariable / get_device_module handling in upstream pytorch to recognize torch.xpu.amp.autocast dtype kwarg correctly (parity with torch.cuda.amp.autocast).
- File fix in pytorch/pytorch alongside PR #171856.
