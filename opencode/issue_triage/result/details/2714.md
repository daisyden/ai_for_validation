## Root Cause

test/dynamo/test_ctx_manager.py::test_cuda_amp_autocast and test_autocast_float64 assert
exported.dtype == torch.float64 after tracing an autocast(device_type, dtype=torch.float64) region
but observe torch.float32 on XPU. The test extends the original CUDA-only test to XPU, but dynamo's
autocast handling (torch/_dynamo/variables/ctx_manager.py AutocastModeVariable) currently records
the CUDA autocast dtype and does not propagate the XPU autocast dtype into the exported graph, so
the fake-tensor output dtype stays at the default float32 instead of the set float64.

## Fix Approach

- Update torch/_dynamo/variables/ctx_manager.py AutocastModeVariable to respect the 'xpu' device_type argument (mirror the 'cuda' branch to set torch.xpu.set_autocast_dtype / push autocast state), and ensure the XPU autocast dtype is honored during fake-tensor propagation.
- Add XPU to the allowed device_type set in dynamo autocast tracing and verify at::autocast::set_autocast_dtype for kXPU is surfaced through the traced graph.
