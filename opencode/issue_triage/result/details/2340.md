## Root Cause

test_sac_ilp.py exercises torch.distributed._tools.runtime_estimator, whose `_roofline_estimate` at
torch/distributed/_tools/runtime_estimator.py:296 hard-asserts `torch.cuda.is_available()` for
accessing device peak FLOPS/BW; on XPU this assertion fires (`AssertionError: Roofline estimation
needs to access CUDA capabilities to make estimations`). The estimator has no XPU code path. Marked
duplicate of #2163.

## Fix Approach

- Upstream PR to pytorch/pytorch generalizing runtime_estimator.py to use torch.accelerator / device-agnostic peak-flops & memory-bandwidth APIs (get_device_tflops, get_dram_gbps) so XPU works.
- gate or skip the assert when a non-CUDA accelerator is current.
- Close as duplicate of #2163 once upstream lands.
