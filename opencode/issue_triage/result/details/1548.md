## Root Cause

test_micro_pipeline_tp tests for `fused_all_gather_matmul` async-TP pattern fail with AssertionError
that the fused op is not found in the post-AOT graph. Per maintainer comment (Chao1Han, 2025-12-24),
this is the same gap as issue #1551 - XPU lacks symmetric memory (symm_mem) support, so the inductor
pattern matcher in pytorch/torch/distributed/_symmetric_memory/ cannot fuse the all_gather + matmul
on XPU. Symm_mem backend for XPU is not yet wired through ProcessGroupXCCL.

## Fix Approach

- Implement XPU symmetric memory backend (multi-device IPC handles via Level Zero) and register `symm_mem::fused_all_gather_matmul` for XPU dispatch key.
- Track jointly with #1551/#1547.
- Until then, skip these tests in test/xpu/skip_list_dist*.py.
