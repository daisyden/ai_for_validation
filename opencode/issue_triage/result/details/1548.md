## Root Cause

Same async-TP micro-pipeline fusion infrastructure as #1549/#1551: tests assert that all_gather + mm
is rewritten to symm_mem::fused_all_gather_matmul by torch/_inductor/fx_passes/micro_pipeline_tp.py.
The pass requires a SymmetricMemory-backed fused op registered for the device, which XPU does not
provide because Intel SymmetricMemory needs Level Zero / SYCL symm memory (planned for oneAPI
2026.0, per Chao1Han). The fused op therefore is not inserted into the Inductor-compiled graph and
the assertion fails. test_extra_collectives shares the same root cause.

## Fix Approach

- 1) Blocked on XPU SymmetricMemory enablement in torch-xpu-ops (Level Zero IPC or SYCL symm API). 2) Once available, register fused_all_gather_matmul for XPU and add an Inductor fusion enablement path for device 'xpu' alongside 'cuda'. 3) Until then, add the affected MicroPipelineTPTest cases to third_party/torch-xpu-ops/test/xpu/skip_list_dist*.py.
