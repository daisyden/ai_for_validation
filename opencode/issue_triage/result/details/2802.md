## Root Cause

XPU SDPA is routed through sdp::SDPBackend selection in
third_party/torch-xpu-ops/src/ATen/native/transformers/Attention.cpp:195 and lowers to
aten._scaled_dot_product_fused_attention_overrideable rather than
aten._scaled_dot_product_flash_attention. (1) test_qwen2_7b_sdpa_input_alignment_requires_recompile:
XPU efficient_attention path rejects `is_causal=True` with an explicit attn_mask (stricter check
than CUDA). (2) test_flash_attention_propagate_real_tensors: XPU flash_attention_forward returns
different output[4] (cumulative seqlens / logsumexp layout) than the fake kernel expectation, so
real-tensor propagation finds a mismatch at index 4 (fake=64, real=0). (3)
test_scaled_dot_product_attention_cuda expected-inline: export trace on XPU emits
_scaled_dot_product_fused_attention_overrideable instead of _scaled_dot_product_flash_attention.

## Fix Approach

- (1) Relax the attn_mask+is_causal check for efficient_attention on XPU (Attention.cpp) or fix the test to not pass both. (2) Align fake/meta kernel for aten._scaled_dot_product_flash_attention with the XPU real implementation output tuple — specifically the 5th output (cum_seq_len / philox seed layout).
- adjust torch/_meta_registrations.py or the XPU registration in third_party/torch-xpu-ops/src/ATen/native/transformers/xpu/flash_attn/flash_api.cpp:33 to match the canonical schema, or ensure XPU path falls back to fused_attention_overrideable for fake propagation. (3) Add XPU-specific expected-inline in test_export.py or generalize the test to accept either SDPA op variant depending on device.
