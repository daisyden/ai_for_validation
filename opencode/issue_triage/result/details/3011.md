## Root Cause

test_pdist_norm_large is a SLOW upstream test (PYTORCH_TEST_WITH_SLOW=1) that allocates ~4.66 GiB
for the pairwise distance computation in test/test_torch.py via the PdistOps test (test_torch.py
'test_pdist_norm_large'). The runtime XPU device under test only has 5.65 GiB total (likely BMG/Arc
tier), and PyTorch's caching allocator cannot satisfy the contiguous allocation on a small-VRAM
device. This is not a kernel bug but an environmental capacity mismatch — the test gates on
largeTensorTest('10GB') on CUDA but the XPU port lacks an equivalent skip/largeTensorTest decorator.

## Fix Approach

- In torch-xpu-ops test override (test/xpu/test_torch_xpu.py or skip_list_common.py), add skip/skipIfNotEnoughXPUMemory for test_pdist_norm_large_xpu, or upstream a largeTensorTest('10GB') equivalent that queries torch.xpu.get_device_properties().total_memory.
- Mirror the CUDA gating pattern.
