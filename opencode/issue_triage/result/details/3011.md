## Root Cause

Same `test_pdist_norm_large_xpu` at torch-xpu-ops/test/xpu/test_torch_xpu.py:5186 requires 4.66 GiB
of XPU memory (n=50000 -> ~1.25e9 output pairs * 4 bytes). The decorators at lines 5184-5185
(`@largeTensorTest('32GB', device='cpu')`, `@largeTensorTest('5GB', device='cuda')`) only gate on
CUDA memory, not XPU, so the test runs on any XPU device. On a 5.65 GiB Arc/iGPU device the
allocation fails with OOM; the test is effectively not applicable to small-memory XPU parts.

## Fix Approach

- Add an XPU-specific memory gate: replace the CUDA decorator with `@largeTensorTest('5GB', device='xpu')` (or dual-gate), which reads `torch.xpu.get_device_properties().total_memory` and skips automatically.
- Also consider raising the threshold to 6 GiB to include working-set overhead.
- Fix lives in third_party/torch-xpu-ops/test/xpu/test_torch_xpu.py around line 5184.
