## Root Cause

Enhancement request (not a bug). clang-tidy is currently listed in
third_party/torch-xpu-ops/.lintrunner.toml/tools/linter configs but skipped in CI, and the config
has drifted from upstream pytorch's .clang-tidy so many checks (bugprone-narrowing-conversions,
cppcoreguidelines-init-variables, modernize-concat-nested-namespaces,
cppcoreguidelines-pro-type-member-init, performance-unnecessary-value-param,
readability-redundant-casting, clang-diagnostic-deprecated-declarations, etc.) regress silently. The
19 listed check groups fail today across src/ATen/native/xpu/sycl, src/xccl, and src/comm.

## Fix Approach

- Work the checklist in the issue one check at a time: (1) run lintrunner init && lintrunner --take CLANGTIDY -a over src/ to surface violations per check, (2) fix or NOLINT warnings, (3) flip that check from disabled to enabled in .clang-tidy / lintrunner config per PR to keep diffs small.
- Finally re-enable clang-tidy in the CI lint job (.github/workflows/_linter.yml or cpp_lint equivalent) so regressions are gated.
- Track progress via the issue's task list.
