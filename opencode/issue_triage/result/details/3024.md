## Root Cause

Enhancement request to enable a list of clang-tidy checks (bugprone-narrowing-conversions,
cppcoreguidelines-init-variables, modernize-concat-nested-namespaces, etc.) and re-enable clang-tidy
in CI for torch-xpu-ops. Currently clang-tidy is skipped from lintrunner and the config in
third_party/torch-xpu-ops drifted from upstream. This is a CI/lint hygiene tracking task, not a
runtime bug, with no specific failing file:line — it concerns the lintrunner/clang-tidy config
inside torch-xpu-ops itself.

## Fix Approach

- Stage by check: for each listed check, run clang-tidy locally over third_party/torch-xpu-ops/src, fix flagged sites or add narrowly-scoped NOLINT comments / disable via .clang-tidy.
- After each check is clean, remove it from the skipped set in lintrunner config and re-enable clang-tidy job in torch-xpu-ops CI to prevent regression.
