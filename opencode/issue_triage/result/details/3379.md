## Root Cause

torch/distributed/distributed_c10d.py::_get_process_group_uid (line 1476-1484) only queries the NCCL
backend (ProcessGroupNCCL.uid) and falls back to -1 for all other backends; no XCCL branch exists,
so even though ProcessGroupXCCL.cpp:336 calls setGroupUid(options_->group_name), the Python helper
returns -1 instead of the expected 0.

## Fix Approach

- Extend _get_process_group_uid in torch/distributed/distributed_c10d.py to also try pg._get_backend(torch.device('xpu')) and return backend.uid when the backend is ProcessGroupXCCL (mirroring the NCCL branch).
- ensure ProcessGroupXCCL exposes a pybind-bound uid property consistent with NCCL.
