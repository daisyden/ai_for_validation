## Root Cause

Extended op UT
third_party.torch-xpu-ops.test.xpu.extended.test_ops_xpu.TestCommonXPU::test_compare_cpu_softmax_with_dtype_xpu_bool
compares XPU softmax(dtype=...) output against CPU and occasionally fails tolerance for the bool
input path (the cast-to-float intermediate produces small numeric drift vs CPU). Root cause is in
the XPU softmax implementation path
(third_party/torch-xpu-ops/src/ATen/native/xpu/sycl/SoftMaxKernels.cpp) versus CPU reference; the
case has been added to the skip list while investigation continues (labeled 'random'/'skipped' to
keep CI green).

## Fix Approach

- Short-term: the case is already listed in the XPU skip_list_common for extended op tests (PR merged on 2026-03-11).
- the 'random' label only prevents auto-close.
- Long-term: in torch-xpu-ops src/ATen/native/xpu/sycl/SoftMaxKernels.cpp, align the bool-input softmax numerics with CPU (cast bool->float in fp32 and match CPU's reduction order/epsilon) or relax the tolerance for bool dtype in extended/test_ops_xpu.py, then remove the skip.
