## Root Cause

Three sub-buckets in test/xpu/dynamo/test_streams_xpu.py and related: (A)
test_record_stream_problem_basic/_interleaved unconditionally call torch.cuda.* / record_stream on a
CUDA tensor in the ported XPU test, raising "Torch not compiled with CUDA enabled" - a port bug in
PR #3383's test files. (B) 11 stream tests compare generated FX graphs against expected strings that
hard-code 'stream': N; the dynamo stream-tracking pass in pytorch/torch/_dynamo (e.g.,
variables/streams.py) numbers XPU streams off-by-one vs CUDA because the default-stream counter
starts at a different index. (C) test_pre_grad and test_wrap_config_affects_cache_key fail with
frame_count Scalars are not equal, indicating the XPU compile pipeline produces different
recompilation counts than the CUDA-derived expected values.

## Fix Approach

- (A) Port test bodies to use torch.xpu.* and xpu tensors in PR #3383. (B) Either regenerate expected FX-graph strings against XPU output, or fix dynamo upstream so stream-id allocation is deterministic across backends (pytorch/torch/_dynamo/variables/streams.py). (C) Investigate XPU-specific recompilation triggers in inductor/dynamo and align expected frame_count, or parameterize the assertion.
- Update Known_issue.log Cases: format from single-comma _xpu suffix to current two-comma form.
