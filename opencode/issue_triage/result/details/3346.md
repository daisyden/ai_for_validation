## Root Cause

test_Conv2d_naive_groups float16 fails on PVC with max abs diff 1.95e-3 / rel diff 0.09 (vs
tolerance 1e-5/1e-3). The test is registered at
third_party/torch-xpu-ops/test/xpu/nn/test_convolution_xpu.py:1439
(TestConvolutionNNDeviceType.test_Conv2d_naive_groups = conv2d_naive_groups). The grouped conv2d
path lowers to oneDNN convolution primitives; reproduces on both oneDNN 3.10.2 and 3.11.2 and only
on PVC for fp16 (BMG passes), pointing to a PVC-specific fp16 grouped-conv accumulation/precision
regression in oneDNN. Possibly related to issue 3174.

## Fix Approach

- 1) Bisect oneDNN versions and isolate the failing primitive (file an oneDNN issue with a minimal reproducer using the failing groups/strides/padding). 2) Loosen tolerance only for fp16 grouped conv on PVC if oneDNN owners confirm acceptable hardware precision.
- otherwise, force a higher-precision accumulation path (fp32 acc) for grouped fp16 conv on PVC inside the XPU convolution dispatch. 3) Until upstream oneDNN fix lands, keep the test skipped on PVC with a tracking xfail tied to issue 3346/3174 and re-enable once a fixed oneDNN ships.
