## Root Cause

test_Conv2d_naive_groups_xpu_float16 fails only on PVC for fp16 (not bf16/f32, not on BMG); greatest
abs diff 0.00195, rel 0.09 — exceeds default fp16 tolerance. Conv2d on XPU is dispatched through
oneDNN (not a SYCL kernel in torch-xpu-ops); the fp16 grouped-convolution path on PVC produces lower
accumulator precision than CPU reference. Likely related to #3174.

## Fix Approach

- File a oneDNN bug report with PVC fp16 grouped conv repro.
- in the meantime, either bump tolerance for fp16 in the upstream test (atol≈3e-3, rtol≈0.1) under XPU device, or keep it on the skipped list until oneDNN delivers a fix.
- Verify with oneDNN 3.12+ when available.
