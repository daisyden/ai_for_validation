## Root Cause

test_Conv2d_naive_groups_xpu_float16 fails only on PVC (passes on BMG and for fp32/bf16) with up to
0.001953 absolute / 9% relative diff. Conv2d on XPU is dispatched through oneDNN at
aten/src/ATen/native/mkldnn/xpu/Conv.cpp (and detail/Conv.cpp); the inaccuracy reproduces on both
oneDNN 3.10.2 and 3.11.2, so it is a oneDNN fp16 grouped-conv kernel-selection / accumulation
precision issue on PVC, not a torch-xpu-ops bug. Related to existing intel/torch-xpu-ops#3174.

## Fix Approach

- Keep the test skipped on PVC fp16 and file an upstream oneDNN ticket with the failing config (groups, shapes, dtype, PVC arch) referencing #3174.
- Investigate forcing a higher-precision accumulator (fpmath_mode = strict / f32 accumulation) in aten/src/ATen/native/mkldnn/xpu/detail/Conv.cpp for grouped fp16 convs on PVC as an interim mitigation, and re-enable once oneDNN provides a fix.
