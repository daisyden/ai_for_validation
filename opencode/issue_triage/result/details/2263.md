## Root Cause

test/profiler/test_profiler.py::TestProfiler::test_basic_chrome_trace (assertion at line 1982) fails
because the Spans row '__xpu_profiler__ (0)' has ts+dur strictly greater than captureWindowEndTime_.
In libkineto's CuptiActivityProfiler::stopTraceInternal, captureWindowEndTime_ is stamped from 'now'
before sessions_[*]->stop() is called; XpuptiActivityProfilerSession::stop() then records the span's
end using a later SYCL/PTI timestamp, so the emitted Trace span overshoots the declared window end.
Float arithmetic on ts+dur further defeats exact equality.

## Fix Approach

- Follow PR pytorch/pytorch#172219: in XpuptiActivityProfilerSession::stop() (kineto plugin) clamp/receive the captureWindowEndTime_ value passed from stopTraceInternal instead of re-sampling the clock, and update the test helper (_validate_basic_json in test/profiler/test_profiler.py) to accept a small float tolerance when comparing ts+dur against endTs.
- Alternatively whitelist the __xpu_profiler__ Spans entry in the assertion, as suggested by PawelSwider2000 in the issue thread.
