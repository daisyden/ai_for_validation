## Root Cause

FSDP test_set_reduce_scatter_divide_factor uses ReduceOp.PREMUL_SUM (via _make_nccl_premul_sum) for
the reduce_scatter divide factor, but ProcessGroupXCCL's reduce-op mapping does not include
PREMUL_SUM in xcclOpsV1/xcclOpsV2 (third_party/torch-xpu-ops/src/xccl/xccl.h:240,261), so the
std::out_of_range path throws 'Cannot use ReduceOp.PREMUL_SUM with XCCL' (xccl.h:244,265). Although
ProcessGroupXCCL.cpp:91 has applyPreMulSumIfNeeded that converts PREMUL_SUM to SUM after
pre-multiplying, it appears the failing call site (likely a coalesced/_base path or
RECORD_PARAM_COMMS validation) reaches the lookup with the original PREMUL_SUM op before conversion,
or this conversion is not invoked on all reduce_scatter variants used by FSDP foreach_reduce.

## Fix Approach

- Audit all ProcessGroupXCCL collective entry points (especially reduce_scatter_tensor_coalesced and any path FSDP foreach_reduce uses) to ensure applyPreMulSumIfNeeded is called before getXcclReduceOpV1/V2.
- alternatively add PREMUL_SUM to xcclOpsV1/V2 maps mapping it to ccl::reduction::sum/onecclSum after multiplying inputs, mirroring oneCCL scale-up behavior.
- Add a unit test mirroring FSDP test_set_reduce_scatter_divide_factor.
