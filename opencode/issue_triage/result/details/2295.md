## Root Cause

Multiple upstream UTs (test_embedding_bag, test_argminmax_multiple, reductions, inductor
multi_kernel gemm) fail in stock PyTorch CI but pass on local PVC and on nightly XPU wheels.
Reproduction confirmed (comment by wincent8) ties failures to IGC version mismatch: CI image uses
IGC LTS2 while nightly wheels use newer IGC, suggesting a SYCL/IGC compiler regression affecting
reduction/embedding_bag SYCL kernels (e.g.
third_party/torch-xpu-ops/src/ATen/native/xpu/sycl/EmbeddingBag*.cpp, Reduce*.cpp).

## Fix Approach

- Identify the IGC/oneAPI version delta between CI image and nightly build.
- align CI build environment to newer IGC, or file IGC compiler bug with reproducer.
- As workaround, pin CI image IGC to known-good version.
- meanwhile keep WAs from pytorch/pytorch#165886 and #169168 until IGC fix lands.
- Track via Intel compiler (CMPLRLLVM-*) ticket.
