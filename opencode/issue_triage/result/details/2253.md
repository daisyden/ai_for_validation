## action_reason

* Issue is labeled `duplicate` and the reporter+assignee (daisyden, MEMBER) declared it a duplicate of intel/torch-xpu-ops#2289 — consolidate tracking there.

## Root Cause

test_dtypes_* failures on
addmm/addmv/baddbmm/mv/conv{1,2,3}d/conv_transpose1d/histogram(dd)/einsum/mm and
test_python_ref__refs_true_divide_xpu_complex32. The supported_dtypes returned by XPU dispatch
(registered via third_party/torch-xpu-ops/yaml/native/native_functions.yaml and OpInfo entries in
pytorch/torch/testing/_internal/common_methods_invocations.py) do not match the CUDA-declared dtypes
that test_dtypes asserts against. Labelled `duplicate, ut_upstream` — the discrepancy lives in
upstream OpInfo dtype declarations vs XPU registration.

## Fix Approach

- Either expand XPU op registrations to cover the same dtype set as CUDA (e.g. addmm/addmv/baddbmm complex32, conv*d half/bfloat16 paths) or update OpInfo dtypesIfXPU in upstream torch/testing/_internal/common_methods_invocations.py so test_dtypes reflects actual XPU support.
- Drop entries already crossed-out.
- coordinate with the duplicate tracker.
