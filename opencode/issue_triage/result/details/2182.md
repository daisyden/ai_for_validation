## Root Cause

test_transform_bias_rescale_qkv_nested fails: q.size() expected 8 but got 2 at item [2], i.e. the
nested-input output tensor shape is wrong on the seq-len/dim_per_head axis. The XPU implementation
transform_bias_rescale_qkv_xpu in
third_party/torch-xpu-ops/src/ATen/native/transformers/Attention.cpp:44 dispatches to
_transform_bias_rescale_qkv_kernel at sycl/AttentionKernels.cpp:417 which only handles the dense
path; for the NestedTensor branch the per-row size accounting in
NestedTensor_batch_offsets_from_size_tensor / collapse_dims_1_and_2 (AttentionKernels.cpp:391-414)
does not match the CUDA reference, producing incorrect Q/K/V output dims.

## Fix Approach

- Port the CUDA nested-tensor path of native/transformers/cuda/AttentionKernels.cu to AttentionKernels.cpp: add a dedicated nested kernel that reads per-batch sizes via the offsets tensor and writes [B,num_heads,T,dim_per_head] using the variable T per row, instead of the dense [B,T,3*D] indexing.
- Compare element by element against CUDA for the same seed before un-skipping in skip_list_common.py:1678.
