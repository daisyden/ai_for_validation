## Root Cause

transform_bias_rescale_qkv_xpu at
third_party/torch-xpu-ops/src/ATen/native/transformers/Attention.cpp:44-81 mis-sizes the output for
nested-tensor inputs. Line 54-55 computes T as NestedTensor_get_max_size(...)[0], which returns the
rank/first dimension of per-tensor sizes (here 2) rather than the max sequence length across the
batch (expected 8 for the test). The CUDA reference uses NestedTensor_get_max_size(...)[1] (or an
equivalent max-seq-len helper), producing correct_q.size(2)=8; XPU returns q.size(2)=2. The
subsequent round-up to multiple of 8 (line 65) hides the off-by-one-dim only when T happens to
already be 8.

## Fix Approach

- Use the correct max-seq-len index: replace `NestedTensor_get_max_size(...)[0]` with `NestedTensor_get_max_size(...)[1]` (or call native::NestedTensor_get_max_size and index the seq-len dimension as CUDA does in aten/src/ATen/native/transformers/cuda/attention.cu::transform_bias_rescale_qkv_cuda).
- Ensure `T` is computed as max over batch of sizes_ptr[i*stride + 0] (seq-len) before the multiple-of-8 rounding.
- Then remove the entry from test/xpu/skip_list_common.py:1678-1681 and re-run test_transform_bias_rescale_qkv_nested_xpu_{float32,float16,bfloat16}.
