## Root Cause

torch-xpu-ops generates ATen headers for XPU out-of-tree (cmake/Codegen.cmake:6-7) and uses
install_xpu_headers.py (tools/codegen/install_xpu_headers.py:92) plus cmake/Codegen.cmake:40-44 to
inject XPU declarations into the stock PyTorch ATen headers as a workaround. The proper solution is
to register XPU dispatch entries directly in upstream native_functions.yaml so codegen produces XPU
function declarations natively, eliminating the post-processing header rewrite.

## Fix Approach

- Upstream XPU dispatch keys for in-tree operators into pytorch/aten/src/ATen/native/native_functions.yaml so torch-gen emits XPU declarations directly.
- then remove the install_xpu_headers.py header-injection workaround and simplify cmake/Codegen.cmake.
