## Root Cause

AOTAutograd's create_functionalized_rng_ops_wrapper
(torch/_functorch/_aot_autograd/graph_capture_wrappers.py:497) calls
CUDARngStateHelper.get_torch_state_as_tuple, which in torch/_prims_common/__init__.py:2198 hardcodes
`raise RuntimeError("CUDA not available")` and only handles CUDA. When
test_higher_order_ops.py::test_dropout runs on XPU, the functionalized RNG path can't obtain the XPU
generator state and AOT compilation fails. This is purely an upstream PyTorch device-generalization
gap (CUDA-only RNG helper), not a torch-xpu-ops bug.

## Fix Approach

- Submit upstream-pytorch PR generalizing CUDARngStateHelper to a device-agnostic AcceleratorRngStateHelper (or branch on torch.xpu.is_available()) so get_torch_state_as_tuple/set_torch_state_tensor work with XPU generators.
- Update create_functionalized_rng_ops_wrapper to dispatch based on example tensor device.
- Until landed, skip test_dropout/test_dropout_inductor on XPU.
