## Root Cause

AOTAutograd's functionalized RNG wrapper unconditionally calls
CUDARngStateHelper.get_torch_state_as_tuple at
torch/_functorch/_aot_autograd/graph_capture_wrappers.py:497, and that helper
(torch/_prims_common/__init__.py:2192-2215) is hardcoded: it checks torch.cuda.is_available() and
uses torch.cuda.initial_seed / _get_rng_state_offset / set_rng_state. On an XPU-only host this
raises 'RuntimeError: CUDA not available' when compiling any graph that contains dropout
(test/dynamo/test_higher_order_ops.py::ActivationCheckpointingTests::test_dropout[_inductor]).

## Fix Approach

- Generalize CUDARngStateHelper to a device-agnostic helper that dispatches on the current fake/real device: replace the hardcoded torch.cuda.* calls with torch.get_device_module(device).initial_seed() / _get_rng_state_offset() / set_rng_state() (falling back to CPU if neither CUDA nor XPU is available).
- Update the single call site in graph_capture_wrappers.py:497 to pass the device inferred from the graph inputs.
- This is a core PyTorch change, not a torch-xpu-ops change.
- file a PR against pytorch/pytorch.
- Ensure torch.xpu exposes _get_rng_state_offset / _set_rng_state_offset equivalents (already present).
