per @aostrowski-hbn 2026-04-08). The host overhead originates inside SYCL runtime / PTI event
handling (kineto XPU backend in pytorch/torch/csrc/profiler/ and PTI integration), not torch-xpu-ops
kernel code; Intel team is collaborating with SYCL runtime on the overhead reduction and 2026 oneAPI
compiler has profiling improvements.
