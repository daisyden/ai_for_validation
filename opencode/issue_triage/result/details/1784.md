## Root Cause

torch.profiler on XPU shows two problems per the OP: (1) very high host-side overhead and (2) kernel
timestamps not aligning with real device time for phi3-mini fp16. The 2026-04-08 comment from
aostrowski-hbn confirms (2) is no longer reproducible on recent stacks, so the remaining problem is
host overhead introduced by the PTI-SDK/SYCL-runtime callback path used by Kineto's XPU backend
(Kineto itself only consumes PTI events). Fix has to happen in the SYCL runtime / PTI layer, not in
torch-xpu-ops.

## Fix Approach

- Keep the issue open narrowed to the host-overhead item: collaborate with the PTI-SDK / SYCL runtime team already engaged (per jfedorov 2026-02-16) to reduce per-kernel callback cost.
- add a CUDA-vs-XPU overhead micro-benchmark (Stonepia's TODO) to track progress, and update torch/csrc/profiler/kineto_client_interface for XPU to batch/aggregate events if the runtime cannot be fixed short-term.
- Close the kernel-time accuracy subtask since it is already resolved.
