## Root Cause

test/test_view_ops.py::test_ravel invokes torch._empty_affine_quantized(..., device='xpu') which
produces a QuantizedXPU tensor. The op is registered only under dispatch key XPU in
third_party/torch-xpu-ops/yaml/native/native_functions.yaml:7482-7485 (dispatch: XPU:
empty_affine_quantized). At runtime the dispatcher selects QuantizedXPU (mirroring QuantizedCPU for
CPU), so the call falls through and raises NotImplementedError. The minimal repro in the issue
comments confirms it is a missing dispatch-key registration, not a ravel/flatten-specific bug.

## Fix Approach

- In third_party/torch-xpu-ops/yaml/native/native_functions.yaml, change the dispatch of _empty_affine_quantized (and _empty_per_channel_affine_quantized for symmetry) so that the factory is also registered for QuantizedXPU, following the CPU/QuantizedCPU pattern in aten/src/ATen/native/native_functions.yaml.
- After regeneration, RegisterQuantizedXPU will expose the kernel.
- Then remove the WA added in pytorch PR #167728 and re-enable test_ravel_xpu / test_flatten_xpu.
