## Root Cause

torch._empty_affine_quantized has no QuantizedXPU dispatch key registration: error trace shows
kernels for [CPU, XPU, Meta, QuantizedCPU] but not QuantizedXPU. test_ravel_xpu / test_flatten_xpu
and the minimal repro torch._empty_affine_quantized([2], dtype=torch.quint8, device='xpu') hit this
gap. Registration is generated in pytorch/build/aten/src/ATen/RegisterQuantizedCPU_0.cpp:334 with no
QuantizedXPU equivalent; needs a stub in pytorch upstream native_functions.yaml /
RegisterQuantizedXPU codegen path or a CPU fallback for QuantizedXPU.

## Fix Approach

- Register _empty_affine_quantized (and _empty_per_channel_affine_quantized) for QuantizedXPU dispatch key in pytorch/aten/src/ATen/native/quantized/ (mirroring QuantizedCUDA/CPU implementations), or add CPU fallback for QuantizedXPU in torch-xpu-ops.
- Then drop the WA in pytorch/pytorch#167728 and unskip test_ravel_xpu / test_flatten_xpu.
