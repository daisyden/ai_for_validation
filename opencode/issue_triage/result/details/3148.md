## Root Cause

HuggingFace openai/whisper-tiny bfloat16 inference passes accuracy with Triton 3.7.0 (commit
64bb0de3) but fails with Triton 3.7.1+git21033c4e. Accuracy log shows RMSE(res-fp64)=0.04446 vs
RMSE(ref-fp64)=0.01102 (tol=0.001 * multiplier=3), so the regression is entirely driven by the new
Triton code-generation in intel-xpu-backend-for-triton (tracked upstream at
intel/intel-xpu-backend-for-triton#6489). The torch-xpu-ops and PyTorch Inductor code paths
themselves are unchanged.

## Fix Approach

- Primary: wait for the Triton fix in intel/intel-xpu-backend-for-triton#6489 and bump the pinned triton-xpu commit.
- Short-term mitigation (per chuanqi129): relax the accuracy tolerance for whisper-tiny in benchmarks/dynamo/ci_expected_accuracy or the XPU tolerance override table so the nightly CI does not regress while the Triton fix lands.
