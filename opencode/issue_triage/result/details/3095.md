## Root Cause

test_cudacodecache.py exercises Inductor's CUDACodeCache async compile path which depends on the
CUTLASS backend (torch/_inductor/codecache.py CUDACodeCache and torch/_inductor/codegen/cuda/*). XPU
has no CUTLASS-equivalent backend in Inductor, so test_async_compile and test_compilation_error
cannot be meaningfully ported and currently block the upstream UT sweep.

## Fix Approach

- Either (a) add device skip markers for XPU on test_cudacodecache.py upstream until an Inductor CUTLASS-equivalent (e.g., CUTLASS-SYCL or oneDNN graph) backend is wired, or (b) implement an XPUCodeCache async path.
- track upstream and add XPU skips in pytorch test_cudacodecache.py.
