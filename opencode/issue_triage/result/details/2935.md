## Root Cause

XLNetLMHeadModel training with float16 and amp-float16 regressed ~11% and 27% on XPU between PT2.10
and PT2.11. The reporter bisected the regression to PyTorch commit ab32398..44e9924 range; the
maintainer (jianyizh) narrowed it down to the exact guilty commit
pytorch/pytorch@bc4d0bf39cc56f4ff3517d503dd80e225143580b. This is an upstream Inductor/Triton
codegen change that impacts the compiled kernel schedule/layout for this model on Intel GPU, not a
torch-xpu-ops kernel bug.

## Fix Approach

- Revert, guard, or re-tune the change in pytorch/pytorch@bc4d0bf3 for XPU.
- Work with the Inductor/Triton-XPU maintainers to (a) identify which heuristic/codegen path changed (likely tiling, layout, or reduction lowering in inductor), (b) add an XPU-specific config/override in torch/_inductor to restore the prior kernel shape for fp16/amp-fp16 on Intel Max, and (c) add an XPU perf regression guard in .ci/benchmarks for XLNetLMHeadModel to prevent re-regression.
