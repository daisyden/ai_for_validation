## Root Cause

test_batch_isend_irecv_nccl hangs for 300 s because ProcessGroupXCCL does not implement a coalesced
batch_isend_irecv path; only scalar ProcessGroupXCCL::send/recv exist
(ProcessGroupXCCL.cpp:981/1025). When the distributed dispatcher enters _coalescing_manager and
flushes a batch of P2P ops, the XCCL backend issues them without grouping so ordering/coalesce
semantics differ from NCCL, producing a deadlock between the blocking ccl::send and matching
ccl::recv on peer ranks (xccl.cpp:210/238).

## Fix Approach

- Implement groupStart/groupEnd (startCoalescing/endCoalescing) in ProcessGroupXCCL and override batchIsendIrecv to emit the whole list within a single coalesced group like ProcessGroupNCCL::batch_isend_irecv.
- ensure p2p ops use a dedicated p2p communicator and non-blocking sends.
- Short term: skip test_batch_isend_irecv_nccl in XCCL CI allow-list.
