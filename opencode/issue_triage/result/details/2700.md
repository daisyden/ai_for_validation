## Root Cause

test_batch_isend_irecv_nccl times out after 300s on XCCL backend. ProcessGroupXCCL batched P2P
(batch_isend_irecv) is implemented as grouped send/recv calls but appears to deadlock when several
P2P pairs are coalesced under one ncclGroupStart-equivalent in oneCCL, causing the spawned ranks to
hang. The test is gated by BACKEND=xccl yet uses the nccl-named test method indicating XCCL
semantics gap.

## Fix Approach

- Audit ProcessGroupXCCL::groupStart/groupEnd and batched P2P paths in third_party/torch-xpu-ops/src/xccl/ProcessGroupXCCL.cpp.
- ensure ordering matches NCCL semantics.
- If oneCCL lacks reliable batched P2P, skip-list test_batch_isend_irecv_nccl for xccl and file a oneCCL request.
