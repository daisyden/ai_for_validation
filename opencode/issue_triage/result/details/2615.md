## Root Cause

Massive list of fft *_xpu_float16 cases failing with 'RuntimeError: Unsupported dtype Half /
torch.float16'. Root cause is that XPU implements FFT Half/Complex<Half> via Cast + FFT
Float/Complex<Float> (per @CuiYifeng), so the upstream not_power_of_two error path and
unsupported-dtype error are not raised on XPU as the tests expect. pytorch/pytorch#171231 enables
half precision upstream, and torch-xpu-ops PR #2637 added Half/Complex32 FFT support. Most
schema_check/meta cases now pass per github-actions nightly; only the not_power_of_two error-message
cases remain.

## Fix Approach

- Land pytorch/pytorch#171231 and skip/redirect the remaining test_fft_half_and_chalf_not_power_of_two_error cases via torch-xpu-ops skiplist (PR #3121 per @CuiYifeng).
- Update issue #2708 (canonical) and close this duplicate.
