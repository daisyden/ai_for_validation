## Root Cause

Inductor asserted a stride mismatch on the output of
torch.ops.aten._scaled_dot_product_fused_attention_overrideable.default (torchinductor generated
assert_size_stride(buf19, (1,64,512,64), (2097152,32768,64,1)) but the kernel returned the
transposed layout). This was a fake/meta kernel vs. real-kernel stride disagreement in the SDPA
overrideable path on XPU, breaking almost all HuggingFace inference accuracy runs. The fix landed
upstream in pytorch/pytorch#178986 (and follow-up #179239); reporter confirmed the issue is resolved
on pytorch commit 0bdf1e5c and the GitHub issue is now CLOSED.

## Fix Approach

- No action needed - already fixed upstream by pytorch/pytorch#178986 / #179239 which aligned the meta kernel of _scaled_dot_product_fused_attention_overrideable with the actual XPU output stride.
- Confirm the torch-xpu-ops CI picks up the new torch and close the ticket (already CLOSED as of 2026-04-21).
