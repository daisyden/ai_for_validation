## Root Cause

test_vjp_linalg_tensorsolve_xpu_float32 (functorch/test_ops.py via test_ops_xpu.py) fails on Windows
with abs diff 0.665 vs tol 0.009 on a 3x4x4x3 tensorsolve VJP. linalg.tensorsolve dispatches to
linalg.solve which is oneMKL-backed on XPU; the VJP backward composes solve with matmul, and the
float32 LU/solve path on Windows oneMKL produces accumulated error larger than the OpInfo tolerance.
Single-sample, Windows-only, no XPU SYCL kernel involved.
