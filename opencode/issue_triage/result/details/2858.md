## Root Cause

test.test_xpu.TestXpuAutocast.test_autocast_torch_need_autocast_promote on Windows raises
RuntimeError 'The program was built for 1 devices' (driver 32.0.101.8331). The error originates in
the SYCL/Level-Zero runtime when a kernel built for one device is reused under a multi-device
context, surfaced through torch.xpu autocast in test/test_xpu.py. This is a runtime/driver-level
mismatch on Windows, not an operator bug.

## Fix Approach

- Confirm driver/SYCL runtime version on the Windows MTL CI host.
- rebuild kernels per visible device or constrain ZE_AFFINITY_MASK in the test harness.
- If reproducible with newer driver, file a driver/oneAPI bug.
- until then keep skip and add a guard in torch.xpu autocast init to handle single-device-built kernels gracefully.
