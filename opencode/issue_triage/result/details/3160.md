## Root Cause

Multiple UT failures on Windows share one root error: 'Failed to find C compiler. Please specify via
CC environment variable or set triton.knobs.build.impl'. These tests exercise torch.compile /
inductor paths (test_content_store, multihead_attention compile path, test_reductions_compiled,
test_dynamic_shapes inductor backend, test_multi_grad_all_hooks). Triton's Windows build backend
requires a host C compiler to JIT-link kernels; the Windows CI image lacks CC/cl.exe on PATH so
triton.knobs.build fails before any kernel runs. This is an environment/CI setup problem, not an XPU
kernel bug.

## Fix Approach

- Update Windows CI provisioning (runner.yml / setup scripts under .github/ci_commit_pins or torch-xpu-ops CI config) to expose MSVC cl.exe on PATH and export CC, or call vcvarsall.bat before pytest.
- Alternatively, skip the affected inductor-dependent tests on Windows via @skipIfWindows until the toolchain is available.
- No source code change in kernels is required.
