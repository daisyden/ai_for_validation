## Root Cause

Duplicate of issue 2613. Same four tests (`test_argmax_argmin2/3_xpu`, `_with_duplicates`,
`_with_nan`) in inductor/test_compile_subprocess.py fail with `AssertionError: Tensor-likes are not
equal!` and identical diff pattern (3/6 mismatched, abs diff 1). Root cause is the XPU argmax/argmin
tie-break and NaN handling in
third_party/torch-xpu-ops/src/ATen/native/xpu/sycl/ReduceArgMaxKernel.cpp and
ReduceArgMinKernel.cpp; the parallel reduction combine does not deterministically return the lowest
index on equal values, causing index mismatch vs eager reference.

## Fix Approach

- Close as duplicate of #2613 (or consolidate skip list into a single issue).
- Apply the same fix: update argmax/argmin reduce combine ops to prefer lower index on ties and propagate NaN per CUDA semantics.
- Once fixed, unskip all four tests and remove this issue's entry from the skip list.
