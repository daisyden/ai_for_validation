## Root Cause

test_cudnn_weight_format at third_party/torch-xpu-ops/test/xpu/test_nn_xpu.py:4296-4297 is gated by
`@unittest.skipIf(not TEST_CUDNN, 'CUDNN not available')`. cuDNN is a CUDA-only library and never
available on XPU, so the test always skips. Test was copied unmodified from CUDA-side test_nn.py
without an XPU-equivalent gate or removal. This is a test-list maintenance issue, not a real bug.

## Fix Approach

- Either remove this test from torch-xpu-ops/test/xpu/test_nn_xpu.py since cuDNN can never be available on XPU, or replace the cudnn-format check with an XPU-relevant equivalent (e.g., oneDNN RNN weight contiguity warning) and also rewrite the hard-coded device='cuda' tensors at line 4308-4309 to use device_type.
