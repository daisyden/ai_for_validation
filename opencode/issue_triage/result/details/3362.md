## Root Cause

test_cudnn_weight_format in third_party/torch-xpu-ops/test/xpu/test_nn_xpu.py (line 4296-4297) is
gated by `@unittest.skipIf(not TEST_CUDNN, "CUDNN not available")` and the body hardcodes
`device="cuda"` (lines 4308-4309), so on an XPU-only machine it always SKIPS. The test was inherited
verbatim from upstream PyTorch test_nn.py without being adapted to XPU (no swap of TEST_CUDNN gate,
no use of device_type, no cuDNN-equivalent oneDNN check). It's a test-harness/porting gap, not a
kernel bug.

## Fix Approach

- In test_nn_xpu.py, either (a) port test_cudnn_weight_format to XPU by replacing the @skipIf(not TEST_CUDNN) decorator with an XPU-appropriate gate (e.g., always run on XPU since RNN uses oneDNN) and replacing hardcoded device="cuda" with device_type/"xpu".
- or (b) explicitly add it to the skip/expected-skip list in torch-xpu-ops test selectors so it is no longer reported as a failing/skipped case.
- Mirror the same fix for other inherited cudnn-gated RNN tests (test_cudnn_rnn_dropout_states_device, test_cudnn_forward_exception, test_cudnn_weight_tying).
