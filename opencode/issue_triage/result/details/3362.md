## Root Cause

test_cudnn_weight_format at third_party/torch-xpu-ops/test/xpu/test_nn_xpu.py:4297 was copy-ported
from upstream CUDA tests but is gated on `@unittest.skipIf(not TEST_CUDNN, 'CUDNN not available')`
(line 4296) and uses hardcoded `device='cuda'` literals (lines 4308-4316). On XPU CI the skip
predicate is always true so the test is permanently SKIPPED rather than exercising any XPU code
path. This is a test-porting hygiene issue, not a real functional bug.

## Fix Approach

- Either drop test_cudnn_weight_format from test_nn_xpu.py (it is intrinsically a cuDNN test), or rewrite it to use `device=device_type` and remove the TEST_CUDNN guard so it actually validates XPU RNN weight-format warnings.
- track in the bug_fix_stage5 cleanup queue.
