## Root Cause

test/quantization/core/test_quantized_op.py:1460 unconditionally calls
torch.from_numpy(X).to(device='cuda') inside test_max_pool2d_cudnn, triggering AssertionError 'Torch
not compiled with CUDA enabled' on XPU CI. The test is hardcoded to CUDA + cuDNN quantized maxpool
path which has no XPU equivalent. Issue is test-harness selection in torch-xpu-ops bringing this
CUDA-only test into the XPU UT list.

## Fix Approach

- Add test_max_pool2d_cudnn to the XPU skip list (test/xpu/skip_list_common.py or the per-file skip dict for test_quantized_op_xpu.py).
- No upstream change required since the test name explicitly says cudnn.
- it should remain CUDA-only.
