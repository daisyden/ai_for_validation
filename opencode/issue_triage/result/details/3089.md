## Root Cause

test/quantization/core/test_quantized_op.py::TestQuantizedOps.test_max_pool2d_cudnn hard-codes
device='cuda' at line 1463 (`a = torch.from_numpy(X).to(device="cuda")`). When torch-xpu-ops reuses
this upstream test under third_party/torch-xpu-ops/test/xpu/, it runs on an XPU-only PyTorch build
and hits torch/cuda/__init__.py::_lazy_init (line 405) which raises 'Torch not compiled with CUDA
enabled'. The test is inherently cuDNN/CUDA-specific and was never intended to run on XPU.

## Fix Approach

- Add test_max_pool2d_cudnn to the torch-xpu-ops skip list (test/xpu/skip_list_common.py or the quantization-specific skip file) with reason 'CUDA/cuDNN-only test'.
- Alternatively upstream a @skipIfNoCUDA / @onlyCUDA decorator on the TestCase method in test/quantization/core/test_quantized_op.py.
- No XPU kernel work is required.
