## Root Cause

Twelve AOTDispatch tests in test/xpu/functorch/test_aotdispatch_xpu.py
(test_mark_activations_dynamic, test_mark_outputs_dynamic_*, plus the WithCache/WithDynamo variants)
regressed between PyTorch commits 8f248fd2 (last good) and 3a5ec45192 (current). Failures are
'Expected hasattr(out, _dynamo_propagated_dynamic_indices) to be True' raised at lines
4342/4363/4455 of test_aotdispatch_xpu.py — the dynamic-indices propagation pass in AOTAutograd no
longer attaches the attribute on output / saved-activation tensors. The test is mirrored from
upstream pytorch/test/functorch/test_aotdispatch.py, so the fix lives in
torch/_functorch/_aot_autograd/runtime_wrappers.py rather than in torch-xpu-ops.
