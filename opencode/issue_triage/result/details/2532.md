## Root Cause

test_compile_int4_mm_* fail under torch.compile with 'AssertionError: wrong number of dimensions2
for op: torch.ops.aten._convert_weight_to_int4pack.default' from the inductor-generated
assert_size_stride check (buf1 shape (s54//8,1,32,1)). Root cause is dtype/shape divergence: XPU's
_convert_weight_to_int4pack in
third_party/torch-xpu-ops/src/ATen/native/xpu/sycl/WeightInt4PackKernel.cpp expects weight as kInt
(4D) while the CUDA reference expects kByte (different rank), so the meta/fake-tensor stride
inferred by Inductor mismatches the runtime XPU output rank.

## Fix Approach

- Align the XPU implementation of _convert_weight_to_int4pack with the CUDA contract (input kByte, matching output rank/shape), or update the meta kernel registration so Inductor's assert_size_stride matches actual XPU output.
- After fixing, remove the eight test_compile_int4_mm_* entries from the skiplist.
