## Root Cause

Test test_linalg_lstsq_input_checks expects the CUDA-only error message 'driver other than gels is
not supported on CUDA' but XPU path falls through the else branch in get_default_lstsq_driver at
aten/src/ATen/native/BatchLinearAlgebra.cpp:3522-3533 where only kCPU and (else implicitly is_cuda)
cases are handled. XPU drivers (gelsy/gelsd/gelss) are also unsupported, but the message the test
expects is hard-coded to 'CUDA'.

## Fix Approach

- Upstream pytorch fix: change the else branch at BatchLinearAlgebra.cpp:3528-3533 to test for any non-CPU device (or explicitly handle XPU) and emit a device-agnostic error such as 'driver other than gels is not supported on CUDA/XPU'.
- Update the test expected regex to match both.
- Then unskip the four lstsq_input_checks XPU tests.
