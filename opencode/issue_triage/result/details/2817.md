## Root Cause

test_linalg_lstsq_input_checks expects the CUDA error string 'driver` other than `gels` is not
supported on CUDA' whenever device != 'cpu' (test/test_linalg.py:607-609). On XPU,
aten/src/ATen/native/BatchLinearAlgebra.cpp:3522 dispatches on `input.device() == at::kCPU`; since
XPU is not CPU it falls into the else branch that was written specifically for CUDA, but actually
XPU's lstsq goes through CPU fallback, so on XPU the CPU branch is taken at line 3523-3527 and
raises 'parameter `driver` should be one of (gels, gelsy, gelsd, gelss)'. The mismatch is between
the test's CUDA-specific regex and the CPU-style message produced by the XPU path.

## Fix Approach

- Update test/test_linalg.py around line 607 so that only 'cuda' devices expect the CUDA-specific message, and XPU (which falls back to CPU-style driver validation) expects the CPU-style regex.
- or, in aten/src/ATen/native/BatchLinearAlgebra.cpp:3522, change the dispatch to accept all allowed drivers when `input.device().is_cpu() || input.device().type() == kXPU` so the error message is consistent.
- Prefer the test-side fix, since XPU genuinely supports the full driver set via CPU fallback.
