## Root Cause

Feature request (PT2.15) to enable Float8 training (torchao/float8) on XPU. It depends on the same
upstream XPU `_scaled_mm` implementation (pytorch/pytorch PR #165978) and AO inference PR
pytorch/ao#3332; without an XPU-dispatched FP8 GEMM via oneDNN, Float8Linear/scaling-aware training
paths fall back and fail. native_functions.yaml:7402 confirms scaled-grouped variants also lack XPU
dispatch.

## Fix Approach

- Track and land `_scaled_mm_xpu` via oneDNN FP8 matmul (rowwise + tensorwise scaling), enable Float8 training UTs with the emulate path first, then flip to native kernels.
- Ensure `torch.float8_e4m3fn/e5m2` dtype support through the XPU allocator/casts and add CI coverage under Inductor-XPU-UT-Data.
- Report progress to pytorch/ao#2917.
