## Root Cause

pytorch commit 366b45070d83f808d763c6ac189c600448607e36 updated conv_transpose{1,2,3}d OpInfo
sample_inputs / supported dtypes (broadening the dtype set). test_dtypes in test_ops_xpu then
cross-checks the OpInfo-declared dtypes against what the XPU backend actually accepts, and
conv_transpose1d/2d/3d on XPU (implemented via oneDNN convolution in torch-xpu-ops) does not support
the newly added dtypes, so test_dtypes reports a mismatch.

## Fix Approach

- Align the XPU OpInfo dtype overrides for nn.functional.conv_transpose{1,2,3}d: update the xpu dtype override list in third_party/torch-xpu-ops/test/xpu (op_db / _xpu_computation_op_list) to match the backend's actual dtype support, and/or extend the backend to cover the added dtypes where the oneDNN primitive allows.
- Owner already noted 'Can fix it by aligning the op_info dtypes.'
