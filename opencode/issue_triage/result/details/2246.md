## action_reason

- Issue is already CLOSED as duplicate of intel/torch-xpu-ops#3081 by MEMBER tszulist-hbn on 2026-04-28; remaining bsr_dense_addmm failures are now tracked under #3081 (OPEN, assigned to tszulist-hbn). No further action required on this issue.

## Root Cause

test_sparse_csr_xpu.TestSparseCompressedTritonKernelsXPU.test_triton_kernel_op_bsr_dense_addmm/_int_bsr_dense_addmm
fails because torch/sparse/_triton_ops_meta.py:138,486,579,741,940 unconditionally calls
torch.cuda.get_device_name() to look up tuned meta-parameters; on an XPU-only build this hits
torch/cuda/__init__.py:404 raising 'Torch not compiled with CUDA enabled'. The triton sparse meta
lookup and tune_bsr_dense_addmm path assume CUDA. Issue is now CLOSED as duplicate of #3081.

## Fix Approach

- Port torch/sparse/_triton_ops_meta.py to be device-agnostic: replace torch.cuda.get_device_name() with a helper dispatching on the input tensor's device (xpu/cuda), and add an XPU device-name → tuned-meta table.
- Upstream PR to pytorch/pytorch.
- Tracked under #3081.
