## Root Cause

AOTAutograd's RNG functionalization wrapper calls `CUDARngStateHelper.get_torch_state_as_tuple` at
torch/_functorch/_aot_autograd/graph_capture_wrappers.py:580, which in
torch/_prims_common/__init__.py:2192-2202 unconditionally checks `torch.cuda.is_available()` and
raises 'CUDA not available' for XPU devices. The helper is hard-coded to CUDA and uses
`torch.cuda.initial_seed()` / `torch.cuda._get_rng_state_offset()`, with no XPU branch. PyTorch PR
#174370 (linked in issue) tracks the upstream fix.

## Fix Approach

- Land the upstream PyTorch fix (pytorch/pytorch#174370) generalizing CUDARngStateHelper to dispatch on device type (use torch.xpu.initial_seed / torch.xpu._get_rng_state_offset for XPU).
- After upstream merge, unskip the test in torch-xpu-ops UT skip-list.
