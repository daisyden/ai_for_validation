## Root Cause

AOTAutograd's create_functionalized_rng_ops_wrapper at
torch/_functorch/_aot_autograd/graph_capture_wrappers.py:580 unconditionally calls
`CUDARngStateHelper.get_torch_state_as_tuple`, which raises `RuntimeError('CUDA not available')` at
torch/_prims_common/__init__.py:2198 when running on an XPU-only host. The helper is CUDA-specific
and has no XPU branch, so any compiled function that needs functionalized RNG state (e.g. dropout
under torch.compile in test_higher_order_ops.test_dropout) fails. The issue body links upstream fix
pytorch/pytorch#174370 as the ToDo.

## Fix Approach

- Wait for upstream PR pytorch/pytorch#174370 to land (generalizes RngStateHelper to dispatch by device, supporting XPU), then rebase and remove any temporary skip.
- no torch-xpu-ops side change needed.
