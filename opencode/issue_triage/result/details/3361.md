## Root Cause

Dynamo/AOTAutograd's functionalized RNG path is hard-coded to CUDA. In
torch/_functorch/_aot_autograd/graph_capture_wrappers.py:580 (and 583/607) it calls
CUDARngStateHelper.get_torch_state_as_tuple, which in torch/_prims_common/__init__.py:2198 raises
'RuntimeError: CUDA not available' on XPU.
test/dynamo/test_higher_order_ops.py::ActivationCheckpointingTests.test_dropout (line 6923) fails at
compile time because the activation-checkpoint dropout path triggers functionalized RNG capture
without device-agnostic handling.

## Fix Approach

- Land/track the upstream PR pytorch/pytorch#174370 which generalizes CUDARngStateHelper to support XPU (and other accelerators) so AOTAutograd RNG functionalization works on non-CUDA devices.
- After upstream merge, remove the test from torch-xpu-ops skip list and re-enable in CI.
