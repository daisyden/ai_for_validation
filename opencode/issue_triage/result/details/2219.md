## Root Cause

Casting a clamped tensor through float8_e4m3fn -> float16 produces NaN on XPU vs 288.0 on A100. Root
cause is a SYCL/IGC compiler bug (CMPLRLLVM-71964) in the float8_e4m3fn conversion that generates a
NaN at the boundary of the e4m3 max value; the NaN then propagates through MaxNanFunctor in
third_party/torch-xpu-ops/src/ATen/native/xpu/sycl/ReduceMaxValuesKernels.cpp:26-30 (which
intentionally propagates NaN). Workaround landed via PR #2387; reopened by @EikanWang to track the
proper compiler fix.

## Fix Approach

- Track CMPLRLLVM-71964 resolution in the oneAPI/SYCL compiler, then revert the workaround in torch-xpu-ops PR #2387 and replace with the compiler-provided correct float8_e4m3fn conversion.
- Add a regression test using the attached clamped_value.pt.
