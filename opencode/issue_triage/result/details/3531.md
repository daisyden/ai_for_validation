## Root Cause

test/xpu/export/test_converter_xpu.py:1457 invokes the upstream TS->EP converter on a quantized
model and asserts allclose between the original TorchScript output and the ExportedProgram output;
on XPU this allclose returns False (test_converter_xpu.py:119). The captured warning from
torch/_export/non_strict_utils.py:654 about Dim.AUTO 0/1 specialization indicates a dynamic-shape
handling divergence in the export path. The failure originates in upstream PT2 export+quantized
lowering rather than in any specific torch-xpu-ops kernel; it is enabled-but-failing on XPU because
no XPU-specific quantized fallback equivalent to CUDA's path exists.
