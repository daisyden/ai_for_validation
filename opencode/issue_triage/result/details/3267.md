## Root Cause

test_fused_mixed_precision_* (Adam/AdamW) fail with 'params, grads, exp_avgs, and exp_avg_sqs must
have same dtype, device, and layout' raised in torch/optim/adam.py:879 (_fused_adam). The XPU
fused-adam kernels in
third_party/torch-xpu-ops/src/ATen/native/xpu/sycl/FusedAdam{,W,Amsgrad,WAmsgrad}Kernels.cpp do not
implement the mixed-precision path that allows params (fp32) and grads/state in different dtypes, so
a dtype-uniformity check rejects the call. test_basic_vitals failures in test_torch_xpu are a
separate unrelated test infra issue.

## Fix Approach

- Add mixed-precision support in the XPU fused Adam/AdamW SYCL kernels (FusedAdamKernels.cpp etc.) following the CUDA implementation, allowing fp32 master params with bf16/fp16 grads & state.
- Update the dispatch wrapper to drop the same-dtype TORCH_CHECK when fused mixed-precision path is taken.
- File a separate issue for test_basic_vitals.
