## Root Cause

Another release/2.10 Windows BMG failure bundle. Categories observed: (1) `Failed to find C
compiler`/`Compiler: cl is not found` InductorError in test_compile_vmap_hessian,
test_multihead_attention fast_path compile tests — Triton's C-shim builder
(torch/_inductor/runtime/compile_worker) cannot locate cl.exe on the runner. (2)
test_Conv2d_zeros_stride2_pad2_xpu `AttributeError: 'NoneType' object has no attribute 'clone'` in
torch/testing/_internal/common_nn.py expanded-weights path — XPU backward returns None for a
gradient input the expanded_weights harness assumes to be defined (likely conv2d zero-padding bias
grad in `src/ATen/native/xpu/Convolution.cpp`). (3) test_zeros_large_xpu_complex128 OOM — genuine 32
GiB allocation on an 11.6 GiB B580; test needs XPU-memory gating. (4) Multiple `collection failure`
entries (test_matmul_cuda, test_fake_tensor, test_ops, etc.) point to an import-time exception on
Windows (likely a missing attribute when common_device_type is imported).

## Fix Approach

- (a) Fix the runner: install MSVC and expose cl.exe so Triton/inductor compile works, or skip those tests on Windows. (b) File a Conv2d expanded-weights sub-bug — ensure xpu conv2d grad_bias is materialized (not None) in `aten/src/ATen/native/xpu/Convolution.cpp`/`ConvolutionKernels.cpp`, mirroring the CUDA path. (c) Add `@largeTensorTest('32GB', device='xpu')` / `skipIfXpuMemoryLessThan` to test_zeros_large for complex128. (d) Run a single `pytest --collect-only` on Windows to capture the collection exception and fix the missing Windows import (often an `@skipCUDAIf`/`@dtypesIfCUDA` decorator referenced without CUDA).
