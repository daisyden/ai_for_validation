## Root Cause

test/nn/test_pooling.py:2059-2074 test_pooling_large is decorated @onlyCUDA but its body hard-codes
device='cuda' (line 2064) and torch.cuda.synchronize() (line 2068). When the XPU test harness in
torch-xpu-ops/test/xpu maps the test to TestPoolingNNDeviceTypeXPU, @onlyCUDA is overridden so the
body executes and triggers 'Torch not compiled with CUDA enabled' on Windows BMG.

## Fix Approach

- Either skip this test for XPU in torch-xpu-ops test/xpu/skip_list (BMG-Windows), or upstream a fix in test/nn/test_pooling.py to replace the hard-coded 'cuda' literal with the device parameter and use torch.accelerator.synchronize() so the test can run on XPU under largeTensorTest.
