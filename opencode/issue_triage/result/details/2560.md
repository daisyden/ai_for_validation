## Root Cause

addcmul with a CPU scalar tensor argument hits the iter.device(arg).is_xpu() assert at
third_party/torch-xpu-ops/src/ATen/native/xpu/sycl/Loops.h:626. The XPU addcmul_kernel in
PointwiseOpsKernels.cpp:48 does not promote/wrap a 0-dim CPU scalar tensor before building the
TensorIterator, while CUDA's gpu_kernel allows cpu_scalar tensors. test_cuda_vitals_gpu_only and
test_addcmul_use_cpu_scalar_True_* all hit this.

## Fix Approach

- In addcmul_kernel (and addcdiv) in PointwiseOpsKernels.cpp, detect 0-dim CPU scalar inputs and convert via .item()/Scalar overload, or extend Loops.h gpu_kernel to allow CPU scalar tensors like CUDA does.
- Mirror at::native::gpu_kernel_with_scalars handling.
