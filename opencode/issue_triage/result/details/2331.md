## Root Cause

Inductor's `_get_exceeding_shared_memory_checker` in torch/_inductor/choices.py only queries CUDA
device props (`shared_memory_per_block_optin`); on XPU it returns None so configs exceeding SLM
aren't pruned, producing 2 configs vs expected 1 in test_triton_heuristics.py:301. Upstream PR
pytorch/pytorch#172314 generalizes this to query SYCL/XPU shared memory. UT still fails on
torch-xpu-ops side until rebased.
