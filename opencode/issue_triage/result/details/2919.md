## Root Cause

55 TestCompositeComplianceXPU.test_cow_input_* tests (matmul/bmm/addmm/conv*/linalg.*/SDPA/etc.)
trigger unintended COW materialization because oneDNN-backed XPU kernels obtain mutable data
pointers from inputs even on read-only paths. Previous PRs pytorch#174374, torch-xpu-ops#2857/#2865
partially fixed RO/RW wrappers, but full resolution requires oneDNN to expose a const-pointer
(read-only) memory descriptor API which is currently unavailable.

## Fix Approach

- Track the SH-team JIRA against oneDNN for a const-pointer-aware memory API.
- once available, plumb it through src/ATen/native/xpu/Blas.cpp, Conv*.cpp, linalg solver wrappers, and SDPA oneDNN paths so RO inputs avoid mutable_data_ptr().
- Until then, keep the test list skipped and document the dependency.
