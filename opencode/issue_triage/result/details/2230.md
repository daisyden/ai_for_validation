## Root Cause

TestSparseCompressedTritonKernelsXPU::test_triton_sampled_addmm_* fails with NotImplementedError:
'aten::sparse_sampled_addmm' is not registered for the SparseCsrXPU backend. There is no XPU
implementation under third_party/torch-xpu-ops/src/ATen/native/sparse/ (the sparse subdir contains
only the xpu/ stub) and no CPU fallback for the SparseCsrXPU dispatch key, so the test in
test/xpu/test_sparse_csr_xpu.py:5139 cannot dispatch. Likely duplicate of #3081.

## Fix Approach

- Either register a SparseCsrXPU kernel for aten::sparse_sampled_addmm in torch-xpu-ops (mirroring the SparseCsrCPU implementation) or add a SparseCsrXPU CPU fallback registration.
- Until then keep the cases on the skip list and consolidate into #3081.
