## Root Cause

torch.compile (inductor backend) on XPU produces incorrect x.grad for slice_scatter backward
(max_diff ~3.67 vs CPU). Issue references upstream pytorch/pytorch#180164. The slice_scatter
backward decomposition lives in torch/_decomp and inductor lowering in pytorch/pytorch
(torch/_inductor/decomposition.py / lowering.py); not a torch-xpu-ops SYCL kernel — eager XPU
slice_scatter is correct, only the inductor-compiled path is wrong, indicating a Dynamo/Inductor
lowering bug in upstream PyTorch.

## Fix Approach

- Track upstream pytorch/pytorch#180164.
- Add a regression unit test under inductor/xpu suite.
- Bisect inductor lowering of aten.slice_scatter backward (zero-fill of the un-scattered region) to verify the gradient mask region for the input x is being correctly zeroed in the codegen-generated kernel.
- fix in pytorch repo, not torch-xpu-ops.
