## Root Cause

The PyTorch test exercising gpu_cpp_wrapper + complex add on XPU raises
torch._inductor.exc.InductorError: NotImplementedError. The complex add decomposition in
torch/_inductor/decomposition.py returns NotImplemented to request a fallback, which the CUDA
cpp_wrapper/AOTI path handles but the XPU cpp_wrapper path does not: the NotImplemented is not
plumbed through to an aten fallback op, so Inductor lowering escalates it into a hard InductorError.
Upstream already landed a temporary skip (pytorch/pytorch#178477) rather than a real XPU fix.

## Fix Approach

- In the XPU cpp_wrapper / AOTI codegen (torch/_inductor/codegen/cpp_wrapper_gpu.py and XPU-specific wrapper/AOTI shims) ensure that when a decomposition returns NotImplemented we fall back to the aten op (aten.add for complex) via the standard fallback_kernel path used by CUDA.
- Validate by removing the skip in pytorch/pytorch#178477 and re-running gpu_cpp_wrapper complex add tests.
