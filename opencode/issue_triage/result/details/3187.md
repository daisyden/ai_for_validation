## Root Cause

When using the XPU gpu_cpp_wrapper (AOTI cpp_wrapper) Inductor path, the complex add decomposition
in torch/_inductor/decomposition.py returns NotImplemented to force a fallback, but the XPU
cpp_wrapper fallback plumbing does not gracefully handle this and raises InductorError:
NotImplementedError. PyTorch upstream temporarily skipped the test in pytorch/pytorch#178477; real
fix needed in XPU cpp_wrapper fallback selection.
