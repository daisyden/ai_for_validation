## Root Cause

test_einsum_random_xpu_float64 in test/test_linalg.py:4408 hits 'RuntimeError: could not construct a
memory descriptor using strides' from einsum's matmul path. On XPU einsum dispatches into oneDNN
matmul, which rejects certain non-contiguous/zero-stride descriptor configurations produced by
einsum's reshape/permute. Fix referenced in pytorch/pytorch#176875 to make einsum contiguify
operands before calling the oneDNN-backed matmul.

## Fix Approach

- Land/cherry-pick upstream pytorch/pytorch#176875 which makes einsum contiguous before bmm/matmul on XPU.
- then remove test_einsum_random_xpu_float64 from the skiplist.
