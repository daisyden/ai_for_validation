## Root Cause

test_content_store.py tests on Windows BMG fail with 'torch._inductor.exc.InductorError:
RuntimeError: Failed to find C compiler. Please specify via CC environment variable or set
triton.knobs.build.impl' — Inductor's Triton CPU/host wrapper compile path cannot locate cl.exe/MSVC
on the BMG Windows CI runner. Same failure on PVC 2.9 indicates this is a Windows CI environment
configuration regression, not a kernel correctness issue. Tests touch torch._inductor compile flow
even though they are content-store tests, because torch.compile is invoked indirectly.

## Fix Approach

- Fix the Windows CI runner setup: ensure Visual Studio/MSVC cl.exe is on PATH and CC env var is set before running pytest, or adjust the test launcher to call vcvarsall.bat.
- Add Windows-specific skip in torch-xpu-ops test list for test_content_store.py until the CI image has the C compiler properly configured.
- No code change in torch-xpu-ops src needed.
