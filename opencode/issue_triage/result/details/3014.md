## Root Cause

Upstream UT op_ut::test_torch_xpu.TestTorchDeviceTypeXPU.test_pdist_norm_large_xpu fails with bare
'AssertionError: False is not true' under PYTORCH_TEST_WITH_SLOW=1, indicating numerical mismatch vs
CPU reference for torch.pdist on large inputs. The XPU implementation lives in
third_party/torch-xpu-ops/src/ATen/native/xpu/sycl/DistanceKernels.cpp (pdist_forward_kernel /
cdist), which uses tiled reductions over pairs; for the 'norm_large' shape it diverges from CPU.
Closely related to issue 3013 (same test, different symptom).

## Fix Approach

- Reproduce locally with PYTORCH_TEST_WITH_SLOW=1 pytest test_torch_xpu.py -k test_pdist_norm_large_xpu, capture max-abs/rel diff and shapes.
- Inspect DistanceKernels.cpp pdist reduction: ensure the accumulator is float (not the input scalar_t) for fp16/bf16 inputs, ensure tile boundary handling for the 'large' shape doesn’t skip pairs, and align p-norm intermediate accumulation with aten/src/ATen/native/Distance.cpp CPU semantics.
