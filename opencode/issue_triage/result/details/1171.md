## Root Cause

On LNL/BMG Windows, `test_MaxUnpool_index_errors_case2_xpu` expects the SYCL device assertion
message `Assertion 'maxind >= 0 && maxind < outputImageSize' failed` (raised in
third_party/torch-xpu-ops/src/ATen/native/xpu/sycl/MaxUnpoolingKernels.cpp's SYCL_KERNEL_ASSERT) but
only receives `AssertHandler::printMessage` without the assertion text. This is an IGC/compiler
issue (JIRA PYTORCHDGQ-5888) where assertion message strings are stripped on Windows. Already fixed
upstream via pytorch/pytorch#167960.
