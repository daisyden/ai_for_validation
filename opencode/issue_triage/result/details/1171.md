## Root Cause

test_MaxUnpool_index_errors_case2_xpu on LNL/BMG Windows expects the device-side assert string
'Assertion `maxind >= 0 && maxind < outputImageSize` failed' (emitted by SYCL_KERNEL_ASSERT in
MaxUnpooling kernel, third_party/torch-xpu-ops/src/ATen/native/xpu/sycl/MaxUnpoolingKernels.cpp) but
the Windows SYCL runtime/IGC strips the assertion text, leaving only 'AssertHandler::printMessage'.
Tracked internally as PYTORCHDGQ-5888 (compiler issue). A workaround PR (pytorch#167960) has landed
to relax the assertion matching in the test.

## Fix Approach

- Confirm pytorch#167960 is included in torch-xpu-ops test expectations (loosen error_msg match on Windows) and close the issue.
- continue to push driver/IGC team to emit assertion strings on Windows so the strict match can be restored.
