## Root Cause

aten::triangular_solve.X has no SparseCsrXPU dispatch.
aten/src/ATen/native/native_functions.yaml:9717-9723 only registers SparseCsrCPU and SparseCsrCUDA
(triangular_solve_out_sparse_csr_cuda implemented with cuSPARSE at
aten/src/ATen/native/sparse/cuda/SparseBlasImpl.cpp:1173).
torch-xpu-ops/yaml/native/native_functions.yaml defines no SparseCsrXPU entry for triangular_solve,
so every block_triangular_solve and sparse_triangular_solve variant (int32/int64 x float/complex)
reports NotImplementedError on SparseCsrXPU.

## Fix Approach

- Implement triangular_solve_out_sparse_csr_xpu in third_party/torch-xpu-ops/src/ATen/native/sparse/xpu/ backed by oneMKL sparse trsv/trsm (the SYCL sparse BLAS APIs mirror cuSPARSE's csrsv2/csrsm2 used by the CUDA implementation).
- Register the new function as SparseCsrXPU under triangular_solve.X in the XPU yaml.
- Handle BSR via block->CSR conversion path as the CUDA implementation does.
