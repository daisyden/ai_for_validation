## Root Cause

torchbenchmark/models/moco/__init__.py:80 rejects any device other than cuda because MoCo is a
DDP-based contrastive-learning model requiring a functional NCCL-equivalent collective backend. XPU
support depends on ProcessGroupXCCL/oneCCL DDP being production-ready and on benchmark harness
changes. PR pytorch/benchmark#2616 posted 2025-05-08 but not yet landed.

## Fix Approach

- Land pytorch/benchmark#2616 (device-agnostic MoCo init using XCCL when xpu is selected), ensure ProcessGroupXCCL supports the collectives MoCo uses (all_gather, broadcast with queue), and then re-enable in XPU E2E.
- Track alongside other DDP-dependent torchbench models.
