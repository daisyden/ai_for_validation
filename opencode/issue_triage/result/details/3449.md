## Root Cause

torch.compile/Inductor on XPU silently produces wrong results for
adaptive_avg_pool2d(...).flatten(1).sum(dim=-1) (compiled_max_diff ~10.76 vs eager ~1e-14). The bug
matches the upstream Inductor contiguous-check / exact-stride fusion bug fixed by
pytorch/pytorch#180898 (issue pytorch/pytorch#180848); the underlying defect is in upstream
Inductor's stride/contiguity handling around adaptive_avg_pool2d output, not in any torch-xpu-ops
SYCL kernel. XPU eager path is correct, isolating the fault to the inductor lowering / pattern-match
in pytorch/pytorch.
