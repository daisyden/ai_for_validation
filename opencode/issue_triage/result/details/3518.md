## Root Cause

test_torchvision_roi_ops.py::TestRoIAlign::test_autocast fails with two parameterizations
(rois_dtype1-x_dtype0-True-True/False), showing 90.9% mismatched elements with greatest absolute
diff ~1e-3 (>1e-5 atol). RoIAlign on XPU is implemented in
third_party/torch-xpu-ops/src/ATen/native/xpu/sycl/RoiAlignKernels.cpp; under autocast the
input/rois dtype combination (likely fp16 input, fp32 rois) is reduced/accumulated at lower
precision than the autocast contract requires, producing small numerical drift vs reference.

## Fix Approach

- Inspect RoiAlignKernels.cpp accumulator type: ensure bilinear-interpolation accumulators use at::acc_type<scalar_t, true> (fp32 accumulator for fp16/bf16 inputs) regardless of input dtype, matching the CUDA/torchvision reference.
- Add an autocast-specific cast of rois to the input dtype if needed.
- Add a regression unit test mirroring the failing parametrization.
- If accuracy gap is small, an alternative is to relax tolerance for the autocast variant in test_torchvision_roi_ops.py.
