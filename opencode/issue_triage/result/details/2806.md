## Root Cause

CompiledAOTI in Inductor only whitelists certain device types; XPU is rejected at
torch/_inductor/output_code.py:985 with 'unsupported device type xpu'. The new aot_compile_fullgraph
path (test/dynamo/test_aot_compile.py::test_aot_compile_with_aoti) routes through
CompiledAOTI.__post_init__, which raises for XPU. Fix is upstream PyTorch PR #178385 (referenced in
the issue comment).
