## Root Cause

torch.linalg.ldl_solve on XPU segfaults (exit 139) with invalid pivot values instead of raising
RuntimeError. linalg_ldl_solve.out is registered as CPU fallback in
third_party/torch-xpu-ops/src/ATen/native/xpu/XPUFallback.template:246, so the call falls back to
CPU and emits the standard fallback warning. Upstream pytorch/pytorch#181032 added '1 <= |pivot| <=
N' validation in the CPU implementation; this fix has not been picked up by the torch
2.13.0.dev20260422 build used here.

## Fix Approach

- Confirm pytorch commit be51081 / PR #181032 is in the XPU nightly.
- If not, rebase/wait for the upstream fix to be picked up - no torch-xpu-ops change is required since this op uses CPU fallback.
- Add a regression UT mirroring pytorch/pytorch#163450 in test/xpu to guard against future regressions.
