## Root Cause

Host stalls when many kernels are queued while xpu.Event recording is enabled
(PTI_ENABLE_COLLECTION=1). Repro creates 20000 paired torch.xpu.Event(enable_timing=True) records;
once kernel submissions exceed the Level Zero default event pool size of 256 the L0 driver blocks
event allocation, serializing host submission. Tracked as GSD-12059 against the L0/PTI driver;
PyTorch event recording path c10/xpu/XPUEvent.cpp / torch/csrc/xpu surfaces but does not cause it.

## Fix Approach

- Driver-side fix in Level Zero / PTI to grow or recycle the event pool (GSD-12059).
- Mitigations on the framework side: cap outstanding timing events, recycle Event objects, or warn when PTI collection is on with high event-record rates.
