## Root Cause

Same root cause as #3176: torch/sparse/_triton_ops.py:32-36 check_device enforces `t.device.type ==
"cuda"`, so test_triton_sampled_addmm (test_sparse_csr_xpu.py:5069) fails with 'sampled_addmm(): all
inputs are expected to be on the same GPU device' on XPU tensors. The torch.sparse.sampled_addmm
Triton path in PyTorch is CUDA-gated at the device-check layer rather than the kernel-dispatch
layer.

## Fix Approach

- In torch/sparse/_triton_ops.py:34 change the check to `t.device.type in ("cuda", "xpu")` (same pattern already applied for other _triton_ops helpers per #3176 fix).
- Then drop the test_triton_sampled_addmm entries from the XPU skip list.
