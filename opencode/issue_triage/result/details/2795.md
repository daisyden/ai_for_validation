## Root Cause

torch-xpu-ops/src/ATen/native/xpu/SummaryOps.cpp:45 unconditionally calls
globalContext().alertNotDeterministic("_histc_xpu") for all dtypes. CUDA only marks the float path
as nondeterministic (integer histc on CUDA bypasses the alert), so XPU diverges from CUDA behavior
and incorrectly raises for integer inputs under torch.use_deterministic_algorithms(True).

## Fix Approach

- In SummaryOps.cpp, gate the alertNotDeterministic call on a floating-point dtype check (e.g., self.is_floating_point()) so integer histc is treated as deterministic, matching the CUDA implementation in pytorch/aten/src/ATen/native/cuda/SummaryOps.cu.
