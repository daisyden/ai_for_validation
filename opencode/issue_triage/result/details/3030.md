## Root Cause

test_cpu_gpu_parity_nn_ConvTranspose2d_xpu_complex32 fails backward check with greatest absolute
diff 1.22e-4 (tolerance 1e-5) at a single element (1/96). ComplexHalf (complex32) is flagged
experimental upstream (ATen/EmptyTensor.cpp:54 warning) and ConvTranspose2d backward on XPU lowers
to oneDNN/deconv SYCL kernel that accumulates complex32 as two fp16 components, giving only ~3 bits
less mantissa than CPU which uses fp32 accumulation. The mismatch is pure precision drift in
complex-half accumulation, not a functional correctness bug.

## Fix Approach

- Short-term: skip/xfail this dtype entry in third_party/torch-xpu-ops/test/xpu/skip_list_common.py (or modules-specific skip list) with a tolerance-relaxation note, matching how other complex32 parity tests are handled.
- Long-term: promote complex32 grad accumulation to fp32 inside the XPU ConvTranspose backward kernel (accumulate real/imag parts in float) before casting back, or bump the per-dtype tolerance in test_modules.py dtype_precisions for complex32 parity checks.
