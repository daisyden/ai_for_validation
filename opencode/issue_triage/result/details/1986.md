## Root Cause

torch.xpu._sleep is missing as an XPU runtime API counterpart to torch.cuda._sleep, blocking
torch.get_device_module(device_type)._sleep(). The function needs to be added to PyTorch's
torch/xpu/__init__.py and a backing C++ implementation registered (analogous to
aten/src/ATen/cuda/Sleep.cpp) that submits a SYCL kernel cycling for N cycles on the active stream.

## Fix Approach

- Implement torch.xpu._sleep in pytorch/pytorch: add Python binding in torch/xpu/__init__.py and C++ helper in torch/csrc/xpu mirroring CUDA Sleep.cpp using a small SYCL kernel that spins for the requested cycles on the current XPU stream.
