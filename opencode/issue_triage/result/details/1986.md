## Root Cause

torch.cuda._sleep(cycles) exists at torch/cuda/__init__.py:237 (calling torch._C._cuda_sleep), but
there is no equivalent torch.xpu._sleep. Code paths using
torch.get_device_module(device_type)._sleep() therefore fail on XPU. An underlying SYCL kernel that
busy-waits for N GPU clock cycles must be added and bound to Python via torch._C._xpu_sleep.

## Fix Approach

- Implement a _sleep SYCL kernel in third_party/torch-xpu-ops (analogue of aten/src/ATen/cuda/Sleep.cu), expose it as torch._C._xpu_sleep via a pybind binding under torch/csrc/xpu/Module.cpp, and add the Python wrapper def _sleep(cycles): torch._C._xpu_sleep(cycles) in torch/xpu/__init__.py.
- Add a unit test mirroring test_cuda_sleep.
