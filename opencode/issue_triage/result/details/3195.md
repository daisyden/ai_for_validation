## Root Cause

test_sdpa_unbacked_no_dde in third_party/torch-xpu-ops/test/xpu/test_transformers_xpu.py:2150-2174
uses torch._dynamo.mark_dynamic on q/k/v and torch.compile(backend='eager') to exercise the
TORCH_GUARD_OR_FALSE path in attention.cpp for empty-tensor checks. The pytest worker 'gw0' crashes
on setup rather than failing, and the run passes locally — symptom of an xdist/forkserver
interaction with torch._dynamo.reset() and XPU context init (labelled 'random'), not a numerics
problem in the SDPA kernel itself. No specific XPU SDPA kernel line is exercised before the crash
occurs during test collection/setup.

- Add to dynamic_skip under 'random' label until root-caused.
