## Root Cause

logaddexp/logaddexp2 lacks complex dtype dispatch on XPU:
third_party/torch-xpu-ops/src/ATen/native/xpu/sycl/LogAddExpKernels.cpp only registers real floating
types via AT_DISPATCH_FLOATING_TYPES_AND2, so calls with complex32/64/128 raise NotImplementedError.
CUDA recently added complex support so corresponding XPU OpInfo tests now exercise complex dtypes.
Comment confirms kernel is implemented; remaining work is enabling tests upstream.

## Fix Approach

- Extend LogAddExpKernels.cpp dispatch to AT_DISPATCH_FLOATING_AND_COMPLEX_TYPES_AND* (mirroring CUDA's logaddexp_kernel_cuda complex impl), implement complex log-add-exp formula, then enable the cases by removing the WA in pytorch/pytorch#171238 and from xpu test skip lists.
