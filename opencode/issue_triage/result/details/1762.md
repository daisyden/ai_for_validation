## Root Cause

Feature request (module: build): users frequently hit AOT compile failures because their installed
GPU driver ships an older `ocloc`/IGC that cannot target the configured `TORCH_XPU_ARCH_LIST`. There
is currently no preflight check in torch-xpu-ops CMake (`third_party/torch-xpu-ops/cmake/`) that
invokes ocloc against each AOT target before the long SYCL build starts, so failures surface only
after hours of compilation.

## Fix Approach

- Add a CMake `try_compile`/`execute_process` step in the torch-xpu-ops top-level CMake that runs `ocloc compile -device <arch>` with a minimal stub for every entry in `TORCH_XPU_ARCH_LIST`.
- fail fast with a clear message pointing at the required driver/IGC version.
- Gate behind an option (e.g. `TORCH_XPU_CHECK_AOT=ON` by default) and document it in the build README.
