## Root Cause

`test_scaled_dot_product_attention_fused_kernels_packed_type_nested_is_contiguous_False_xpu` and the
many `test_fused_kernels_nested_broadcasting_kernel1_*` cases all run SDPA over non-contiguous
NestedTensors. The XPU SDPA entry in
third_party/torch-xpu-ops/src/ATen/native/transformers/Attention.cpp
(check_for_seq_len_1_nested_tensor / _scaled_dot_product_fused_attention_overrideable branch, lines
~83–200) rejects non-contiguous nested inputs and returns the generic 'If inputs are nested tensors
they must be contiguous' check from aten/src/ATen/native/transformers/sdp_utils_cpp.cpp. XPU SDPA
does not implement the nested-tensor variant (kernel1 path), and the team has decided nested-tensor
SDPA is out of scope — future varlen attention will be handled via `_flash_attention_forward`.

## Fix Approach

- Not a target.
- Add all listed nested-tensor SDPA cases to the XPU skip list (test/xpu/skip_list_common.py or test_transformers_xpu skip map) with a rationale comment pointing to the planned `_flash_attention_forward` varlen path.
- Revisit only when varlen flash-attention lands for XPU.
