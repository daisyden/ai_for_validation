## Root Cause

Tests expect RuntimeError for invalid input checks but XPU kernels don't raise them.
test__int_mm_errors expects constraints 'self.size(0) needs to be greater than 16' and
'size(1)/size(1) multiple of 8' (CUTLASS/CUDA-specific in test/test_linalg.py:7971-7997), which are
absent from _int_mm_out_xpu in aten/src/ATen/native/mkldnn/xpu/Blas.cpp:556-605 (oneDNN
quantized_matmul has no such restrictions). test_swap_basic (test_torch_xpu.py:12595) and
test_linalg_lstsq_input_checks also expect input-validation errors that the XPU code paths skip.
Some variants (vdot/dot errors, addmm etc.) already pass in nightly.

## Fix Approach

- For test__int_mm_errors: either mark test as @onlyCUDA upstream (since XPU/oneDNN has no size-multiple-of-8/>16 requirement) or add matching TORCH_CHECKs to _int_mm_out_xpu so the XPU message format matches the regex.
- For test_linalg_lstsq_input_checks, add the missing input validation TORCH_CHECKs in the XPU lstsq dispatch (mirror aten/src/ATen/native/BatchLinearAlgebra.cpp).
- For test_swap_basic investigate XPU TensorImpl weakref/PyObjectSlot handling in torch/csrc/autograd/python_variable.cpp to ensure 'has weakref' / 'Expected single reference' checks fire for xpu tensors.
