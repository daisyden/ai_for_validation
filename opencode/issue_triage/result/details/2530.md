## Root Cause

Several aten op error-path tests assert that XPU raises RuntimeError but it doesn't — e.g.,
test__int_mm_errors at test/test_linalg.py:7736 expects _int_mm to validate shapes/dtypes and raise,
but the XPU impl silently accepts; test_linalg_lstsq_input_checks and test_swap_basic show similar
missing-checks. Root cause is missing input-validation TORCH_CHECK in XPU op implementations (e.g.,
_int_mm in third_party/torch-xpu-ops/src/ATen/native/xpu/Blas.cpp / sycl/Int4MM kernels) compared to
CUDA reference.

## Fix Approach

- Add the missing TORCH_CHECK guards mirroring CUDA's _int_mm/lstsq/swap input validators in the XPU op entry points under torch-xpu-ops/src/ATen/native/xpu/.
- Cross-check against CUDA implementation in aten/src/ATen/native/cuda/Blas.cpp and replicate dtype/shape/contiguity checks.
- Re-enable the tests once parity is achieved.
