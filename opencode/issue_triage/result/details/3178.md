## Root Cause

Six newly-activated tests failed after upstream SDPA/vmap test-coverage expansion. Per @pponikox's
2026-04-10 comment, test_vmap_xpu randomness_backend tests expected CUDA-style behavior but on XPU
_fused_sdp_choice picks EFFICIENT_ATTENTION which falls back to MATH, and FLASH_ATTENTION with
dropout>0 raises a 'no kernel' error; test_nestedtensor_xpu SDPA cases
(test_sdpa_autocast/flop_counter/with_packed_in_proj) use SDPA kernels that are simply not
implemented for NestedTensor on XPU. The struck-through TestSDPAFailureModesXPU/TestTransformersXPU
cases already self-recovered in nightly. So the root cause is test-list desync with actual XPU SDPA
backend support, not a kernel bug.

## Fix Approach

- Land the fix already described by @pponikox: in test/xpu/functorch/test_vmap_xpu.py adjust expected backend/exception for EFFICIENT_ATTENTION (fallback to MATH) and for FLASH_ATTENTION+dropout (expect no-kernel error).
- in test/xpu/test_nestedtensor_xpu.py mark the four SDPA NestedTensor tests as skipped on XPU, mirroring existing skips in the same file.
- No kernel change required.
