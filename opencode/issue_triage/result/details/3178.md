## Root Cause

Six newly enabled tests fail in CI: two functorch test_vmap_xpu randomness tests (backend1/backend2)
had expectations out of sync with XPU SDPA backend support (EFFICIENT_ATTENTION falls back to MATH;
FLASH_ATTENTION with dropout>0 raises no-kernel error), and four test_nestedtensor_xpu SDPA tests
(autocast, flop_counter, with_packed_in_proj fp16/fp32) exercise NestedTensor SDPA paths that are
unsupported on XPU. Per maintainer pponikox, fix is test-list adjustment in torch-xpu-ops test/xpu/.

## Fix Approach

- Update test_vmap_xpu.py randomness expectations to match XPU SDPA backend semantics, and add explicit XPU skips for unsupported NestedTensor SDPA cases in test_nestedtensor_xpu.py (consistent with existing skip pattern in that file).
- Track underlying NestedTensor SDPA support as a separate feature gap.
