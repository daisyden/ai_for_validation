## Root Cause

test_c10d_xccl.py::XcclProcessGroupWithDispatchedCollectivesTests::test_all_to_all_single calls
_test_all_to_all_single (test/distributed/test_c10d_common.py:2194) which expects
assertRaisesRegex(ValueError, 'Tensors must be contiguous') for a non-contiguous input. The XCCL
ProcessGroup (third_party/torch-xpu-ops/src/xccl/ProcessGroupXCCL.cpp alltoall_base path) does not
validate contiguity and silently accepts/copies the tensor, so no ValueError is raised. NCCL/Gloo
paths perform this check; XCCL is missing the parity guard.

## Fix Approach

- Add a contiguity check in ProcessGroupXCCL::alltoall_base (and matching alltoall) to throw TORCH_CHECK_VALUE(input.is_contiguous() && output.is_contiguous(), 'Tensors must be contiguous') before dispatching to oneCCL, mirroring ProcessGroupNCCL.
- Re-run test_c10d_xccl.py to confirm the assertRaisesRegex passes.
