## Root Cause

In third_party/torch-xpu-ops/src/xccl/ProcessGroupXCCL.cpp:1983-1984,
ProcessGroupXCCL::alltoall_base calls checkSingleTensor(outputTensor, true) /
checkSingleTensor(inputTensor, true) with the second argument (p2p flag) set to true. In
checkSingleTensor (line 58-83), p2p=true downgrades the non-contiguous check from a ValueError throw
(line 80) to a TORCH_WARN_ONCE (lines 75-78), so non-contiguous all_to_all_single inputs are
silently accepted. The shared common-test _test_all_to_all_single expects 'Tensors must be
contiguous' to be raised (as NCCL does), hence 'AssertionError: ValueError not raised'.

## Fix Approach

- In ProcessGroupXCCL.cpp alltoall_base (lines 1983-1984), drop the p2p=true argument so the calls become checkSingleTensor(outputTensor) / checkSingleTensor(inputTensor).
- alltoall is a collective, not a P2P op, so contiguous enforcement must match NCCL's behavior.
- Re-run test/distributed/test_c10d_xccl.py::XcclProcessGroupWithDispatchedCollectivesTests::test_all_to_all_single to confirm ValueError is now raised.
