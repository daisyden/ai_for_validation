## Root Cause

In eval mode (train=false), the XPU batch_norm kernel at
third_party/torch-xpu-ops/src/ATen/native/xpu/sycl/BatchNormKernels.cpp:4157-4162 resizes save_mean
to running_mean.sizes() and copies running stats into it, and computes save_invstd from running_var.
CPU/CUDA semantics return size-0 tensors for save_mean/save_invstd in eval mode (per upstream
pytorch#85960 and PR pytorch#178209), creating a backend parity break that fails shape assertions.

## Fix Approach

- In the else branch at BatchNormKernels.cpp:4157-4162, replace the resize/copy with at::native::resize_output(save_mean, {0}) and at::native::resize_output(save_invstd, {0}).
- skip the copy_ and calc_invstd calls in eval mode.
- Mirror upstream PR 178209 behavior.
- Add a parity test in test/xpu/test_nn_xpu.py.
