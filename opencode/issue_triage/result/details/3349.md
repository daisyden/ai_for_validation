## Root Cause

In the eval-mode (training=False) branch of batch_norm_kernel,
third_party/torch-xpu-ops/src/ATen/native/xpu/sycl/BatchNormKernels.cpp:4157-4161 resizes save_mean
to running_mean.sizes() and copies running_mean into it, then computes save_invstd from running_var.
CPU/CUDA semantics return size-0 tensors for save_mean/save_invstd in eval mode (per upstream issue
pytorch/pytorch#85960 and PR #178209). The XPU eval branch should leave save_mean/save_invstd as
empty tensors instead of populating them, breaking shape parity with CPU.

## Fix Approach

- In BatchNormKernels.cpp:4157-4161 replace the resize/copy/calc_invstd logic with at::native::resize_output(save_mean, {0}) and resize_output(save_invstd, {0}), then call batch_norm_elementwise using running_mean/running_var directly (or compute invstd into a local tensor).
- Mirror the CPU/CUDA implementations referenced in pytorch/pytorch#85960 and PR #178209 and add an XPU regression test for shape parity.
