## Root Cause

torch.native_batch_norm in eval mode (training=False) returns shape [C] save_mean/save_invstd on XPU
but size-0 tensors on CPU. In
third_party/torch-xpu-ops/src/ATen/native/xpu/sycl/BatchNormKernels.cpp around lines 4155 and 4161
the eval-mode branch resizes save_invstd to running_var sizes and fills it via
batch_norm_calc_invstd (and similarly populates save_mean), violating the documented contract that
these outputs are empty when not training. Upstream reference: pytorch/pytorch#85960; related PR
pytorch/pytorch#178209.

## Fix Approach

- In BatchNormKernels.cpp eval-mode branch (~L4140-4165), short-circuit before resize_/calc_invstd so that save_mean and save_invstd are returned as empty tensors (resize to {0}) when training==false, matching CPU/CUDA.
- Add a unit test in third_party/torch-xpu-ops/test/xpu mirroring the reporter's repro to assert shape parity across devices, and check downstream consumers (autograd backward) don't depend on the populated values in eval.
