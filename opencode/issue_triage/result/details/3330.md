## Root Cause

torch.std on float32 input with magnitude ~1e20 returns inf on XPU while CPU returns finite ~1e19.
CPU uses Welford's online algorithm with numerically stable accumulation; XPU's std reduction in
third_party/torch-xpu-ops/src/ATen/native/xpu/sycl/ReduceMomentKernels.cpp likely accumulates
sum-of-squares in float32, overflowing at ~3.4e38 when squaring values near 1e20. The Welford
combine on XPU may also be promoting to float incorrectly versus double on CPU.

## Fix Approach

- Audit ReduceMomentKernels.cpp Welford reducer to ensure mean/M2 accumulators use higher precision (acc_type / double for float32 input on devices with has_fp64).
- Compare with CUDA's WelfordOps in pytorch/aten/src/ATen/native/cuda/ReduceMomentKernel.cu and align dtype promotion.
