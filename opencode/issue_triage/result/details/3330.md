## Root Cause

torch.std on float32 input uses Welford reduction with accscalar_t = at::acc_type_device<float,
kXPU> which is float on XPU
(third_party/torch-xpu-ops/src/ATen/native/xpu/sycl/ReduceMomentKernels.cpp:31-35). For inputs with
magnitudes ~1e20, the m2 (sum of squared deviations) accumulator overflows float range and produces
inf, while CPU promotes the float32 reduction accumulator to double and stays finite.

## Fix Approach

- Promote the accumulator type for float32 std/var on XPU to double (mirroring CPU behavior) by specializing std_var_template for scalar_t=float to use accscalar_t=double, or numerically stabilize the Welford update against overflow.
- Adjust test tolerance and document expected divergence on extreme magnitudes if upstream-CUDA matches XPU behavior.
