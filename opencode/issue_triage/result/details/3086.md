## Root Cause

test/test_cuda_nvml_based_avail.py exercises CUDA-specific nvml-based availability detection
(torch.cuda._device_count_nvml). XPU has no nvml equivalent, and these tests are CUDA-only by
design. They are surfacing in the XPU UT list because torch-xpu-ops' CI harness is sweeping
test_cuda_nvml_based_avail.py without filtering.

## Fix Approach

- Exclude test_cuda_nvml_based_avail.py from the XPU UT runner (test/xpu/run_test_with_skip.py or skip_list_common.py).
- These tests are intentionally CUDA-only and should not be retargeted to XPU.
- no source code fix is required.
