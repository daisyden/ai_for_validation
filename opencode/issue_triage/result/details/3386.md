## Root Cause

Multiple Dynamo/Inductor tests (`test_autocast_sdpa`, `test_assume_32_bit_indexing`,
`test_parsing_sdpa`, `test_inductor_lowering`, plus three `test_T244632748` variants in
test_proxy_tensor_xpu.py) fail in the Triton→ZEBIN compilation stage with `ocloc failed with error
code 250` and `Loading of IGC library has failed! Filename: libigc.so.2 ... Error code = -6`. The
failure originates in the GPU compiler driver layer (intel-igc-cm 1.0.224 / intel-ocloc 25.18.33578)
where `libigc.so.2` cannot be loaded or initialized; this is environmental (driver/IGC packaging or
LD path) rather than a torch-xpu-ops kernel bug. Same error pattern as #2979.

## Fix Approach

- 1) Treat as a driver/runtime environment issue: verify `libigc.so.2` is installed and reachable on `LD_LIBRARY_PATH`, and that the CI image's intel-igc-cm/ocloc/level-zero versions are mutually compatible (pin to a known-good combination, e.g. matching the validated PVC stack). 2) Add a smoke test in CI that runs `ocloc compile` on a trivial kernel before the UT shard and fails fast with a clear diagnostic if IGC init returns -6. 3) Cross-link #2979/#1762 and request the SYCL/IGC team confirm whether this is a regression vs. ocloc 25.18.33578. 4) Keep the listed cases dynamic-skipped here until the IGC packaging issue is resolved.
- no torch-xpu-ops source change is required.
