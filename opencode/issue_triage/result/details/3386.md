## Root Cause

Inductor's Triton-to-ZEBIN path invokes ocloc, which fails with `Loading of IGC library has failed!
libigc.so.2 ... IGC initialization failure. Error code = -6` (ocloc exit 250). This is an
environment/driver issue: the installed intel-igc-cm 1.0.224 / intel-ocloc 25.18.33578 stack cannot
load libigc.so.2 in the CI image, blocking any Inductor compile path on XPU (test_autocast_sdpa,
test_assume_32_bit_indexing, test_inductor_lowering, test_parsing_sdpa).

## Fix Approach

- Fix the CI driver stack: install matching libigc/intel-compute-runtime versions so libigc.so.2 loads, verify with `ocloc compile` smoke test.
- same root cause as #2979.
- No PyTorch/torch-xpu-ops code change required, but consider adding a startup health check in CI that fails fast with a clear message when ocloc cannot init IGC.
