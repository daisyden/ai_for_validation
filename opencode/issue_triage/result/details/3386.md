## Root Cause

Inductor's Triton-to-ZEBIN lowering invokes ocloc which fails to load libigc.so.2 (IGC init error
-6, ocloc exit code 250), so every test that JIT-compiles a Triton kernel via torch.compile aborts
with InductorError. This is an environment/driver-level IGC packaging problem, not a code defect in
torch-xpu-ops. Evidence: error 'Loading of IGC library has failed! Filename: libigc.so.2' with
intel-igc-cm 1.0.224 / intel-ocloc 25.18.33578; same pattern as intel/torch-xpu-ops#2979.

## Fix Approach

- Align intel-igc-cm/intel-ocloc/compute-runtime package versions in the CI image (install matching libigc-core/libigc2 and verify LD_LIBRARY_PATH/OCLOC_EXTRA_OPTIONS), and gate these tests until the driver stack is updated.
- track under the existing #2979 umbrella issue.
