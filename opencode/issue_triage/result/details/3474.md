## Root Cause

torch._scaled_mm_v2 on XPU rejects DeepSeek-style block-wise scales in argument validation with
ValueError('Invalid scaling configuration') at
aten/src/ATen/native/mkldnn/xpu/ScaledBlas.cpp:100,661 and aten/src/ATen/native/ScaledBlas.cpp:54,
while the upstream test in test_scaled_matmul_cuda.py:1662-1700 expects
NotImplementedError('DeepSeek-style scaling is only supported in CUDA for SM90'). The XPU port
currently skips via _get_torch_cuda_version()<(12,9), which trivially returns (0,0) on XPU — a
misleading skip. The error type/message mismatch is the entire failure mode.

## Fix Approach

- Preferred: in the XPU _scaled_mm_v2 dispatch (aten/src/ATen/native/mkldnn/xpu/ScaledBlas.cpp), detect blockwise scales early and raise NotImplementedError('DeepSeek-style scaling is not supported on XPU') before the generic ValueError.
- then generalize the test to a device-agnostic assertRaisesRegex(NotImplementedError, ...).
- Alternative: replace the stale CUDA-version skip in test/xpu/test_scaled_matmul_cuda_xpu.py with an explicit @skipXPU('blockwise scales not implemented') reason.
- Long-term: implement blockwise scaling in oneDNN-backed scaled_mm.
