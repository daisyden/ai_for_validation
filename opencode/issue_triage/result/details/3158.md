## Root Cause

test_triton_bsr_scatter_mm (test/test_sparse_csr.py:3905) catches `triton.compiler.OutOfResources`,
but newer Triton moved the symbol to `triton.runtime.autotuner.OutOfResources` (PyTorch already
adapted in torch/_inductor/runtime/triton_compat.py:19 and select_algorithm.py:4899). The bare
`triton.compiler.OutOfResources` reference therefore raises AttributeError before the autotune retry
path runs. Bug lives in upstream pytorch/pytorch test file, not torch-xpu-ops.

## Fix Approach

- Submit upstream PR to pytorch/pytorch updating test/test_sparse_csr.py:3905 to import OutOfResources via triton_compat (or `from triton.runtime.autotuner import OutOfResources` with fallback to `triton.compiler.OutOfResources`).
- cherry-pick into torch-xpu-ops skip list until the fix lands.
