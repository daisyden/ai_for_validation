## Root Cause

test/xpu/test_sparse_csr_xpu.py:5274 catches 'triton.compiler.OutOfResources', but upstream Triton
moved this class to triton.runtime.autotuner (see torch/_inductor/runtime/triton_compat.py:19 which
imports 'from triton.runtime.autotuner import OutOfResources'). The attribute no longer exists on
triton.compiler, so the except clause itself raises AttributeError in
test_triton_bsr_scatter_mm_blocksize_{16,16x32,32,64}_xpu_float32. This is a stale test copied from
the older upstream test_sparse_csr.py (which has the same outdated reference at line 3905).

## Fix Approach

- Replace 'triton.compiler.OutOfResources' with the compat import at third_party/torch-xpu-ops/test/xpu/test_sparse_csr_xpu.py:5274, e.g. 'from torch._inductor.runtime.triton_compat import OutOfResources' and 'except OutOfResources:'.
- Mirror the same fix in upstream test/test_sparse_csr.py:3905 and benchmarks/sparse/triton_ops.py:388.
