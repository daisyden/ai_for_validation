## Root Cause

The SYCL TLA flash-attention forward/backward kernels currently require dropout > 0.0 and
TORCH_CHECK fail otherwise:
third_party/torch-xpu-ops/src/ATen/native/transformers/xpu/flash_attn/sycltla/mha_fwd.cpp:382 (and
mha_bwd.cpp:1208). Upstream test_fused_attention_vs_math_ref_grads_cudagraph parameterizes
dropout_p=0.22 but the dispatch doesn't gate on the dropout value, so the fused-kernel path is
selected and aborts with 'FlashAttentionForwardXPU does not only support dropout > 0.0 yet'. Pure
feature-gap in the SYCL TLA kernel, surfaces as ~50 UT failures.

## Fix Approach

- Either (a) implement dropout support in mha_fwd.cpp / mha_bwd.cpp sycltla kernels, or (b) fix the SDPA backend selector to fall back to the math/efficient path when dropout_p > 0.0 so FlashAttentionForwardXPU is not chosen.
- correct the (grammatically broken) error message in the meantime.
