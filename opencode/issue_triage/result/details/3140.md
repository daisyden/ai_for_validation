## Root Cause

The sycltla FlashAttention forward entry point rejects any non-zero dropout with TORCH_CHECK(dropout
== 0.0, 'FlashAttentionForwardXPU does not only support dropout > 0.0 yet') at
third_party/torch-xpu-ops/src/ATen/native/transformers/xpu/flash_attn/sycltla/mha_fwd.cpp:380-382
(matching backward check in mha_bwd.cpp:1208). The SDPA cudagraph tests parametrize dropout_p in
{0.0, 0.22}; all dropout_p=0.22 variants hit this check. The SYCL-TLA kernel simply has no
Philox-style dropout mask implementation integrated with its online softmax.

## Fix Approach

- Implement dropout in the sycltla FlashAttention kernels: add a Philox4x32 RNG state (seed/offset plumbed through FLASH_FWD_params and FLASH_BWD_params), apply the mask/scale after the online softmax in mha_fwd.cpp and invert it in mha_bwd.cpp, then remove the two TORCH_CHECKs.
- Alternative short-term mitigation: at the ATen dispatch layer route dropout>0 cases to the mem-efficient/math SDPA backend (similar to the headdim fallback in #3141) so user models do not crash.
