## Root Cause

test/test_ops.py::test_dtypes (line ~1802) fails for einsum, mm, nn_functional_embedding_bag,
nn_functional_linear, inner because XPU kernels accept more dtypes than listed in OpInfo.dtypes
(e.g. int8/uint8/int16/int32/int64 work on XPU for einsum/inner/mm while OpInfo only advertises
float types). test_python_ref__refs_true_divide_xpu_complex32 fails because complex32 is enabled on
XPU's true_divide path but not advertised in _refs OpInfo. Source of truth is
torch/testing/_internal/opinfo/definitions/*; XPU dtype set differs from CUDA. Duplicate of #2255.

## Fix Approach

- Land the dtype alignment already prototyped in pytorch/pytorch PR #161246 (commit 7f545509) which extends OpInfo.dtypesIfXPU for einsum, inner, mm, tensordot, linalg_lu_factor_ex, nn_functional_embedding_bag, nn_functional_linear and true_divide to match actual XPU kernel support.
- No XPU kernel change needed -- metadata only.
- Once merged, remove the skipped label from the listed test cases in third_party/torch-xpu-ops/test/xpu/skip_list_common.py.
