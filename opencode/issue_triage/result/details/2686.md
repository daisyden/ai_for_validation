## Root Cause

torch/testing/_internal/distributed/distributed_test.py:4382 (_test_DistributedDataParallel) invokes
DDP with static_graph=True, and the resulting parameters diverge from the reference single-GPU run
(up to 39% relative diff). The frost-intel investigation confirms the same failure occurs with the
NCCL backend (see pytorch/pytorch#112277 and #77317 where the test is already disabled upstream), so
this is not an XCCL-specific correctness bug but a long-standing static_graph DDP test instability
exposed by the XPU path once it runs the test. XCCL allreduce/bucketing behavior simply follows the
same static-graph code path that upstream has flagged as broken for years.

## Fix Approach

- Mirror upstream behavior by disabling test_DistributedDataParallel on XPU/XCCL in torch-xpu-ops distributed skip list (test/xpu/distributed/ skip file) referencing pytorch/pytorch#77317 and #112277.
- Do not invest in an XCCL-only workaround.
- track upstream resolution of the static_graph DDP correctness bug and re-enable once fixed.
