## Root Cause

torch.linalg.pinv has no XPU backend registration in torch-xpu-ops and falls back to CPU at
build/xpu/ATen/RegisterXPU_0.cpp:54525 (as shown in the user warning). pinv is a composite op
typically decomposed via SVD; implementation depends on oneMKL LAPACK (gesvd/gesdd) plus linalg_inv
kernels on XPU. The PR for this op missed the PT2.12 branch cut and is now being re-targeted to
PT2.13 per maintainer discussion in issue comments.

## Fix Approach

- Land the pending implementation PR that registers linalg_pinv (and its _out variant) via oneMKL-backed SVD + matmul composition in third_party/torch-xpu-ops/src/ATen/native/xpu/.
- Ensure fp32/fp64/complex dtype coverage, add op_db/xfail cleanup in test/xpu, and validate against PEFT LoRA-FA finetuning (op accounts for ~10.7% of step time).
- Target milestone PT2.13.
