## Root Cause

torch.linalg.pinv has no XPU backend; falls back to CPU at RegisterXPU_0.cpp:54525, used by PEFT
LoRA-FA (lorafa.py:149) consuming ~10.7% of finetune step time. pinv internally calls SVD, and per
@mwiktor-intel's comment on PR #2982/#3264, oneMKL gesvd is slower than the CPU fallback, so a naive
XPU implementation is not a perf win.
