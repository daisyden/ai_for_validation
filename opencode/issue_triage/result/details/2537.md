## Root Cause

Tests were ported from CUDA skiplist with @expectedFailure/skipCUDAIfRocm-style markers that map to
XPU, but the XPU behavior actually succeeds (e.g. CUDA-specific grid_y > 65535 limitation does not
exist on SYCL; complex32 pow/mul ref-executor comparisons happen to pass; memory_format
ConvTranspose/Conv2d float64 tests pass). pytest therefore reports 'Failed: Unexpected success' e.g.
in third_party/torch-xpu-ops/test/xpu/test_modules_xpu.py and test_ops_xpu.py via the generic
expected_failure decorator list in test/test_modules.py / test/test_ops.py
(common_modules/common_methods_invocations). The harness reports at test collection time, not in a
specific SYCL kernel.

## Fix Approach

- For each listed test, remove the obsolete XPU expected_failure entry from the xpu skip/xfail table (e.g. third_party/torch-xpu-ops/test/xpu/skip_list_common.py or the per-op decorators in common_modules.py/common_methods_invocations.py), or convert it to a strict=False xfail.
- For test_upsamplingNearest2d_launch_fail_xpu specifically drop the CUDA-launch-limit xfail (SYCL has no 65535 y-dim limit).
- Re-run to confirm PASS.
- nightly has already cleared test_upsamplingNearest2d_launch_fail_xpu.
