## Root Cause

Eight UTs (test_memory_format_nn_(Lazy)Conv(Transpose)2d_xpu_float64,
test_cpu_gpu_parity_nn_ConvTranspose1d/2d_xpu_complex32,
test_python_ref_executor__refs_pow/mul_executor_aten_xpu_complex32) report 'Unexpected success'
because they are decorated @expectedFailure for CUDA in upstream test_modules.py / test_ops.py, but
XPU does not hit the same failure mode (e.g., SYCL has no 65535 grid_y limit, complex32 ref-executor
numerics differ). The tests pass on XPU and the expectedFailure marker triggers Unexpected success.

## Fix Approach

- Add per-device skip/xfail-only-for-CUDA decorators (e.g., @skipIfXPU on the expectedFailure or use dtypesIfXPU/onlyCUDA) in the relevant upstream tests, or add overrides in torch-xpu-ops skiplist to mark these tests as expected-pass on XPU.
