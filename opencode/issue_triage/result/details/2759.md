## Root Cause

Two clusters of skipped failures introduced 2026-01-22: (a)
test_errors_nn_NLLLoss_xpu_{float32,float64} - 'AssertionError: RuntimeError not raised' in
test/test_modules.py:879, meaning the XPU NLLLoss path silently accepts an input that CPU/CUDA
reject (input validation missing in the XPU LossNLL kernel registered through torch-xpu-ops); (b)
test_addcmul_alpha_one_fma_parity_xpu_{float32,float64} - Tensor-likes not equal at
test/test_foreach.py:1454, FMA parity mismatch in the foreach addcmul XPU kernel
(third_party/torch-xpu-ops/src/ATen/native/xpu/sycl/ForeachPointwiseOpScalarKernels.cpp /
ForeachBinaryOpScalarKernels.cpp) due to differing fused-multiply-add behavior vs the non-foreach
reference. NLLLoss subset already passed on nightly per bot.

## Fix Approach

- For NLLLoss errors: add input-validation TORCH_CHECK calls in the XPU NLLLoss kernel (third_party/torch-xpu-ops/src/ATen/native/xpu/LossNLLKernel.cpp) to mirror CUDA error_inputs.
- For foreach addcmul: align the fused-multiply-add path in the foreach SYCL kernel with the non-foreach addcmul (use the same accumulation order / contraction policy) so alpha=1 parity holds.
- remove WAs in pytorch PR #169354 once fixed.
- Then drop entries from skip list.
