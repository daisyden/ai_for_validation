## Root Cause

Two distinct active failures remain (NLLLoss entries passed in nightly and are struck out):
test_foreach.TestForeachXPU.test_addcmul_alpha_one_fma_parity_xpu_{float32,float64} asserts
`Tensor-likes are not equal` at test/test_foreach.py:1454. The test is decorated `@onlyCUDA`
upstream but is being collected for XPU via the XPU test generator, and checks bitwise parity
between the `alpha=1` fast path of `torch._foreach_addcmul` and `input + tensor1 * tensor2`; on XPU
the foreach_addcmul kernel in
third_party/torch-xpu-ops/src/ATen/native/xpu/sycl/ForeachPointwiseOpScalarKernels.cpp does not use
a true FMA for the alpha==1 path, producing 1-ULP differences from the reference computation used in
the test.

## Fix Approach

- Two-step. (1) Immediate: gate the test so it does not run on XPU by replacing `@onlyCUDA` enforcement in the XPU test generator (third_party/torch-xpu-ops/test/xpu/test_foreach_xpu.py), or add an XPU-specific skip since the test is explicitly CUDA-only.
- this removes the WA added in pytorch/pytorch#169354. (2) Proper fix: update the XPU foreach addcmul functor in ForeachPointwiseOpScalarKernels.cpp to use `sycl::fma(a, b, c)` on the alpha==1 branch so XPU matches CUDA FMA semantics, then enable the test on XPU.
