## Root Cause

Feature request (PT2.15) to enable MX-format training (torchao/prototype/mx_formats) on XPU. The
enabling path is blocked on the XPU `_scaled_mm` implementation landing (pytorch/pytorch PR #165978)
and the AO inference PR pytorch/ao#3248; `native_functions.yaml:7402` shows `_scaled_grouped_mm` has
no XPU dispatch and `_scaled_mm` XPU dispatch exists in name (native_functions.yaml) but the MX
E8M0/E4M3 emulate path is not yet wired in torch-xpu-ops.

## Fix Approach

- Land/track XPU `_scaled_mm` (PR #165978) via oneDNN matmul with scale support, enable the MX training UTs in `torchao/prototype/mx_formats` using the `emulate=True` path first, then replace with native scaled kernels once available.
- Add XPU entries under torch-xpu-ops test/xpu and report status in pytorch/ao#2917.
