## Root Cause

Umbrella skip list of regression UTs (WW10 2026-3-2). Triage by maintainers (jmamzax, 2026-04-15/17)
shows the dataloader/autograd process-spawn tests (test_get_worker_info, test_no_segfault,
test_large_sampler_indices, test_timeout, test_proper_exit, test_pynode_destruction_deadlock) only
fail on runner dut7901 when only 7 of 8 GPUs come online, an invalid ZE_AFFINITY_MASK configuration
is used (mask=7 instead of 2), and runs stretch to 5h+ causing 20-60s process-startup assertions to
time out (test/test_dataloader.py:1468/1492/1514/2076/2751, test/test_autograd.py:10915). The
test_compile_forward_clone NestedTensor failure is a dynamo data-dependent guard error Ne(u1,u0) in
test_nestedtensor_xpu.py:8909, and the unused scatter path hits AttributeError: module 'torch._C'
has no attribute '_scatter' because torch/nn/parallel/comm.py:209 calls a CUDA-only binding not
exposed for XPU.

## Fix Approach

- CI infra: fix dut7901 so all 8 Max-1100 GPUs enumerate reliably (driver/udev) and keep the corrected --tx popen ZE_AFFINITY_MASK=0..7 configuration.
- add a runner pre-check that aborts if `clinfo --list | grep Graphics` under each ZE_AFFINITY_MASK returns empty.
- Since the latest three runs on dut7901 (4/20) all pass with 8 GPUs, close this umbrella and re-file only the remaining genuine bugs: (a) port test_scatter away from torch._C._scatter for XPU or add an XPU binding mirroring cuda/Module.cpp's THCPModule_scatter, (b) mark test_compile_forward_clone_xpu_float32 with @expectedFailureXPU or file a dynamo UserError fix upstream, (c) matrix_rank warnings count test is already fixed in nightly.
