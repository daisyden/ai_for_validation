## Root Cause

test/inductor/test_inplace_padding.py::InplacePaddingTest.test_linear_and_cel_max_autotune (line 262
-> test_linear_and_cel at line 245) produces NaN in the bf16 CompiledFunctionBackward path while
eager ref is finite. Root cause is in the upstream Inductor max-autotune bf16
Linear+CrossEntropyLoss epilogue fusion for XPU: the Triton-generated kernel under max_autotune
writes into an in-place-padded tail of the grad activations with uninitialized padding, and those
padded values feed the subsequent softmax/log-softmax, producing NaN for the whole last rows
(observable as all-NaN tail rows in the actual output). Not an XPU aten op bug; it is exposed only
under inductor max_autotune + inplace padding after PR pytorch#162370.
