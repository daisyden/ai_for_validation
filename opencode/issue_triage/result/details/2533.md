## Root Cause

test/quantization/core/test_quantized_op.py:1310 defines test_qsoftmax_qnnpack which internally
calls self.test_qsoftmax(). When common_device_type instantiates TestQuantizedOps for XPU,
test_qsoftmax is renamed to test_qsoftmax_xpu, so self.test_qsoftmax no longer exists and
AttributeError is raised. qnnpack is an ARM/x86 CPU quantized backend and is not relevant to XPU at
all.

## Fix Approach

- Skip test_qsoftmax_qnnpack_xpu in third_party/torch-xpu-ops/test/xpu/skip_list_common.py (quantization section) with reason 'qnnpack is CPU-only.
- device-instantiation renames test_qsoftmax to test_qsoftmax_xpu causing AttributeError'.
- Alternatively upstream a fix in test_quantized_op.py to call self.test_qsoftmax_xpu when device is xpu, or decorate test_qsoftmax_qnnpack with @onlyCPU so it is not instantiated for xpu.
