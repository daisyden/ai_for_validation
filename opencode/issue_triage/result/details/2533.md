## Root Cause

test_qsoftmax_qnnpack_xpu in test/quantization/core/test_quantized_op.py:1308 calls
self.test_qsoftmax(), but test_qsoftmax is gated to only register for QNNPACK/CPU and is not present
on TestQuantizedOpsXPU, raising AttributeError. The XPU test class instantiation pulls in the
qnnpack wrapper test without the underlying test method, indicating a test-framework filtering gap
rather than an XPU op bug.

## Fix Approach

- Upstream PyTorch fix: guard test_qsoftmax_qnnpack with @skipIfNoQNNPACK plus device filter, or skip qsoftmax tests for non-CPU device types via instantiate_device_type_tests filter.
- Alternatively keep the entry on the torch-xpu-ops skiplist.
