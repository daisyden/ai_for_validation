## Root Cause

test_matrix_ops.py::test_scaled_mm fails inside DTensor sharding propagation:
aten._scaled_mm.default with 1x128 block-wise FP8 scales is sharded across a 4-way mesh, producing
local scale shapes (16,4) and (4,128), which _scaled_mm's shape validator rejects ('For
(BlockWise1x128, BlockWise1x128), scale_a should be (16, 1), scale_b should be (1, 128)'). The bug
is in upstream torch/distributed/tensor/_dispatch.py:277 / DTensor sharding rules for _scaled_mm —
they shard the K dim of the scale tensors, which is invalid for block-wise FP8. Not an XPU kernel
issue.

## Fix Approach

- File / track upstream pytorch fix in torch.distributed.tensor sharding propagation rule for aten._scaled_mm to either replicate the K-dim of the scale tensors or reject sharding along block-K.
- Skip test_scaled_mm on XPU until upstream lands.
