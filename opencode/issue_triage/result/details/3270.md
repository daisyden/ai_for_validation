## Root Cause

DTensor's sharding propagation for aten._scaled_mm with FP8 rejects scale shapes (16,4) and (4,128)
as 'Invalid scaling configuration' in torch/distributed/tensor/_dispatch.py:277. The check in
torch/distributed/tensor/_ops/_matrix_ops.py validates only
tensorwise/rowwise/blockwise(1x128,128x128) layouts and does not yet handle the new BlockWise128x128
x BlockWise128x128 (or sharded blockwise) layout produced when scales are sharded across the mesh.
This is a DTensor op-level gap, not XPU-specific; CUDA path likely hits the same unless the
validator was updated.

## Fix Approach

- Extend the _scaled_mm sharding rule in torch/distributed/tensor/_ops/_matrix_ops.py to accept the (BlockWise128x128, BlockWise128x128) scale shape and properly compute output sharding for sharded scales.
- Submit fix to pytorch/pytorch upstream.
- afterward re-enable test_scaled_mm in torch-xpu-ops distributed CI.
