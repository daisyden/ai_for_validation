## Root Cause

Two unimplemented XCCL features. (1) The worker control-plane handler
'/handler/dump_nccl_trace_pickle' is registered only by
torch/csrc/distributed/c10d/FlightRecorderCuda.cpp:16 and there is no equivalent XCCL registration
in third_party/torch-xpu-ops/src/xccl/FlightRecorderXCCL.cpp, so
test_control_plane.py::test_dump_nccl_trace_pickle[_with_json|_with_params] returns 404. (2) FSDP
force-sum tests expect a per-rank 'nccl_log' file written by ProcessGroupNCCL when
TORCH_NCCL_DEBUG/NCCL_DEBUG_FILE is set; ProcessGroupXCCL does not emit such a log, so
test_fully_shard_force_sum_* fails with FileNotFoundError for nccl_log.

## Fix Approach

- (1) In FlightRecorderXCCL.cpp register a 'dump_nccl_trace_pickle' (or an alias) handler via ::c10d::control_plane::registerHandler, paralleling FlightRecorderCuda.cpp, that serializes the XCCL flight recorder via pickle. (2) Add debug-log emission in ProcessGroupXCCL: honour TORCH_NCCL_DEBUG_FILE env and write collective op trace to that path on init/destruction.
- Alternatively, register device-aware skip entries for these three tests in test/xpu/skip_list_dist_local.py until parity is implemented.
