## Root Cause

FSDP force-sum reduction tests and elastic control_plane tests rely on the
/handler/dump_nccl_trace_pickle HTTP handler and on writing nccl_log files. The handler is
registered only in torch/csrc/distributed/c10d/FlightRecorderCuda.cpp:16 for CUDA/NCCL; the XCCL
counterpart third_party/torch-xpu-ops/src/xccl/FlightRecorderXCCL.cpp does not register an
equivalent dump_xccl/nccl_trace_pickle handler nor does ProcessGroupXCCL emit nccl_log on
reduce-scatter, so FileNotFoundError is raised.

## Fix Approach

- In FlightRecorderXCCL.cpp register a /handler/dump_nccl_trace_pickle (or aliased dump_xccl_trace_pickle) handler analogous to FlightRecorderCuda.cpp:16, and add nccl_log emission paths in ProcessGroupXCCL reduce-scatter/all-reduce when TORCH_NCCL_DEBUG=INFO.
- then enable the listed FSDP and control_plane tests.
