## Root Cause

Upstream PyTorch PR pytorch/pytorch#168335 (commit d50721ab12) reordered Inductor's pattern matcher
so that early_patterns.apply(gm.graph) runs first and removes 'pointless' view operations before the
main fusion pass. This breaks the large contiguous graph the pattern matcher previously saw for T5,
splitting a single fused softmax/embedding/dropout kernel into two smaller kernels
(triton_red_fused__softmax_add_permute_slice_unsqueeze_view_* and
triton_poi_fused__softmax__to_copy_add_native_dropout_permute_slice_unsqueeze_view_*). Affects both
XPU and CUDA; a ~20% inductor slowdown shows on T5ForConditionalGeneration and T5Small in amp
fp16/bf16 training.

## Fix Approach

- Cherry-pick upstream fix pytorch/pytorch#169257 into release/2.11 to restore pattern matcher ordering so view-removal runs after the fused softmax/dropout pattern is matched.
- tracked upstream as pytorch/pytorch#177118.
- After the cherry-pick, re-run the T5 huggingface benchmark (amp fp16/bf16 training) to confirm the inductor ratio returns to ~1.0.
