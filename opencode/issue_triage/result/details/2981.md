## Root Cause

T5 inductor training perf dropped ~20% on release/2.11 BMG. Bisected to pytorch/pytorch@d50721a (PR
#168335), which made early_patterns.apply(gm.graph) run before the main pattern matcher, removing
'pointless' view ops and breaking the large fused kernel
triton_per_fused__softmax__to_copy_add_arange_embedding_... into two smaller kernels
(triton_red_fused__softmax_add_... + triton_poi_fused__softmax__to_copy_add_...). Same drop
reproduced on CUDA, confirming upstream Inductor pattern-matcher ordering bug. Tracked as
pytorch/pytorch#177118 and a candidate fix is PR #169257.

## Fix Approach

- Cherry-pick pytorch/pytorch#169257 (or the eventual fix for #177118) into release/2.11 to restore pre-pattern-matcher view ordering so the full softmax+embedding fusion is preserved.
- No torch-xpu-ops change needed.
- verify T5ForConditionalGeneration / T5Small inductor ratio returns to ~1.0.
